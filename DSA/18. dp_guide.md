# 1. What is Dynamic Programming?

## The Core Idea

**Dynamic Programming (DP)** solves complex problems by breaking them into overlapping subproblems, solving each subproblem **once**, and **storing** the result to avoid recomputation.

```
Two conditions for DP:
1. OPTIMAL SUBSTRUCTURE: Optimal solution contains optimal solutions to subproblems
2. OVERLAPPING SUBPROBLEMS: Same subproblems computed repeatedly

If BOTH hold → DP is the right tool!
```

## Why DP? — The Fibonacci Example

```
Fibonacci: fib(n) = fib(n-1) + fib(n-2)

NAIVE RECURSION:              DP WITH MEMO:
fib(5)                        fib(5)
├─ fib(4)                     ├─ fib(4) → compute once, store
│   ├─ fib(3)                 └─ fib(3) → already stored, return!
│   │   ├─ fib(2)
│   │   │   ├─ fib(1)         Time: O(2ⁿ) → O(n)
│   │   │   └─ fib(0)         Space: O(n) → O(1) with optimization
│   │   └─ fib(1)
│   └─ fib(2)  ← RECOMPUTED!
│       ├─ fib(1)
│       └─ fib(0)
└─ fib(3)  ← RECOMPUTED AGAIN!
    ├─ fib(2)
    └─ fib(1)

Naive: O(2ⁿ)    DP: O(n)   ← MASSIVE improvement!
```

## DP vs Greedy vs Backtracking

```
BACKTRACKING: Explore ALL possibilities
  → Good for: "Find ALL solutions", combinatorial problems
  → Problem: Exponential time

GREEDY: Make LOCALLY OPTIMAL choice at each step
  → Good for: "Always correct" local decisions (Huffman, Dijkstra)
  → Problem: Doesn't always give global optimum

DYNAMIC PROGRAMMING: Overlapping subproblems + optimal substructure
  → Good for: Optimization ("min/max/count"), overlapping computations
  → Problem: Requires careful state definition

Key question: "If I solve smaller subproblems optimally,
              can I combine them to solve the bigger problem optimally?"
  YES → DP
```

# 2. The Two Approaches

## Top-Down (Memoization) — Recursive + Cache

```
Start from the original problem, break it down recursively.
When you compute a subproblem, CACHE the result.
On future calls, return cached result instead of recomputing.

fib(5)
  → calls fib(4) and fib(3)
  → fib(4) calls fib(3) and fib(2)
  → fib(3) already cached! Return immediately ✅

Natural to write (follows the recurrence directly).
```

```java
// TOP-DOWN: Fibonacci
int[] memo;

public int fib(int n) {
    memo = new int[n + 1];
    Arrays.fill(memo, -1);
    return solve(n);
}

private int solve(int n) {
    if (n <= 1) return n;                // Base case
    if (memo[n] != -1) return memo[n];  // Return cached
    memo[n] = solve(n-1) + solve(n-2);  // Compute and cache
    return memo[n];
}
```

## Bottom-Up (Tabulation) — Iterative + Table

```
Start from base cases, build up to the answer iteratively.
Fill a table in order such that when we need dp[i],
all dependencies dp[j] (j < i) are already computed.

fib: dp[0]=0, dp[1]=1, dp[2]=1, dp[3]=2, dp[4]=3, dp[5]=5

No recursion, no stack overhead. Usually faster in practice.
```

```java
// BOTTOM-UP: Fibonacci
public int fib(int n) {
    if (n <= 1) return n;
    int[] dp = new int[n + 1];
    dp[0] = 0;
    dp[1] = 1;
    for (int i = 2; i <= n; i++) {
        dp[i] = dp[i-1] + dp[i-2];  // Build from base cases up
    }
    return dp[n];
}

// OPTIMIZED: Only need last 2 values
public int fibOptimized(int n) {
    if (n <= 1) return n;
    int prev2 = 0, prev1 = 1;
    for (int i = 2; i <= n; i++) {
        int curr = prev1 + prev2;
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
```

## Which to Use?

```
TOP-DOWN (Memoization):
  ✅ Easier to think about (mirrors the recurrence)
  ✅ Only computes needed subproblems (lazy evaluation)
  ❌ Recursion overhead, stack overflow risk for large n
  ❌ Harder to optimize space

BOTTOM-UP (Tabulation):
  ✅ No recursion — faster, no stack risk
  ✅ Easier to optimize space (rolling array)
  ✅ Clear about computation order
  ❌ Need to figure out correct filling order

RECOMMENDATION: Learn both. Start with top-down to understand, 
                convert to bottom-up for efficiency.
```

# 3. The 5-Step Framework

**Use these 5 steps for ANY DP problem:**

```
STEP 1: Define the STATE
  What does dp[i] (or dp[i][j]) represent?
  This is the most important step — get this right, rest follows.

STEP 2: Write the RECURRENCE
  How does dp[i] relate to smaller subproblems?
  dp[i] = f(dp[i-1], dp[i-2], ...)

STEP 3: Identify BASE CASES
  What are the smallest/simplest inputs?
  dp[0] = ?, dp[1] = ?

STEP 4: Determine COMPUTATION ORDER
  Which direction to fill the table?
  (Usually left to right, or outer to inner for 2D)

STEP 5: Extract ANSWER
  Which dp value(s) give the final answer?
  dp[n]? max(dp[i])? dp[m][n]?
```

## Example — Applying the Framework

**Problem: Climbing Stairs (LC #70)**

```
STEP 1: STATE
  dp[i] = number of ways to climb i stairs

STEP 2: RECURRENCE
  To reach stair i, we came from stair i-1 (took 1 step)
                      OR stair i-2 (took 2 steps)
  dp[i] = dp[i-1] + dp[i-2]

STEP 3: BASE CASES
  dp[0] = 1 (one way to be at stair 0: do nothing)
  dp[1] = 1 (one way to reach stair 1: take 1 step)

STEP 4: ORDER
  Fill left to right: dp[0], dp[1], dp[2], ..., dp[n]

STEP 5: ANSWER
  dp[n]
```

# 4. Pattern 1 — 1D Linear DP

## The Pattern

State depends on a **linear sequence** (array, string, stairs).
`dp[i]` depends on `dp[i-1]`, `dp[i-2]`, or earlier values.

```
dp[i] = optimal value for the first i elements
         or for ending at element i
```

## 🟢 LC #70 — Climbing Stairs

```java
public int climbStairs(int n) {
    if (n <= 2) return n;
    int prev2 = 1, prev1 = 2;
    for (int i = 3; i <= n; i++) {
        int curr = prev1 + prev2;
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
```

## 🟢 LC #198 — House Robber

**Problem:** Rob houses in a row. Can't rob adjacent houses. Maximize amount robbed.

**State:** `dp[i]` = max money robbing from houses 0 to i

**Recurrence:**
```
For house i, two choices:
  1. Rob house i:    dp[i] = dp[i-2] + nums[i]  (can't use i-1)
  2. Skip house i:   dp[i] = dp[i-1]
  dp[i] = max(dp[i-2] + nums[i], dp[i-1])
```

```java
public int rob(int[] nums) {
    int n = nums.length;
    if (n == 1) return nums[0];

    int prev2 = nums[0];
    int prev1 = Math.max(nums[0], nums[1]);

    for (int i = 2; i < n; i++) {
        int curr = Math.max(prev1, prev2 + nums[i]);
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
```

## 🟡 LC #213 — House Robber II (Circular)

**Problem:** Houses are in a circle — first and last are adjacent.

**Key Insight:** Either rob house 0 (skip last) OR skip house 0 (can rob last). Run House Robber I twice.

```java
public int rob(int[] nums) {
    int n = nums.length;
    if (n == 1) return nums[0];
    if (n == 2) return Math.max(nums[0], nums[1]);
    // Case 1: include first house (indices 0 to n-2)
    // Case 2: include last house (indices 1 to n-1)
    return Math.max(robRange(nums, 0, n-2), robRange(nums, 1, n-1));
}

private int robRange(int[] nums, int start, int end) {
    int prev2 = 0, prev1 = 0;
    for (int i = start; i <= end; i++) {
        int curr = Math.max(prev1, prev2 + nums[i]);
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
```

## 🟡 LC #300 — Longest Increasing Subsequence (LIS)

**Problem:** Find the length of the longest strictly increasing subsequence.

**State:** `dp[i]` = length of LIS ending at index i

**Recurrence:**
```
dp[i] = max(dp[j] + 1) for all j < i where nums[j] < nums[i]
```

```java
// O(n²) DP approach
public int lengthOfLIS(int[] nums) {
    int n = nums.length;
    int[] dp = new int[n];
    Arrays.fill(dp, 1);  // Each element is an LIS of length 1 by itself
    int maxLen = 1;

    for (int i = 1; i < n; i++) {
        for (int j = 0; j < i; j++) {
            if (nums[j] < nums[i]) {
                dp[i] = Math.max(dp[i], dp[j] + 1);
            }
        }
        maxLen = Math.max(maxLen, dp[i]);
    }
    return maxLen;
}

// O(n log n) Binary Search approach
public int lengthOfLISFast(int[] nums) {
    List<Integer> tails = new ArrayList<>();  // tails[i] = smallest tail of IS of length i+1

    for (int num : nums) {
        int pos = Collections.binarySearch(tails, num);
        if (pos < 0) pos = -(pos + 1);  // Insertion point
        if (pos == tails.size()) tails.add(num);
        else tails.set(pos, num);
    }
    return tails.size();
}
```

## 🟡 LC #139 — Word Break

**Problem:** Can string s be segmented using words from wordDict?

**State:** `dp[i]` = can we segment s[0..i-1] using wordDict?

```java
public boolean wordBreak(String s, List<String> wordDict) {
    Set<String> wordSet = new HashSet<>(wordDict);
    int n = s.length();
    boolean[] dp = new boolean[n + 1];
    dp[0] = true;  // Empty string is always valid

    for (int i = 1; i <= n; i++) {
        for (int j = 0; j < i; j++) {
            // If dp[j] is true AND s[j..i-1] is a valid word → dp[i] = true
            if (dp[j] && wordSet.contains(s.substring(j, i))) {
                dp[i] = true;
                break;
            }
        }
    }
    return dp[n];
}
```

## 🟡 LC #322 — Coin Change (Minimum Coins)

**Problem:** Minimum coins to make target amount. (Covered in Unbounded Knapsack too)

```java
public int coinChange(int[] coins, int amount) {
    int[] dp = new int[amount + 1];
    Arrays.fill(dp, amount + 1);  // Initialize with impossible value
    dp[0] = 0;

    for (int i = 1; i <= amount; i++) {
        for (int coin : coins) {
            if (coin <= i) {
                dp[i] = Math.min(dp[i], 1 + dp[i - coin]);
            }
        }
    }
    return dp[amount] > amount ? -1 : dp[amount];
}
```

# 5. Pattern 2 — Kadane's / Subarray DP

## The Pattern

Find **optimal contiguous subarray** (max sum, max product, etc.).

```
dp[i] = best result for subarray ENDING at index i
         (local optimum at each position)
Answer = max(dp[i]) for all i   (global optimum)
```

---

## 🟡 LC #53 — Maximum Subarray (Kadane's Algorithm)

```
dp[i] = max subarray sum ending at i
dp[i] = max(nums[i], dp[i-1] + nums[i])
      = "Start fresh at i" vs "Extend subarray from i-1"
```

```java
public int maxSubArray(int[] nums) {
    int maxSoFar = nums[0];
    int currentMax = nums[0];

    for (int i = 1; i < nums.length; i++) {
        currentMax = Math.max(nums[i], currentMax + nums[i]);
        maxSoFar = Math.max(maxSoFar, currentMax);
    }
    return maxSoFar;
}
```

---

## 🟡 LC #152 — Maximum Product Subarray

**Key:** Track both max AND min (negative × negative = positive!)

```java
public int maxProduct(int[] nums) {
    int maxSoFar = nums[0];
    int maxEnd = nums[0];
    int minEnd = nums[0];

    for (int i = 1; i < nums.length; i++) {
        int num = nums[i];
        // All three candidates: num alone, extend max, extend min (neg*neg)
        int tempMax = Math.max(num, Math.max(maxEnd * num, minEnd * num));
        int tempMin = Math.min(num, Math.min(maxEnd * num, minEnd * num));
        maxEnd = tempMax;
        minEnd = tempMin;
        maxSoFar = Math.max(maxSoFar, maxEnd);
    }
    return maxSoFar;
}
```

## 🟡 LC #918 — Maximum Sum Circular Subarray

```java
public int maxSubarraySumCircular(int[] nums) {
    int totalSum = 0;
    int maxSum = nums[0], curMax = 0;
    int minSum = nums[0], curMin = 0;

    for (int num : nums) {
        curMax = Math.max(num, curMax + num);
        maxSum = Math.max(maxSum, curMax);
        curMin = Math.min(num, curMin + num);
        minSum = Math.min(minSum, curMin);
        totalSum += num;
    }
    // If all negative → return maxSum (circular case is empty = invalid)
    return maxSum > 0 ? Math.max(maxSum, totalSum - minSum) : maxSum;
}
```

# 6. Pattern 3 — 0/1 Knapsack

## The Pattern

**Binary choice per item:** include (1) or exclude (0).
Classic setup: N items, each with weight and value, capacity W.

```
dp[i][w] = max value using first i items with capacity w

For item i:
  Option 1: Skip item i   → dp[i][w] = dp[i-1][w]
  Option 2: Take item i   → dp[i][w] = dp[i-1][w - weight[i]] + value[i]
                            (only if weight[i] <= w)

dp[i][w] = max(Option 1, Option 2)
```

## Classic 0/1 Knapsack

```java
public int knapsack01(int W, int[] weights, int[] values, int n) {
    // dp[i][w] = max value with first i items and capacity w
    int[][] dp = new int[n + 1][W + 1];

    for (int i = 1; i <= n; i++) {
        for (int w = 0; w <= W; w++) {
            dp[i][w] = dp[i-1][w];  // Skip item i
            if (weights[i-1] <= w) {
                dp[i][w] = Math.max(dp[i][w],
                    dp[i-1][w - weights[i-1]] + values[i-1]);  // Take item i
            }
        }
    }
    return dp[n][W];
}

// SPACE OPTIMIZED: 1D rolling array
public int knapsack01Optimized(int W, int[] weights, int[] values, int n) {
    int[] dp = new int[W + 1];
    for (int i = 0; i < n; i++) {
        // Traverse RIGHT TO LEFT to avoid using item twice!
        for (int w = W; w >= weights[i]; w--) {
            dp[w] = Math.max(dp[w], dp[w - weights[i]] + values[i]);
        }
    }
    return dp[W];
}
```

**Why right-to-left for 0/1 knapsack?**
```
If we go left-to-right, dp[w - weight] has already been updated
in this iteration → we'd be using the same item twice (unbounded!)

Right-to-left: dp[w - weight] is still from the PREVIOUS item's row ✅
```

## 🟡 LC #416 — Partition Equal Subset Sum

**Problem:** Can we partition array into two subsets with equal sum?

**Key:** Find subset summing to total/2 → Boolean 0/1 Knapsack

```java
public boolean canPartition(int[] nums) {
    int total = 0;
    for (int n : nums) total += n;
    if (total % 2 != 0) return false;  // Odd sum → impossible

    int target = total / 2;
    boolean[] dp = new boolean[target + 1];
    dp[0] = true;  // Empty subset sums to 0

    for (int num : nums) {
        for (int w = target; w >= num; w--) {  // Right to left!
            dp[w] = dp[w] || dp[w - num];
        }
    }
    return dp[target];
}
```

## 🟡 LC #474 — Ones and Zeroes

**Problem:** Find max subset of strings where total 0s ≤ m and 1s ≤ n.
This is a **2D knapsack** — capacity is a 2D constraint.

```java
public int findMaxForm(String[] strs, int m, int n) {
    int[][] dp = new int[m + 1][n + 1];  // dp[i][j] = max strings with i zeros, j ones

    for (String s : strs) {
        int zeros = 0, ones = 0;
        for (char c : s.toCharArray()) {
            if (c == '0') zeros++; else ones++;
        }

        // Right to left (0/1 knapsack: each string used at most once)
        for (int i = m; i >= zeros; i--) {
            for (int j = n; j >= ones; j--) {
                dp[i][j] = Math.max(dp[i][j], dp[i - zeros][j - ones] + 1);
            }
        }
    }
    return dp[m][n];
}
```

## 🟡 LC #494 — Target Sum

**Problem:** Assign + or - to each number. Count ways to reach target.

**Key:** Let P = sum of positive numbers, N = sum of negative numbers.
P - N = target, P + N = total → P = (target + total) / 2
→ Count subsets summing to P.

```java
public int findTargetSumWays(int[] nums, int target) {
    int total = 0;
    for (int n : nums) total += n;
    if ((total + target) % 2 != 0 || Math.abs(target) > total) return 0;
    int sum = (total + target) / 2;

    int[] dp = new int[sum + 1];
    dp[0] = 1;  // One way to make sum 0: use no elements

    for (int num : nums) {
        for (int w = sum; w >= num; w--) {
            dp[w] += dp[w - num];
        }
    }
    return dp[sum];
}
```

## 🟡 LC #1049 — Last Stone Weight II

**Problem:** Split stones into two groups. Minimize difference of their sums.

**Key:** Minimize |S1 - S2| where S1 + S2 = total → maximize S1 ≤ total/2.

```java
public int lastStoneWeightII(int[] stones) {
    int total = 0;
    for (int s : stones) total += s;
    int target = total / 2;

    boolean[] dp = new boolean[target + 1];
    dp[0] = true;

    for (int stone : stones) {
        for (int w = target; w >= stone; w--) {
            dp[w] = dp[w] || dp[w - stone];
        }
    }

    for (int w = target; w >= 0; w--) {
        if (dp[w]) return total - 2 * w;  // Minimize diff = total - 2*S1
    }
    return 0;
}
```

# 7. Pattern 4 — Unbounded Knapsack

## The Pattern

Same as 0/1 Knapsack but each item can be used **unlimited times**.
Only change: traverse left-to-right in the 1D array!

```
0/1 Knapsack:        for w = W downto weight[i]    ← right to left
Unbounded Knapsack:  for w = weight[i] to W        ← left to right
```

```java
        if (coin <= i) dp[i] = Math.min(dp[i], 1 + dp[i - coin]);
    }
}
```

## 🟡 LC #518 — Coin Change II (Count Ways)

**Problem:** Count distinct combinations to make amount. (Order doesn't matter)

```java
public int change(int amount, int[] coins) {
    int[] dp = new int[amount + 1];
    dp[0] = 1;

    // Outer loop: coins (ensures combinations, not permutations)
    for (int coin : coins) {
        for (int i = coin; i <= amount; i++) {  // Left to right = unbounded
            dp[i] += dp[i - coin];
        }
    }
    return dp[amount];
}
```

**Why coins in outer loop?**
```
If amount in outer loop:
  We count [1,2] and [2,1] as different → PERMUTATIONS

If coins in outer loop:
  When processing coin=2, coin=1 choices are already fixed
  → COMBINATIONS (order doesn't matter) ✅
```

## 🟡 LC #377 — Combination Sum IV (Permutations Count)

```java
public int combinationSum4(int[] nums, int target) {

    // Amount in outer loop → ORDER MATTERS (permutations)

        for (int num : nums) {
            if (num <= i) dp[i] += dp[i - num];

}
```

# 8. Pattern 5 — DP on Strings

## The Pattern

Two strings s and t. `dp[i][j]` = answer for first i chars of s and first j chars of t.

```
If s[i] == t[j]:  dp[i][j] = dp[i-1][j-1] + something
If s[i] != t[j]:  dp[i][j] = best of (dp[i-1][j], dp[i][j-1], dp[i-1][j-1] + 1)
```

---

## 🟡 LC #1143 — Longest Common Subsequence (LCS)

**Problem:** Length of longest subsequence common to both strings.

```
dp[i][j] = LCS of s[0..i-1] and t[0..j-1]

If s[i-1] == t[j-1]: dp[i][j] = dp[i-1][j-1] + 1
Else:                 dp[i][j] = max(dp[i-1][j], dp[i][j-1])
```

```java
public int longestCommonSubsequence(String text1, String text2) {
    int m = text1.length(), n = text2.length();
    int[][] dp = new int[m + 1][n + 1];

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (text1.charAt(i-1) == text2.charAt(j-1)) {
                dp[i][j] = dp[i-1][j-1] + 1;
            } else {
                dp[i][j] = Math.max(dp[i-1][j], dp[i][j-1]);
            }
        }
    }
    return dp[m][n];
}
```

## 🟡 LC #72 — Edit Distance

**Problem:** Minimum operations (insert, delete, replace) to transform word1 to word2.

```
dp[i][j] = min edits to transform word1[0..i-1] to word2[0..j-1]

If word1[i-1] == word2[j-1]: dp[i][j] = dp[i-1][j-1]   (no op needed)
Else:
  Insert:  dp[i][j-1] + 1
  Delete:  dp[i-1][j] + 1
  Replace: dp[i-1][j-1] + 1
  dp[i][j] = min of all three
```

```java
public int minDistance(String word1, String word2) {
    int m = word1.length(), n = word2.length();
    int[][] dp = new int[m + 1][n + 1];

    // Base cases
    for (int i = 0; i <= m; i++) dp[i][0] = i;  // Delete all of word1
    for (int j = 0; j <= n; j++) dp[0][j] = j;  // Insert all of word2

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (word1.charAt(i-1) == word2.charAt(j-1)) {
                dp[i][j] = dp[i-1][j-1];
            } else {
                dp[i][j] = 1 + Math.min(dp[i-1][j-1],       // Replace
                               Math.min(dp[i-1][j],           // Delete
                                        dp[i][j-1]));         // Insert
            }
        }
    }
    return dp[m][n];
}
```

```java
public int longestPalindromeSubseq(String s) {
    int n = s.length();
    int[][] dp = new int[n][n];

    // All single chars are palindromes of length 1
    for (int i = 0; i < n; i++) dp[i][i] = 1;

    // Build for increasing lengths
    for (int len = 2; len <= n; len++) {
        for (int i = 0; i <= n - len; i++) {
            int j = i + len - 1;
            if (s.charAt(i) == s.charAt(j)) {
                dp[i][j] = dp[i+1][j-1] + 2;
            } else {
                dp[i][j] = Math.max(dp[i+1][j], dp[i][j-1]);
            }
        }
    }
    return dp[0][n-1];
}
```

## 🟡 LC #5 — Longest Palindromic Substring

```java
public String longestPalindrome(String s) {
    int n = s.length(), start = 0, maxLen = 1;
    boolean[][] dp = new boolean[n][n];

    // All single chars
    for (int i = 0; i < n; i++) dp[i][i] = true;

    // Length 2
    for (int i = 0; i < n-1; i++) {
        if (s.charAt(i) == s.charAt(i+1)) {
            dp[i][i+1] = true; start = i; maxLen = 2;
        }
    }

    // Length 3+
    for (int len = 3; len <= n; len++) {
        for (int i = 0; i <= n - len; i++) {
            int j = i + len - 1;
            if (s.charAt(i) == s.charAt(j) && dp[i+1][j-1]) {
                dp[i][j] = true;
                if (len > maxLen) { maxLen = len; start = i; }
            }
        }
    }
    return s.substring(start, start + maxLen);
}
```

## 🟡 LC #115 — Distinct Subsequences

**Problem:** Count ways s's subsequences match t.

```java
public int numDistinct(String s, String t) {
    int m = s.length(), n = t.length();
    long[][] dp = new long[m + 1][n + 1];

    for (int i = 0; i <= m; i++) dp[i][0] = 1;  // Empty t matches anywhere

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {

        }
```

# 9. Pattern 6 — DP on Intervals

## The Pattern

`dp[i][j]` = optimal answer for subproblem on range [i, j].
Try every possible **split point k** in [i, j].

```
dp[i][j] = best over all k in [i, j]:
            f(dp[i][k], dp[k+1][j], cost(i, j, k))

Fill order: increasing interval length!
for len = 1 to n:
  for i = 0 to n-len:
    j = i + len - 1
```

---

## 🔴 LC #312 — Burst Balloons

**Problem:** Burst all balloons. Bursting balloon i earns `nums[i-1]*nums[i]*nums[i+1]`. Maximize coins.

**Key Insight:** Think of k as the LAST balloon burst in range [i,j] (not the first).
Then left and right boundaries are still intact when k is burst.

```java
public int maxCoins(int[] nums) {
    int n = nums.length;
    // Add boundary balloons
    int[] balls = new int[n + 2];
    balls[0] = balls[n + 1] = 1;
    for (int i = 0; i < n; i++) balls[i + 1] = nums[i];
    int m = n + 2;

    // dp[i][j] = max coins from bursting ALL balloons strictly between i and j
    int[][] dp = new int[m][m];

    for (int len = 2; len < m; len++) {
        for (int i = 0; i < m - len; i++) {
            int j = i + len;
            for (int k = i + 1; k < j; k++) {
                // k is the LAST burst in (i, j) — boundaries balls[i] and balls[j] still alive
                int coins = balls[i] * balls[k] * balls[j];
                dp[i][j] = Math.max(dp[i][j], dp[i][k] + coins + dp[k][j]);
            }
        }
    }
    return dp[0][m - 1];
}
```

# 10. Pattern 7 — DP on Trees

## The Pattern

Process tree bottom-up (postorder). Each node computes based on children.
Often need to track two states: use this node OR skip this node.

## 🟡 LC #337 — House Robber III

**Problem:** Rob houses in a binary tree — can't rob adjacent (parent-child) nodes.

**State per node:** `[rob, skip]`
- `rob` = max if we ROB this node
- `skip` = max if we SKIP this node

```java
public int rob(TreeNode root) {
    int[] result = dfs(root);
    return Math.max(result[0], result[1]);
}

// Returns [rob_this_node, skip_this_node]
private int[] dfs(TreeNode node) {
    if (node == null) return new int[]{0, 0};

    int[] left  = dfs(node.left);
    int[] right = dfs(node.right);

    // Rob this node: can't rob children (use their skip values)
    int rob  = node.val + left[1] + right[1];
    // Skip this node: children can be robbed or skipped (take max)
    int skip = Math.max(left[0], left[1]) + Math.max(right[0], right[1]);

    return new int[]{rob, skip};
}
```

## 🔴 LC #968 — Binary Tree Cameras

**Problem:** Place cameras on nodes to monitor entire tree. Minimum cameras.

**State per node:** 0=not covered, 1=has camera, 2=covered (no camera)

```java
int cameras = 0;

public int minCameraCover(TreeNode root) {
    cameras = 0;
    return dfs(root) == 0 ? cameras + 1 : cameras;
}

// 0: not monitored, 1: has camera, 2: monitored (by child camera)
private int dfs(TreeNode node) {
    if (node == null) return 2;  // Null = covered (don't need camera)

    int left  = dfs(node.left);
    int right = dfs(node.right);

    if (left == 0 || right == 0) {
        cameras++;     // Child not covered → must place camera here
        return 1;
    }
    if (left == 1 || right == 1) return 2;  // Child has camera → covered
    return 0;  // Both children covered (no camera) → this node not covered
}
```

# 11. Pattern 8 — DP on Grids

## The Pattern

2D grid where each cell depends on adjacent cells.
`dp[r][c]` = optimal value reaching/at cell (r, c).

```
Most common transitions:
  dp[r][c] = dp[r-1][c] + grid[r][c]        (from top)
  dp[r][c] = min(dp[r-1][c], dp[r][c-1]) + grid[r][c]   (from top or left)
```

```java
public int uniquePaths(int m, int n) {
    int[][] dp = new int[m][n];
    // First row and first column: only one path (always go right/down)
    for (int i = 0; i < m; i++) dp[i][0] = 1;
    for (int j = 0; j < n; j++) dp[0][j] = 1;

    for (int i = 1; i < m; i++) {
        for (int j = 1; j < n; j++) {
            dp[i][j] = dp[i-1][j] + dp[i][j-1];  // From top + from left
        }
    }
    return dp[m-1][n-1];
}
```

## 🟡 LC #221 — Maximal Square

**Problem:** Find the largest square of 1s in binary matrix.

**Key Insight:** dp[i][j] = side length of largest square with bottom-right corner at (i,j)

```
If matrix[i][j] == '1':
  dp[i][j] = min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) + 1
  
Why min? The square must fit in all three directions.
```

```java
public int maximalSquare(char[][] matrix) {
    int m = matrix.length, n = matrix[0].length;
    int[][] dp = new int[m + 1][n + 1];
    int maxSide = 0;

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (matrix[i-1][j-1] == '1') {
                dp[i][j] = Math.min(dp[i-1][j],
                           Math.min(dp[i][j-1], dp[i-1][j-1])) + 1;
                maxSide = Math.max(maxSide, dp[i][j]);
            }
        }
    }
    return maxSide * maxSide;
}
```

```java
public int atMostNGivenDigitSet(String[] digits, int n) {
    String s = Integer.toString(n);
```

# 13. Pattern 10 — Bitmask DP

## The Pattern

When n is small (≤ 20) and we need to track which elements have been used.
Use bitmask: bit i set = element i is used.

```
dp[mask] = optimal value having used the elements in 'mask'
dp[mask | (1 << i)] = transition: add element i
```

---

## 🔴 LC #847 — Shortest Path Visiting All Nodes

```java
public int shortestPathLength(int[][] graph) {
    int n = graph.length;
    int fullMask = (1 << n) - 1;

    // BFS: state = (node, visited mask)
    Queue<int[]> queue = new ArrayDeque<>();
    boolean[][] visited = new boolean[n][1 << n];

    for (int i = 0; i < n; i++) {
        int mask = 1 << i;
        queue.offer(new int[]{i, mask, 0});  // [node, mask, distance]
        visited[i][mask] = true;
    }

    while (!queue.isEmpty()) {
        int[] curr = queue.poll();
        int node = curr[0], mask = curr[1], dist = curr[2];

        if (mask == fullMask) return dist;

        for (int next : graph[node]) {
            int newMask = mask | (1 << next);
            if (!visited[next][newMask]) {
                visited[next][newMask] = true;
                queue.offer(new int[]{next, newMask, dist + 1});
            }
        }
    }
    return -1;
}
```

## 🔴 LC #1125 — Smallest Sufficient Team

```java
public int[] smallestSufficientTeam(String[] req_skills, List<List<String>> people) {
    int n = req_skills.length, m = people.size();
    Map<String, Integer> skillIndex = new HashMap<>();
    for (int i = 0; i < n; i++) skillIndex.put(req_skills[i], i);

    int fullMask = (1 << n) - 1;
    int[] dp = new int[fullMask + 1];  // dp[mask] = bitmask of people for this skill set
    int[] prevMask = new int[fullMask + 1];
    int[] prevPerson = new int[fullMask + 1];
    Arrays.fill(dp, Integer.MAX_VALUE);
    dp[0] = 0;

    for (int mask = 0; mask <= fullMask; mask++) {
        if (dp[mask] == Integer.MAX_VALUE) continue;
        for (int p = 0; p < m; p++) {
            int skillMask = 0;
            for (String skill : people.get(p)) {
                if (skillIndex.containsKey(skill)) skillMask |= 1 << skillIndex.get(skill);
            }
            int newMask = mask | skillMask;
            if (Integer.bitCount(dp[mask]) + 1 < Integer.bitCount(dp[newMask])) {
                dp[newMask] = dp[mask] | (1 << p);
                prevMask[newMask] = mask;
                prevPerson[newMask] = p;
            }
        }
    }

    List<Integer> result = new ArrayList<>();
    for (int mask = fullMask; mask != 0; mask = prevMask[mask]) {
        result.add(prevPerson[mask]);
    }
    return result.stream().mapToInt(Integer::intValue).toArray();
}
```

# 15. How to Identify DP Patterns

```
KEYWORDS → DP PATTERN
──────────────────────────────────────────────────────────────
"max/min" + "subarray/path"    → Kadane's or Grid DP
"count ways to..."             → Combinatorial DP
"can we reach..." / "possible" → Boolean DP (true/false)
"longest/shortest subsequence" → LCS / LIS family
"all subsequences"             → Usually DP, not backtrack
"partition into..."            → 0/1 Knapsack
"unlimited/any number of uses" → Unbounded Knapsack
"two strings, transform"       → Edit Distance / LCS
"range [i,j] optimal split"    → Interval DP
"tree + max/min at each node"  → Tree DP
"grid, reach bottom-right"     → Grid DP
"small n ≤ 20, all subsets"    → Bitmask DP
"stock buy/sell"               → State Machine DP
"k transactions allowed"       → 2D DP (day × transactions)

PATTERN SELECTION:
┌──────────────────────────────────────────────────────┐
│ Linear sequence, dp[i] from dp[i-k..i-1]?            │
│   → 1D LINEAR DP                                     │
├──────────────────────────────────────────────────────┤
│ Two strings / arrays? dp[i][j]?                      │
│   → LCS FAMILY (edit distance, LCS, etc.)            │
├──────────────────────────────────────────────────────┤
│ Items + capacity, binary choice?                     │
│   → 0/1 KNAPSACK (traverse right-to-left)           │
├──────────────────────────────────────────────────────┤
│ Items + capacity, unlimited use?                     │
│   → UNBOUNDED KNAPSACK (traverse left-to-right)     │
├──────────────────────────────────────────────────────┤
│ Optimal split of range [i,j]?                        │
│   → INTERVAL DP (fill by increasing length)          │
├──────────────────────────────────────────────────────┤
│ Tree structure, bottom-up?                           │
│   → TREE DP (postorder, return array)                │
├──────────────────────────────────────────────────────┤
│ 2D grid, move right/down?                            │
│   → GRID DP                                         │
├──────────────────────────────────────────────────────┤
│ n ≤ 20, track subset of elements?                   │
│   → BITMASK DP                                      │
├──────────────────────────────────────────────────────┤
│ Multiple discrete states (hold/sold/cooldown)?       │
│   → STATE MACHINE DP                                │
└──────────────────────────────────────────────────────┘
```

## ⚡ Top-Down Template

```java
int[] memo = new int[MAXN];
Arrays.fill(memo, -1);

int solve(int i, /* other params */) {
    if (baseCase) return baseValue;
    if (memo[i] != -1) return memo[i];
    memo[i] = /* recurrence using solve(i-1, ...) */;
    return memo[i];
}
```

## ⚡ Bottom-Up Template

```java
int[] dp = new int[n + 1];
// Initialize base cases
dp[0] = baseValue;
// Fill in order
for (int i = 1; i <= n; i++) {
    dp[i] = /* recurrence using dp[i-1], dp[i-2], ... */;
}
return dp[n];
```

## ⚡ 0/1 Knapsack Template (right-to-left)

```java
int[] dp = new int[W + 1];
for (int item : items) {
    for (int w = W; w >= item.weight; w--) {  // RIGHT TO LEFT
        dp[w] = Math.max(dp[w], dp[w - item.weight] + item.value);
    }
}
```

## ⚡ Unbounded Knapsack Template (left-to-right)

```java
int[] dp = new int[target + 1];
dp[0] = 1; // or 0, depending on problem
for (int item : items) {
    for (int w = item; w <= target; w++) {  // LEFT TO RIGHT
        dp[w] = Math.max(dp[w], dp[w - item] + value); // or += for counting
    }
}
```

## ⚡ LCS Template

```java
int[][] dp = new int[m + 1][n + 1];
for (int i = 1; i <= m; i++) {
    for (int j = 1; j <= n; j++) {
        if (s1.charAt(i-1) == s2.charAt(j-1)) dp[i][j] = dp[i-1][j-1] + 1;
        else dp[i][j] = Math.max(dp[i-1][j], dp[i][j-1]);
    }
}
```

## ⚡ State Machine DP (Stock Problems)

```java
// Stock with cooldown (LC #309)
int hold = -prices[0], sold = 0, rest = 0;
for (int i = 1; i < prices.length; i++) {
    int prevHold = hold, prevSold = sold, prevRest = rest;
    hold = Math.max(prevHold, prevRest - prices[i]);
    sold = prevHold + prices[i];
    rest = Math.max(prevRest, prevSold);
}
return Math.max(sold, rest);
```

## ⚡ The 5 Questions to Ask Any DP Problem

```
1. What is dp[i] / dp[i][j]?
   → Define clearly in English before writing code

2. What is the recurrence?
   → How does dp[i] depend on smaller values?

3. What are the base cases?
   → dp[0], dp[1], dp[0][j], dp[i][0] = ?

4. What order to fill?
   → Left to right? Bottom-up? By length?

5. Where is the answer?
   → dp[n]? max(dp[i])? dp[m][n]?
```

> **The Golden Rules of DP:**
>
> 1. **Define the state precisely.** "dp[i] = ?" must be a complete English sentence before you write code.
>
> 2. **Trust the recurrence.** Once your state and recurrence are correct, the code writes itself.
>
> 3. **Base cases matter.** Off-by-one in base cases breaks everything. Use 1-indexed dp arrays (dp[0] = base) to avoid confusion.
>
> 4. **0/1 Knapsack → right-to-left. Unbounded → left-to-right.** This is the one trick that fixes most knapsack bugs.
>
> 5. **When in doubt, try top-down first.** It's easier to think about and debug. Then optimize to bottom-up.
>
> 6. **"Optimize" means identify which previous values you need.** If dp[i] only needs dp[i-1] and dp[i-2], you only need 2 variables — not an array.
