# Part 1 — Foundations

# 1. What is Cyclic Sort?

## The Problem It Solves

```
SETTING: Array of n numbers where values are in range [1, n] or [0, n-1].

OBSERVATION: In a perfectly sorted array:
  value 1 belongs at index 0
  value 2 belongs at index 1
  value 3 belongs at index 2
  ...
  value i belongs at index i-1

If the array is [3, 1, 5, 4, 2]:
  3 belongs at index 2 (not index 0)
  1 belongs at index 0 ✓ (already correct)
  5 belongs at index 4 (not index 2)
  4 belongs at index 3 ✓ (already correct — wait, it's at index 3 ✓)
  2 belongs at index 1 (not index 4)

After placing each number at its correct index:
  [1, 2, 3, 4, 5]

Now ANY deviation from "value i at index i-1" reveals:
  → A missing number (index has wrong value)
  → A duplicate number (two indices have same value)
```

## The Big Picture

```
CYCLIC SORT places each number directly at its correct position.
After sorting, scan the array to find anomalies.

UNLIKE regular sorting (O(n log n)), cyclic sort does this in O(n) time
and O(1) extra space — because it only uses values already in the array.

NAME ORIGIN: "Cyclic" because placing one element often displaces another,
creating a cycle of swaps until each element reaches its home.

Example:
[3, 1, 5, 4, 2]

Place 3 at index 2: swap arr[0] and arr[2] → [5, 1, 3, 4, 2]
Place 5 at index 4: swap arr[0] and arr[4] → [2, 1, 3, 4, 5]
Place 2 at index 1: swap arr[0] and arr[1] → [1, 2, 3, 4, 5]
Place 1 at index 0: ✓ already correct, move to next
...

Result: [1, 2, 3, 4, 5] ✓
```

# 2. The Core Insight — Why It Works

## The Correctness Proof (Simple)

```
CLAIM: After cyclic sort, for array with values in [1, n]:
  arr[i] == i+1  for all i where no duplicate/missing exists.

PROOF:
  We process each index i. At each step:
    - If arr[i] is already at its correct position → move on.
    - Else → swap arr[i] to its correct position arr[arr[i]-1].
  
  After swap, either:
    a) arr[i] is now correct (the element we swapped in belongs here), OR
    b) arr[i] is a duplicate of what's now at arr[arr[i]-1]
       → we can't make progress by swapping, so move on.
  
  Every swap places AT LEAST ONE element in its correct position.
  There are n elements and at most n swaps total → O(n).

WHY O(n) AND NOT O(n²)?
  Each element is swapped AT MOST ONCE into its correct position.
  Once placed correctly, it stays there.
  So total swaps ≤ n, total work = O(n).

Key question at each step:
  "Is arr[i] at its correct position?"
  → For range [1,n]: correct position of value v is index v-1.
     Check: arr[i] == i+1? If not: arr[arr[i]-1] == arr[i]? (already placed = duplicate)
  → For range [0,n-1]: correct position of value v is index v.
     Check: arr[i] == i? If not: arr[arr[i]] == arr[i]? (duplicate)
```

## Visual Walkthrough

```
Array: [4, 3, 1, 2, 5], range [1, 5]
Correct positions: value v → index v-1

i=0: arr[0]=4, correct pos=3, arr[3]=2. 4≠2 → swap(0,3)
     [2, 3, 1, 4, 5]
i=0: arr[0]=2, correct pos=1, arr[1]=3. 2≠3 → swap(0,1)
     [3, 2, 1, 4, 5]
i=0: arr[0]=3, correct pos=2, arr[2]=1. 3≠1 → swap(0,2)
     [1, 2, 3, 4, 5]
i=0: arr[0]=1, correct pos=0. 1==0+1 ✓ → move to i=1
i=1: arr[1]=2, correct pos=1. 2==1+1 ✓ → move to i=2
i=2: arr[2]=3, correct pos=2. 3==2+1 ✓ → move to i=3
i=3: arr[3]=4, correct pos=3. 4==3+1 ✓ → move to i=4
i=4: arr[4]=5, correct pos=4. 5==4+1 ✓ → done

Result: [1, 2, 3, 4, 5] ✓ Sorted perfectly!
```

# 3. When to Use Cyclic Sort

```
USE CYCLIC SORT WHEN ALL OF THESE ARE TRUE:

✅ Array contains integers in a KNOWN RANGE
   → [1, n], [0, n-1], [1, n+1], etc.

✅ Array length n is related to the range
   → "array of n numbers" where values should be in [1,n]

✅ Problem asks about MISSING or DUPLICATE numbers

✅ O(n) time and O(1) space is required or preferred

TRIGGER KEYWORDS:
  "find the missing number"
  "find all missing numbers"
  "find the duplicate"
  "find all duplicates"
  "numbers in range [1,n]"
  "array of size n containing numbers 1 to n"
  "first missing positive"

DO NOT USE CYCLIC SORT WHEN:
  ❌ Values are not in a predictable range (e.g., could be any integer)
  ❌ Values can be negative (unless you handle separately)
  ❌ Array has values much larger than n (use hashing instead)
  ❌ Array is read-only (can't swap in place)

ALTERNATIVE APPROACHES (and when cyclic sort beats them):
  Sorting:    O(n log n) — cyclic sort is O(n)
  HashSet:    O(n) time but O(n) space — cyclic sort uses O(1)
  XOR tricks: O(n), O(1) but only works for single missing/duplicate
  Cyclic sort: O(n) time, O(1) space — the sweet spot for this problem class
```

# 4. The Algorithm Step by Step

## For Range [1, n] (Most Common)

```
SETUP:
  nums[i] should be at index nums[i] - 1
  (value 1 → index 0, value 2 → index 1, etc.)

ALGORITHM:
  i = 0
  while i < n:
    j = nums[i] - 1        ← correct index for nums[i]
    if nums[i] != nums[j]: ← not at correct position AND not a duplicate
      swap(nums[i], nums[j])
      (DON'T increment i — the new nums[i] might also need to move)
    else:
      i++                  ← move to next (either correct or duplicate)

WHY check nums[i] != nums[j] instead of nums[i] != i+1?
  If nums[i] == nums[j]: nums[i] is already at its correct position,
  OR it's a duplicate of what's at index j.
  Either way, swapping would be useless (and infinite loop!).
  So we just increment i.

AFTER SORTING: scan for anomalies
  For each index i:
    if nums[i] != i+1: → i+1 is missing, nums[i] is duplicate (or vice versa)
```

## For Range [0, n-1]

```
SETUP:
  nums[i] should be at index nums[i]
  (value 0 → index 0, value 1 → index 1, etc.)

ALGORITHM:
  i = 0
  while i < n:
    j = nums[i]            ← correct index for nums[i]
    if nums[i] != i:       ← not at correct position
      if nums[i] != nums[j]: swap(nums[i], nums[j])
      else: i++            ← duplicate, can't progress
    else:
      i++
```

# 5. Cyclic Sort Template (Memorize This)

```java
// ============================================================
// TEMPLATE 1: Range [1, n] — most common
// ============================================================
void cyclicSort(int[] nums) {
    int i = 0;
    while (i < nums.length) {
        int j = nums[i] - 1;           // Correct index for nums[i]
        if (nums[i] != nums[j]) {      // Not at correct position AND not duplicate
            swap(nums, i, j);           // Place nums[i] at its correct position
            // DON'T increment i — check new value at i
        } else {
            i++;                        // Already correct OR duplicate — move on
        }
    }
}

// ============================================================
// TEMPLATE 2: Range [0, n-1]
// ============================================================
void cyclicSort0(int[] nums) {
    int i = 0;
    while (i < nums.length) {
        if (nums[i] != i && nums[i] != nums[nums[i]]) {
            swap(nums, i, nums[i]);
        } else {
            i++;
        }
    }
}

// ============================================================
// SWAP HELPER
// ============================================================
void swap(int[] arr, int i, int j) {
    int temp = arr[i];
    arr[i] = arr[j];
    arr[j] = temp;
}

// ============================================================
// POST-SORT SCAN PATTERNS
// ============================================================

// Find ONE missing number (range [1,n], one number missing):
for (int i = 0; i < nums.length; i++) {
    if (nums[i] != i + 1) return i + 1;  // i+1 is missing
}
return nums.length + 1;  // n+1 is missing

// Find ALL missing numbers:
List<Integer> missing = new ArrayList<>();
for (int i = 0; i < nums.length; i++) {
    if (nums[i] != i + 1) missing.add(i + 1);
}

// Find ONE duplicate:
for (int i = 0; i < nums.length; i++) {
    if (nums[i] != i + 1) return nums[i];  // nums[i] appeared twice
}

// Find ALL duplicates:
List<Integer> duplicates = new ArrayList<>();
for (int i = 0; i < nums.length; i++) {
    if (nums[i] != i + 1) duplicates.add(nums[i]);
}
```

# Part 2 — The 6 Problem Types

# 6. Type 1 — Find the Missing Number

## Problem Setup

```
Array of n numbers, values in [0, n].
Exactly ONE number from 0 to n is missing.
Find it.

Example: [3, 0, 1] → missing 2
Example: [9,6,4,2,3,5,7,0,1] → missing 8

RANGE IS [0, n], not [1, n] → value v belongs at index v.
```

---

## 🟢 LC #268 — Missing Number

```java
public int missingNumber(int[] nums) {
    int n = nums.length;
    int i = 0;

    // Cyclic sort: place each number at its correct index
    // Value v should be at index v (range [0, n])
    // Skip n because it can't fit in indices 0..n-1
    while (i < n) {
        int j = nums[i];  // Correct index for nums[i]
        if (nums[i] < n && nums[i] != nums[j]) {
            swap(nums, i, j);
        } else {
            i++;
        }
    }

    // Scan: find index where value doesn't match index
    for (int k = 0; k < n; k++) {
        if (nums[k] != k) return k;
    }
    return n;  // n itself is missing (all indices 0..n-1 correct)
}

void swap(int[] arr, int i, int j) {
    int t = arr[i]; arr[i] = arr[j]; arr[j] = t;
}
```

**Dry run:**
```
nums = [3, 0, 1], n = 3

i=0: nums[0]=3. j=3. 3 < n? 3 < 3? NO → i++
i=1: nums[1]=0. j=0. 0 < 3 AND nums[1]=0 != nums[0]=3 → swap(1,0)
     [0, 3, 1]
i=1: nums[1]=3. j=3. 3 < 3? NO → i++
i=2: nums[2]=1. j=1. 1 < 3 AND nums[2]=1 != nums[1]=3 → swap(2,1)
     [0, 1, 3]
i=2: nums[2]=3. j=3. 3 < 3? NO → i++
i=3: done

Scan: nums[0]=0 ✓, nums[1]=1 ✓, nums[2]=3≠2 → return 2 ✓
```

**Alternative O(1) space solution (no cyclic sort needed):**
```java
public int missingNumber(int[] nums) {
    int n = nums.length;
    int expected = n * (n + 1) / 2;  // Sum of 0 to n
    int actual = 0;
    for (int num : nums) actual += num;
    return expected - actual;
}
```

**Which to use?** The sum formula is simpler for this specific problem. But cyclic sort generalizes better to harder problems. Know both.

# 7. Type 2 — Find All Missing Numbers

## Problem Setup

```
Array of n numbers, values in [1, n].
MULTIPLE numbers may be missing (some numbers appear twice instead).
Find ALL missing numbers.

Example: [4,3,2,7,8,2,3,1] → missing [5, 6]
  After sort: [1,2,3,3,4,7,8,2]  (positions 4,5 wrong)
  Wait — let's trace properly...
  After cyclic sort: [1,2,3,4,3,2,7,8]
  Index 4: nums[4]=3 ≠ 5 → 5 is missing
  Index 5: nums[5]=2 ≠ 6 → 6 is missing
```

---

## 🟡 LC #448 — Find All Numbers Disappeared in an Array

```java
public List<Integer> findDisappearedNumbers(int[] nums) {
    int n = nums.length;

    // Step 1: Cyclic sort — place each number at index nums[i]-1
    int i = 0;
    while (i < n) {
        int j = nums[i] - 1;           // Correct index for nums[i]
        if (nums[i] != nums[j]) {      // Not at correct position AND not duplicate
            swap(nums, i, j);
        } else {
            i++;
        }
    }

    // Step 2: Collect indices where nums[i] != i+1 (those are missing numbers)
    List<Integer> missing = new ArrayList<>();
    for (int k = 0; k < n; k++) {
        if (nums[k] != k + 1) {
            missing.add(k + 1);        // k+1 is the missing number
        }
    }
    return missing;
}

void swap(int[] arr, int i, int j) {
    int t = arr[i]; arr[i] = arr[j]; arr[j] = t;
}
```

# 8. Type 3 — Find the Duplicate Number

## Problem Setup

```
Array of n+1 numbers, values in [1, n].
Exactly ONE number appears TWICE.
Find it. Must use O(1) extra space.

Example: [1, 3, 4, 2, 2] → duplicate is 2
Example: [3, 1, 3, 4, 2] → duplicate is 3

KEY INSIGHT: After cyclic sort, the index where nums[i] ≠ i+1
reveals the duplicate (nums[i] is the duplicate because its correct
position is already occupied by an identical value).
```

---

## 🟡 LC #287 — Find the Duplicate Number

```java
public int findDuplicate(int[] nums) {
    int n = nums.length;  // n+1 elements, values in [1,n]

    // Step 1: Cyclic sort
    int i = 0;
    while (i < n) {
        int j = nums[i] - 1;
        if (nums[i] != nums[j]) {
            swap(nums, i, j);
        } else {
            i++;
        }
    }

    // Step 2: Find index where value doesn't match
    // That value is the duplicate (its correct index is taken by itself)
    for (int k = 0; k < n; k++) {
        if (nums[k] != k + 1) {
            return nums[k];  // This value appeared twice
        }
    }

    return -1;
}

void swap(int[] arr, int i, int j) {
    int t = arr[i]; arr[i] = arr[j]; arr[j] = t;
}
```

**Dry run:**
```
nums = [1, 3, 4, 2, 2], n = 5

i=0: nums[0]=1, j=0. nums[0]==nums[0] ✓ i++
i=1: nums[1]=3, j=2. nums[1]≠nums[2]: swap(1,2) → [1, 4, 3, 2, 2]
i=1: nums[1]=4, j=3. nums[1]≠nums[3]: swap(1,3) → [1, 2, 3, 4, 2]
i=1: nums[1]=2, j=1. nums[1]==nums[1] ✓ i++
i=2: nums[2]=3, j=2. ✓ i++
i=3: nums[3]=4, j=3. ✓ i++
i=4: nums[4]=2, j=1. nums[4]==nums[1]=2 → DUPLICATE! i++

Final: [1, 2, 3, 4, 2]

Scan: k=4: nums[4]=2 ≠ 5 → return 2 ✓
```

**Note:** LC #287 has additional constraints (don't modify array, O(1) space). If those apply, use Floyd's cycle detection instead. But if modification is allowed, cyclic sort is the cleanest.

# 9. Type 4 — Find All Duplicates

## Problem Setup

```
Array of n numbers, values in [1, n].
Some numbers appear TWICE, others appear once.
Find ALL duplicates.

Example: [4,3,2,7,8,2,3,1] → duplicates are [2, 3]

After cyclic sort: [1, 2, 3, 4, 3, 2, 7, 8]
Scan: index 4: nums[4]=3 ≠ 5 → 3 is duplicate
      index 5: nums[5]=2 ≠ 6 → 2 is duplicate
```

---

## 🟡 LC #442 — Find All Duplicates in an Array

```java
public List<Integer> findDuplicates(int[] nums) {
    int n = nums.length;

    // Step 1: Cyclic sort
    int i = 0;
    while (i < n) {
        int j = nums[i] - 1;
        if (nums[i] != nums[j]) {
            swap(nums, i, j);
        } else {
            i++;
        }
    }

    // Step 2: Collect nums[i] where nums[i] ≠ i+1
    // These are the duplicates (their correct position is occupied by themselves)
    List<Integer> duplicates = new ArrayList<>();
    for (int k = 0; k < n; k++) {
        if (nums[k] != k + 1) {
            duplicates.add(nums[k]);
        }
    }
    return duplicates;
}
```

**Comparing Types 2 and 4:**
```
Same cyclic sort step!
Different scan step:
  Type 2 (missing):   add k+1    to result (the index's expected value)
  Type 4 (duplicate): add nums[k] to result (the value sitting at wrong index)
  
Type 5 (missing + duplicate together): collect both!
```

```java
public int[] findErrorNums(int[] nums) {
```

# 11. Type 6 — First Missing Positive

## Problem Setup

```
Array of ARBITRARY integers (can be negative, zero, > n).
Find the SMALLEST MISSING POSITIVE integer.

Example: [3, 4, -1, 1] → answer is 2
Example: [1, 2, 0]     → answer is 3
Example: [7, 8, 9, 11] → answer is 1

KEY INSIGHT: The answer is always in [1, n+1].
  (If all of 1..n are present, answer is n+1. Otherwise, answer ≤ n.)
  
So we only care about values in [1, n].
Ignore values ≤ 0 and values > n.
Apply cyclic sort only to valid values.
```

---

## 🔴 LC #41 — First Missing Positive

```java
public int firstMissingPositive(int[] nums) {
    int n = nums.length;

    // Step 1: Cyclic sort — only move valid values (1 to n)
    int i = 0;
    while (i < n) {
        int j = nums[i] - 1;           // Correct index for nums[i]
        // Only process if nums[i] is in [1, n] AND not already at correct spot
        if (nums[i] > 0 && nums[i] <= n && nums[i] != nums[j]) {
            swap(nums, i, j);
        } else {
            i++;
        }
    }

    // Step 2: Find first index where nums[i] != i+1
    for (int k = 0; k < n; k++) {
        if (nums[k] != k + 1) {
            return k + 1;              // k+1 is missing
        }
    }
    return n + 1;                      // All of 1..n present, answer is n+1
}
```

```
i=2: 0 ≤ 0, invalid → i++

Scan: k=0: 1==1 ✓, k=1: 2==2 ✓, k=2: 0≠3 → return 3 ✓
```

# Part 3 — Variations and Edge Cases

# 12. Array with Range [1, n] vs [0, n]

```
RANGE [1, n]: value v belongs at index v-1
  Correct position check: nums[i] == i + 1
  Swap target: j = nums[i] - 1
  Applicable to: LC #442, #448, #645

RANGE [0, n-1]: value v belongs at index v
  Correct position check: nums[i] == i
  Swap target: j = nums[i]
  Watch out: j can equal i when value equals index!
  Applicable to: LC #268 (mixed: values in [0,n], array size n)

RANGE [0, n] with one missing (LC #268):
  n+1 values (0 to n), but only n slots.
  Handle n specially: it can't fit anywhere, so skip it.
  if nums[i] < n: try to place, else skip.
```

```java
// Template for range [0, n-1]
void cyclicSortZeroIndexed(int[] nums) {
    int i = 0;
    while (i < nums.length) {
        if (nums[i] != i && nums[nums[i]] != nums[i]) {
            swap(nums, i, nums[i]);
        } else {
            i++;
        }
    }
}

// Template for range [1, n] with values possibly outside [1, n]
void cyclicSortWithFilter(int[] nums, int n) {
    int i = 0;
    while (i < n) {
        int j = nums[i] - 1;
        if (nums[i] >= 1 && nums[i] <= n && nums[i] != nums[j]) {
            swap(nums, i, j);
        } else {
            i++;
        }
    }
}
```

# 13. Multiple Missing Numbers

```
When multiple numbers are missing, cyclic sort handles it seamlessly.
After sorting, every index with nums[i] != i+1 indicates a missing number.

Example: [2, 3, 4, 7, 7, 7] → n=6, values in [1,6]
  After sort: [7, 2, 3, 4, 7, 7]  (indices 0, 4, 5 are wrong)
  Missing: 1, 5, 6
  Duplicates: 7 (appears 3 times at wrong positions)

EDGE CASE: What if the same number appears many times?
  Cyclic sort handles it — once a value is placed at its correct index,
  all duplicates at other indices become "stuck" and get skipped.
```

# 14. Multiple Duplicates

```
Cyclic sort handles multiple duplicates cleanly.
After sort: each value v that appears k times will be:
  - At index v-1 once (correct position)
  - At k-1 other wrong indices

Example: [2,2,2,3] → n=4, one 1 missing, 2 appears 3 times
  After sort: [2, 2, 2, 3]
  Index 0: nums[0]=2 ≠ 1 → missing 1, duplicate 2
  Index 1: nums[1]=2 ≠ 2? Wait, 2==2 ✓ (it's at correct spot)
  Actually: [2, 2, 2, 3] — index 0 has 2 (wrong), index 1 has 2 (correct)

The key: after cyclic sort, index i with nums[i] != i+1 means:
  value i+1 is missing (possibly appearing 0 times)
  nums[i] is a duplicate (appears more than once)
```

# 15. Arrays with Values Outside [1, n]

```
When array contains values < 1 or > n (like in LC #41):
  → Simply SKIP them during cyclic sort.
  → They're irrelevant (first missing positive is always in [1, n+1]).
  → After sort, they sit in wrong positions, correctly indicating missing values.

IMPORTANT: The condition nums[i] >= 1 && nums[i] <= n guards against:
  1. Negative values / zero (can't be placed in [0,n-1])
  2. Values > n (would access out-of-bounds index)
  3. Values that are already correct (avoid infinite loop)

Example with negatives and large values:
  [100, -1, 2, 1] → answer = 3

  i=0: 100 > 4, skip → i++
  i=1: -1 < 1, skip → i++
  i=2: 2∈[1,4], j=1. nums[2]≠nums[1] → swap → [100, 2, -1, 1]
  i=2: -1 < 1, skip → i++
  i=3: 1∈[1,4], j=0. nums[3]≠nums[0] → swap → [1, 2, -1, 100]
  i=3: 100 > 4, skip → i++

  Final: [1, 2, -1, 100]
  Scan: k=2: -1 ≠ 3 → return 3 ✓
```

## 🟢 LC #268 — Missing Number

```java
// Array has n distinct numbers from [0, n]. Find the missing one.
public int missingNumber(int[] nums) {
    int n = nums.length;
    int i = 0;
    while (i < n) {
        int j = nums[i];                      // Value v goes to index v (range [0,n])
        if (nums[i] < n && nums[i] != i) {   // Skip n (no slot for it) and already-correct
            swap(nums, i, j);
        } else {
            i++;
        }
    }
    for (int k = 0; k < n; k++) {
        if (nums[k] != k) return k;
    }
    return n;
}
void swap(int[] a, int i, int j) { int t=a[i]; a[i]=a[j]; a[j]=t; }
```

## 🟡 LC #287 — Find the Duplicate Number

```java
// n+1 numbers, values in [1,n]. One number appears twice. Find it.
// Constraints: don't modify array, O(1) space → use Floyd's cycle detection.
// If modification allowed → use cyclic sort (shown below).

// APPROACH A: Cyclic sort (if modification allowed)
public int findDuplicateCyclicSort(int[] nums) {
    int i = 0;
    while (i < nums.length) {
        int j = nums[i] - 1;
        if (nums[i] != nums[j]) swap(nums, i, j);
        else i++;
    }
    for (int k = 0; k < nums.length; k++)
        if (nums[k] != k + 1) return nums[k];
    return -1;
}

// APPROACH B: Floyd's Cycle Detection (no modification, O(1) space)
// Treat array as linked list: index i points to nums[i]
// Duplicate value = entry point of cycle
public int findDuplicate(int[] nums) {
    int slow = nums[0], fast = nums[0];
    // Phase 1: detect cycle
    do {
        slow = nums[slow];
        fast = nums[nums[fast]];
    } while (slow != fast);
    // Phase 2: find cycle entry (= duplicate)
    fast = nums[0];
    while (slow != fast) {
        slow = nums[slow];
        fast = nums[fast];
    }
    return slow;
}

// APPROACH C: Binary search on value (O(n log n), no modification)
public int findDuplicateBinarySearch(int[] nums) {
    int lo = 1, hi = nums.length - 1;
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        int count = 0;
        for (int n : nums) if (n <= mid) count++;
        // By pigeonhole: if count > mid, duplicate is in [1, mid]
        if (count > mid) hi = mid;
        else lo = mid + 1;
    }
    return lo;
}
```

## Bonus — LC #442 Variant: Without Modifying Array

```java
// If array can't be modified, use negation trick:
// For each value v in [1,n], negate nums[v-1] to mark v as seen.
// After marking, negative indices indicate present values.
public List<Integer> findDuplicatesNoModify(int[] nums) {
    List<Integer> result = new ArrayList<>();
    for (int i = 0; i < nums.length; i++) {
        int idx = Math.abs(nums[i]) - 1;
        if (nums[idx] < 0) result.add(idx + 1);  // Already negated = duplicate
        else nums[idx] = -nums[idx];               // Negate to mark as seen
    }
    return result;
}
```

# 19. Cyclic Sort Interview Cheat Sheet

## ⚡ The Single Master Template

```java
// CYCLIC SORT — handles ALL 6 types with minor variations
public void cyclicSort(int[] nums) {
    int i = 0;
    while (i < nums.length) {
        int j = nums[i] - 1;  // Target index for nums[i]

        // Condition varies by problem:
        // Standard [1,n]:  nums[i] != nums[j]
        // With filter:     nums[i] >= 1 && nums[i] <= n && nums[i] != nums[j]
        // Range [0,n-1]:   nums[i] != i && nums[i] != nums[nums[i]]

        if (/* condition */) {
            swap(nums, i, j);
            // DO NOT increment i — new value at i might need moving too
        } else {
            i++;  // Either correct position OR duplicate — move on
        }
    }
}

void swap(int[] a, int i, int j) { int t=a[i]; a[i]=a[j]; a[j]=t; }
```

## ⚡ Condition Reference Table

```
Problem Type          | Swap Condition                          | Post-Sort Return
─────────────────────────────────────────────────────────────────────────────────
Range [1,n], no filter | nums[i] != nums[j]                     | varies
Range [1,n], filter    | nums[i]≥1 && nums[i]≤n && nums[i]!=nums[j] | varies
Range [0,n-1]          | nums[i]!=i && nums[nums[i]]!=nums[i]   | varies

Find ONE missing [1,n] | nums[i] != nums[j]                     | first i: nums[i]≠i+1 → i+1
Find ALL missing       | nums[i] != nums[j]                     | all i: nums[i]≠i+1 → i+1
Find ONE duplicate     | nums[i] != nums[j]                     | first i: nums[i]≠i+1 → nums[i]
Find ALL duplicates    | nums[i] != nums[j]                     | all i: nums[i]≠i+1 → nums[i]
Missing + Duplicate    | nums[i] != nums[j]                     | i: nums[i]≠i+1 → [nums[i], i+1]
First Missing Positive | nums[i]≥1 && nums[i]≤n && nums[i]!=nums[j] | first i: nums[i]≠i+1 → i+1
```

## ⚡ Complexity Analysis

```
TIME COMPLEXITY: O(n)
  - Outer while loop runs n iterations (i goes from 0 to n-1).
  - Each swap places at least one element at its correct position.
  - An element is swapped into its correct position at most ONCE.
  - Total swaps ≤ n, total work = O(n).
  
  NOTE: The inner "re-check same i" does NOT make it O(n²).
  Although we don't increment i after a swap, the total number of swaps
  across the ENTIRE algorithm is still ≤ n. Amortized O(1) per index.

SPACE COMPLEXITY: O(1)
  - Sorting is done in-place with only a constant extra variables.
  - Output lists are not counted as extra space.

COMPARE TO ALTERNATIVES:
  Approach          | Time      | Space    | Notes
  ─────────────────────────────────────────────────────────
  Sorting           | O(n logn) | O(1)     | Modifies array
  HashSet           | O(n)      | O(n)     | Doesn't modify
  XOR               | O(n)      | O(1)     | Only for SINGLE missing
  Sum formula       | O(n)      | O(1)     | Only for SINGLE missing
  Negation trick    | O(n)      | O(1)     | Modifies array, brittle
  Cyclic sort       | O(n)      | O(1)     | BEST for this problem class
```

## ⚡ The Most Important Things to Remember

```
1. THE KEY CONDITION:
   Swap when: nums[i] != nums[j]   (where j = nums[i] - 1)
   NOT when:  nums[i] != i + 1
   
   WHY: If nums[i] == nums[j], it means either:
     a) nums[i] is already at correct position (nums[i] == i+1), OR
     b) nums[i] is a duplicate of what's at position j.
   In either case, swapping is useless (would be an infinite loop for case b).

2. DON'T INCREMENT AFTER SWAP:
   After swapping, the new value at index i might ALSO need to be moved.
   Only increment when the current position is "done" (correct or stuck duplicate).

3. POST-SORT SCAN LOGIC:
   Missing → return i+1  (the expected value that's absent)
   Duplicate → return nums[i]  (the value sitting at the wrong index)

4. FIRST MISSING POSITIVE GUARD:
   Must check: nums[i] >= 1 && nums[i] <= n
   Without this: negative values and large values cause index out-of-bounds.

5. RANGE MATTERS:
   [1, n] → j = nums[i] - 1  (subtract 1 to convert to 0-based index)
   [0, n-1] → j = nums[i]    (no subtraction needed)
   [0, n] with missing → handle n separately (no slot for it)
```

## ⚡ Common Mistakes and Fixes

```java
// ❌ MISTAKE 1: Incrementing i unconditionally
while (i < n) {
    int j = nums[i] - 1;
    swap(nums, i, j);
    i++;  // WRONG! After swap, new nums[i] might need to move too.
}

// ✅ FIX: Only increment when no swap needed
while (i < n) {
    int j = nums[i] - 1;
    if (nums[i] != nums[j]) swap(nums, i, j);
    else i++;
}


// ❌ MISTAKE 2: Wrong swap condition (causes infinite loop on duplicates)
while (i < n) {
    int j = nums[i] - 1;
    if (nums[i] != i + 1) swap(nums, i, j);  // WRONG for duplicates!
    else i++;
    // If nums[i] = nums[j] = 3 (duplicate), this loops forever!
}

// ✅ FIX: Check nums[i] != nums[j], not nums[i] != i+1
if (nums[i] != nums[j]) swap(nums, i, j);


// ❌ MISTAKE 3: Forgetting bounds check for LC #41
while (i < n) {
    int j = nums[i] - 1;
    if (nums[i] != nums[j]) swap(nums, i, j);  // WRONG if nums[i] ≤ 0 or > n
    // j could be negative or >= n → ArrayIndexOutOfBoundsException!
    else i++;
}

// ✅ FIX: Guard with range check
if (nums[i] > 0 && nums[i] <= n && nums[i] != nums[j]) swap(nums, i, j);


// ❌ MISTAKE 4: Wrong scan for missing vs duplicate
// For missing numbers: add i+1 (the EXPECTED value)
// For duplicates:      add nums[i] (the ACTUAL value at wrong position)
for (int k = 0; k < n; k++) {
    if (nums[k] != k + 1) {
        missing.add(nums[k]);   // ❌ WRONG — this adds the duplicate, not missing
        duplicate.add(k + 1);   // ❌ WRONG — this adds the missing, not duplicate
    }
}

// ✅ FIX:
for (int k = 0; k < n; k++) {
    if (nums[k] != k + 1) {
        missing.add(k + 1);    // ✅ k+1 is the value that SHOULD be here
        duplicate.add(nums[k]); // ✅ nums[k] is the value ACTUALLY here (the duplicate)
    }
}
```

```
     This is exactly how LC #41 (First Missing Positive) works."
```
