# 1. What is a Queue?

## The Simple Explanation

A **Queue** is a linear data structure that follows the **FIFO** principle:

```
FIFO = First In, First Out
```

Imagine people standing in a line (queue) at a ticket counter:
- The person who arrived **FIRST** gets the ticket **FIRST**
- New people join from the **BACK** (rear)
- People leave from the **FRONT**

```
FRONT                                    REAR
  │                                        │
  ▼                                        ▼
┌─────┬─────┬─────┬─────┬─────┐
│  1  │  2  │  3  │  4  │  5  │
└─────┴─────┴─────┴─────┴─────┘
  │                                        │
dequeue()                            enqueue(6)
removes 1                            adds 6 at back
```

## The Four Core Operations

| Operation | Also Called | What it Does | End |
|-----------|-------------|--------------|-----|
| `enqueue(x)` | `offer(x)`, `add(x)` | Add element to rear | Rear |
| `dequeue()` | `poll()`, `remove()` | Remove element from front | Front |
| `peek()` | `front()` | View front without removing | Front |
| `isEmpty()` | — | Check if queue is empty | — |

# 2. Queue vs Stack — The Core Difference

This is the **most important distinction** to understand. Interviewers test this.

```
┌──────────────────────────────────────────────────────────────┐
│                         STACK                                │
│                       (LIFO)                                 │
│                                                              │
│  push(1) push(2) push(3)                                     │
│  ─────────────────────►                                      │
│                   ┌───┐                                      │
│                   │ 3 │  ← TOP (pop removes this first)      │
│                   ├───┤                                      │
│                   │ 2 │                                      │
│                   ├───┤                                      │
│                   │ 1 │  ← BOTTOM                            │
│                   └───┘                                      │
│  pop() → 3, then 2, then 1  (reverse of insertion order)    │
└──────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────┐
│                         QUEUE                                │
│                       (FIFO)                                 │
│                                                              │
│  enqueue(1) enqueue(2) enqueue(3)                            │
│                                                              │
│  FRONT                              REAR                     │
│  ┌─────┬─────┬─────┐                                        │
│  │  1  │  2  │  3  │                                        │
│  └─────┴─────┴─────┘                                        │
│  dequeue() → 1, then 2, then 3  (same as insertion order)   │
└──────────────────────────────────────────────────────────────┘
```

| Feature | Stack | Queue |
|---------|-------|-------|
| Principle | LIFO (Last In First Out) | FIFO (First In First Out) |
| Insert at | Top | Rear |
| Remove from | Top | Front |
| Real life | Stack of plates | Line at cinema |
| Algorithm use | DFS, Backtracking | BFS, Scheduling |
| Java class | `ArrayDeque` (as stack) | `ArrayDeque` (as queue) |

# 3. Real Life Examples of Queue

Understanding real-world examples makes interview explanations natural and clear.

## Example 1: Printer Queue
```
Print requests come in:  Doc1 → Doc2 → Doc3 → Doc4

Queue:  [Doc1, Doc2, Doc3, Doc4]
         FRONT                REAR

Printer processes:
  dequeue() → prints Doc1
  dequeue() → prints Doc2
  dequeue() → prints Doc3

New request arrives:
  enqueue(Doc5) → Queue: [Doc4, Doc5]

Prints in ORDER they were submitted — Fair! ✅
```

## Example 2: CPU Process Scheduling
```
Processes arrive:  P1(at t=0) → P2(at t=1) → P3(at t=2)

Ready Queue: [P1, P2, P3]

CPU Round-Robin:
  Execute P1 for 1 unit → dequeue P1
  Execute P2 for 1 unit → dequeue P2
  P1 not done? → enqueue P1 again

Queue: [P3, P1]  ← P3 waited patiently, now gets CPU
```

## Example 3: BFS Maze Solving
```
Start at S, find path to E:

┌───┬───┬───┬───┐
│ S │   │   │   │
├───┼───┼───┼───┤
│   │ █ │ █ │   │
├───┼───┼───┼───┤
│   │   │   │ E │
└───┴───┴───┴───┘

BFS explores level by level using Queue:
Level 0: [S]
Level 1: [right of S, below S]          ← neighbors of S
Level 2: [right-right, below-right, ...]  ← their neighbors
...continues until E found

This GUARANTEES shortest path — only possible with Queue (BFS)!
```

## Example 4: Customer Service Call Center
```
Customers call in:  C1 → C2 → C3 → C4 → C5

Call Queue: [C1, C2, C3, C4, C5]
             FRONT              REAR

Agent available → dequeue() → serves C1 (waited longest)
Another call: C6 → enqueue(C6) → [C2, C3, C4, C5, C6]
```

# 4. How Queue Works Internally

## Array-Based Queue Problem (Naive)

```
Initial array:  [_, _, _, _, _]   front=0, rear=-1

enqueue(A):  [A, _, _, _, _]   front=0, rear=0
enqueue(B):  [A, B, _, _, _]   front=0, rear=1
enqueue(C):  [A, B, C, _, _]   front=0, rear=2

dequeue():   [_, B, C, _, _]   front=1, rear=2  ← A removed
dequeue():   [_, _, C, _, _]   front=2, rear=2  ← B removed
enqueue(D):  [_, _, C, D, _]   front=2, rear=3
enqueue(E):  [_, _, C, D, E]   front=2, rear=4
enqueue(F):  ???  ARRAY FULL!  ← But positions 0,1 are empty! WASTED SPACE!
```

**The Problem:** Front keeps moving right, wasting left spaces.

## Solution: Circular Array

```
Use modulo arithmetic to wrap around:
  rear = (rear + 1) % capacity
  front = (front + 1) % capacity

Circular view:

    ┌─── 0 ───┐
  7 │         │ 1
    │         │
  6 │         │ 2
    │         │
  5 │         │ 3
    └─── 4 ───┘

enqueue fills slots around the circle.
When rear reaches end, it wraps to beginning!
```

## Linked List-Based Queue

```
front → [A|→] → [B|→] → [C|null]  ← rear

enqueue(D):
front → [A|→] → [B|→] → [C|→] → [D|null]  ← rear

dequeue():
front → [B|→] → [C|→] → [D|null]  ← rear   (A removed)
```

**Advantages of Linked List:** No wasted space, dynamic size, no wraparound complexity.
**Disadvantage:** Extra memory per node for pointer.

# 5. Implement Queue from Scratch

## 5.1 Queue Using Circular Array

```java
public class CircularArrayQueue {

    private int[] arr;
    private int front;     // Index of front element
    private int rear;      // Index of rear element
    private int size;      // Current number of elements
    private int capacity;  // Max capacity

    public CircularArrayQueue(int capacity) {
        this.capacity = capacity;
        arr = new int[capacity];
        front = 0;
        rear = -1;
        size = 0;
    }

    // ENQUEUE: Add element to the rear
    // Time: O(1)
    public void enqueue(int value) {
        if (isFull()) {
            throw new RuntimeException("Queue is Full! Cannot enqueue " + value);
        }
        // Circular increment: wrap around using modulo
        rear = (rear + 1) % capacity;
        arr[rear] = value;
        size++;
        System.out.println("Enqueued: " + value + " | front=" + front + " rear=" + rear);
    }

    // DEQUEUE: Remove and return front element
    // Time: O(1)
    public int dequeue() {
        if (isEmpty()) {
            throw new RuntimeException("Queue is Empty! Cannot dequeue.");
        }
        int value = arr[front];
        // Circular increment for front
        front = (front + 1) % capacity;
        size--;
        return value;
    }

    // PEEK: View front element without removing
    // Time: O(1)
    public int peek() {
        if (isEmpty()) {
            throw new RuntimeException("Queue is Empty!");
        }
        return arr[front];
    }

    public boolean isEmpty() { return size == 0; }
    public boolean isFull()  { return size == capacity; }
    public int size()        { return size; }

    // Display all elements from front to rear
    public void display() {
        if (isEmpty()) { System.out.println("Queue is empty"); return; }
        System.out.print("Queue (front → rear): ");
        for (int i = 0; i < size; i++) {
            System.out.print(arr[(front + i) % capacity] + " ");
        }
        System.out.println();
    }

    public static void main(String[] args) {
        CircularArrayQueue q = new CircularArrayQueue(5);

        q.enqueue(10);
        q.enqueue(20);
        q.enqueue(30);
        q.display();            // Queue (front → rear): 10 20 30

        System.out.println("Dequeue: " + q.dequeue()); // 10
        System.out.println("Peek: "    + q.peek());    // 20

        q.enqueue(40);
        q.enqueue(50);
        q.enqueue(60); // Wraps around! rear goes to index 0
        q.display();   // Queue (front → rear): 20 30 40 50 60

        System.out.println("Size: " + q.size()); // 5
    }
}
```

## 5.2 Queue Using Linked List

```java
public class LinkedListQueue {

    // Node class: each node has data and points to next node
    private static class Node {
        int data;
        Node next;

        Node(int data) {
            this.data = data;
            this.next = null;
        }
    }

    private Node front; // Points to front (dequeue end)
    private Node rear;  // Points to rear  (enqueue end)
    private int size;

    public LinkedListQueue() {
        front = null;
        rear = null;
        size = 0;
    }

    // ENQUEUE: Add new node at the rear
    // Time: O(1)
    public void enqueue(int value) {
        Node newNode = new Node(value);

        if (rear == null) {
            // Queue was empty: front and rear both point to new node
            front = newNode;
            rear = newNode;
        } else {
            // Attach new node after current rear, update rear
            rear.next = newNode;
            rear = newNode;
        }
        size++;
    }

    // DEQUEUE: Remove and return front node's value
    // Time: O(1)
    public int dequeue() {
        if (isEmpty()) {
            throw new RuntimeException("Queue is empty!");
        }
        int value = front.data;
        front = front.next; // Move front pointer forward

        if (front == null) {
            rear = null; // Queue became empty
        }
        size--;
        return value;
    }

    // PEEK: Return front node's value without removing
    // Time: O(1)
    public int peek() {
        if (isEmpty()) throw new RuntimeException("Queue is empty!");
        return front.data;
    }

    public boolean isEmpty() { return front == null; }
    public int size()        { return size; }

    // Display from front to rear
    public void display() {
        System.out.print("Queue (front → rear): ");
        Node current = front;
        while (current != null) {
            System.out.print(current.data + " ");
            current = current.next;
        }
        System.out.println();
    }

    public static void main(String[] args) {
        LinkedListQueue q = new LinkedListQueue();

        q.enqueue(1);
        q.enqueue(2);
        q.enqueue(3);
        q.display();  // Queue (front → rear): 1 2 3

        System.out.println("Dequeue: " + q.dequeue()); // 1
        System.out.println("Peek: "    + q.peek());    // 2
        System.out.println("Size: "    + q.size());    // 2

        q.enqueue(4);
        q.display();  // Queue (front → rear): 2 3 4
    }
}
```

# 6. Types of Queues

## 6.1 Simple Queue
The standard queue — FIFO. Add at rear, remove from front. We've already covered this.

## 6.2 Circular Queue

**What:** Queue where the rear connects back to the front, forming a circle. Reuses spaces left by dequeued elements.

**Why important:** Fixed-size buffer problems (e.g., network packet buffers).

---

### 🟡 LC #622 — Design Circular Queue

**Problem:** Design a circular queue with operations: `enQueue`, `deQueue`, `Front`, `Rear`, `isEmpty`, `isFull`.

```java
public class MyCircularQueue {  // LC #622

    private int[] arr;
    private int front;
    private int rear;
    private int size;
    private int capacity;

    public MyCircularQueue(int k) {
        capacity = k;
        arr = new int[k];
        front = 0;
        rear = -1;
        size = 0;
    }

    // Add element to rear
    public boolean enQueue(int value) {
        if (isFull()) return false;
        rear = (rear + 1) % capacity; // Circular wrap
        arr[rear] = value;
        size++;
        return true;
    }

    // Remove front element
    public boolean deQueue() {
        if (isEmpty()) return false;
        front = (front + 1) % capacity; // Circular wrap
        size--;
        return true;
    }

    public int Front() {
        return isEmpty() ? -1 : arr[front];
    }

    public int Rear() {
        return isEmpty() ? -1 : arr[rear];
    }

    public boolean isEmpty() { return size == 0; }
    public boolean isFull()  { return size == capacity; }

    public static void main(String[] args) {
        MyCircularQueue cq = new MyCircularQueue(3);
        System.out.println(cq.enQueue(1)); // true
        System.out.println(cq.enQueue(2)); // true
        System.out.println(cq.enQueue(3)); // true
        System.out.println(cq.enQueue(4)); // false (full)
        System.out.println(cq.Rear());     // 3
        System.out.println(cq.isFull());   // true
        System.out.println(cq.deQueue());  // true
        System.out.println(cq.enQueue(4)); // true
        System.out.println(cq.Rear());     // 4
    }
}
```

**Complexity:** All operations O(1), Space O(k)

## 6.3 Double-Ended Queue (Deque)

**What:** A queue where you can add/remove from BOTH ends.

```
FRONT ←→ REAR

addFirst(x)    →  adds to front
addLast(x)     →  adds to rear
removeFirst()  →  removes from front
removeLast()   →  removes from rear
peekFirst()    →  view front
peekLast()     →  view rear
```

**Why powerful:** Can behave as both Stack AND Queue. Essential for sliding window maximum problems.

---

### 🟡 LC #641 — Design Circular Deque

```java
public class MyCircularDeque {  // LC #641

    private int[] arr;
    private int front;
    private int rear;
    private int size;
    private int capacity;

    public MyCircularDeque(int k) {
        capacity = k;
        arr = new int[k];
        front = 0;
        rear = k - 1; // Rear starts at end for easy front insertion
        size = 0;
    }

    // Insert at front
    public boolean insertFront(int value) {
        if (isFull()) return false;
        front = (front - 1 + capacity) % capacity; // Move front back (circular)
        arr[front] = value;
        size++;
        return true;
    }

    // Insert at rear
    public boolean insertLast(int value) {
        if (isFull()) return false;
        rear = (rear + 1) % capacity;
        arr[rear] = value;
        size++;
        return true;
    }

    // Delete from front
    public boolean deleteFront() {
        if (isEmpty()) return false;
        front = (front + 1) % capacity;
        size--;
        return true;
    }

    // Delete from rear
    public boolean deleteLast() {
        if (isEmpty()) return false;
        rear = (rear - 1 + capacity) % capacity;
        size--;
        return true;
    }

    public int getFront() { return isEmpty() ? -1 : arr[front]; }
    public int getRear()  { return isEmpty() ? -1 : arr[rear]; }
    public boolean isEmpty() { return size == 0; }
    public boolean isFull()  { return size == capacity; }
}
```

## 6.4 Priority Queue

**What:** Each element has a **priority**. The element with the **highest (or lowest) priority** is dequeued first — NOT based on insertion order.

```
Insert:  Task(priority=3), Task(priority=1), Task(priority=5), Task(priority=2)

Min Heap (smallest first):
  dequeue() → priority=1
  dequeue() → priority=2
  dequeue() → priority=3
  dequeue() → priority=5

Max Heap (largest first):
  dequeue() → priority=5
  dequeue() → priority=3
  dequeue() → priority=2
  dequeue() → priority=1
```

**Internally uses a Heap** (complete binary tree satisfying heap property).

```java
import java.util.*;

public class PriorityQueueDemo {

    public static void main(String[] args) {

        // ===== MIN-HEAP (default in Java) =====
        // Smallest element always at front
        PriorityQueue<Integer> minHeap = new PriorityQueue<>();
        minHeap.offer(5);
        minHeap.offer(1);
        minHeap.offer(3);
        minHeap.offer(2);

        System.out.println("Min-Heap order:");
        while (!minHeap.isEmpty()) {
            System.out.print(minHeap.poll() + " "); // 1 2 3 5
        }

        // ===== MAX-HEAP (reverse comparator) =====
        // Largest element always at front
        PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Collections.reverseOrder());
        maxHeap.offer(5);
        maxHeap.offer(1);
        maxHeap.offer(3);
        maxHeap.offer(2);

        System.out.println("\nMax-Heap order:");
        while (!maxHeap.isEmpty()) {
            System.out.print(maxHeap.poll() + " "); // 5 3 2 1
        }

        // ===== CUSTOM OBJECT PRIORITY =====
        // Sort by second element of array
        PriorityQueue<int[]> customPQ = new PriorityQueue<>((a, b) -> a[1] - b[1]);
        customPQ.offer(new int[]{1, 10}); // [task_id, priority]
        customPQ.offer(new int[]{2, 5});
        customPQ.offer(new int[]{3, 8});

        System.out.println("\nCustom Priority order:");
        while (!customPQ.isEmpty()) {
            int[] item = customPQ.poll();
            System.out.println("Task " + item[0] + " priority " + item[1]);
        }
        // Task 2 priority 5 → Task 3 priority 8 → Task 1 priority 10
    }
}
```

# 7. Java's Built-in Queue APIs

## The Interface Hierarchy (Important for Interviews!)

```
Iterable
  └── Collection
        └── Queue (interface)
              ├── LinkedList (implements Queue)
              ├── ArrayDeque (implements Deque)
              └── PriorityQueue
                    Deque (interface extends Queue)
                      ├── ArrayDeque ← BEST for most problems
                      └── LinkedList
```

## Queue Interface Methods

```java
import java.util.*;

Queue<Integer> queue = new LinkedList<>();
// OR
Queue<Integer> queue = new ArrayDeque<>(); // ← PREFERRED (faster)

// ===== TWO SETS OF METHODS =====

// Set 1: THROWS EXCEPTION if operation fails
queue.add(x);     // enqueue — throws if full
queue.remove();   // dequeue — throws if empty
queue.element();  // peek    — throws if empty

// Set 2: RETURNS null/false if operation fails (SAFER, use this!)
queue.offer(x);   // enqueue — returns false if full
queue.poll();     // dequeue — returns null if empty
queue.peek();     // peek    — returns null if empty

// Other methods
queue.isEmpty();  // true if empty
queue.size();     // number of elements
queue.contains(x); // true if x is in queue
```

## Deque Interface Methods (Use as Queue OR Stack)

```java
Deque<Integer> deque = new ArrayDeque<>();

// ===== AS A QUEUE (FIFO) =====
deque.offerLast(x);   // enqueue (add to rear)
deque.pollFirst();    // dequeue (remove from front)
deque.peekFirst();    // peek at front

// ===== AS A STACK (LIFO) =====
deque.push(x);        // same as addFirst — add to front
deque.pop();          // same as removeFirst — remove from front
deque.peek();         // same as peekFirst — view front

// ===== DEQUE SPECIFIC =====
deque.offerFirst(x);  // add to front
deque.offerLast(x);   // add to rear
deque.pollFirst();    // remove from front
deque.pollLast();     // remove from rear
deque.peekFirst();    // view front
deque.peekLast();     // view rear
deque.isEmpty();
deque.size();
```

## Complete Cheat Sheet: Which to Use When

```java
// Situation 1: Simple FIFO queue in interview
Queue<Integer> q = new ArrayDeque<>();
q.offer(x);   // enqueue
q.poll();     // dequeue
q.peek();     // front element

// Situation 2: Sliding window (need both ends)
Deque<Integer> dq = new ArrayDeque<>();
dq.offerLast(x);   // add to rear
dq.pollFirst();    // remove from front
dq.pollLast();     // remove from rear
dq.peekFirst();    // view front
dq.peekLast();     // view rear

// Situation 3: Priority (need sorted order)
PriorityQueue<Integer> pq = new PriorityQueue<>(); // min-heap
PriorityQueue<Integer> maxPQ = new PriorityQueue<>(Collections.reverseOrder()); // max-heap
pq.offer(x);   // add
pq.poll();     // remove smallest (or largest for maxPQ)
pq.peek();     // view smallest

// Situation 4: BFS (most common usage)
Queue<TreeNode> bfsQ = new LinkedList<>(); // or ArrayDeque
bfsQ.offer(root);
while (!bfsQ.isEmpty()) {
    TreeNode node = bfsQ.poll();
    if (node.left != null)  bfsQ.offer(node.left);
    if (node.right != null) bfsQ.offer(node.right);
}
```

# 8. Queue Operations & Complexity

```
Operation     | Array Queue | Linked List | ArrayDeque | PriorityQueue
--------------|-------------|-------------|------------|---------------
enqueue/offer |   O(1)      |    O(1)     |   O(1)     |  O(log n)
dequeue/poll  |   O(1)      |    O(1)     |   O(1)     |  O(log n)
peek/front    |   O(1)      |    O(1)     |   O(1)     |  O(1)
isEmpty       |   O(1)      |    O(1)     |   O(1)     |  O(1)
size          |   O(1)      |    O(1)     |   O(1)     |  O(1)
contains      |   O(n)      |    O(n)     |   O(n)     |  O(n)
Space         |   O(n)      |    O(n)     |   O(n)     |  O(n)
```

**Key Insight:** Basic queue operations (enqueue, dequeue, peek) are O(1). PriorityQueue is O(log n) because it maintains heap order on every insert/delete.

## 🟢 LC #232 — Implement Queue Using Two Stacks

We covered this in the stack guide — here's the queue perspective.

**Core idea:** Stack reverses order. Reverse twice = original order = FIFO!

```
push 1, 2, 3 into inbox stack:
inbox:  3 (top)
        2
        1 (bottom)

Transfer to outbox: inbox pops into outbox, reversing order:
outbox: 1 (top)  ← FIFO! First pushed = first popped
        2
        3 (bottom)
```

```java
import java.util.*;

class MyQueue {  // LC #232

    Deque<Integer> inbox  = new ArrayDeque<>(); // New elements go here
    Deque<Integer> outbox = new ArrayDeque<>(); // Elements ready to dequeue

    public void push(int x) {
        inbox.push(x);
    }

    public int pop() {
        refill();
        return outbox.pop();
    }

    public int peek() {
        refill();
        return outbox.peek();
    }

    // Only refill outbox when it is empty (amortizes cost)
    private void refill() {
        if (outbox.isEmpty()) {
            while (!inbox.isEmpty()) {
                outbox.push(inbox.pop()); // Reverse order = FIFO
            }
        }
    }

    public boolean empty() {
        return inbox.isEmpty() && outbox.isEmpty();
    }
}
```

**Complexity:** Push O(1), Pop amortized O(1), Space O(n)

## 🟢 LC #933 — Number of Recent Calls

**Problem:** Count the number of calls in the last `3000` milliseconds.
Each `ping(t)` records a call at time t and returns how many calls were in range `[t-3000, t]`.

**Why Queue?** Old pings expire from the front (FIFO). New pings added to rear.

```
ping(1)    → queue: [1]         → count = 1
ping(100)  → queue: [1, 100]    → count = 2
ping(3001) → queue: [1, 100, 3001] → count = 3
ping(3002) → queue: [1, 100, 3001, 3002]
           → remove 1 (3002-3000=2, 1<2 → expired)
           → queue: [100, 3001, 3002] → count = 3
```

```java
import java.util.*;

class RecentCounter {  // LC #933

    private Queue<Integer> queue;

    public RecentCounter() {
        queue = new ArrayDeque<>();
    }

    public int ping(int t) {
        queue.offer(t); // Add new ping

        // Remove all pings older than t - 3000
        while (queue.peek() < t - 3000) {
            queue.poll();
        }

        return queue.size(); // Remaining pings are all in [t-3000, t]
    }

    public static void main(String[] args) {
        RecentCounter rc = new RecentCounter();
        System.out.println(rc.ping(1));    // 1
        System.out.println(rc.ping(100));  // 2
        System.out.println(rc.ping(3001)); // 3
        System.out.println(rc.ping(3002)); // 3
    }
}
```

**Complexity:** Each element added and removed at most once → amortized O(1), Space O(n)

## 🟢 LC #950 — Reveal Cards in Increasing Order

**Problem:** Given a deck of cards, reveal them in sorted increasing order using a specific process:
- Take the top card and reveal it
- Move next top card to the bottom of the deck
- Repeat

Find the initial deck arrangement.

**Key Insight:** Simulate the process in reverse using a queue of indices.

```java
import java.util.*;

class RevealCards {  // LC #950

    public int[] deckRevealedIncreasing(int[] deck) {
        int n = deck.length;
        Arrays.sort(deck); // Sort in increasing order

        // Queue of indices: represents positions in the result
        Queue<Integer> indexQueue = new ArrayDeque<>();
        for (int i = 0; i < n; i++) indexQueue.offer(i);

        int[] result = new int[n];

        for (int card : deck) {
            // Place current smallest card at the next available position
            result[indexQueue.poll()] = card;

            // Simulate "move top card to bottom" (skip next index)
            if (!indexQueue.isEmpty()) {
                indexQueue.offer(indexQueue.poll());
            }
        }

        return result;
    }

    public static void main(String[] args) {
        RevealCards sol = new RevealCards();
        System.out.println(Arrays.toString(
            sol.deckRevealedIncreasing(new int[]{17,13,11,2,3,5,7})));
        // [2, 13, 3, 11, 5, 17, 7]
    }
}
```

## 🟢 LC #346 — Moving Average from Data Stream

**Problem:** Calculate the moving average of the last `size` numbers in a stream.

```java
import java.util.*;

class MovingAverage {  // LC #346

    private Queue<Integer> window;
    private int capacity;
    private double sum;

    public MovingAverage(int size) {
        window = new ArrayDeque<>();
        capacity = size;
        sum = 0;
    }

    public double next(int val) {
        if (window.size() == capacity) {
            sum -= window.poll(); // Remove oldest element from sum
        }
        window.offer(val); // Add new element
        sum += val;
        return sum / window.size();
    }

    public static void main(String[] args) {
        MovingAverage ma = new MovingAverage(3);
        System.out.println(ma.next(1));  // 1.0   (1/1)
        System.out.println(ma.next(10)); // 5.5   (1+10)/2
        System.out.println(ma.next(3));  // 4.67  (1+10+3)/3
        System.out.println(ma.next(5));  // 6.0   (10+3+5)/3  ← 1 dropped
    }
}
```

# 10. BFS Pattern with Queue

## The BFS Template — Memorize This!

BFS (Breadth-First Search) is the **most common use of Queue** in DSA. Every BFS problem uses this exact template.

```java
// UNIVERSAL BFS TEMPLATE
void bfs(StartNode start) {
    Queue<Node> queue = new ArrayDeque<>();
    Set<Node> visited = new HashSet<>(); // Prevent revisiting

    queue.offer(start);
    visited.add(start);
    int level = 0; // Current BFS level (distance from start)

    while (!queue.isEmpty()) {
        int levelSize = queue.size(); // Number of nodes at current level

        // Process ALL nodes at current level before moving to next
        for (int i = 0; i < levelSize; i++) {
            Node current = queue.poll();

            // === PROCESS current node here ===

            // Add all unvisited neighbors to queue
            for (Node neighbor : getNeighbors(current)) {
                if (!visited.contains(neighbor)) {
                    visited.add(neighbor);
                    queue.offer(neighbor);
                }
            }
        }

        level++; // Increment after processing entire level
    }
}
```

## Why Queue Guarantees Shortest Path

```
Graph:     A
          / \
         B   C
        / \   \
       D   E   F

BFS starting from A:

Level 0: Queue=[A]     → process A  → add B, C
Level 1: Queue=[B,C]   → process B  → add D, E
                       → process C  → add F
Level 2: Queue=[D,E,F] → process D,E,F

When we first reach any node, it's via the SHORTEST path!
(Because we explore all distance-1 nodes before distance-2, etc.)

DFS might take A→B→D→... and find F much later via longer path.
BFS reaches F at level 2 = shortest distance 2 from A ✅
```

## 🟢 LC #102 — Binary Tree Level Order Traversal

**Problem:** Return nodes level by level as a 2D list.

**Input:** `root = [3,9,20,null,null,15,7]`
**Output:** `[[3],[9,20],[15,7]]`

```
        3          ← Level 0
       / \
      9   20       ← Level 1
         /  \
        15    7    ← Level 2
```

```java
import java.util.*;

class LevelOrderTraversal {  // LC #102

    static class TreeNode {
        int val; TreeNode left, right;
        TreeNode(int val) { this.val = val; }
    }

    public List<List<Integer>> levelOrder(TreeNode root) {
        List<List<Integer>> result = new ArrayList<>();
        if (root == null) return result;

        Queue<TreeNode> queue = new ArrayDeque<>();
        queue.offer(root); // Start with root

        while (!queue.isEmpty()) {
            int levelSize = queue.size(); // How many nodes at this level?
            List<Integer> currentLevel = new ArrayList<>();

            // Process EXACTLY levelSize nodes — all at current level
            for (int i = 0; i < levelSize; i++) {
                TreeNode node = queue.poll();
                currentLevel.add(node.val);

                // Add children for next level
                if (node.left  != null) queue.offer(node.left);
                if (node.right != null) queue.offer(node.right);
            }

            result.add(currentLevel);
        }

        return result;
    }
}
```

**Complexity:** Time O(n) — visit each node once, Space O(w) — w is max width of tree

## 🟡 LC #103 — Binary Tree Zigzag Level Order Traversal

**Problem:** Level order but alternate direction each level (left→right, then right→left, etc.)

**Output for above tree:** `[[3],[20,9],[15,7]]`

```java
import java.util.*;

class ZigzagLevelOrder {  // LC #103

    static class TreeNode {
        int val; TreeNode left, right;
        TreeNode(int val) { this.val = val; }
    }

    public List<List<Integer>> zigzagLevelOrder(TreeNode root) {
        List<List<Integer>> result = new ArrayList<>();
        if (root == null) return result;

        Queue<TreeNode> queue = new ArrayDeque<>();
        queue.offer(root);
        boolean leftToRight = true; // Direction toggle

        while (!queue.isEmpty()) {
            int levelSize = queue.size();
            // Use LinkedList to add at front or back based on direction
            LinkedList<Integer> currentLevel = new LinkedList<>();

            for (int i = 0; i < levelSize; i++) {
                TreeNode node = queue.poll();

                if (leftToRight) {
                    currentLevel.addLast(node.val);  // Normal order
                } else {
                    currentLevel.addFirst(node.val); // Reverse order
                }

                if (node.left  != null) queue.offer(node.left);
                if (node.right != null) queue.offer(node.right);
            }

            result.add(currentLevel);
            leftToRight = !leftToRight; // Toggle direction
        }

        return result;
    }
}
```

**Complexity:** Time O(n), Space O(w)

## 🟡 LC #662 — Maximum Width of Binary Tree

**Problem:** Find the maximum width of the binary tree across all levels.
Width = number of nodes between leftmost and rightmost (including null nodes).

**Key trick:** Number nodes like an array heap — left child = 2*i, right child = 2*i+1. Width = last_index - first_index + 1.

```java
import java.util.*;

class MaxWidthBinaryTree {  // LC #662

    static class TreeNode {
        int val; TreeNode left, right;
        TreeNode(int val) { this.val = val; }
    }

    public int widthOfBinaryTree(TreeNode root) {
        if (root == null) return 0;

        int maxWidth = 0;
        // Queue stores [node, index] pairs
        Queue<Object[]> queue = new ArrayDeque<>();
        queue.offer(new Object[]{root, 1}); // Root has index 1

        while (!queue.isEmpty()) {
            int levelSize = queue.size();
            int firstIndex = 0, lastIndex = 0;

            for (int i = 0; i < levelSize; i++) {
                Object[] pair = queue.poll();
                TreeNode node = (TreeNode) pair[0];
                int index = (int) pair[1];

                if (i == 0) firstIndex = index;           // First node of level
                if (i == levelSize - 1) lastIndex = index; // Last node of level

                // Left child = 2*index, Right child = 2*index+1
                if (node.left  != null) queue.offer(new Object[]{node.left,  2 * index});
                if (node.right != null) queue.offer(new Object[]{node.right, 2 * index + 1});
            }

            maxWidth = Math.max(maxWidth, lastIndex - firstIndex + 1);
        }

        return maxWidth;
    }
}
```

## 🟡 LC #994 — Rotting Oranges

**Problem:** Grid of 0 (empty), 1 (fresh), 2 (rotten). Each minute, fresh oranges adjacent to rotten ones become rotten. Return minimum minutes until all oranges rot, or -1 if impossible.

**Why BFS?** Multi-source BFS from ALL rotten oranges simultaneously.

```
Initial:
2 1 1
1 1 0
0 1 1

Minute 0: Rotten=[top-left corner]
Minute 1: [1,0],[0,1] become rotten
Minute 2: [1,1],[0,2] become rotten
Minute 3: [2,1] becomes rotten
Minute 4: [2,2] becomes rotten

Answer: 4
```

```java
import java.util.*;

class RottingOranges {  // LC #994

    public int orangesRotting(int[][] grid) {
        int rows = grid.length, cols = grid[0].length;
        Queue<int[]> queue = new ArrayDeque<>();
        int fresh = 0; // Count of fresh oranges

        // Step 1: Find all initially rotten oranges and count fresh ones
        for (int r = 0; r < rows; r++) {
            for (int c = 0; c < cols; c++) {
                if (grid[r][c] == 2) {
                    queue.offer(new int[]{r, c}); // Add all rotten to queue (multi-source)
                } else if (grid[r][c] == 1) {
                    fresh++;
                }
            }
        }

        if (fresh == 0) return 0; // No fresh oranges

        int minutes = 0;
        int[][] dirs = {{0,1},{0,-1},{1,0},{-1,0}};

        // Step 2: BFS from all rotten oranges simultaneously
        while (!queue.isEmpty() && fresh > 0) {
            minutes++;
            int levelSize = queue.size();

            for (int i = 0; i < levelSize; i++) {
                int[] cell = queue.poll();

                for (int[] dir : dirs) {
                    int nr = cell[0] + dir[0];
                    int nc = cell[1] + dir[1];

                    if (nr >= 0 && nr < rows && nc >= 0 && nc < cols
                            && grid[nr][nc] == 1) { // Fresh orange
                        grid[nr][nc] = 2;           // Make it rotten
                        queue.offer(new int[]{nr, nc});
                        fresh--;
                    }
                }
            }
        }

        return fresh == 0 ? minutes : -1; // -1 if some fresh remain
    }
}
```

**Complexity:** Time O(m×n), Space O(m×n)

## 🟡 LC #127 — Word Ladder

**Problem:** Transform `beginWord` to `endWord` by changing one letter at a time. Each intermediate word must be in wordList. Find minimum transformations.

**Input:** `begin="hit"`, `end="cog"`, `wordList=["hot","dot","dog","lot","log","cog"]`
**Output:** `5` (hit→hot→dot→dog→cog)

```java
import java.util.*;

class WordLadder {  // LC #127

    public int ladderLength(String beginWord, String endWord, List<String> wordList) {
        Set<String> wordSet = new HashSet<>(wordList);
        if (!wordSet.contains(endWord)) return 0;

        Queue<String> queue = new ArrayDeque<>();
        queue.offer(beginWord);
        Set<String> visited = new HashSet<>();
        visited.add(beginWord);
        int steps = 1;

        while (!queue.isEmpty()) {
            int levelSize = queue.size(); // Process one full level per step

            for (int i = 0; i < levelSize; i++) {
                String word = queue.poll();

                // Try changing each character in the word
                char[] chars = word.toCharArray();
                for (int j = 0; j < chars.length; j++) {
                    char original = chars[j];

                    for (char c = 'a'; c <= 'z'; c++) {
                        if (c == original) continue;
                        chars[j] = c;
                        String newWord = new String(chars);

                        if (newWord.equals(endWord)) return steps + 1; // Found!

                        if (wordSet.contains(newWord) && !visited.contains(newWord)) {
                            queue.offer(newWord);
                            visited.add(newWord);
                        }
                    }
                    chars[j] = original; // Restore character
                }
            }
            steps++;
        }

        return 0; // No path found
    }
}
```

## 🟡 LC #542 — 01 Matrix (Distance to Nearest 0)

**Problem:** For each cell, find the distance to the nearest 0.

**Strategy:** Multi-source BFS starting from ALL 0-cells simultaneously.

```java
import java.util.*;

class ZeroOneMatrix {  // LC #542

    public int[][] updateMatrix(int[][] mat) {
        int rows = mat.length, cols = mat[0].length;
        int[][] dist = new int[rows][cols];
        Queue<int[]> queue = new ArrayDeque<>();
        int[][] dirs = {{0,1},{0,-1},{1,0},{-1,0}};

        // Initialize: 0-cells have distance 0 (add to queue)
        //             1-cells have unknown distance (set to MAX)
        for (int r = 0; r < rows; r++) {
            for (int c = 0; c < cols; c++) {
                if (mat[r][c] == 0) {
                    queue.offer(new int[]{r, c});
                    dist[r][c] = 0;
                } else {
                    dist[r][c] = Integer.MAX_VALUE; // Unknown
                }
            }
        }

        // BFS from all 0-cells simultaneously
        while (!queue.isEmpty()) {
            int[] cell = queue.poll();
            for (int[] dir : dirs) {
                int nr = cell[0] + dir[0];
                int nc = cell[1] + dir[1];
                // If neighbor can be reached with shorter distance
                if (nr >= 0 && nr < rows && nc >= 0 && nc < cols
                        && dist[nr][nc] > dist[cell[0]][cell[1]] + 1) {
                    dist[nr][nc] = dist[cell[0]][cell[1]] + 1;
                    queue.offer(new int[]{nr, nc});
                }
            }
        }

        return dist;
    }
}
```

**Complexity:** Time O(m×n), Space O(m×n)

```java
import java.util.*;

class BusRoutes {  // LC #815

                        if (!visitedStops.contains(nextStop)) {
                            visitedStops.add(nextStop);
                            queue.offer(nextStop);
                        }
                    }
                }
            }
        }
```

# 13. Sliding Window with Deque

## Why Deque for Sliding Window?

The **Monotonic Deque** pattern solves a crucial problem: finding the **maximum or minimum in a sliding window** in O(1) per query, O(n) total.

```
Normal approach (brute force): For each window, scan all k elements → O(n×k)
Deque approach: Each element added/removed at most once → O(n)
```

## The Key Idea

```
For sliding window MAXIMUM:
- Maintain a DECREASING deque (front is always max)
- When adding new element:
  → Remove from FRONT any elements outside the window
  → Remove from BACK any elements SMALLER than current (they're useless — current is bigger and newer)
- Front of deque = maximum in current window

Window: [1, 3, -1, -3, 5, 3, 6, 7], k=3

        Window         Deque(indices)  Max
i=0     [1]            [0]             -
i=1     [1,3]          [1]             -      (remove 0 since 3>1)
i=2     [1,3,-1]       [1,2]           3      (first full window)
i=3     [3,-1,-3]      [1,2,3]         3      (1 still in window)
i=4     [-1,-3,5]      [4]             5      (remove 1,2,3 since 5>all)
i=5     [-3,5,3]       [4,5]           5
i=6     [5,3,6]        [6]             6      (remove 4,5 since 6>both)
i=7     [3,6,7]        [7]             7      (remove 6 since 7>6)
```

## 🔴 LC #239 — Sliding Window Maximum

**Problem:** Find the maximum in every window of size k.

**Input:** `nums=[1,3,-1,-3,5,3,6,7]`, `k=3`
**Output:** `[3,3,5,5,6,7]`

```java
import java.util.*;

class SlidingWindowMaximum {  // LC #239

    public int[] maxSlidingWindow(int[] nums, int k) {
        int n = nums.length;
        int[] result = new int[n - k + 1];
        // Deque stores INDICES (not values). Front = index of max element.
        Deque<Integer> deque = new ArrayDeque<>();

        for (int i = 0; i < n; i++) {

            // Step 1: Remove indices that are outside the window from FRONT
            while (!deque.isEmpty() && deque.peekFirst() < i - k + 1) {
                deque.pollFirst();
            }

            // Step 2: Remove indices from BACK whose values are less than current
            // (They can never be maximum while current element is in the window)
            while (!deque.isEmpty() && nums[deque.peekLast()] < nums[i]) {
                deque.pollLast();
            }

            // Step 3: Add current index to back
            deque.offerLast(i);

            // Step 4: Record result once first window is complete
            if (i >= k - 1) {
                result[i - k + 1] = nums[deque.peekFirst()]; // Front is always max
            }
        }

        return result;
    }

    public static void main(String[] args) {
        SlidingWindowMaximum sol = new SlidingWindowMaximum();
        System.out.println(Arrays.toString(
            sol.maxSlidingWindow(new int[]{1,3,-1,-3,5,3,6,7}, 3)));
        // [3, 3, 5, 5, 6, 7]
    }
}
```

**Complexity:** Time O(n) — each element added/removed once, Space O(k)

## 🟡 LC #1438 — Longest Continuous Subarray With Absolute Diff ≤ Limit

**Problem:** Find longest subarray where `max - min ≤ limit`.
Use two monotonic deques — one for max, one for min.

```java
import java.util.*;

class LongestSubarrayAbsDiff {  // LC #1438

    public int longestSubarray(int[] nums, int limit) {
        Deque<Integer> maxDeque = new ArrayDeque<>(); // Decreasing — front is max
        Deque<Integer> minDeque = new ArrayDeque<>(); // Increasing — front is min
        int left = 0;
        int result = 0;

        for (int right = 0; right < nums.length; right++) {
            // Maintain max deque (decreasing)
            while (!maxDeque.isEmpty() && nums[maxDeque.peekLast()] < nums[right]) {
                maxDeque.pollLast();
            }
            maxDeque.offerLast(right);

            // Maintain min deque (increasing)
            while (!minDeque.isEmpty() && nums[minDeque.peekLast()] > nums[right]) {
                minDeque.pollLast();
            }
            minDeque.offerLast(right);

            // Shrink window from left if max - min > limit
            while (nums[maxDeque.peekFirst()] - nums[minDeque.peekFirst()] > limit) {
                left++;
                if (maxDeque.peekFirst() < left) maxDeque.pollFirst();
                if (minDeque.peekFirst() < left) minDeque.pollFirst();
            }

            result = Math.max(result, right - left + 1);
        }

        return result;
    }
}
```

## 🟡 LC #862 — Shortest Subarray with Sum at Least K

**Problem:** Find shortest subarray with sum ≥ k (array can have negatives).

```java
import java.util.*;

class ShortestSubarraySumK {  // LC #862

    public int shortestSubarray(int[] nums, int k) {
        int n = nums.length;
        long[] prefix = new long[n + 1]; // Prefix sums (long to avoid overflow)
        for (int i = 0; i < n; i++) prefix[i+1] = prefix[i] + nums[i];

        Deque<Integer> deque = new ArrayDeque<>(); // Monotonic increasing prefix sum
        int result = Integer.MAX_VALUE;

        for (int i = 0; i <= n; i++) {
            // If current prefix - front prefix ≥ k, found a valid subarray
            while (!deque.isEmpty() && prefix[i] - prefix[deque.peekFirst()] >= k) {
                result = Math.min(result, i - deque.pollFirst());
            }

            // Maintain increasing deque (remove larger prefix sums from back)
            while (!deque.isEmpty() && prefix[i] <= prefix[deque.peekLast()]) {
                deque.pollLast();
            }

            deque.offerLast(i);
        }

        return result == Integer.MAX_VALUE ? -1 : result;
    }
}
```

## 🟡 LC #215 — Kth Largest Element in an Array

**Problem:** Find the kth largest element in an unsorted array.

**Strategy:** Min-heap of size k. The smallest in the heap = kth largest overall.

```java
import java.util.*;

class KthLargestElement {  // LC #215

    public int findKthLargest(int[] nums, int k) {
        // Min-heap of size k
        PriorityQueue<Integer> minHeap = new PriorityQueue<>();

        for (int num : nums) {
            minHeap.offer(num);
            // Keep heap size at exactly k
            if (minHeap.size() > k) {
                minHeap.poll(); // Remove the smallest — we only want top k
            }
        }

        // Top of min-heap = kth largest (k-1 elements are larger)
        return minHeap.peek();
    }

    public static void main(String[] args) {
        KthLargestElement sol = new KthLargestElement();
        System.out.println(sol.findKthLargest(new int[]{3,2,1,5,6,4}, 2)); // 5
        System.out.println(sol.findKthLargest(new int[]{3,2,3,1,2,4,5,5,6}, 4)); // 4
    }
}
```

**Complexity:** Time O(n log k), Space O(k)

## 🟡 LC #373 — Find K Pairs with Smallest Sums

**Problem:** Given two sorted arrays, find k pairs (one from each) with smallest sums.

```java
import java.util.*;

class KSmallestPairs {  // LC #373

    public List<List<Integer>> kSmallestPairs(int[] nums1, int[] nums2, int k) {
        List<List<Integer>> result = new ArrayList<>();
        if (nums1.length == 0 || nums2.length == 0) return result;

        // Min-heap: [sum, i (index in nums1), j (index in nums2)]
        PriorityQueue<int[]> minHeap = new PriorityQueue<>((a, b) -> a[0] - b[0]);

        // Start with all pairs (nums1[i], nums2[0]) for i=0..min(k-1,n1-1)
        for (int i = 0; i < Math.min(k, nums1.length); i++) {
            minHeap.offer(new int[]{nums1[i] + nums2[0], i, 0});
        }

        while (!minHeap.isEmpty() && result.size() < k) {
            int[] curr = minHeap.poll();
            int i = curr[1], j = curr[2];

            result.add(Arrays.asList(nums1[i], nums2[j]));

            // Add next pair with same nums1[i] but next nums2 element
            if (j + 1 < nums2.length) {
                minHeap.offer(new int[]{nums1[i] + nums2[j+1], i, j+1});
            }
        }

        return result;
    }
}
```

## 🟡 LC #1642 — Furthest Building You Can Reach

**Problem:** With bricks and ladders, reach as far as possible in buildings array.

```java
class FurthestBuilding {  // LC #1642

    public int furthestBuilding(int[] heights, int bricks, int ladders) {
        // Min-heap stores the ladder climbs (so we can swap smallest ladder use with bricks)
        PriorityQueue<Integer> ladderClimbs = new PriorityQueue<>();

        for (int i = 0; i < heights.length - 1; i++) {
            int diff = heights[i+1] - heights[i];
            if (diff <= 0) continue; // Going down or same height — no cost

            // Use a ladder for this climb
            ladderClimbs.offer(diff);

            // If we used more ladders than we have, swap the smallest ladder use with bricks
            if (ladderClimbs.size() > ladders) {
                bricks -= ladderClimbs.poll(); // Replace smallest ladder climb with bricks
            }

            if (bricks < 0) return i; // Can't afford — stuck here
        }

        return heights.length - 1; // Reached the end!
    }
}
```

## 🔴 LC #295 — Find Median from Data Stream

**Problem:** Design a data structure that supports `addNum` and `findMedian` efficiently.

**Strategy:** Two heaps — max-heap for lower half, min-heap for upper half.

```
Lower half: max-heap  (top = largest in lower half)
Upper half: min-heap  (top = smallest in upper half)

Maintain: |maxHeap.size - minHeap.size| ≤ 1

Median: if equal sizes → average of both tops
        if unequal    → top of larger heap

Example: [1, 2, 3, 4, 5]
maxHeap: [3, 2, 1]   ← max-heap, top=3
minHeap: [4, 5]      ← min-heap, top=4
Median: (3+4)/2 = 3.5 ✅
```

```java
import java.util.*;

class MedianFinder {  // LC #295

    private PriorityQueue<Integer> maxHeap; // Lower half, max at top
    private PriorityQueue<Integer> minHeap; // Upper half, min at top

    public MedianFinder() {
        maxHeap = new PriorityQueue<>(Collections.reverseOrder()); // Max-heap
        minHeap = new PriorityQueue<>(); // Min-heap
    }

    public void addNum(int num) {
        // Step 1: Add to maxHeap first
        maxHeap.offer(num);

        // Step 2: Balance: maxHeap's top must be ≤ minHeap's top
        if (!minHeap.isEmpty() && maxHeap.peek() > minHeap.peek()) {
            minHeap.offer(maxHeap.poll()); // Move max of lower to upper half
        }

        // Step 3: Balance sizes: maxHeap can have at most 1 extra element
        if (maxHeap.size() > minHeap.size() + 1) {
            minHeap.offer(maxHeap.poll());
        } else if (minHeap.size() > maxHeap.size()) {
            maxHeap.offer(minHeap.poll());
        }
    }

    public double findMedian() {
        if (maxHeap.size() == minHeap.size()) {
            return (maxHeap.peek() + minHeap.peek()) / 2.0;
        }
        return maxHeap.peek(); // maxHeap has 1 extra element
    }

    public static void main(String[] args) {
        MedianFinder mf = new MedianFinder();
        mf.addNum(1);
        mf.addNum(2);
        System.out.println(mf.findMedian()); // 1.5
        mf.addNum(3);
        System.out.println(mf.findMedian()); // 2.0
    }
}
```

**Complexity:** addNum O(log n), findMedian O(1)

## 🔴 LC #23 — Merge K Sorted Lists

**Problem:** Merge k sorted linked lists into one sorted list.

```java
import java.util.*;

class MergeKSortedLists {  // LC #23

    static class ListNode {
        int val; ListNode next;
        ListNode(int val) { this.val = val; }
    }

    public ListNode mergeKLists(ListNode[] lists) {
        // Min-heap ordered by node value
        PriorityQueue<ListNode> minHeap = new PriorityQueue<>((a, b) -> a.val - b.val);

        // Add the head of each list to the heap
        for (ListNode list : lists) {
            if (list != null) minHeap.offer(list);
        }

        ListNode dummy = new ListNode(0);
        ListNode current = dummy;

        while (!minHeap.isEmpty()) {
            ListNode smallest = minHeap.poll(); // Get smallest node across all lists
            current.next = smallest;
            current = current.next;

            // Add next node from the same list
            if (smallest.next != null) minHeap.offer(smallest.next);
        }

        return dummy.next;
    }
}
```

**Complexity:** Time O(n log k), Space O(k) — n total nodes, k lists

## 🔴 LC #1670 — Design Front Middle Back Queue

**Problem:** Design a queue that supports push and pop at front, middle, and back in O(1).

```java
import java.util.*;

class FrontMiddleBackQueue {  // LC #1670

    // Use two deques: front half and back half
    // Maintain: back.size() == front.size() OR back.size() == front.size() + 1
    private Deque<Integer> front;
    private Deque<Integer> back;

    public FrontMiddleBackQueue() {
        front = new ArrayDeque<>();
        back = new ArrayDeque<>();
    }

    // Balance: back should have same size or 1 more than front
    private void balance() {
        if (front.size() > back.size()) {
            back.addFirst(front.removeLast());
        } else if (back.size() > front.size() + 1) {
            front.addLast(back.removeFirst());
        }
    }

    public void pushFront(int val) {
        front.addFirst(val);
        balance();
    }

    public void pushMiddle(int val) {
        if (front.size() == back.size()) {
            back.addFirst(val); // back gets the middle
        } else {
            front.addLast(val); // front gets the middle
        }
        // No need to balance — we chose correctly
    }

    public void pushBack(int val) {
        back.addLast(val);
        balance();
    }

    public int popFront() {
        if (front.isEmpty() && back.isEmpty()) return -1;
        int val = front.isEmpty() ? back.removeFirst() : front.removeFirst();
        balance();
        return val;
    }

    public int popMiddle() {
        if (front.isEmpty() && back.isEmpty()) return -1;
        // Middle is: last of front (if equal sizes) OR first of back (if back is larger)
        int val = (front.size() == back.size()) ? front.removeLast() : back.removeFirst();
        // No balance needed — we chose from correct half
        return val;
    }

    public int popBack() {
        if (back.isEmpty()) return -1;
        int val = back.removeLast();
        balance();
        return val;
    }
}
```

## 🔴 LC #407 — Trapping Rain Water II (3D)

**Problem:** 3D version of trapping rain water on a heightmap matrix.

**Strategy:** BFS with min-heap — water fills from lowest boundary inward.

```java
import java.util.*;

class TrapRainWater3D {  // LC #407

    public int trapRainWater(int[][] heightMap) {
        if (heightMap == null || heightMap.length < 3 || heightMap[0].length < 3) return 0;

        int rows = heightMap.length, cols = heightMap[0].length;
        boolean[][] visited = new boolean[rows][cols];
        // Min-heap: [height, row, col]
        PriorityQueue<int[]> minHeap = new PriorityQueue<>((a, b) -> a[0] - b[0]);

        // Add all border cells to heap as initial boundaries
        for (int r = 0; r < rows; r++) {
            for (int c = 0; c < cols; c++) {
                if (r == 0 || r == rows-1 || c == 0 || c == cols-1) {
                    minHeap.offer(new int[]{heightMap[r][c], r, c});
                    visited[r][c] = true;
                }
            }
        }

        int water = 0;
        int maxBoundary = 0;
        int[][] dirs = {{0,1},{0,-1},{1,0},{-1,0}};

        while (!minHeap.isEmpty()) {
            int[] cell = minHeap.poll();
            int h = cell[0], r = cell[1], c = cell[2];
            maxBoundary = Math.max(maxBoundary, h);

            for (int[] dir : dirs) {
                int nr = r + dir[0], nc = c + dir[1];
                if (nr >= 0 && nr < rows && nc >= 0 && nc < cols && !visited[nr][nc]) {
                    visited[nr][nc] = true;
                    // Water trapped = boundary height - cell height (if positive)
                    water += Math.max(0, maxBoundary - heightMap[nr][nc]);
                    minHeap.offer(new int[]{heightMap[nr][nc], nr, nc});
                }
            }
        }

        return water;
    }
}
```

## 🔴 LC #630 — Course Schedule III

**Problem:** Maximize number of courses taken given (duration, lastDay) pairs.

**Strategy:** Greedy + Max-heap. Sort by deadline, greedily take courses. If over time, drop the longest duration course.

```java
import java.util.*;

class CourseScheduleIII {  // LC #630

    public int scheduleCourse(int[][] courses) {

        Arrays.sort(courses, (a, b) -> a[1] - b[1]);

        // Max-heap to track durations of taken courses

            if (totalTime > deadline) {
                totalTime -= maxHeap.poll(); // Drop most expensive course
            }
        }

        return maxHeap.size(); // Number of courses we kept
    }
```

## 🔴 LC #1499 — Max Value of Equation

**Problem:** Find maximum value of `yi + yj + |xi - xj|` where `xi < xj` and `xj - xi ≤ k`.

**Strategy:** Since xi < xj, `|xi-xj| = xj - xi`. Equation becomes `(yi-xi) + (yj+xj)`. Use monotonic deque.

```java
import java.util.*;

class MaxValueEquation {  // LC #1499

    public int findMaxValueOfEquation(int[][] points, int k) {
        // Deque stores [yi - xi, xi] — decreasing by (yi - xi)
        Deque<int[]> deque = new ArrayDeque<>();
        int maxVal = Integer.MIN_VALUE;

        for (int[] point : points) {
            int xj = point[0], yj = point[1];

            // Remove points outside window (xj - xi > k)
            while (!deque.isEmpty() && xj - deque.peekFirst()[1] > k) {
                deque.pollFirst();
            }

            // Calculate value with best previous point
            if (!deque.isEmpty()) {
                maxVal = Math.max(maxVal, yj + xj + deque.peekFirst()[0]);
            }

            // Maintain decreasing deque by (yi - xi) value
            while (!deque.isEmpty() && deque.peekLast()[0] <= yj - xj) {
                deque.pollLast();
            }

            deque.offerLast(new int[]{yj - xj, xj});
        }

        return maxVal;
    }
}
```

## 🚀 Quick API Reference

```java
// ===== SIMPLE QUEUE =====
Queue<Integer> q = new ArrayDeque<>();
```

## 🧠 BFS Template (Memorize!)

```java
// STANDARD BFS FOR SHORTEST PATH
Queue<Node> queue = new ArrayDeque<>();
Set<Node> visited = new HashSet<>();

queue.offer(start);
visited.add(start);
int distance = 0;

while (!queue.isEmpty()) {
    int levelSize = queue.size(); // CRITICAL: snapshot size before processing

    for (int i = 0; i < levelSize; i++) {
        Node node = queue.poll();

        if (node == target) return distance; // Found!

        for (Node neighbor : getNeighbors(node)) {
            if (!visited.contains(neighbor)) {
                visited.add(neighbor);
                queue.offer(neighbor);
            }
        }
    }

    distance++; // Increment AFTER processing entire level
}

return -1; // Not reachable
```

## 🧠 Monotonic Deque Template (Sliding Window Max)

```java
// SLIDING WINDOW MAXIMUM
Deque<Integer> deque = new ArrayDeque<>(); // Stores indices
int[] result = new int[n - k + 1];

for (int i = 0; i < n; i++) {
    // Remove out-of-window indices from FRONT
    while (!deque.isEmpty() && deque.peekFirst() < i - k + 1)
        deque.pollFirst();

    // Remove smaller elements from BACK (useless candidates)
    while (!deque.isEmpty() && nums[deque.peekLast()] < nums[i])
        deque.pollLast();

    deque.offerLast(i);

    if (i >= k - 1)
        result[i - k + 1] = nums[deque.peekFirst()]; // Front = max
}
```

## 🧠 Priority Queue Templates

```java
// TOP K LARGEST ELEMENTS (use min-heap of size k)
PriorityQueue<Integer> minHeap = new PriorityQueue<>();
for (int num : nums) {
    minHeap.offer(num);
    if (minHeap.size() > k) minHeap.poll(); // Drop smallest
}
// minHeap.peek() = kth largest

// K-WAY MERGE (merge k sorted arrays/lists)
PriorityQueue<int[]> pq = new PriorityQueue<>((a,b) -> a[0]-b[0]); // [value, list_idx, elem_idx]
for (int i = 0; i < lists.length; i++) {
    if (!lists[i].isEmpty()) pq.offer(new int[]{lists[i].get(0), i, 0});
}
while (!pq.isEmpty()) {
    int[] curr = pq.poll();
    result.add(curr[0]);
    // Add next element from same list
    if (curr[2]+1 < lists[curr[1]].size())
        pq.offer(new int[]{lists[curr[1]].get(curr[2]+1), curr[1], curr[2]+1});
}

// TWO HEAPS (for median)
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Collections.reverseOrder()); // lower half
PriorityQueue<Integer> minHeap = new PriorityQueue<>(); // upper half
// Always: maxHeap.size() == minHeap.size() OR maxHeap.size() == minHeap.size() + 1
```

## 🔍 Recognition Keywords

```
When you see these words → THINK QUEUE:

"level by level"         → Tree BFS with queue
"shortest path"          → Graph BFS with queue
"minimum steps"          → BFS with queue
"nearest/closest"        → BFS / Multi-source BFS
"spread" / "propagation" → Multi-source BFS
"rotting" / "infection"  → Multi-source BFS
"sliding window max/min" → Monotonic Deque
"kth largest/smallest"   → Priority Queue (heap)
"top k"                  → Priority Queue
"median stream"          → Two Heaps
"scheduling"             → Priority Queue
"order of arrival"       → Queue (FIFO)
"first in first out"     → Queue
"waiting line"           → Queue
```

## 🧠 Decision Framework

```
Does the problem need level-by-level processing?
│
├── YES → Tree BFS or Graph BFS
│          └── Use Queue, process levelSize nodes per loop
│
├── Need SHORTEST PATH in unweighted graph?
│   → BFS with Queue
│
├── Need SHORTEST PATH in weighted graph?
│   → Dijkstra with Priority Queue
│
├── Need MAX/MIN over a sliding window?
│   → Monotonic Deque
│       Keep decreasing deque for MAX
│       Keep increasing deque for MIN
│
├── Need Kth largest/smallest?
│   → Min-Heap of size k  (for kth largest)
│   → Max-Heap of size k  (for kth smallest)
│
├── Need top K elements?
│   → Priority Queue
│
├── Need median from stream?
│   → Two Heaps (max-heap + min-heap)
│
├── Need to merge K sorted sequences?
│   → Priority Queue (K-way merge)
│
├── Multiple sources spreading simultaneously?
│   → Multi-Source BFS with Queue
│
└── FIFO ordering needed?
    → Simple Queue
```

## ⚡ Pattern → Data Structure Map

| Problem Pattern | Data Structure | Why |
|-----------------|----------------|-----|
| Level order traversal | Queue | FIFO explores level by level |
| Shortest path (unweighted) | Queue (BFS) | Level = distance |
| Shortest path (weighted) | Priority Queue (Dijkstra) | Explore by distance |
| Sliding window max/min | Monotonic Deque | O(1) front/back access |
| Kth largest | Min-Heap size k | Efficiently track top k |
| Kth smallest | Max-Heap size k | Efficiently track bottom k |
| Merge k sorted | Min-Heap | Always get global minimum |
| Median from stream | Two Heaps | Balance halves |
| Task scheduling | Priority Queue | Priority-based selection |
| Spread/infection | Multi-source BFS | Simultaneous expansion |
