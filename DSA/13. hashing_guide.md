# 1. What is Hashing?

## The Core Idea

**Hashing** converts a key (string, integer, object) into an **index** in an array using a **hash function**. This allows O(1) average time for insert, delete, and lookup.

```
Without hashing:
  "apple" → search through all keys → O(n)

With hashing:
  "apple" → hash("apple") = 42 → array[42] → O(1)!

The Magic:
┌──────────────────────────────────────────────────┐
│                                                  │
│  KEY        HASH FUNCTION      INDEX             │
│  "apple"  → h("apple") = 42 → array[42] = value │
│  "banana" → h("banana") = 17 → array[17] = value│
│  "cherry" → h("cherry") = 89 → array[89] = value│
│                                                  │
│  Lookup: h("apple") = 42 → go to array[42] → ✅ │
└──────────────────────────────────────────────────┘
```

## Real-World Uses

```
Database indexing    → Find record by primary key in O(1)
Caching (Redis)      → Store key-value pairs for fast retrieval
Set membership       → "Have I seen this element before?" O(1)
Counting frequency   → How many times does each word appear?
Detecting duplicates → O(n) instead of O(n²)
Symbol tables        → Compilers store variables in hash tables
Routing tables       → Network routers use hashing for fast lookup
```

# 2. Hash Functions — How They Work

## Properties of a Good Hash Function

```
1. DETERMINISTIC: Same input always produces same output
2. FAST: O(1) to compute
3. UNIFORM: Distributes keys evenly across array (minimizes collisions)
4. AVALANCHE EFFECT: Small change in input → large change in output

BAD HASH FUNCTION:
  h(key) = key % 10   → "apple"=1, "maple"=1 → TOO MANY COLLISIONS

GOOD HASH FUNCTION (polynomial rolling):
  h("abc") = 'a'*31² + 'b'*31¹ + 'c'*31⁰ = 97*961 + 98*31 + 99
           = 93217 + 3038 + 99 = 96354

Why 31? It's prime, odd, and fits in 5 bits.
Powers of small primes distribute well.
```

## Java's hashCode()

```java
// String.hashCode() in Java:
// h = 0
// for each char c: h = h * 31 + c

"abc".hashCode()   // = 96354
"hello".hashCode() // = 99162322

// Integer.hashCode() just returns the integer itself
Integer.hashCode(42) // = 42

// Object.hashCode() returns memory address (bad for business logic)
// Override hashCode() when using custom objects as keys!

// Example: custom class as HashMap key
class Point {
    int x, y;
    @Override
    public int hashCode() {
        return 31 * x + y;  // Must be consistent with equals()
    }
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof Point)) return false;
        Point p = (Point) o;
        return x == p.x && y == p.y;
    }
}
// Rule: If a.equals(b), then a.hashCode() == b.hashCode() MUST hold!
```

# 3. Collision Resolution

## What is a Collision?

```
Two different keys map to the same index:
  h("abc") = 5
  h("xyz") = 5  ← COLLISION!

Both need to go to index 5. How?
```

## Method 1: Chaining (Java's HashMap uses this)

```
Each array slot holds a LINKED LIST (or tree for long chains).
All keys with same hash go into the same chain.

index 5: → ["abc":"value1"] → ["xyz":"value2"] → null

Lookup "abc": go to index 5, search chain for "abc" → O(1) avg, O(n) worst

Java 8+: When chain length > 8, converts to RED-BLACK TREE → O(log n) worst!
```

```java
// Conceptual view of Java HashMap internals:
//
// Array of size 16 initially:
// [0]: null
// [1]: Entry("apple","fruit") → null
// [2]: null
// [3]: Entry("cat","animal") → Entry("bat","animal") → null  ← chain!
// ...
//
// When load factor > 0.75: resize to double capacity, rehash all entries
```

## Method 2: Open Addressing (Linear Probing)

```
When collision at index i, try i+1, i+2, ... (wrap around)

h("abc") = 5, but index 5 is taken → try 6 → try 7... until empty slot

PROS: Better cache performance (no pointer chasing)
CONS: Clustering (consecutive filled slots)
USED IN: Python dictionaries, some databases
```

## Method 3: Double Hashing

```
When collision at index i: try i + k*h2(key) for k=1,2,3...
Uses a second hash function h2 to determine step size.
Better distribution than linear probing.
```

# 4. Load Factor and Rehashing

```
LOAD FACTOR = n / capacity
  n = number of entries currently stored
  capacity = size of internal array

Java's default load factor = 0.75 (75% full → resize)

When load factor exceeds threshold:
  1. Create new array of DOUBLE size
  2. Rehash ALL existing entries into new array
  3. This is O(n) but happens rarely → amortized O(1) per insert

INITIAL CAPACITY = 16 (always power of 2 for fast modulo: index = hash & (cap-1))

Why 0.75? Trade-off:
  Lower LF = less collisions but wastes memory
  Higher LF = more collisions but saves memory
  0.75 is empirically the sweet spot

// Specify initial capacity if you know size:
Map<String, Integer> map = new HashMap<>(1000);  // Avoids resizing
```

# 5. Time and Space Complexity

```
Operation   | Average | Worst Case | Notes
------------|---------|------------|--------------------------------------
get(key)    | O(1)    | O(n)       | Worst: all keys in same bucket
put(key,val)| O(1)    | O(n)       | Java 8: O(log n) with tree buckets
remove(key) | O(1)    | O(n)       |
containsKey | O(1)    | O(n)       |
size()      | O(1)    | O(1)       | Tracked separately
iteration   | O(n)    | O(n)       | Must visit all buckets

Space: O(n) for n entries (plus overhead for empty buckets)

Java 8+ improvement:
  Chain length > 8 → converts to Red-Black Tree
  Worst case: O(log n) instead of O(n)
  This handles hash flooding attacks (intentional collisions)
```

```java
import java.util.*;

public class HashMapGuide {
    public static void main(String[] args) {

        // ===== CREATE =====
        Map<String, Integer> map = new HashMap<>();
        Map<String, Integer> withCapacity = new HashMap<>(100);      // Initial capacity
        Map<String, Integer> copy = new HashMap<>(anotherMap);        // Copy constructor

        // ===== PUT / ADD =====
        map.put("apple", 1);
        map.put("banana", 2);
        map.put("apple", 99);     // OVERWRITES previous value for "apple"

        // ===== GET =====
        int val = map.get("apple");          // Returns 99
        int val2 = map.getOrDefault("cherry", 0);  // Returns 0 (not found)

        // ===== CHECK EXISTENCE =====
        map.containsKey("apple");    // true
        map.containsValue(99);       // true (slower: O(n))

        // ===== REMOVE =====
        map.remove("banana");                    // Remove by key
        map.remove("apple", 99);                 // Remove only if key=apple AND value=99

        // ===== SIZE =====
        map.size();     // Number of key-value pairs
        map.isEmpty();  // true if size() == 0

        // ===== ITERATE =====
        // 1. Iterate entries (most common)
        for (Map.Entry<String, Integer> entry : map.entrySet()) {
            System.out.println(entry.getKey() + " → " + entry.getValue());
        }

        // 2. Iterate keys only
        for (String key : map.keySet()) {
            System.out.println(key);
        }

        // 3. Iterate values only
        for (int value : map.values()) {
            System.out.println(value);
        }

        // 4. forEach (Java 8+)
        map.forEach((key, value) -> System.out.println(key + "=" + value));

        // ===== ADVANCED OPERATIONS =====

        // putIfAbsent: only put if key doesn't exist
        map.putIfAbsent("cherry", 3);       // Adds cherry=3
        map.putIfAbsent("apple", 100);      // Does NOT overwrite (apple exists)

        // getOrDefault: return default if key missing
        int count = map.getOrDefault("durian", 0);

        // computeIfAbsent: compute value lazily if key missing
        map.computeIfAbsent("elderberry", k -> k.length());  // Sets elderberry=10

        // merge: add to existing value or put new
        map.merge("apple", 1, Integer::sum);  // apple = 99 + 1 = 100

        // compute: recompute value for key
        map.compute("apple", (k, v) -> v == null ? 1 : v + 1);

        // FREQUENCY COUNTING (most common pattern):
        String[] words = {"apple", "banana", "apple", "cherry", "banana", "apple"};
        Map<String, Integer> freq = new HashMap<>();
        for (String word : words) {
            freq.merge(word, 1, Integer::sum);  // Clean one-liner!
            // OR:
            // freq.put(word, freq.getOrDefault(word, 0) + 1);
        }
        System.out.println(freq); // {apple=3, banana=2, cherry=1}
    }
}
```

# 7. HashSet — Complete Guide

```java
import java.util.*;

public class HashSetGuide {
    public static void main(String[] args) {

        // ===== CREATE =====
        Set<Integer> set = new HashSet<>();
        Set<Integer> fromList = new HashSet<>(Arrays.asList(1, 2, 3, 4, 5));

        // ===== ADD / REMOVE =====
        set.add(1);
        set.add(2);
        set.add(1);         // Duplicate — ignored! Set has no duplicates
        set.remove(2);

        // ===== CHECK =====
        set.contains(1);    // true — O(1)
        set.contains(99);   // false
        set.size();         // 1
        set.isEmpty();      // false

        // ===== SET OPERATIONS =====
        Set<Integer> a = new HashSet<>(Arrays.asList(1, 2, 3, 4));
        Set<Integer> b = new HashSet<>(Arrays.asList(3, 4, 5, 6));

        // UNION: a ∪ b
        Set<Integer> union = new HashSet<>(a);
        union.addAll(b);    // {1, 2, 3, 4, 5, 6}

        // INTERSECTION: a ∩ b
        Set<Integer> intersection = new HashSet<>(a);
        intersection.retainAll(b);   // {3, 4}

        // DIFFERENCE: a - b
        Set<Integer> difference = new HashSet<>(a);
        difference.removeAll(b);     // {1, 2}

        // IS SUBSET: a ⊆ b?
        b.containsAll(a);   // false

        // ===== ITERATE =====
        for (int num : set) System.out.println(num);

        // ===== COMMON PATTERN: Remove duplicates from array =====
        int[] arr = {1, 2, 2, 3, 3, 3, 4};
        Set<Integer> unique = new HashSet<>();
        for (int n : arr) unique.add(n);
        // unique = {1, 2, 3, 4}

        // ===== COMMON PATTERN: Check if seen before =====
        boolean hasDuplicate(int[] nums) {
            Set<Integer> seen = new HashSet<>();
            for (int num : nums) {
                if (!seen.add(num)) return true;  // add returns false if already present
            }
            return false;
        }
    }
}
```

# 8. LinkedHashMap — Insertion Order

```java
// LinkedHashMap maintains INSERTION ORDER (unlike HashMap)
// Used for: LRU cache, maintaining order of first appearance

Map<String, Integer> linked = new LinkedHashMap<>();
linked.put("banana", 2);
linked.put("apple", 1);
linked.put("cherry", 3);

// Iteration in INSERTION ORDER:
for (String key : linked.keySet()) {
    System.out.println(key);  // banana, apple, cherry (insertion order!)
}

// ACCESS ORDER mode — moves recently accessed to end (perfect for LRU!)
Map<String, Integer> lruMap = new LinkedHashMap<>(16, 0.75f, true) {
    @Override
    protected boolean removeEldestEntry(Map.Entry<String, Integer> eldest) {
        return size() > 3;  // Remove oldest when size exceeds 3
    }
};

lruMap.put("a", 1);
lruMap.put("b", 2);
lruMap.put("c", 3);
lruMap.get("a");    // Access "a" → moves to end
lruMap.put("d", 4); // Exceeds size 3 → removes "b" (oldest unaccessed)
// lruMap = {c=3, a=1, d=4}
```

# 10. Choosing the Right Map/Set

```
WHICH STRUCTURE TO USE?

HashMap  → FASTEST, no order, O(1) avg. Use by default.
LinkedHashMap → Insertion or access order. LRU cache, order matters.
TreeMap  → Sorted keys, O(log n). Need floor/ceiling/range queries.

HashSet  → Fast membership check, no order.
LinkedHashSet → Membership + insertion order.
TreeSet  → Sorted membership, floor/ceiling, range.

CHEAT SHEET:
Need O(1) lookup?                → HashMap / HashSet
Need sorted keys?                → TreeMap / TreeSet
Need insertion-order iteration?  → LinkedHashMap / LinkedHashSet
Need LRU cache?                  → LinkedHashMap (access order mode)
Need floor/ceiling/range?        → TreeMap / TreeSet

COMMON PATTERNS:
Count frequency         → Map<T, Integer>
Check membership        → Set<T>
Group elements          → Map<K, List<V>>
Track seen elements     → Set<T>
Count distinct values   → Set<T>.size()
```

# 11. Custom HashMap Implementation

```java
public class MyHashMap {

    static class Entry {
        int key, val;
        Entry next;
        Entry(int key, int val) { this.key = key; this.val = val; }
    }

    private static final int CAPACITY = 1024;
    private Entry[] buckets;

    public MyHashMap() {
        buckets = new Entry[CAPACITY];
    }

    // Hash function: map key to bucket index
    private int hash(int key) {
        return key % CAPACITY;
    }

    // PUT: insert or update key-value pair
    public void put(int key, int value) {
        int idx = hash(key);
        Entry head = buckets[idx];

        // Search chain for existing key
        for (Entry e = head; e != null; e = e.next) {
            if (e.key == key) {
                e.val = value;  // Update existing
                return;
            }
        }

        // Key not found: insert at front of chain
        Entry newEntry = new Entry(key, value);
        newEntry.next = head;
        buckets[idx] = newEntry;
    }

    // GET: retrieve value for key
    public int get(int key) {
        int idx = hash(key);
        for (Entry e = buckets[idx]; e != null; e = e.next) {
            if (e.key == key) return e.val;
        }
        return -1;  // Not found
    }

    // REMOVE: delete key-value pair
    public void remove(int key) {
        int idx = hash(key);
        Entry head = buckets[idx];

        if (head == null) return;

        if (head.key == key) {
            buckets[idx] = head.next;  // Remove head
            return;
        }

        // Find and remove from chain
        for (Entry e = head; e.next != null; e = e.next) {
            if (e.next.key == key) {
                e.next = e.next.next;  // Skip the node
                return;
            }
        }
    }

    public static void main(String[] args) {
        MyHashMap map = new MyHashMap();
        map.put(1, 1);
        map.put(2, 2);
        System.out.println(map.get(1));    // 1
        System.out.println(map.get(3));    // -1
        map.put(2, 1);
        System.out.println(map.get(2));    // 1 (updated)
        map.remove(2);
        System.out.println(map.get(2));    // -1
    }
}
```

## 🟡 LC #347 — Top K Frequent Elements

```java
public int[] topKFrequent(int[] nums, int k) {
    // Step 1: Count frequencies
    Map<Integer, Integer> freq = new HashMap<>();
    for (int num : nums) freq.merge(num, 1, Integer::sum);

    // Step 2: Use min-heap of size k
    PriorityQueue<Integer> minHeap = new PriorityQueue<>(
        (a, b) -> freq.get(a) - freq.get(b)
    );

    for (int num : freq.keySet()) {
        minHeap.offer(num);
        if (minHeap.size() > k) minHeap.poll();  // Remove least frequent
    }

    int[] result = new int[k];
    for (int i = k - 1; i >= 0; i--) result[i] = minHeap.poll();
    return result;
}

// Alternative: Bucket Sort O(n)
public int[] topKFrequentBucket(int[] nums, int k) {
    Map<Integer, Integer> freq = new HashMap<>();
    for (int num : nums) freq.merge(num, 1, Integer::sum);

    // Bucket: index = frequency
    List<Integer>[] bucket = new List[nums.length + 1];
    for (int num : freq.keySet()) {
        int f = freq.get(num);
        if (bucket[f] == null) bucket[f] = new ArrayList<>();
        bucket[f].add(num);
    }

    int[] result = new int[k];
    int idx = 0;
    for (int f = nums.length; f >= 0 && idx < k; f--) {
        if (bucket[f] != null) {
            for (int num : bucket[f]) {
                if (idx < k) result[idx++] = num;
            }
        }
    }
    return result;
}
```

## 🟡 LC #451 — Sort Characters By Frequency

```java
public String frequencySort(String s) {
    Map<Character, Integer> freq = new HashMap<>();
    for (char c : s.toCharArray()) freq.merge(c, 1, Integer::sum);

    // Sort characters by frequency (descending)
    List<Character> chars = new ArrayList<>(freq.keySet());
    chars.sort((a, b) -> freq.get(b) - freq.get(a));

    StringBuilder sb = new StringBuilder();
    for (char c : chars) {
        for (int i = 0; i < freq.get(c); i++) sb.append(c);
    }
    return sb.toString();
}
```

# 13. Pattern 2 — Two Sum / Complement Pattern

## The Key Insight

```
For each element x, check if its COMPLEMENT (target - x) exists in a set/map.
This turns O(n²) brute force into O(n).

Phase 1: Build set/map of elements seen so far
Phase 2: For each new element, check if complement exists
```

---

## 🟢 LC #1 — Two Sum

```java
public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> seen = new HashMap<>();  // value → index

    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];

        if (seen.containsKey(complement)) {
            return new int[]{seen.get(complement), i};
        }

        seen.put(nums[i], i);  // Store AFTER checking (handle same element)
    }

    return new int[]{};  // No solution
}
```

## 🟡 LC #15 — 3Sum (Sort + Two Pointers, but also hashable)

```java
// HashMap approach for 3Sum
public List<List<Integer>> threeSum(int[] nums) {
    Arrays.sort(nums);
    Set<List<Integer>> result = new HashSet<>();  // Dedup using set

    for (int i = 0; i < nums.length - 2; i++) {
        Set<Integer> seen = new HashSet<>();
        for (int j = i + 1; j < nums.length; j++) {
            int complement = -nums[i] - nums[j];
            if (seen.contains(complement)) {
                List<Integer> triple = Arrays.asList(nums[i], complement, nums[j]);
                Collections.sort(triple);
                result.add(triple);
            }
            seen.add(nums[j]);
        }
    }
    return new ArrayList<>(result);
}
```

## 🟡 LC #454 — 4Sum II

```java
// Count tuples (i,j,k,l) such that A[i]+B[j]+C[k]+D[l] == 0
public int fourSumCount(int[] nums1, int[] nums2, int[] nums3, int[] nums4) {
    Map<Integer, Integer> sumAB = new HashMap<>();

    // Store all pairwise sums of A and B
    for (int a : nums1) {
        for (int b : nums2) {
            sumAB.merge(a + b, 1, Integer::sum);
        }
    }

    int count = 0;
    // For each C+D, check if -(C+D) exists in sumAB
    for (int c : nums3) {
        for (int d : nums4) {
            count += sumAB.getOrDefault(-(c + d), 0);
        }
    }

    return count;
}
```

## 🟡 LC #560 — Subarray Sum Equals K

```java
// Count subarrays with sum equal to k
public int subarraySum(int[] nums, int k) {
    Map<Integer, Integer> prefixCount = new HashMap<>();
    prefixCount.put(0, 1);  // Empty subarray has sum 0

    int count = 0, runningSum = 0;

    for (int num : nums) {
        runningSum += num;
        // If (runningSum - k) appeared before, those subarrays sum to k
        count += prefixCount.getOrDefault(runningSum - k, 0);
        prefixCount.merge(runningSum, 1, Integer::sum);
    }

    return count;
}
```

# 14. Pattern 3 — Grouping / Anagram Pattern

## The Key Insight

```
Use a CANONICAL FORM as the key.
Elements with the same canonical form belong to the same group.

For anagrams: canonical = sorted string ("eat"→"aet", "tea"→"aet")
For equal content: canonical = sorted array, XOR, etc.
```

---

## 🟡 LC #49 — Group Anagrams

```java
public List<List<String>> groupAnagrams(String[] strs) {
    Map<String, List<String>> groups = new HashMap<>();

    for (String s : strs) {
        // Canonical form: sorted characters
        char[] chars = s.toCharArray();
        Arrays.sort(chars);
        String key = new String(chars);

        groups.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
    }

    return new ArrayList<>(groups.values());
}

// Optimized: use character count as key (avoids sorting)
```

```java
public List<Integer> findAnagrams(String s, String p) {
    List<Integer> result = new ArrayList<>();
    if (s.length() < p.length()) return result;

    int[] pCount = new int[26];
    int[] window = new int[26];

    for (char c : p.toCharArray()) pCount[c - 'a']++;

    int left = 0, matches = 0;

    for (int right = 0; right < s.length(); right++) {
        window[s.charAt(right) - 'a']++;

        // Check if right character now matches p's requirement
        if (window[s.charAt(right) - 'a'] == pCount[s.charAt(right) - 'a']) matches++;

        // Shrink window if too large
        if (right - left + 1 > p.length()) {
            char leftChar = s.charAt(left);
            if (window[leftChar - 'a'] == pCount[leftChar - 'a']) matches--;
            window[leftChar - 'a']--;
            left++;
        }

        if (matches == 26) result.add(left);
    }

    return result;
}
```

## The Key Insight

```
Prefix sum + HashMap enables range query in O(1).

Core trick: If prefix[j] - prefix[i] = k,
            then subarray from i+1 to j sums to k.

So: for each prefix sum P, check if (P - k) has appeared before.
```

```java
public boolean checkSubarraySum(int[] nums, int k) {
    // prefix sum mod k → first index it appeared
    Map<Integer, Integer> modIndex = new HashMap<>();
    modIndex.put(0, -1);  // Empty prefix has mod = 0 at index -1

    int sum = 0;
    for (int i = 0; i < nums.length; i++) {
        sum = (sum + nums[i]) % k;

        if (modIndex.containsKey(sum)) {
            if (i - modIndex.get(sum) >= 2) return true;  // Subarray length >= 2
        } else {
            modIndex.put(sum, i);  // Store FIRST occurrence
        }
    }
    return false;
}
```

## 🔴 LC #76 — Minimum Window Substring

```java
public String minWindow(String s, String t) {
    Map<Character, Integer> need = new HashMap<>();
    for (char c : t.toCharArray()) need.merge(c, 1, Integer::sum);

    int left = 0, formed = 0, required = need.size();
    int minLen = Integer.MAX_VALUE, minLeft = 0;
    Map<Character, Integer> window = new HashMap<>();

    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        window.merge(c, 1, Integer::sum);

        if (need.containsKey(c) && window.get(c).equals(need.get(c))) formed++;

        // Shrink window when all required chars are included
        while (formed == required) {
            if (right - left + 1 < minLen) {
                minLen = right - left + 1;
                minLeft = left;
            }
            char leftChar = s.charAt(left);
            window.merge(leftChar, -1, Integer::sum);
            if (need.containsKey(leftChar) && window.get(leftChar) < need.get(leftChar)) formed--;
            left++;
        }
    }

    return minLen == Integer.MAX_VALUE ? "" : s.substring(minLeft, minLeft + minLen);
}
```

## 🔴 LC #146 — LRU Cache

```java
// Least Recently Used Cache
// get and put both O(1) using LinkedHashMap

class LRUCache extends LinkedHashMap<Integer, Integer> {
    private int capacity;

    public LRUCache(int capacity) {
        super(capacity, 0.75f, true);  // accessOrder = true (move accessed to end)
        this.capacity = capacity;
    }

    public int get(int key) {
        return getOrDefault(key, -1);
    }

    public void put(int key, int value) {
        super.put(key, value);
    }

    @Override
    protected boolean removeEldestEntry(Map.Entry<Integer, Integer> eldest) {
        return size() > capacity;
    }
}

// OR: Manual implementation with Doubly LL + HashMap
class LRUCacheManual {
    class Node {
        int key, val;
        Node prev, next;
        Node(int k, int v) { key = k; val = v; }
    }

    int capacity;
    Map<Integer, Node> map;
    Node head, tail;  // Dummy nodes

    public LRUCacheManual(int capacity) {
        this.capacity = capacity;
        map = new HashMap<>();
        head = new Node(0, 0);
        tail = new Node(0, 0);
        head.next = tail;
        tail.prev = head;
    }

    public int get(int key) {
        if (!map.containsKey(key)) return -1;
        Node node = map.get(key);
        remove(node);
        addToTail(node);
        return node.val;
    }

    public void put(int key, int val) {
        if (map.containsKey(key)) {
            Node node = map.get(key);
            node.val = val;
            remove(node);
            addToTail(node);
        } else {
            if (map.size() == capacity) {
                Node lru = head.next;
                remove(lru);
                map.remove(lru.key);
            }
            Node newNode = new Node(key, val);
            addToTail(newNode);
            map.put(key, newNode);
        }
    }

    private void remove(Node node) {
        node.prev.next = node.next;
        node.next.prev = node.prev;
    }

    private void addToTail(Node node) {
        node.prev = tail.prev;
        node.next = tail;
        tail.prev.next = node;
        tail.prev = node;
    }
}
```

## 🔴 LC #460 — LFU Cache

```java
class LFUCache {
    int capacity, minFreq;
    Map<Integer, Integer> keyVal;       // key → value
    Map<Integer, Integer> keyFreq;      // key → frequency
    Map<Integer, LinkedHashSet<Integer>> freqKeys; // freq → ordered set of keys

    public LFUCache(int capacity) {
        this.capacity = capacity;
        keyVal  = new HashMap<>();
        keyFreq = new HashMap<>();
        freqKeys = new HashMap<>();
    }

    public int get(int key) {
        if (!keyVal.containsKey(key)) return -1;
        incrementFreq(key);
        return keyVal.get(key);
    }

    public void put(int key, int value) {
        if (capacity <= 0) return;
        if (keyVal.containsKey(key)) {
            keyVal.put(key, value);
            incrementFreq(key);
            return;
        }
        if (keyVal.size() == capacity) evict();
        keyVal.put(key, value);
        keyFreq.put(key, 1);
        freqKeys.computeIfAbsent(1, k -> new LinkedHashSet<>()).add(key);
        minFreq = 1;
    }

    private void incrementFreq(int key) {
        int f = keyFreq.get(key);
        keyFreq.put(key, f + 1);
        freqKeys.get(f).remove(key);
        if (freqKeys.get(f).isEmpty()) {
            freqKeys.remove(f);
            if (minFreq == f) minFreq++;
        }
        freqKeys.computeIfAbsent(f + 1, k -> new LinkedHashSet<>()).add(key);
    }

    private void evict() {
        LinkedHashSet<Integer> minKeys = freqKeys.get(minFreq);
        int evict = minKeys.iterator().next();
        minKeys.remove(evict);
        if (minKeys.isEmpty()) freqKeys.remove(minFreq);
        keyVal.remove(evict);
        keyFreq.remove(evict);
    }
}
```

```java
public String fractionToDecimal(int numerator, int denominator) {
    if (numerator == 0) return "0";

    StringBuilder sb = new StringBuilder();
    if ((numerator < 0) ^ (denominator < 0)) sb.append('-');

    long num = Math.abs((long) numerator);
    long den = Math.abs((long) denominator);

    sb.append(num / den);
    long remainder = num % den;
    if (remainder == 0) return sb.toString();

    sb.append('.');
    Map<Long, Integer> seen = new HashMap<>();  // remainder → position in decimal

        if (seen.containsKey(remainder)) {
            sb.insert(seen.get(remainder), '(');
            sb.append(')');
            break;
        }
        seen.put(remainder, sb.length());
        remainder *= 10;
        sb.append(remainder / den);
        remainder %= den;
    }

    return sb.toString();
}
```

## 🟡 LC #721 — Accounts Merge

```java
public List<List<String>> accountsMerge(List<List<String>> accounts) {
    Map<String, String> emailToName = new HashMap<>();
    Map<String, String> parent = new HashMap<>();

    // Initialize union-find
    for (List<String> account : accounts) {
        String name = account.get(0);
        for (int i = 1; i < account.size(); i++) {
            String email = account.get(i);
            emailToName.put(email, name);
            parent.put(email, email);  // Self-parent initially
        }
    }

    // Union emails in same account
    for (List<String> account : accounts) {
        String first = account.get(1);
        for (int i = 2; i < account.size(); i++) {
            union(parent, first, account.get(i));
        }
    }

    // Group by root parent
    Map<String, List<String>> groups = new HashMap<>();
    for (String email : parent.keySet()) {
        String root = find(parent, email);
        groups.computeIfAbsent(root, k -> new ArrayList<>()).add(email);
    }

    List<List<String>> result = new ArrayList<>();
    for (Map.Entry<String, List<String>> entry : groups.entrySet()) {
        List<String> emails = entry.getValue();
        Collections.sort(emails);
        emails.add(0, emailToName.get(entry.getKey()));
        result.add(emails);
    }

    return result;
}

private String find(Map<String, String> parent, String x) {
    if (!parent.get(x).equals(x)) parent.put(x, find(parent, parent.get(x)));
    return parent.get(x);
}

private void union(Map<String, String> parent, String x, String y) {
    parent.put(find(parent, x), find(parent, y));
}
```

```java
public int firstMissingPositive(int[] nums) {

    // Mark out-of-range values
    for (int i = 0; i < n; i++) if (nums[i] <= 0 || nums[i] > n) nums[i] = n + 1;

        if (val <= n) nums[val - 1] = -Math.abs(nums[val - 1]);
    }

    // First positive index + 1 is missing
    for (int i = 0; i < n; i++) if (nums[i] > 0) return i + 1;
    return n + 1;
}
```

```java
public int maxPoints(int[][] points) {

    int max = 2;

        Map<String, Integer> slopes = new HashMap<>();

    }

}
```

## 🔴 LC #691 — Stickers to Spell Word

```java
public int minStickers(String[] stickers, String target) {
    int n = target.length();
    int[] dp = new int[1 << n];  // dp[mask] = min stickers to cover chars in mask
    Arrays.fill(dp, -1);
    dp[0] = 0;

    for (int mask = 0; mask < (1 << n); mask++) {
        if (dp[mask] == -1) continue;

        for (String sticker : stickers) {
            int cur = mask;
            for (char c : sticker.toCharArray()) {
                for (int i = 0; i < n; i++) {
                    if ((cur >> i & 1) == 0 && target.charAt(i) == c) {
                        cur |= (1 << i);
                        break;
                    }
                }
            }
            if (dp[cur] == -1 || dp[cur] > dp[mask] + 1) dp[cur] = dp[mask] + 1;
        }
    }

    return dp[(1 << n) - 1];
}
```

# 22. How to Identify Hashing Problems

```
Keywords → Hashing Pattern
──────────────────────────────────────────────────────────────
"two sum" / "pair with sum k"     → Map: value → index
"count occurrences"               → Map: element → count
"find duplicate"                  → Set: check before add
"group / anagram / same letters"  → Map: canonical → list
"subarray sum = k"                → Map: prefix sum → count
"longest substring"               → Map/Set in sliding window
"most frequent"                   → Map + heap/bucket sort
"O(1) lookup"                     → HashMap
"O(1) insert+delete+random"       → HashMap + ArrayList
"LRU cache"                       → LinkedHashMap or DLL + Map
"first unique"                    → LinkedHashMap or freq map
"isomorphic / bijection"          → Two maps (both directions)
"connected components"            → Union-Find with map keys
"sorted by key"                   → TreeMap
"insertion order"                 → LinkedHashMap

DECISION TREE:
├── Need O(1) lookup?             → HashMap / HashSet
├── Need to count elements?       → Map<E, Integer>
├── Need to group elements?       → Map<K, List<E>>
├── Check membership efficiently? → HashSet
├── Need sorted keys?             → TreeMap
├── Need insertion order?         → LinkedHashMap
├── Subarray with target sum?     → HashMap + prefix sum
├── Longest substring with constraint? → HashMap + sliding window
└── Cache design?                 → LRU: LinkedHashMap, LFU: two maps
```

# 23. Interview Cheat Sheet

## ⚡ Most Common HashMap Operations

```java
// FREQUENCY COUNT (most used pattern)
Map<T, Integer> freq = new HashMap<>();
for (T item : items) freq.merge(item, 1, Integer::sum);
// OR:
freq.put(item, freq.getOrDefault(item, 0) + 1);

// GROUPING
Map<K, List<V>> groups = new HashMap<>();
groups.computeIfAbsent(key, k -> new ArrayList<>()).add(value);

// TWO SUM COMPLEMENT
Map<Integer, Integer> seen = new HashMap<>();
for (int i = 0; i < nums.length; i++) {
    if (seen.containsKey(target - nums[i])) return answer;
    seen.put(nums[i], i);
}

// PREFIX SUM + MAP
Map<Integer, Integer> prefixCount = new HashMap<>();
prefixCount.put(0, 1);
int sum = 0;
for (int num : nums) {
    sum += num;
    count += prefixCount.getOrDefault(sum - k, 0);
    prefixCount.merge(sum, 1, Integer::sum);
}

// SLIDING WINDOW WITH MAP
Map<Character, Integer> window = new HashMap<>();
int left = 0;
for (int right = 0; right < s.length(); right++) {
    window.merge(s.charAt(right), 1, Integer::sum);
    while (violatesConstraint(window)) {
        window.merge(s.charAt(left), -1, Integer::sum);
        if (window.get(s.charAt(left)) == 0) window.remove(s.charAt(left));
        left++;
    }
    maxLen = Math.max(maxLen, right - left + 1);
}
```

## ⚡ HashMap vs TreeMap vs LinkedHashMap

```
HashMap:        O(1) avg  — unordered   → DEFAULT CHOICE
TreeMap:        O(log n)  — sorted keys → need floor/ceil/range
LinkedHashMap:  O(1) avg  — ordered     → LRU cache / order matters

// TreeMap navigation:
treeMap.floorKey(k)   // largest key ≤ k
treeMap.ceilingKey(k) // smallest key ≥ k
treeMap.lowerKey(k)   // largest key < k
treeMap.higherKey(k)  // smallest key > k
treeMap.firstKey()    // minimum key
treeMap.lastKey()     // maximum key
```

## ⚡ Complexity Reference

```
HashMap / HashSet:
  get/put/remove/contains: O(1) average, O(n) worst
  Iteration: O(n + capacity)

TreeMap / TreeSet:
  get/put/remove/contains: O(log n)
  floor/ceiling/first/last: O(log n)
  Iteration: O(n) in sorted order

LinkedHashMap / LinkedHashSet:
  Same as HashMap but maintains insertion/access order
  Iteration: O(n) in order
```

## ⚡ The 5 Golden Rules of Hashing

```
1. GETORDEFAULT for counting:
   map.getOrDefault(key, 0) + 1  — never get NPE on first occurrence.

2. TWO-MAP BIJECTION for isomorphic/pattern problems:
   Map<A,B> forward + Map<B,A> backward. Check both directions.

3. MERGE for clean frequency counting:
   map.merge(key, 1, Integer::sum)  — one-liner, cleaner than getOrDefault.

4. PREFIX SUM pattern for subarray sums:
   Always initialize with put(0, 1) or put(0, -1) for empty prefix.

5. WHEN TO USE SET vs MAP:
   SET: just checking existence ("have I seen this?")
   MAP: storing additional info ("how many times? at what index?")
```
