# 1. What is a Heap?

## The Core Idea

A **Heap** is a special tree-based data structure that satisfies the **heap property**:

```
MAX-HEAP: Every parent is GREATER THAN or equal to its children
          → Root is always the MAXIMUM element

MIN-HEAP: Every parent is LESS THAN or equal to its children
          → Root is always the MINIMUM element

MAX-HEAP example:           MIN-HEAP example:
        90                          1
       /  \                        / \
      75   80                     3   5
     / \  / \                    / \ / \
    40 60 70 50                 6  8 9  7

get_max() = 90 → O(1)          get_min() = 1 → O(1)
```

## The Heap's Superpower

```
HEAP gives you:
  → O(1)      to peek at max/min
  → O(log n)  to insert
  → O(log n)  to remove max/min
  → O(n)      to build from array

Compare to:
  SORTED ARRAY: O(1) peek, O(n) insert (must shift)
  BST:          O(log n) for all, but complex implementation

HEAP wins when you need REPEATED extraction of max/min
with occasional insertions.

Real world: Priority queue (hospital triage), Dijkstra's algorithm,
            Huffman coding, event simulation, scheduling.
```

# 2. Heap vs Sorted Array vs BST

```
Operation       | Heap      | Sorted Array | BST (balanced)
----------------|-----------|--------------|----------------
Get min/max     | O(1)   ✅ | O(1)         | O(log n)
Insert          | O(log n)✅| O(n) ❌      | O(log n)
Delete min/max  | O(log n)✅| O(1) then O(n) shift | O(log n)
Search any      | O(n) ❌   | O(log n)     | O(log n)
Build from arr  | O(n)   ✅ | O(n log n)   | O(n log n)
Space           | O(n)      | O(n)         | O(n)

KEY INSIGHT: Heap is the BEST structure when you only need
             repeated access to the min or max — nothing else.
             If you need arbitrary search → use BST.
```

# 3. Binary Heap — Internal Structure

## Stored as an Array (Not a Tree!)

```
Even though we VISUALIZE a heap as a tree, it's STORED as an ARRAY.
This makes it extremely cache-friendly and memory-efficient.

Tree:               Array:
        1           [_, 1, 3, 5, 6, 8, 9, 7]
       / \           0  1  2  3  4  5  6  7
      3   5          ↑
     / \ / \         index 0 unused (1-indexed for clean formulas)
    6  8 9  7

Index relationships (1-indexed):
  Parent of node i:    i / 2
  Left child of i:     2 * i
  Right child of i:    2 * i + 1

OR (0-indexed):
  Parent of node i:    (i - 1) / 2
  Left child of i:     2 * i + 1
  Right child of i:    2 * i + 2

WHY ARRAY? No pointers needed. Children of arr[i] are always
           arr[2i+1] and arr[2i+2] — direct calculation!
```

# 4. Heap Operations — How They Work

## Insert (Heapify Up / Bubble Up)

```
Add new element at the END of array, then move it UP.

Insert 2 into min-heap [1, 3, 5, 6, 8, 9, 7]:

Step 1: Add at end:
  [1, 3, 5, 6, 8, 9, 7, 2]
  Tree:       1
             / \
            3   5
           / \ / \
          6  8 9  7
               \
                2  ← added here

Step 2: Compare with parent (index 7, parent = index 3 = value 6):
  2 < 6 → SWAP
  [1, 3, 5, 2, 8, 9, 7, 6]

Step 3: Compare with new parent (index 3, parent = index 1 = value 3):
  2 < 3 → SWAP
  [1, 2, 5, 3, 8, 9, 7, 6]

Step 4: Compare with new parent (index 1, parent = index 0 = value 1):
  2 > 1 → STOP

Final: [1, 2, 5, 3, 8, 9, 7, 6] ✅
```

## Extract Min (Heapify Down / Bubble Down)

```
Remove root (minimum), replace with LAST element, move it DOWN.

Extract min from [1, 2, 5, 3, 8, 9, 7, 6]:

Step 1: Remove root (1), put last element (6) at root:
  [6, 2, 5, 3, 8, 9, 7]
  Tree:       6
             / \
            2   5
           / \ / \
          3  8 9  7

Step 2: Compare 6 with children (2 and 5). Min child = 2 < 6 → SWAP:
  [2, 6, 5, 3, 8, 9, 7]

Step 3: Compare 6 with children (3 and 8). Min child = 3 < 6 → SWAP:
  [2, 3, 5, 6, 8, 9, 7]

Step 4: 6 is at index 3 (leaf), no children → STOP

Final min-heap: [2, 3, 5, 6, 8, 9, 7] ✅
Extracted: 1
```

# 5. Implement Heap from Scratch

```java
public class MinHeap {

    private int[] heap;
    private int size;
    private int capacity;

    public MinHeap(int capacity) {
        this.capacity = capacity;
        heap = new int[capacity + 1]; // 1-indexed
        size = 0;
    }

    // ═══ HELPER METHODS ═══════════════════════════════════

    private int parent(int i)    { return i / 2; }
    private int leftChild(int i) { return 2 * i; }
    private int rightChild(int i){ return 2 * i + 1; }
    private boolean hasLeft(int i)  { return leftChild(i)  <= size; }
    private boolean hasRight(int i) { return rightChild(i) <= size; }

    private void swap(int i, int j) {
        int temp = heap[i]; heap[i] = heap[j]; heap[j] = temp;
    }

    // ═══ INSERT — O(log n) ════════════════════════════════

    public void insert(int val) {
        if (size == capacity) throw new RuntimeException("Heap is full");
        heap[++size] = val;      // Add at end
        bubbleUp(size);          // Restore heap property
    }

    private void bubbleUp(int i) {
        while (i > 1 && heap[parent(i)] > heap[i]) {
            swap(i, parent(i));  // Parent too large → swap
            i = parent(i);       // Move up
        }
    }

    // ═══ EXTRACT MIN — O(log n) ═══════════════════════════

    public int extractMin() {
        if (size == 0) throw new RuntimeException("Heap is empty");
        int min = heap[1];       // Root is minimum
        heap[1] = heap[size--];  // Replace root with last element
        bubbleDown(1);           // Restore heap property
        return min;
    }

    private void bubbleDown(int i) {
        while (hasLeft(i)) {
            int smallest = leftChild(i);

            // Find smaller child
            if (hasRight(i) && heap[rightChild(i)] < heap[smallest]) {
                smallest = rightChild(i);
            }

            if (heap[i] <= heap[smallest]) break; // Heap property satisfied

            swap(i, smallest);   // Swap with smaller child
            i = smallest;        // Move down
        }
    }

    // ═══ PEEK — O(1) ═════════════════════════════════════

    public int peekMin() {
        if (size == 0) throw new RuntimeException("Heap is empty");
        return heap[1];
    }

    // ═══ BUILD HEAP — O(n) ════════════════════════════════
    // Much faster than inserting n elements one by one (O(n log n))

    public void buildHeap(int[] arr) {
        size = arr.length;
        heap = new int[size + 1];
        for (int i = 0; i < size; i++) heap[i + 1] = arr[i];

        // Heapify from last internal node DOWN to root
        for (int i = size / 2; i >= 1; i--) {
            bubbleDown(i);
        }
        // WHY O(n)? Most nodes are near the bottom (leaves don't need bubbling)
        // Mathematical proof: sum of (height × nodes at that height) = O(n)
    }

    // ═══ HEAP SORT — O(n log n) ═══════════════════════════

    public static int[] heapSort(int[] arr) {
        // Build max-heap then extract max repeatedly
        int n = arr.length;
        // Build max-heap
        for (int i = n / 2 - 1; i >= 0; i--) siftDown(arr, n, i);
        // Extract elements
        for (int i = n - 1; i > 0; i--) {
            int temp = arr[0]; arr[0] = arr[i]; arr[i] = temp; // Move max to end
            siftDown(arr, i, 0); // Restore heap for remaining elements
        }
        return arr;
    }

    private static void siftDown(int[] arr, int n, int i) {
        int largest = i;
        int left = 2*i+1, right = 2*i+2;
        if (left  < n && arr[left]  > arr[largest]) largest = left;
        if (right < n && arr[right] > arr[largest]) largest = right;
        if (largest != i) {
            int tmp = arr[i]; arr[i] = arr[largest]; arr[largest] = tmp;
            siftDown(arr, n, largest);
        }
    }

    public int size()    { return size; }
    public boolean isEmpty() { return size == 0; }

    public static void main(String[] args) {
        MinHeap heap = new MinHeap(10);
        heap.insert(5); heap.insert(3); heap.insert(8);
        heap.insert(1); heap.insert(4);

        System.out.println("Min: " + heap.peekMin());       // 1
        System.out.println("Extract: " + heap.extractMin()); // 1
        System.out.println("Min: " + heap.peekMin());       // 3

        // Build heap
        MinHeap heap2 = new MinHeap(10);
        heap2.buildHeap(new int[]{5, 3, 8, 1, 4, 2, 7});
        System.out.println("Min after build: " + heap2.peekMin()); // 1
    }
}
```

# 6. PriorityQueue — Complete Guide

## Java's PriorityQueue is a MIN-HEAP by default

```java
import java.util.*;

public class PriorityQueueGuide {
    public static void main(String[] args) {

        // ===== MIN-HEAP (default) — smallest element at top =====
        PriorityQueue<Integer> minHeap = new PriorityQueue<>();
        minHeap.offer(5);
        minHeap.offer(1);
        minHeap.offer(3);
        minHeap.offer(2);

        System.out.println(minHeap.peek());  // 1 (min) — O(1)
        System.out.println(minHeap.poll());  // 1 — removes and returns min — O(log n)
        System.out.println(minHeap.poll());  // 2
        System.out.println(minHeap.size());  // 2

        // ===== MAX-HEAP — largest element at top =====
        PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Collections.reverseOrder());
        // OR: new PriorityQueue<>((a, b) -> b - a);
        maxHeap.offer(5); maxHeap.offer(1); maxHeap.offer(3);
        System.out.println(maxHeap.poll());  // 5 (max)

        // ===== METHODS =====
        PriorityQueue<Integer> pq = new PriorityQueue<>();

        pq.offer(x);     // Insert — O(log n). Returns true.
        pq.add(x);       // Same as offer but throws on failure.
        pq.poll();       // Remove & return min — O(log n). Returns null if empty.
        pq.remove();     // Same as poll but throws if empty.
        pq.peek();       // View min — O(1). Returns null if empty.
        pq.element();    // Same as peek but throws if empty.
        pq.size();       // Number of elements — O(1).
        pq.isEmpty();    // True if empty — O(1).
        pq.contains(x);  // Check membership — O(n) LINEAR!
        pq.remove(x);    // Remove specific element — O(n) LINEAR!
        pq.clear();      // Remove all elements.

        // ===== INITIAL CAPACITY =====
        PriorityQueue<Integer> sized = new PriorityQueue<>(100); // Avoids resizing

        // ===== BUILD FROM COLLECTION =====
        List<Integer> list = Arrays.asList(5, 3, 1, 4, 2);
        PriorityQueue<Integer> fromList = new PriorityQueue<>(list); // O(n) build
        System.out.println(fromList.poll()); // 1

        // ===== ITERATE (NOT in order!) =====
        PriorityQueue<Integer> pq2 = new PriorityQueue<>(Arrays.asList(3,1,4,1,5));
        for (int n : pq2) System.out.print(n + " "); // NOT guaranteed sorted order!
        // To iterate in order: while(!pq.isEmpty()) System.out.print(pq.poll() + " ");
    }
}
```

```java
// ===== SORT OBJECTS =====
// By distance from origin
PriorityQueue<int[]> byDist = new PriorityQueue<>(
    (a, b) -> (a[0]*a[0] + a[1]*a[1]) - (b[0]*b[0] + b[1]*b[1])
);

// By value desc, then by index asc (multi-key)
PriorityQueue<int[]> multiKey = new PriorityQueue<>((a, b) -> {
    if (a[0] != b[0]) return b[0] - a[0];  // value DESC
    return a[1] - b[1];                     // index ASC
});

// ===== STRINGS =====
PriorityQueue<String> byLen = new PriorityQueue<>((a, b) -> a.length() - b.length());
PriorityQueue<String> byLenDesc = new PriorityQueue<>((a, b) -> b.length() - a.length());
PriorityQueue<String> lexMax = new PriorityQueue<>((a, b) -> b.compareTo(a));
```

# 8. Common Heap Patterns in Java

```java
// ===== PATTERN: TOP K — min-heap of size k =====
// Keep only k largest elements; min-heap's top = k-th largest
PriorityQueue<Integer> topK = new PriorityQueue<>(); // min-heap
for (int num : nums) {
    topK.offer(num);
    if (topK.size() > k) topK.poll(); // Remove smallest when size exceeds k
}
// topK.peek() = k-th largest

// ===== PATTERN: K-WAY MERGE =====
// Merge k sorted arrays/lists — always extract global min
PriorityQueue<int[]> merge = new PriorityQueue<>((a, b) -> a[0] - b[0]);
// Initialize: add first element from each list
// Loop: extract min, add next from same list

// ===== PATTERN: TWO HEAPS for MEDIAN =====
PriorityQueue<Integer> lower = new PriorityQueue<>(Collections.reverseOrder()); // max-heap
PriorityQueue<Integer> upper = new PriorityQueue<>(); // min-heap
// lower: bottom half (max at top = smaller median candidate)
// upper: top half (min at top = larger median candidate)
// Invariant: lower.size() == upper.size() OR lower.size() == upper.size() + 1
// Median: lower.size() > upper.size() ? lower.peek() : (lower.peek() + upper.peek()) / 2.0

// ===== PATTERN: LAZY DELETION =====
// When you can't efficiently remove from heap, mark as "deleted"
Set<Integer> deleted = new HashSet<>();
PriorityQueue<Integer> lazyPQ = new PriorityQueue<>();
// ... operations ...
// On poll: skip deleted elements
while (!lazyPQ.isEmpty() && deleted.contains(lazyPQ.peek())) lazyPQ.poll();
```

# 9. Pattern 1 — Top K Elements (Min-Heap of size K)

## The Key Insight

```
To find K LARGEST: use a MIN-heap of size K.
  → Keep evicting the smallest when size > K
  → What remains = K largest, top of heap = K-th largest

To find K SMALLEST: use a MAX-heap of size K.
  → Keep evicting the largest when size > K
  → What remains = K smallest, top of heap = K-th smallest

WHY? Counter-intuitive but correct:
  Min-heap keeps the K largest because whenever a new element
  is larger than the smallest-so-far (heap top), the smallest
  gets evicted and the new one takes its place.
  After processing all elements: heap has the K largest remaining.
```

## 🟡 LC #347 — Top K Frequent Elements

```java
public int[] topKFrequent(int[] nums, int k) {
    // Step 1: Count frequencies
    Map<Integer, Integer> freq = new HashMap<>();
    for (int n : nums) freq.merge(n, 1, Integer::sum);

    // Step 2: Min-heap of size k, ordered by frequency
    PriorityQueue<Integer> minHeap = new PriorityQueue<>(
        (a, b) -> freq.get(a) - freq.get(b)
    );

    for (int num : freq.keySet()) {
        minHeap.offer(num);
        if (minHeap.size() > k) minHeap.poll(); // Evict least frequent
    }

    // Step 3: Build result
    int[] result = new int[k];
    for (int i = k - 1; i >= 0; i--) result[i] = minHeap.poll();
    return result;
}
```

**Complexity:** Time O(n log k), Space O(n + k)

```java
public int[][] kClosest(int[][] points, int k) {
    // Max-heap of size k by distance — keep k SMALLEST distances
    PriorityQueue<int[]> maxHeap = new PriorityQueue<>(
        (a, b) -> (b[0]*b[0]+b[1]*b[1]) - (a[0]*a[0]+a[1]*a[1])

        maxHeap.offer(point);

    }
```

## 🟡 LC #692 — Top K Frequent Words

```java
public List<String> topKFrequent(String[] words, int k) {
    Map<String, Integer> freq = new HashMap<>();
    for (String w : words) freq.merge(w, 1, Integer::sum);

    // Min-heap: lowest freq first; for equal freq, reverse lex order (so lex-smaller stays)
    PriorityQueue<String> minHeap = new PriorityQueue<>(
        (a, b) -> freq.get(a).equals(freq.get(b))
                ? b.compareTo(a)       // Reverse lex: lex-larger gets evicted
                : freq.get(a) - freq.get(b) // Lower freq gets evicted
    );

    for (String word : freq.keySet()) {
        minHeap.offer(word);
        if (minHeap.size() > k) minHeap.poll();
    }

    LinkedList<String> result = new LinkedList<>();
    while (!minHeap.isEmpty()) result.addFirst(minHeap.poll()); // Reverse order
    return result;
}
```

## 🟡 LC #1642 — Furthest Building You Can Reach

```java
public int furthestBuilding(int[] heights, int bricks, int ladders) {
    // Use ladders for the K largest climbs (keep in min-heap)
    PriorityQueue<Integer> ladderJumps = new PriorityQueue<>(); // min-heap

    for (int i = 0; i < heights.length - 1; i++) {
        int diff = heights[i+1] - heights[i];
        if (diff <= 0) continue; // Going down — free

        ladderJumps.offer(diff);

        if (ladderJumps.size() > ladders) {
            // Too many jumps for ladders — use bricks for the smallest jump
            bricks -= ladderJumps.poll();
        }

        if (bricks < 0) return i; // Out of bricks!
    }

    return heights.length - 1; // Made it to the end!
}
```

## 🟡 LC #378 — Kth Smallest Element in a Sorted Matrix

```java
public int kthSmallest(int[][] matrix, int k) {
    int n = matrix.length;
    // Min-heap: [value, row, col]
    PriorityQueue<int[]> minHeap = new PriorityQueue<>((a, b) -> a[0] - b[0]);

    // Start with first element of each row
    for (int i = 0; i < Math.min(n, k); i++) {
        minHeap.offer(new int[]{matrix[i][0], i, 0});
    }

    for (int i = 0; i < k - 1; i++) {
        int[] curr = minHeap.poll();
        int row = curr[1], col = curr[2];
        if (col + 1 < n) {
            minHeap.offer(new int[]{matrix[row][col+1], row, col+1});
        }
    }

    return minHeap.poll()[0];
}
```

# 11. Pattern 3 — Merge K Sorted Lists / Arrays

## The Key Insight

```
K sorted lists → always want the global minimum next.
Use min-heap seeded with the first element from each list.
Extract min → add next from same list → repeat.

Time: O(n log k) where n = total elements, k = number of lists
      Each of n elements does one heap operation of O(log k)

vs naive merge all then sort: O(n log n)
  Heap approach wins when k << n
```

## 🔴 LC #23 — Merge K Sorted Lists

```java
public ListNode mergeKLists(ListNode[] lists) {
    // Min-heap ordered by node value
    PriorityQueue<ListNode> minHeap = new PriorityQueue<>(
        (a, b) -> a.val - b.val
    );

    // Seed heap with head of each list
    for (ListNode list : lists) {
        if (list != null) minHeap.offer(list);
    }

    ListNode dummy = new ListNode(0);
    ListNode curr = dummy;

    while (!minHeap.isEmpty()) {
        ListNode node = minHeap.poll();    // Get global minimum
        curr.next = node;
        curr = curr.next;
        if (node.next != null) {
            minHeap.offer(node.next);      // Add next from same list
        }
    }

    return dummy.next;
}
```

## 🟡 LC #373 — Find K Pairs with Smallest Sums

```java
public List<List<Integer>> kSmallestPairs(int[] nums1, int[] nums2, int k) {
    List<List<Integer>> result = new ArrayList<>();
    if (nums1.length == 0 || nums2.length == 0) return result;

    // Min-heap: [sum, i1 (index in nums1), i2 (index in nums2)]
    PriorityQueue<int[]> minHeap = new PriorityQueue<>((a, b) -> a[0] - b[0]);

    // Start with pairs (nums1[i], nums2[0]) for all i
    for (int i = 0; i < Math.min(nums1.length, k); i++) {
        minHeap.offer(new int[]{nums1[i] + nums2[0], i, 0});
    }

    while (!minHeap.isEmpty() && result.size() < k) {
        int[] curr = minHeap.poll();
        int i = curr[1], j = curr[2];
        result.add(Arrays.asList(nums1[i], nums2[j]));

        // Add next pair from same nums1 element
        if (j + 1 < nums2.length) {
            minHeap.offer(new int[]{nums1[i] + nums2[j+1], i, j+1});
        }
    }

    return result;
}
```

```java
public int[] smallestRange(List<List<Integer>> nums) {
    // Min-heap: [value, listIndex, elementIndex]
    PriorityQueue<int[]> minHeap = new PriorityQueue<>((a, b) -> a[0] - b[0]);
    int maxVal = Integer.MIN_VALUE;

    // Seed with first element from each list
    for (int i = 0; i < nums.size(); i++) {
        int val = nums.get(i).get(0);
        minHeap.offer(new int[]{val, i, 0});
        maxVal = Math.max(maxVal, val);
    }

    int[] result = {minHeap.peek()[0], maxVal};

    while (true) {
        int[] curr = minHeap.poll();
        int val = curr[0], li = curr[1], ei = curr[2];

        // Move to next element in same list
        if (ei + 1 >= nums.get(li).size()) break; // One list exhausted

        int next = nums.get(li).get(ei + 1);
        minHeap.offer(new int[]{next, li, ei + 1});
        maxVal = Math.max(maxVal, next);

        // Update result if range is smaller
        if (maxVal - minHeap.peek()[0] < result[1] - result[0]) {
            result[0] = minHeap.peek()[0];
            result[1] = maxVal;
        }
    }

    return result;
}
```

# 12. Pattern 4 — Two Heaps (Median / Balance)

## The Key Insight

```
Maintain two heaps:
  lower = MAX-heap  (bottom half of all elements, max at top)
  upper = MIN-heap  (top half of all elements, min at top)

Invariant: lower.size() == upper.size()     → median = avg of tops
           lower.size() == upper.size() + 1 → median = lower.peek()

        lower (max-heap)  upper (min-heap)
        [1, 2, 3]         [4, 5, 6]
         ↑                  ↑
        max=3             min=4

Median = (3 + 4) / 2.0 = 3.5 ✅

Adding 7:
  lower = [1,2,3], upper = [4,5,6,7] → unbalanced!
  Move upper.peek() (4) to lower:
  lower = [1,2,3,4], upper = [5,6,7]  → unbalanced other way!
  Hmm... let's add 2 instead:
  lower = [1,2,3,4], upper = [5,6,7] → size 4 vs 3 → OK!
  Median = lower.peek() = 4 ✅
```

## 🔴 LC #295 — Find Median from Data Stream

```java
class MedianFinder {
    private PriorityQueue<Integer> lower; // Max-heap: lower half
    private PriorityQueue<Integer> upper; // Min-heap: upper half

    public MedianFinder() {
        lower = new PriorityQueue<>(Collections.reverseOrder()); // max-heap
        upper = new PriorityQueue<>(); // min-heap
    }

    public void addNum(int num) {
        // Step 1: Add to lower (max-heap)
        lower.offer(num);

        // Step 2: Ensure every lower element <= every upper element
        if (!upper.isEmpty() && lower.peek() > upper.peek()) {
            upper.offer(lower.poll()); // Move lower's max to upper
        }

        // Step 3: Balance sizes: lower can have at most 1 extra
        if (lower.size() > upper.size() + 1) {
            upper.offer(lower.poll());
        } else if (upper.size() > lower.size()) {
            lower.offer(upper.poll());
        }
    }

    public double findMedian() {
        if (lower.size() == upper.size()) {
            return (lower.peek() + upper.peek()) / 2.0;
        }
        return lower.peek(); // lower has 1 extra
    }
}
```

```java
// SIMPLER: Just using delay map for lazy deletion:
public double[] medianSlidingWindowClean(int[] nums, int k) {
    double[] result = new double[nums.length - k + 1];
    TreeMap<Integer, Integer> lo = new TreeMap<>(Collections.reverseOrder());
    TreeMap<Integer, Integer> hi = new TreeMap<>();
    int loSz = 0, hiSz = 0;

    for (int i = 0; i < nums.length; i++) {
        // Add nums[i] to appropriate side
        int num = nums[i];
        if (loSz == 0 || num <= lo.firstKey()) { addMap(lo, num); loSz++; }
        else { addMap(hi, num); hiSz++; }

        // Balance
        while (loSz > hiSz + 1) { addMap(hi, removeFirst(lo)); loSz--; hiSz++; }
        while (hiSz > loSz)     { addMap(lo, removeFirst(hi)); hiSz--; loSz++; }

        // Record median
        if (i >= k - 1) {
            result[i-k+1] = loSz == hiSz
                ? ((double)lo.firstKey() + hi.firstKey()) / 2
                : lo.firstKey();

            // Remove outgoing element
            int out = nums[i-k+1];
            if (out <= lo.firstKey()) { removeMap(lo, out); loSz--; }
            else                       { removeMap(hi, out); hiSz--; }

            // Re-balance after removal
            while (loSz > hiSz + 1) { addMap(hi, removeFirst(lo)); loSz--; hiSz++; }
            while (hiSz > loSz)     { addMap(lo, removeFirst(hi)); hiSz--; loSz++; }
        }
    }
    return result;
}

private void addMap(TreeMap<Integer,Integer> map, int val) {
    map.merge(val, 1, Integer::sum);
}
private void removeMap(TreeMap<Integer,Integer> map, int val) {
    map.merge(val, -1, Integer::sum);
    if (map.get(val) == 0) map.remove(val);
}
private int removeFirst(TreeMap<Integer,Integer> map) {
    int key = map.firstKey(); removeMap(map, key); return key;
}
```

## 🟡 LC #621 — Task Scheduler

```java
public int leastInterval(char[] tasks, int n) {
    int[] freq = new int[26];
    for (char t : tasks) freq[t-'A']++;

    // Max-heap: always execute most frequent task
    PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Collections.reverseOrder());
    for (int f : freq) if (f > 0) maxHeap.offer(f);

    int time = 0;

    while (!maxHeap.isEmpty()) {
        List<Integer> temp = new ArrayList<>();
        int cycle = n + 1; // Execute up to n+1 tasks in each cycle

        for (int i = 0; i < cycle && !maxHeap.isEmpty(); i++) {
            temp.add(maxHeap.poll() - 1); // Execute task
            time++;
        }

        // Add back tasks that still have remaining count
        for (int f : temp) if (f > 0) maxHeap.offer(f);

        // If heap not empty, we need idle time to fill the cycle
        if (!maxHeap.isEmpty()) time += cycle - temp.size();
    }

    return time;
}
```

```java
public int maxEvents(int[][] events) {
    // Sort by start day
    Arrays.sort(events, (a, b) -> a[0] - b[0]);

    // Min-heap: end days of currently available events
    PriorityQueue<Integer> minHeap = new PriorityQueue<>();

    int day = 0, i = 0, attended = 0;

    while (!minHeap.isEmpty() || i < events.length) {
        // Move to next event start day if heap is empty
        if (minHeap.isEmpty()) day = events[i][0];

        // Add all events starting on or before today
        while (i < events.length && events[i][0] <= day) {
            minHeap.offer(events[i][1]); // Add end day
            i++;
        }

        // Attend event ending soonest (greedy)
        minHeap.poll();
        attended++;
        day++;

        // Remove expired events
        while (!minHeap.isEmpty() && minHeap.peek() < day) minHeap.poll();
    }

    return attended;
}
```

## 🟡 LC #2462 — Total Cost to Hire K Workers

```java
public long totalCost(int[] costs, int k, int candidates) {
    int n = costs.length;
    // Min-heap: [cost, index]
    PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) ->
        a[0] != b[0] ? a[0] - b[0] : a[1] - b[1]);

    int left = 0, right = n - 1;
    long total = 0;

    // Add initial candidates from both ends
    for (int i = 0; i < candidates && left <= right; i++, left++) {
        pq.offer(new int[]{costs[left], left});
    }
    for (int i = 0; i < candidates && left <= right; i++, right--) {
        pq.offer(new int[]{costs[right], right});
    }

    for (int i = 0; i < k; i++) {
        int[] worker = pq.poll();
        total += worker[0];

        // Add next worker from the exhausted side
        if (worker[1] < left && left <= right) {
            pq.offer(new int[]{costs[left], left++});
        } else if (worker[1] > right && left <= right) {
            pq.offer(new int[]{costs[right], right--});
        }
    }

    return total;
}
```

## 🟡 LC #743 — Network Delay Time

```java
public int networkDelayTime(int[][] times, int n, int k) {
    // Build adjacency list: source → [(dest, weight)]
    Map<Integer, List<int[]>> adj = new HashMap<>();
    for (int[] t : times) {
        adj.computeIfAbsent(t[0], x -> new ArrayList<>())
           .add(new int[]{t[1], t[2]});
    }

    // Dijkstra: min-heap [dist, node]
    PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> a[0] - b[0]);
    int[] dist = new int[n + 1];
    Arrays.fill(dist, Integer.MAX_VALUE);
    dist[k] = 0;
    pq.offer(new int[]{0, k});

    while (!pq.isEmpty()) {
        int[] curr = pq.poll();
        int d = curr[0], node = curr[1];

        if (d > dist[node]) continue; // Stale entry — skip!

        for (int[] edge : adj.getOrDefault(node, new ArrayList<>())) {
            int next = edge[0], weight = edge[1];
            if (dist[node] + weight < dist[next]) {
                dist[next] = dist[node] + weight;
                pq.offer(new int[]{dist[next], next});
            }
        }
    }

    int maxDist = 0;
    for (int i = 1; i <= n; i++) {
        if (dist[i] == Integer.MAX_VALUE) return -1;
        maxDist = Math.max(maxDist, dist[i]);
    }
    return maxDist;
}
```

```java
public int swimInWater(int[][] grid) {
    int n = grid.length;
    int[][] dist = new int[n][n];
    for (int[] row : dist) Arrays.fill(row, Integer.MAX_VALUE);
    dist[0][0] = grid[0][0];

    // Min-heap: [time/elevation, row, col]
    PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> a[0] - b[0]);
    pq.offer(new int[]{grid[0][0], 0, 0});
    int[][] dirs = {{0,1},{0,-1},{1,0},{-1,0}};

    while (!pq.isEmpty()) {
        int[] curr = pq.poll();
        int t = curr[0], r = curr[1], c = curr[2];

        if (t > dist[r][c]) continue;
        if (r == n-1 && c == n-1) return t;

        for (int[] d : dirs) {
            int nr = r+d[0], nc = c+d[1];
            if (nr>=0 && nr<n && nc>=0 && nc<n) {
                int newT = Math.max(t, grid[nr][nc]); // Time = max elevation on path
                if (newT < dist[nr][nc]) {
                    dist[nr][nc] = newT;
                    pq.offer(new int[]{newT, nr, nc});
                }
            }
        }
    }
    return -1;
}
```

```java
public String reorganizeString(String s) {
    int[] freq = new int[26];
    for (char c : s.toCharArray()) freq[c-'a']++;

    // Max-heap by frequency
    PriorityQueue<int[]> maxHeap = new PriorityQueue<>((a, b) -> b[1] - a[1]);
    for (int i = 0; i < 26; i++) {
        if (freq[i] > 0) maxHeap.offer(new int[]{i, freq[i]});
    }

    if (maxHeap.peek()[1] > (s.length() + 1) / 2) return ""; // Impossible

    StringBuilder sb = new StringBuilder();
    while (maxHeap.size() >= 2) {
        int[] first  = maxHeap.poll();
        int[] second = maxHeap.poll();
        sb.append((char)('a' + first[0]));
        sb.append((char)('a' + second[0]));
        if (--first[1]  > 0) maxHeap.offer(first);
        if (--second[1] > 0) maxHeap.offer(second);
    }
    if (!maxHeap.isEmpty()) sb.append((char)('a' + maxHeap.poll()[0]));

    return sb.toString();
}
```

```java
public int[] minInterval(int[][] intervals, int[] queries) {
    Arrays.sort(intervals, (a, b) -> a[0] - b[0]); // Sort by start

    int[] sortedQueries = queries.clone();
    Arrays.sort(sortedQueries);

    // Min-heap: [interval length, end]
    PriorityQueue<int[]> minHeap = new PriorityQueue<>((a, b) -> a[0] - b[0]);
    Map<Integer, Integer> result = new HashMap<>();

    int i = 0;
    for (int q : sortedQueries) {
        // Add all intervals starting <= q
        while (i < intervals.length && intervals[i][0] <= q) {
            int len = intervals[i][1] - intervals[i][0] + 1;
            minHeap.offer(new int[]{len, intervals[i][1]});
            i++;
        }
        // Remove intervals ending before q
        while (!minHeap.isEmpty() && minHeap.peek()[1] < q) minHeap.poll();

        result.put(q, minHeap.isEmpty() ? -1 : minHeap.peek()[0]);
    }

    int[] ans = new int[queries.length];
    for (int j = 0; j < queries.length; j++) ans[j] = result.get(queries[j]);
    return ans;
}
```

```java
public int findMaximizedCapital(int k, int w, int[] profits, int[] capital) {
    int n = profits.length;
    // Min-heap by capital required: unlock projects as capital grows
    PriorityQueue<int[]> locked = new PriorityQueue<>((a, b) -> a[0] - b[0]);
    // Max-heap by profit: always pick most profitable available project
    PriorityQueue<Integer> available = new PriorityQueue<>(Collections.reverseOrder());

    for (int i = 0; i < n; i++) locked.offer(new int[]{capital[i], profits[i]});

    for (int i = 0; i < k; i++) {
        // Unlock all projects we can afford
        while (!locked.isEmpty() && locked.peek()[0] <= w) {
            available.offer(locked.poll()[1]);
        }
        if (available.isEmpty()) break; // Can't afford any project

        w += available.poll(); // Do most profitable project
    }

    return w;
}
```

```java
public int mostBooked(int n, int[][] meetings) {
    Arrays.sort(meetings, (a, b) -> a[0] - b[0]);

    // Min-heap of available rooms by room number
    PriorityQueue<Integer> available = new PriorityQueue<>();
    for (int i = 0; i < n; i++) available.offer(i);

    // Min-heap of busy rooms: [endTime, roomNumber]
    PriorityQueue<long[]> busy = new PriorityQueue<>((a, b) ->
        a[0] != b[0] ? Long.compare(a[0], b[0]) : Long.compare(a[1], b[1]));

    int[] count = new int[n];

    for (int[] meeting : meetings) {
        long start = meeting[0], end = meeting[1];

        // Free up rooms that finished before this meeting starts
        while (!busy.isEmpty() && busy.peek()[0] <= start) {
            available.offer((int) busy.poll()[1]);
        }

        if (!available.isEmpty()) {
            int room = available.poll();
            busy.offer(new long[]{end, room});
            count[room]++;
        } else {
            // All rooms busy — wait for earliest to finish
            long[] earliest = busy.poll();
            long newEnd = earliest[0] + (end - start);
            int room = (int) earliest[1];
            busy.offer(new long[]{newEnd, room});
            count[room]++;
        }
    }

    int maxCount = 0, result = 0;
    for (int i = 0; i < n; i++) {
        if (count[i] > maxCount) { maxCount = count[i]; result = i; }
    }
    return result;
}
```

```
├── Need repeated min/max extraction?     → Heap (PriorityQueue)
├── Top K largest?                        → Min-heap size K
├── Top K smallest?                       → Max-heap size K
├── K-th element (streaming)?             → Min-heap size K
├── Merge multiple sorted sources?        → K-way merge (min-heap)
├── Median / balance two halves?          → Two heaps
├── Weighted shortest path?               → Dijkstra (min-heap)
├── Task scheduling with cooldown?        → Max-heap + cycle
└── Greedy: always pick best available?  → Max-heap greedy
```

```java
// TOP K LARGEST (min-heap of size k)
PriorityQueue<Integer> pq = new PriorityQueue<>();
for (int n : nums) {
    pq.offer(n);
    if (pq.size() > k) pq.poll(); // Evict smallest
}
// pq.peek() = kth largest,  pq = top k largest

// TOP K SMALLEST (max-heap of size k)
PriorityQueue<Integer> pq = new PriorityQueue<>(Collections.reverseOrder());
for (int n : nums) {
    pq.offer(n);
    if (pq.size() > k) pq.poll(); // Evict largest
}
// pq.peek() = kth smallest,  pq = k smallest

// K-WAY MERGE
PriorityQueue<int[]> pq = new PriorityQueue<>((a,b) -> a[0]-b[0]); // [val, listIdx, elemIdx]
// Initialize: add first element from each list
// Loop: poll → add to result → add next from same list

// TWO HEAPS (MEDIAN)
PriorityQueue<Integer> lower = new PriorityQueue<>(Collections.reverseOrder()); // max
PriorityQueue<Integer> upper = new PriorityQueue<>(); // min
void addNum(int n) {
    lower.offer(n);
    if (!upper.isEmpty() && lower.peek() > upper.peek()) upper.offer(lower.poll());
    if (lower.size() > upper.size() + 1) upper.offer(lower.poll());
    else if (upper.size() > lower.size()) lower.offer(upper.poll());
}
double median() {
    return lower.size() == upper.size()
           ? (lower.peek() + upper.peek()) / 2.0
           : lower.peek();
}

// DIJKSTRA
PriorityQueue<int[]> pq = new PriorityQueue<>((a,b) -> a[0]-b[0]); // [dist, node]
int[] dist = new int[n]; Arrays.fill(dist, Integer.MAX_VALUE); dist[src] = 0;
pq.offer(new int[]{0, src});
while (!pq.isEmpty()) {
    int[] cur = pq.poll();
    int d = cur[0], u = cur[1];
    if (d > dist[u]) continue; // Stale — MUST CHECK!
    for (int[] e : adj.get(u)) {
        if (dist[u] + e[1] < dist[e[0]]) {
            dist[e[0]] = dist[u] + e[1];
            pq.offer(new int[]{dist[e[0]], e[0]});
        }
    }
}
```

## ⚡ The 5 Golden Rules of Heap Problems

```
1. MIN-HEAP for TOP K LARGEST, MAX-HEAP for TOP K SMALLEST.
   Counter-intuitive but correct: the heap holds the K keepers,
   and you evict the "worst" of those keepers.

2. ALWAYS skip stale entries in Dijkstra:
   if (d > dist[node]) continue;
   Without this, you process the same node multiple times.

3. For K-WAY MERGE, always add the NEXT element from the
   SAME source as the one you just polled. Never add all at once.

4. TWO HEAPS invariant: lower.size() == upper.size() OR lower.size() == upper.size() + 1
   Always maintain this after every insertion. Rebalance immediately.

5. PriorityQueue.remove(element) is O(n) — LINEAR!
   For efficient removal, use TreeMap/TreeSet or lazy deletion with a "deleted" set.
```
