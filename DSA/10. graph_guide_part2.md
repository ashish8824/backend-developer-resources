# 24. How to Identify Graph Patterns

## 🔍 Master Recognition Guide

```
Do you see these keywords?         → Use this
─────────────────────────────────────────────────────
"connected", "reachable"           → BFS/DFS
"shortest path" (unweighted)       → BFS
"shortest path" (weighted)         → Dijkstra
"shortest path" (negative weights) → Bellman-Ford
"all pairs shortest"               → Floyd-Warshall
"minimum spanning tree"            → Kruskal or Prim
"connected components"             → Union-Find or DFS
"cycle detection" (undirected)     → Union-Find or DFS+parent
"cycle detection" (directed)       → DFS with 3 states
"ordering with dependencies"       → Topological Sort
"prerequisites"                    → Topological Sort
"can complete all tasks?"          → Cycle detect in directed
"bipartite" / "2-coloring"         → BFS coloring
"bridge" / "critical edge"         → Tarjan's
"strongly connected"               → Kosaraju's / Tarjan's
"grid" / "matrix" with spreading   → Multi-source BFS
"Eulerian path/circuit"            → Hierholzer's
"visit all nodes/edges once"       → Eulerian path
"all paths" from A to B            → DFS + backtracking
"number of ways" to reach          → BFS/DFS + counting
```

## 🧠 Decision Framework

```
Is input a GRID?
├── Spread/flood fill? → BFS or DFS
├── Shortest path?     → BFS (unweighted) or Dijkstra (weighted)
└── From multiple sources? → Multi-source BFS

Is input an EXPLICIT graph (edges list)?
├── Find shortest path?
│   ├── Unweighted → BFS
│   ├── Weighted, non-negative → Dijkstra
│   ├── Weighted, negative → Bellman-Ford
│   └── All pairs → Floyd-Warshall
│
├── Check connectivity / components?
│   → Union-Find or DFS
│
├── Detect cycle?
│   ├── Undirected → Union-Find or DFS+parent
│   └── Directed   → DFS with 3 states
│
├── Find ordering?
│   → Topological Sort (Kahn's BFS or DFS postorder)
│
├── Find MST?
│   ├── Kruskal: sort edges + Union-Find
│   └── Prim: grow from one node with min-heap
│
└── Advanced?
    ├── Bridges/AP → Tarjan's
    ├── SCC → Kosaraju's
    └── Eulerian → Hierholzer's
```

# 25. Interview Cheat Sheet

## ⚡ Build Graph from Edges (30 seconds)

```java
// Undirected
List<List<Integer>> adj = new ArrayList<>();
for (int i = 0; i < n; i++) adj.add(new ArrayList<>());
for (int[] e : edges) { adj.get(e[0]).add(e[1]); adj.get(e[1]).add(e[0]); }

// Directed
for (int[] e : edges) adj.get(e[0]).add(e[1]);

// Weighted
List<List<int[]>> adj = new ArrayList<>();
for (int i = 0; i < n; i++) adj.add(new ArrayList<>());
for (int[] e : edges) { adj.get(e[0]).add(new int[]{e[1],e[2]}); } // [neighbor, weight]
```

## ⚡ BFS Template

```java
boolean[] visited = new boolean[n];
Queue<Integer> q = new ArrayDeque<>();
visited[src] = true;
q.offer(src);
int dist = 0;

while (!q.isEmpty()) {
    int sz = q.size();
    for (int i = 0; i < sz; i++) {
        int node = q.poll();
        // process node
        for (int nb : adj.get(node)) {
            if (!visited[nb]) { visited[nb] = true; q.offer(nb); }
        }
    }
    dist++;
}
```

---

## ⚡ DFS Template (Recursive)

```java
boolean[] visited = new boolean[n];
void dfs(int node) {
    visited[node] = true;
    // process node
    for (int nb : adj.get(node)) {
        if (!visited[nb]) dfs(nb);
    }
}
// Call: for (int i=0; i<n; i++) if (!visited[i]) dfs(i);
```

---

## ⚡ Union-Find Template

```java
int[] parent, rank;
void init(int n) {
    parent = new int[n]; rank = new int[n];
    for (int i=0; i<n; i++) parent[i] = i;
}
int find(int x) {
    if (parent[x] != x) parent[x] = find(parent[x]);
    return parent[x];
}
boolean union(int x, int y) {
    int px = find(x), py = find(y);
    if (px == py) return false;
    if (rank[px] < rank[py]) { int t=px; px=py; py=t; }
    parent[py] = px;
    if (rank[px] == rank[py]) rank[px]++;
    return true;
}
```

## ⚡ Topological Sort (Kahn's BFS)

```java
int[] inDeg = new int[n];
for (int[] e : edges) inDeg[e[1]]++;
Queue<Integer> q = new ArrayDeque<>();
for (int i=0; i<n; i++) if (inDeg[i]==0) q.offer(i);
List<Integer> order = new ArrayList<>();
while (!q.isEmpty()) {
    int node = q.poll(); order.add(node);
    for (int nb : adj.get(node)) if (--inDeg[nb]==0) q.offer(nb);
}
boolean hasCycle = order.size() != n;
```

---

## ⚡ Dijkstra Template

```java
int[] dist = new int[n];
Arrays.fill(dist, Integer.MAX_VALUE);
dist[src] = 0;
PriorityQueue<int[]> pq = new PriorityQueue<>((a,b)->a[0]-b[0]);
pq.offer(new int[]{0, src});
while (!pq.isEmpty()) {
    int[] cur = pq.poll();
    int d = cur[0], u = cur[1];
    if (d > dist[u]) continue;
    for (int[] e : adj.get(u)) {
        int v = e[0], w = e[1];
        if (dist[u] + w < dist[v]) {
            dist[v] = dist[u] + w;
            pq.offer(new int[]{dist[v], v});
        }
    }
}
```

## ⚡ Kruskal's MST Template

```java
Arrays.sort(edges, (a,b)->a[2]-b[2]); // Sort by weight
init(n); // UnionFind init
int cost = 0, edgesUsed = 0;
for (int[] e : edges) {
    if (union(e[0], e[1])) {
        cost += e[2]; edgesUsed++;
        if (edgesUsed == n-1) break;
    }
}
```

## ⚡ Cycle Detection (Directed) Template

```java
// 0=unvisited, 1=in-progress, 2=done
int[] state = new int[n];
boolean hasCycle = false;
void dfs(int u) {
    state[u] = 1;
    for (int v : adj.get(u)) {
        if (state[v] == 1) { hasCycle = true; return; }
        if (state[v] == 0) dfs(v);
    }
    state[u] = 2;
}
```

## ⚡ Grid BFS Template

```java
int[][] dirs = {{0,1},{0,-1},{1,0},{-1,0}};
boolean[][] visited = new boolean[rows][cols];
Queue<int[]> q = new ArrayDeque<>();
q.offer(new int[]{startR, startC});
visited[startR][startC] = true;
while (!q.isEmpty()) {
    int[] cell = q.poll();
    for (int[] d : dirs) {
        int nr = cell[0]+d[0], nc = cell[1]+d[1];
        if (nr>=0 && nr<rows && nc>=0 && nc<cols && !visited[nr][nc] && condition) {
            visited[nr][nc] = true;
            q.offer(new int[]{nr, nc});
        }
    }
}
```

> **The Golden Rules of Graph Problems:**
>
> 1. **Always mark nodes as visited BEFORE adding to BFS queue** (not after polling). This prevents adding the same node multiple times.
>
> 2. **For Dijkstra, skip stale entries:** `if (d > dist[node]) continue;`
>
> 3. **For directed cycle detection, use 3 states** (unvisited/in-progress/done). 2 states only works for undirected graphs.
>
> 4. **Union-Find is perfect when you need to** repeatedly check if two nodes are in the same component and merge components.
>
> 5. **Topological sort only works on DAGs.** If a cycle exists, Kahn's algorithm will leave nodes with non-zero in-degree — use this to detect cycles.
>
> 6. **Grid problems are just graph problems** — no need to build explicit adjacency lists. Use direction arrays.
>
> 7. **BFS guarantees shortest path in UNWEIGHTED graphs.** For weighted graphs, use Dijkstra.
