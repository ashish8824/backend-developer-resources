# 1. What is Recursion?

## The Core Idea

**Recursion** is when a function calls **itself** to solve a smaller version of the same problem.

```
Real world analogy: Russian Nesting Dolls (Matryoshka)

Open doll → find smaller doll → open it → find smaller doll
→ ... → find the SMALLEST doll (no more dolls inside)
→ Now you know you've reached the bottom

In code:
solve(bigProblem) {
    solve(smallerProblem)   ← calls itself with smaller input
    solve(evenSmaller)
    ...
    if (smallest possible) return directly  ← BASE CASE
}
```

## Fibonacci — The Clearest Example

```
fib(5) = fib(4) + fib(3)
fib(4) = fib(3) + fib(2)
fib(3) = fib(2) + fib(1)
fib(2) = fib(1) + fib(0)
fib(1) = 1   ← BASE CASE (stop here)
fib(0) = 0   ← BASE CASE (stop here)

Key insight: fib(n) IS DEFINED in terms of fib(n-1) and fib(n-2)
             This self-referential definition IS recursion.
```

## Two Absolute Requirements

```
Every valid recursive function MUST have:

1. BASE CASE — the condition where recursion STOPS
   → Without this → infinite recursion → StackOverflowError!

2. RECURSIVE CASE — the problem gets SMALLER each call
   → Without this → never reaches base case → StackOverflowError!

Think of it like a staircase:
  - Base case = the ground (you stop here)
  - Each recursive call = one step DOWN the staircase
  - You must get smaller each step or you never reach the ground
```

# 2. How the Call Stack Works

## The Most Important Thing to Understand

When a function calls itself, Java's **call stack** stores each call's local variables and return address. When the base case is hit, the stack **unwinds** back up.

```
factorial(4):

CALL STACK BUILDS UP:
┌─────────────────────┐
│ factorial(1) = 1    │ ← TOP (base case reached)
├─────────────────────┤
│ factorial(2) = 2*?  │ ← waiting for factorial(1)
├─────────────────────┤
│ factorial(3) = 3*?  │ ← waiting for factorial(2)
├─────────────────────┤
│ factorial(4) = 4*?  │ ← waiting for factorial(3)
└─────────────────────┘  BOTTOM (original call)

CALL STACK UNWINDS:
factorial(1) returns 1
factorial(2) = 2 * 1 = 2  returns 2
factorial(3) = 3 * 2 = 6  returns 6
factorial(4) = 4 * 6 = 24 returns 24  ← FINAL ANSWER
```

## Stack Overflow

```java
// This causes StackOverflowError — NO BASE CASE!
public int infinite(int n) {
    return infinite(n - 1);  // n keeps decreasing but never stops!
}

// This also causes StackOverflowError — n never decreases!
public int stuck(int n) {
    return stuck(n);  // Same input each time, never reaches base case!
}

// Each function call takes memory on the stack.
// Java's default stack size is ~512KB - 1MB.
// Deep recursion (n > ~5000-10000) may overflow.
// Solution: Use iteration, or increase stack size.
```

# 3. The Anatomy of Every Recursive Function

```java
returnType solve(parameters) {

    // ═══ PART 1: BASE CASE ═══════════════════════
    // What is the SIMPLEST possible input?
    // What do we return without needing to recurse?
    if (baseCaseCondition) {
        return baseCaseValue;
    }

    // ═══ PART 2: RECURSIVE CASE ══════════════════
    // How do we make the problem SMALLER?
    // What do we do with the result of the smaller problem?
    result = solve(smallerInput);  // Recursive call
    return combineWithCurrentData(result);
}
```

## Applied to Factorial

```java
public int factorial(int n) {
    // BASE CASE: 0! = 1 and 1! = 1
    if (n <= 1) return 1;

    // RECURSIVE CASE: n! = n × (n-1)!
    return n * factorial(n - 1);  // Problem shrinks: n → n-1
}
```

## Applied to Sum of Array

```java
public int arraySum(int[] arr, int index) {
    // BASE CASE: past the end of array
    if (index == arr.length) return 0;

    // RECURSIVE CASE: current element + sum of rest
    return arr[index] + arraySum(arr, index + 1);  // index grows toward end
}
```

# 4. Recursion vs Iteration

Every recursive solution has an equivalent iterative solution (using an explicit stack). Understanding when to use which is key.

```
RECURSION:
✅ Natural for tree/graph traversal
✅ Cleaner code for divide & conquer
✅ Elegant for problems defined recursively (Fibonacci, factorial)
❌ Stack overflow risk for deep recursion
❌ Function call overhead
❌ Harder to optimize space

ITERATION:
✅ No stack overflow risk
✅ Usually faster (no call overhead)
✅ Better space efficiency
❌ More complex code for tree/graph
❌ Requires explicit stack for some problems

RULE OF THUMB:
→ Tree / Graph traversal → Recursion (natural) or BFS Queue
→ Simple loops (factorial, sum) → Iteration
→ Divide & Conquer (merge sort) → Recursion
→ Very deep recursion (n > 10000) → Convert to iteration
```

## Side-by-Side Comparison

```java
// ===== FACTORIAL =====
// Recursive — O(n) space (call stack)
public int factRecursive(int n) {
    if (n <= 1) return 1;
    return n * factRecursive(n - 1);
}

// Iterative — O(1) space
public int factIterative(int n) {
    int result = 1;
    for (int i = 2; i <= n; i++) result *= i;
    return result;
}

// ===== FIBONACCI =====
// Recursive — O(2^n) time, O(n) space
public int fibRecursive(int n) {
    if (n <= 1) return n;
    return fibRecursive(n-1) + fibRecursive(n-2);
}

// Iterative — O(n) time, O(1) space
public int fibIterative(int n) {
    if (n <= 1) return n;
    int a = 0, b = 1;
    for (int i = 2; i <= n; i++) {
        int c = a + b; a = b; b = c;
    }
    return b;
}
```

# 5. Common Mistakes and How to Avoid Them

## Mistake 1: Missing Base Case

```java
// ❌ WRONG — no base case → infinite recursion
public int sum(int n) {
    return n + sum(n - 1);
}

// ✅ CORRECT — base case added
public int sum(int n) {
    if (n == 0) return 0;  // BASE CASE
    return n + sum(n - 1);
}
```

## Mistake 2: Base Case Never Reached

```java
// ❌ WRONG — n never reaches 0 (decrements by 2, skips 0 if odd)
public int sum(int n) {
    if (n == 0) return 0;
    return n + sum(n - 2);  // What if n starts odd? Goes to -1, -3, ...
}

// ✅ CORRECT
public int sum(int n) {
    if (n <= 0) return 0;  // Handles all cases ≤ 0
    return n + sum(n - 2);
}
```

## Mistake 3: Wrong Return Value

```java
// ❌ WRONG — recursive call result ignored
public int factorial(int n) {
    if (n <= 1) return 1;
    factorial(n - 1);  // Result thrown away!
    return n;          // Always returns n
}

// ✅ CORRECT
public int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);  // Use the result!
}
```

## Mistake 4: Modifying Shared State Without Restoring

```java
// ❌ WRONG in backtracking — forgets to backtrack
public void generateSubsets(int[] nums, int i, List<Integer> current) {
    if (i == nums.length) { result.add(new ArrayList<>(current)); return; }
    current.add(nums[i]);
    generateSubsets(nums, i + 1, current);
    // FORGOT: current.remove(current.size() - 1);  ← MUST UNDO!
    generateSubsets(nums, i + 1, current);
}

// ✅ CORRECT — always undo what you did
public void generateSubsets(int[] nums, int i, List<Integer> current) {
    if (i == nums.length) { result.add(new ArrayList<>(current)); return; }
    current.add(nums[i]);
    generateSubsets(nums, i + 1, current);
    current.remove(current.size() - 1);  // UNDO ✅
    generateSubsets(nums, i + 1, current);
}
```

## Mistake 5: Not Copying When Saving State

```java
// ❌ WRONG — adds reference to mutable list
result.add(current);  // current will change later!

// ✅ CORRECT — add a COPY
result.add(new ArrayList<>(current));
```

# 6. Direct vs Indirect Recursion

## Direct Recursion
Function calls itself directly.
```java
public int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);  // Calls itself directly
}
```

## Indirect Recursion (Mutual Recursion)
Function A calls B, B calls A.
```java
public boolean isEven(int n) {
    if (n == 0) return true;
    return isOdd(n - 1);   // A calls B
}

public boolean isOdd(int n) {
    if (n == 0) return false;
    return isEven(n - 1);  // B calls A
}
```

# 7. Tail Recursion

## What is Tail Recursion?

A recursive call is **tail recursive** when it is the **LAST operation** in the function — nothing happens after the recursive call returns.

```java
// NOT TAIL RECURSIVE — multiplication happens AFTER the recursive call
public int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);  // Must come back to multiply by n
}

// TAIL RECURSIVE — recursive call IS the last operation
public int factTail(int n, int accumulator) {
    if (n <= 1) return accumulator;
    return factTail(n - 1, n * accumulator);  // Result passed as argument
}
// Call: factTail(5, 1)
```

## Why Does It Matter?

```
Tail Recursive → Compiler can OPTIMIZE to iteration (no stack growth)
                → Some languages/compilers eliminate the call stack entirely
                → Java does NOT optimize tail calls automatically
                → But the pattern helps you convert to iteration manually

Non-Tail:   factorial(5) → factorial(4) → factorial(3) → ... (stack builds up)
Tail:       factTail(5,1) → factTail(4,5) → factTail(3,20) → ...
            Each call is independent → no need to "remember" previous calls

Converting Tail Recursion to Iteration:
public int factIterative(int n) {
    int accumulator = 1;
    while (n > 1) { accumulator *= n; n--; }
    return accumulator;
}
```

# 8. Head Recursion

The recursive call happens **FIRST**, then operations happen on the way **back up**.

```java
// HEAD RECURSION — process on the way BACK UP (after recursive call)
public void printReverse(int n) {
    if (n == 0) return;    // Base case
    printReverse(n - 1);  // RECURSE FIRST
    System.out.print(n + " ");  // Print on the way BACK
}
// printReverse(5) prints: 1 2 3 4 5

// TAIL RECURSION — process on the way DOWN (before recursive call)
public void printForward(int n) {
    if (n == 0) return;
    System.out.print(n + " ");  // Print FIRST
    printForward(n - 1);        // Then recurse
}
// printForward(5) prints: 5 4 3 2 1
```

**Key insight:** Head recursion naturally **reverses** the order of processing — it's why `printReverse(linkedList)` is so elegant recursively.

# 9. Tree Recursion (Multiple Calls)

When a function makes **more than one** recursive call, it creates a **tree of calls**.

```
Fibonacci tree for fib(5):

              fib(5)
            /        \
         fib(4)      fib(3)
        /     \      /    \
     fib(3) fib(2) fib(2) fib(1)
     /   \   / \   / \
  fib(2) fib(1) ...
  / \
fib(1) fib(0)

Notice fib(3) is computed TWICE, fib(2) three times!
This is why naive Fibonacci is O(2^n) — exponential!
Fix: Memoization (next section)
```

```java
// Tree recursion — exponential without memoization
public int fib(int n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);  // TWO recursive calls → tree
}
```

# 10. Memoization — Recursion + Caching

## What is Memoization?

Store the result of each subproblem so we never compute it twice.

```
Without memo: fib(5) computes fib(3) TWICE, fib(2) THREE times
With memo:    fib(3) computed ONCE, stored, returned instantly on future calls

Time:  O(2^n) → O(n)
Space: O(n) for memo table
```

## Template

```java
Map<Integer, Integer> memo = new HashMap<>();

public int fib(int n) {
    if (n <= 1) return n;

    // Check cache first
    if (memo.containsKey(n)) return memo.get(n);

    // Compute and STORE
    int result = fib(n - 1) + fib(n - 2);
    memo.put(n, result);
    return result;
}
```

## Array-Based Memoization (Faster)

```java
int[] memo;

public int fib(int n) {
    memo = new int[n + 1];
    Arrays.fill(memo, -1);  // -1 = not computed
    return fibHelper(n);
}

private int fibHelper(int n) {
    if (n <= 1) return n;
    if (memo[n] != -1) return memo[n];  // Cache hit!
    memo[n] = fibHelper(n-1) + fibHelper(n-2);
    return memo[n];
}
```

# 11. Pattern 1 — Decrease and Conquer (Linear Recursion)

## What is it?

Solve a problem by reducing it to a **slightly smaller** version of the same problem. One recursive call.

```
Problem of size n → Problem of size n-1 → ... → Base case
```

---

## Classic Examples

```java
// FACTORIAL: n! = n × (n-1)!
public int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}

// SUM of 1 to n: sum(n) = n + sum(n-1)
public int sum(int n) {
    if (n == 0) return 0;
    return n + sum(n - 1);
}

// POWER: base^exp = base × base^(exp-1)
public double power(double base, int exp) {
    if (exp == 0) return 1;
    if (exp < 0) return 1.0 / power(base, -exp);
    return base * power(base, exp - 1);
}

// FAST POWER — O(log n) instead of O(n)
public double fastPower(double base, int exp) {
    if (exp == 0) return 1;
    if (exp < 0) return 1.0 / fastPower(base, -exp);
    if (exp % 2 == 0) {
        double half = fastPower(base, exp / 2);
        return half * half;  // base^(2k) = (base^k)^2
    }
    return base * fastPower(base, exp - 1);
}
```

---

## 🟢 LC #509 — Fibonacci Number

```java
// Naive — O(2^n)
public int fib(int n) {
    if (n <= 1) return n;
    return fib(n-1) + fib(n-2);
}

// With memoization — O(n)
public int fib(int n) {
    int[] memo = new int[n + 2];
    Arrays.fill(memo, -1);
    return fibMemo(n, memo);
}
private int fibMemo(int n, int[] memo) {
    if (n <= 1) return n;
    if (memo[n] != -1) return memo[n];
    return memo[n] = fibMemo(n-1, memo) + fibMemo(n-2, memo);
}

// Iterative — O(n) time, O(1) space (best)
public int fibIterative(int n) {
    if (n <= 1) return n;
    int a = 0, b = 1;
    for (int i = 2; i <= n; i++) { int c = a+b; a=b; b=c; }
    return b;
}
```

## 🟡 LC #50 — Pow(x, n) — Fast Power

```java
public double myPow(double x, int n) {
    if (n == 0) return 1.0;

    // Handle negative exponent
    if (n < 0) {
        x = 1.0 / x;
        n = -(n + 1);  // Avoid overflow: Integer.MIN_VALUE negated overflows
        return x * myPow(x * x, n / 2);
    }

    // Even exponent: x^n = (x^2)^(n/2)
    if (n % 2 == 0) {
        return myPow(x * x, n / 2);
    }

    // Odd exponent: x^n = x * x^(n-1)
    return x * myPow(x, n - 1);
}
```

**Complexity:** Time O(log n), Space O(log n) call stack

# 12. Pattern 2 — Divide and Conquer

## What is it?

Split the problem into **independent halves**, solve each recursively, then **merge** the results.

```
Problem(n) → solve(left half) + solve(right half) → merge
```

---

## Merge Sort — The Classic Example

```java
public int[] mergeSort(int[] arr, int left, int right) {
    if (left >= right) return new int[]{arr[left]};  // Base case: 1 element

    int mid = left + (right - left) / 2;

    // DIVIDE: solve left half and right half independently
    int[] leftSorted  = mergeSort(arr, left, mid);
    int[] rightSorted = mergeSort(arr, mid + 1, right);

    // CONQUER: merge the two sorted halves
    return merge(leftSorted, rightSorted);
}

private int[] merge(int[] left, int[] right) {
    int[] result = new int[left.length + right.length];
    int i = 0, j = 0, k = 0;
    while (i < left.length && j < right.length) {
        if (left[i] <= right[j]) result[k++] = left[i++];
        else                      result[k++] = right[j++];
    }
    while (i < left.length)  result[k++] = left[i++];
    while (j < right.length) result[k++] = right[j++];
    return result;
}
```

---

## Binary Search — Divide and Conquer on Sorted Array

```java
public int binarySearch(int[] arr, int target, int left, int right) {
    if (left > right) return -1;  // Base case: not found

    int mid = left + (right - left) / 2;

    if (arr[mid] == target) return mid;
    if (arr[mid] < target)  return binarySearch(arr, target, mid + 1, right);
    else                    return binarySearch(arr, target, left, mid - 1);
}
```

## 🔴 LC #4 — Median of Two Sorted Arrays (Hard Divide & Conquer)

```java
public double findMedianSortedArrays(int[] nums1, int[] nums2) {
    // Always binary search on smaller array
    if (nums1.length > nums2.length) return findMedianSortedArrays(nums2, nums1);

    int m = nums1.length, n = nums2.length;
    int left = 0, right = m;
    int halfLen = (m + n + 1) / 2;

    while (left <= right) {
        int i = left + (right - left) / 2;
        int j = halfLen - i;

        if (i < m && nums2[j-1] > nums1[i])      left = i + 1;
        else if (i > 0 && nums1[i-1] > nums2[j]) right = i - 1;
        else {
            int maxLeft;
            if (i == 0)      maxLeft = nums2[j-1];
            else if (j == 0) maxLeft = nums1[i-1];
            else             maxLeft = Math.max(nums1[i-1], nums2[j-1]);

            if ((m + n) % 2 == 1) return maxLeft;

            int minRight;
            if (i == m)      minRight = nums2[j];
            else if (j == n) minRight = nums1[i];
            else             minRight = Math.min(nums1[i], nums2[j]);

            return (maxLeft + minRight) / 2.0;
        }
    }
    return 0.0;
}
```

# 13. Pattern 3 — Backtracking via Recursion

## What is it?

Try a choice → recurse → **undo** the choice → try next choice.

```
solve() {
    for each choice:
        MAKE choice
        recurse()
        UNDO choice   ← THE KEY STEP
}
```

*(Full backtracking coverage is in the Backtracking guide. Here we focus on the recursion mechanics.)*

## 🟡 LC #78 — Subsets (Recursion Perspective)

```java
public List<List<Integer>> subsets(int[] nums) {
    List<List<Integer>> result = new ArrayList<>();
    generateSubsets(nums, 0, new ArrayList<>(), result);
    return result;
}

private void generateSubsets(int[] nums, int idx,
                              List<Integer> current, List<List<Integer>> result) {
    result.add(new ArrayList<>(current));  // Every state is a valid subset

    for (int i = idx; i < nums.length; i++) {
        current.add(nums[i]);                            // CHOOSE
        generateSubsets(nums, i + 1, current, result);  // RECURSE
        current.remove(current.size() - 1);             // UNDO
    }
}
```

## 🟡 LC #46 — Permutations

```java
public List<List<Integer>> permute(int[] nums) {
    List<List<Integer>> result = new ArrayList<>();
    permuteHelper(nums, new boolean[nums.length], new ArrayList<>(), result);
    return result;
}

private void permuteHelper(int[] nums, boolean[] used,
                            List<Integer> current, List<List<Integer>> result) {
    if (current.size() == nums.length) {
        result.add(new ArrayList<>(current));
        return;
    }
    for (int i = 0; i < nums.length; i++) {
        if (used[i]) continue;
        used[i] = true;
        current.add(nums[i]);
        permuteHelper(nums, used, current, result);
        current.remove(current.size() - 1);
        used[i] = false;
    }
}
```

# 14. Pattern 4 — Build the Answer Bottom-Up (Return Value)

## What is it?

Each recursive call **returns** a value that its caller uses to build the final answer. The answer is assembled as the call stack **unwinds**.

```
Think: "What should I RETURN to my parent so they can do their job?"
```

---

## 🟢 LC #104 — Maximum Depth of Binary Tree

```java
public int maxDepth(TreeNode root) {
    if (root == null) return 0;  // Base: empty tree has depth 0

    int leftDepth  = maxDepth(root.left);   // What left subtree returns
    int rightDepth = maxDepth(root.right);  // What right subtree returns

    return 1 + Math.max(leftDepth, rightDepth);  // Build answer from children
}
```

**Call stack for tree [1, [2,[4],[5]], [3]]:**
```
maxDepth(1):
  maxDepth(2):
    maxDepth(4): returns 1 (leaf)
    maxDepth(5): returns 1 (leaf)
  returns 1 + max(1,1) = 2
  maxDepth(3): returns 1 (leaf)
returns 1 + max(2,1) = 3  ← FINAL ANSWER
```

---

## 🟢 LC #226 — Invert Binary Tree

```java
public TreeNode invertTree(TreeNode root) {
    if (root == null) return null;  // Base case

    // Invert subtrees first (bottom-up)
    TreeNode left  = invertTree(root.left);
    TreeNode right = invertTree(root.right);

    // Swap the results
    root.left  = right;
    root.right = left;

    return root;
}
```

## 🟡 LC #543 — Diameter of Binary Tree

```java
private int maxDiameter = 0;

public int diameterOfBinaryTree(TreeNode root) {
    maxDiameter = 0;
    height(root);
    return maxDiameter;
}

private int height(TreeNode node) {
    if (node == null) return 0;

    int leftH  = height(node.left);   // Height of left subtree
    int rightH = height(node.right);  // Height of right subtree

    // Diameter through this node = leftH + rightH
    maxDiameter = Math.max(maxDiameter, leftH + rightH);

    return 1 + Math.max(leftH, rightH);  // Return HEIGHT to parent
}
```

**Key insight:** The function returns HEIGHT (for parent's use), but updates DIAMETER (global). This dual-purpose return is a powerful pattern.

## 🔴 LC #124 — Binary Tree Maximum Path Sum

```java
private int globalMax;

public int maxPathSum(TreeNode root) {
    globalMax = Integer.MIN_VALUE;
    maxGain(root);
    return globalMax;
}

private int maxGain(TreeNode node) {
    if (node == null) return 0;

    // Maximum gain from left/right (ignore negative contributions)
    int leftGain  = Math.max(0, maxGain(node.left));
    int rightGain = Math.max(0, maxGain(node.right));

    // Path through this node (can use BOTH sides)
    globalMax = Math.max(globalMax, node.val + leftGain + rightGain);

    // Return: max gain going ONE direction (parent can only extend one way)
    return node.val + Math.max(leftGain, rightGain);
}
```

# 15. Pattern 5 — Accumulator Pattern (Pass State Down)

## What is it?

Instead of building the answer on the way UP, pass an **accumulator** DOWN through recursive calls. The base case returns the accumulated result.

```java
// BUILDING UP (return value pattern)
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);  // Build on way UP
}

// ACCUMULATOR (pass state down)
int factorial(int n, int acc) {
    if (n <= 1) return acc;           // Return accumulated result
    return factorial(n - 1, n * acc); // Pass accumulated result DOWN
}
// Call: factorial(5, 1)
```

```java
public class ArrayStringRecursion {

    }

    // MAX of array
    public static int max(int[] arr, int i) {
        if (i == arr.length - 1) return arr[i];
        return Math.max(arr[i], max(arr, i + 1));
    }

    // CHECK if array is sorted
    public static boolean isSorted(int[] arr, int i) {
        if (i >= arr.length - 1) return true;
        if (arr[i] > arr[i + 1]) return false;
        return isSorted(arr, i + 1);
    }

    // REVERSE an array in-place
    public static void reverse(int[] arr, int left, int right) {
        if (left >= right) return;
        int temp = arr[left]; arr[left] = arr[right]; arr[right] = temp;
        reverse(arr, left + 1, right - 1);
    }

    // BINARY SEARCH
    public static int binarySearch(int[] arr, int target, int left, int right) {
        if (left > right) return -1;
        int mid = left + (right - left) / 2;
        if (arr[mid] == target) return mid;
        if (arr[mid] < target)  return binarySearch(arr, target, mid + 1, right);
        return binarySearch(arr, target, left, mid - 1);
    }

    // CHECK palindrome string

        if (left >= right) return true;
        if (s.charAt(left) != s.charAt(right)) return false;
        return isPalindrome(s, left + 1, right - 1);
    }

        if (i == s.length()) return 0;
        return (s.charAt(i) == c ? 1 : 0) + countChar(s, c, i + 1);
```

```java
public class LinkedListRecursion {

    static class ListNode { int val; ListNode next; ListNode(int v){val=v;} }

    // LENGTH of linked list
    public static int length(ListNode head) {
        if (head == null) return 0;
        return 1 + length(head.next);
    }

    // SUM of linked list
    public static int sum(ListNode head) {
        if (head == null) return 0;
        return head.val + sum(head.next);
    }

    // MAX of linked list
    public static int max(ListNode head) {
        if (head == null) return Integer.MIN_VALUE;
        return Math.max(head.val, max(head.next));
    }

    // PRINT linked list FORWARD (head recursion variant)
    public static void printForward(ListNode head) {
        if (head == null) return;
        System.out.print(head.val + " ");
        printForward(head.next);
    }

    // PRINT linked list in REVERSE (head recursion)
    public static void printReverse(ListNode head) {
        if (head == null) return;
        printReverse(head.next);          // Go to end first
        System.out.print(head.val + " "); // Print on the way back
    }

    // SEARCH
    public static boolean contains(ListNode head, int target) {
        if (head == null) return false;
        if (head.val == target) return true;
        return contains(head.next, target);
    }

    // REVERSE linked list recursively
    public static ListNode reverse(ListNode head) {
        if (head == null || head.next == null) return head;
        ListNode newHead = reverse(head.next);
        head.next.next = head;  // Second node points back to first
        head.next = null;
        return newHead;
    }

    // CHECK if linked list is palindrome recursively
    static ListNode left;
    public static boolean isPalindrome(ListNode head) {
        left = head;
        return check(head);
    }
    private static boolean check(ListNode right) {
        if (right == null) return true;
        if (!check(right.next)) return false;  // Go to end first
        boolean result = left.val == right.val; // Compare outer, then inner
        left = left.next;
        return result;
    }

    // MERGE TWO sorted lists
    public static ListNode mergeSorted(ListNode l1, ListNode l2) {
        if (l1 == null) return l2;
        if (l2 == null) return l1;
        if (l1.val <= l2.val) {
            l1.next = mergeSorted(l1.next, l2);
            return l1;
        } else {
            l2.next = mergeSorted(l1, l2.next);
            return l2;
        }
    }
}
```

# 20. Tree Recursion

```java
public class TreeRecursion {

    static class TreeNode { int val; TreeNode left, right; TreeNode(int v){val=v;} }

    // HEIGHT of tree
    public static int height(TreeNode node) {
        if (node == null) return 0;
        return 1 + Math.max(height(node.left), height(node.right));
    }

    // COUNT nodes
    public static int count(TreeNode node) {
        if (node == null) return 0;
        return 1 + count(node.left) + count(node.right);
    }

    // SUM of all nodes
    public static int sum(TreeNode node) {
        if (node == null) return 0;
        return node.val + sum(node.left) + sum(node.right);
    }

    // IS SAME tree
    public static boolean isSame(TreeNode p, TreeNode q) {
        if (p == null && q == null) return true;
        if (p == null || q == null) return false;
        return p.val == q.val && isSame(p.left, q.left) && isSame(p.right, q.right);
    }

    // IS BALANCED
    public static boolean isBalanced(TreeNode node) {
        return checkHeight(node) != -1;
    }
    private static int checkHeight(TreeNode node) {
        if (node == null) return 0;
        int left = checkHeight(node.left);
        int right = checkHeight(node.right);
        if (left == -1 || right == -1 || Math.abs(left-right) > 1) return -1;
        return 1 + Math.max(left, right);
    }

    // PATH EXISTS from root to leaf with given sum
    public static boolean hasPathSum(TreeNode node, int target) {
        if (node == null) return false;
        if (node.left == null && node.right == null) return node.val == target;
        return hasPathSum(node.left, target - node.val)
            || hasPathSum(node.right, target - node.val);
    }

    // FIND all root-to-leaf paths
    public static void findPaths(TreeNode node, String path, List<String> result) {
        if (node == null) return;
        path += node.val;
        if (node.left == null && node.right == null) { result.add(path); return; }
        findPaths(node.left,  path + "->", result);
        findPaths(node.right, path + "->", result);
    }

    // LOWEST COMMON ANCESTOR (General Binary Tree)
    public static TreeNode lca(TreeNode root, TreeNode p, TreeNode q) {
        if (root == null || root == p || root == q) return root;
        TreeNode left  = lca(root.left,  p, q);
        TreeNode right = lca(root.right, p, q);
        if (left != null && right != null) return root;  // Found in both sides
        return left != null ? left : right;
    }
}
```

# 21. Recursion to Iteration Conversion

## The General Method

Every recursive function can be converted to iterative using an **explicit stack**.

```java
// RECURSIVE DFS
void dfsRecursive(TreeNode node) {
    if (node == null) return;
    visit(node);
    dfsRecursive(node.left);
    dfsRecursive(node.right);
}

// ITERATIVE DFS — explicit stack replaces call stack
void dfsIterative(TreeNode root) {
    if (root == null) return;
    Deque<TreeNode> stack = new ArrayDeque<>();
    stack.push(root);

    while (!stack.isEmpty()) {
        TreeNode node = stack.pop();
        visit(node);
        if (node.right != null) stack.push(node.right);  // Right first!
        if (node.left  != null) stack.push(node.left);   // Left second (processed first)
    }
}
```

## Factorial: Recursion → Iteration

```java
// Recursive (O(n) space)

    if (n <= 1) return 1;
    return n * factorial(n-1);
}

// Iterative (O(1) space)

    for (int i = 2; i <= n; i++) result *= i;
```

## Fibonacci: Recursion → Memoization → Iteration → Space Optimized

```java
// Level 1: Naive recursion — O(2^n)
int fib1(int n) { return n<=1 ? n : fib1(n-1)+fib1(n-2); }

// Level 2: Memoization — O(n) time, O(n) space
Map<Integer,Integer> memo = new HashMap<>();
int fib2(int n) {
    if (n<=1) return n;
    if (memo.containsKey(n)) return memo.get(n);
    return memo.put(n, fib2(n-1)+fib2(n-2)) == null ? memo.get(n) : memo.get(n);
}

// Level 3: Bottom-up DP — O(n) time, O(n) space
int fib3(int n) {
    if (n<=1) return n;
    int[] dp = new int[n+1];
    dp[0]=0; dp[1]=1;
    for (int i=2; i<=n; i++) dp[i] = dp[i-1]+dp[i-2];
    return dp[n];
}

// Level 4: Space optimized — O(n) time, O(1) space
int fib4(int n) {
    if (n<=1) return n;
    int a=0, b=1;
    for (int i=2; i<=n; i++) { int c=a+b; a=b; b=c; }
    return b;
}
```

```java
public boolean isMatch(String s, String p) {
    // Base cases
    if (p.isEmpty()) return s.isEmpty();

    boolean firstMatch = !s.isEmpty() &&
        (p.charAt(0) == s.charAt(0) || p.charAt(0) == '.');

    if (p.length() >= 2 && p.charAt(1) == '*') {
        // '*' can match 0 times: isMatch(s, p.substring(2))
        // OR match 1+ times if first matches: firstMatch && isMatch(s.substring(1), p)
        return isMatch(s, p.substring(2)) ||
               (firstMatch && isMatch(s.substring(1), p));
    } else {
        return firstMatch && isMatch(s.substring(1), p.substring(1));
    }
}

// With memoization
public boolean isMatchMemo(String s, String p) {
    Map<String, Boolean> memo = new HashMap<>();
    return dp(s, p, 0, 0, memo);
}

private boolean dp(String s, String p, int i, int j, Map<String, Boolean> memo) {
    String key = i + "," + j;
    if (memo.containsKey(key)) return memo.get(key);

    boolean result;
    if (j == p.length()) {
        result = i == s.length();
    } else {
        boolean firstMatch = i < s.length() &&
            (p.charAt(j) == s.charAt(i) || p.charAt(j) == '.');
        if (j + 1 < p.length() && p.charAt(j+1) == '*') {
            result = dp(s,p,i,j+2,memo) || (firstMatch && dp(s,p,i+1,j,memo));
        } else {
            result = firstMatch && dp(s,p,i+1,j+1,memo);
        }
    }
    memo.put(key, result);
    return result;
}
```

```java
                 long eval, long mult, String expr, List<String> result) {
    if (pos == num.length()) {
        if (eval == target) result.add(expr);
        return;
```

# 26. How to Think Through Any Recursion Problem

## The 4 Questions Framework

```
Ask yourself these 4 questions IN ORDER:

1. WHAT IS THE BASE CASE?
   → "What is the smallest possible input?"
   → "What can I return immediately without recursing?"

2. WHAT DOES THE RECURSIVE CALL RETURN?
   → "If I call myself on a smaller input, what do I get back?"
   → "How does the smaller answer help build the bigger answer?"

3. HOW DO I MAKE THE PROBLEM SMALLER?
   → "Which parameter decreases each call?"
   → "n-1? Left half? Substring? Next node?"

4. HOW DO I COMBINE THE RESULT?
   → "What do I do with what my recursive call returns?"
   → "Do I add it, max it, return it directly, or use it in backtracking?"
```

## Applied Example: Count Nodes in Tree

```
Question 1 (Base Case):
  "What's the simplest tree?" → null tree → return 0

Question 2 (What does recursive call return?):
  "count(left) returns the count of nodes in left subtree"
  "count(right) returns the count of nodes in right subtree"

Question 3 (Make smaller):
  "I recurse on node.left and node.right — both smaller trees"

Question 4 (Combine):
  "Total = 1 (this node) + count(left) + count(right)"

CODE:
int count(TreeNode node) {
    if (node == null) return 0;              // Q1: base case
    return 1 + count(node.left)              // Q4: combine
             + count(node.right);            // Q2+Q3: recursive calls on smaller input
}
```

# 27. Interview Cheat Sheet

## ⚡ Recursion Template
```java
returnType solve(parameters) {
    // 1. BASE CASE
    if (simplestInput) return simplestAnswer;

    // 2. RECURSIVE CASE (make it smaller!)
    returnType subResult = solve(smallerInput);

    // 3. COMBINE
    return combine(subResult, currentData);
}
```

## ⚡ Memoization Template
```java
Map<String, Integer> memo = new HashMap<>();

int solve(int n, /* other params */) {
    // Base case
    if (n <= 0) return 0;

    // Cache check
    String key = n + "," + otherParam;
    if (memo.containsKey(key)) return memo.get(key);

    // Compute
    int result = solve(n-1) + solve(n-2);  // or whatever

    // Store and return
    memo.put(key, result);
    return result;
}
```

## ⚡ Backtracking Template
```java
void backtrack(state, List<T> current, List<List<T>> result) {
    if (isComplete(state)) {
        result.add(new ArrayList<>(current));  // COPY!
        return;
    }
    for (T choice : getChoices(state)) {
        current.add(choice);              // CHOOSE
        backtrack(next state, current, result);  // EXPLORE
        current.remove(current.size()-1); // UNCHOOSE
    }
}
```

## ⚡ Convert Recursion → Iteration
```java
// Any DFS recursion can become:
Deque<State> stack = new ArrayDeque<>();
stack.push(initialState);
while (!stack.isEmpty()) {
    State curr = stack.pop();
    // process curr
    for (State next : getNextStates(curr)) {
        stack.push(next);
    }
}
```

## ⚡ The Recursion Complexity Rules

```
Time Complexity:
  Linear recursion (1 call, problem shrinks by 1): O(n)
  Binary recursion (2 calls, problem halves):      O(n log n) or O(n)
  Exponential (2 calls, problem shrinks by 1):     O(2^n)
  With memoization:                                O(unique states)

Space Complexity:
  Always at least O(depth of recursion tree)
  Depth = O(n) for linear, O(log n) for divide & conquer

The recurrence T(n) = 2T(n/2) + O(n) → O(n log n)  [Merge Sort]
The recurrence T(n) = T(n-1) + O(1)  → O(n)         [Factorial]
The recurrence T(n) = 2T(n-1) + O(1) → O(2^n)       [Naive Fibonacci]
```

## ⚡ The 5 Golden Rules of Recursion

```
1. ALWAYS define the base case first — before the recursive case.
   "If I can't make it smaller, what do I return?"

2. TRUST YOUR RECURSIVE CALL.
   Assume it works correctly on smaller inputs. Just use the result.
   Don't trace through the entire call tree mentally.

3. MAKE IT SMALLER EVERY CALL.
   The input to recursive call must be strictly smaller.
   n-1, arr[1..], left half, next node — something decreases.

4. For backtracking: ALWAYS UNDO what you did.
   current.remove() after current.add()
   visited[i]=false after visited[i]=true

5. When saving results in recursion, ALWAYS COPY mutable objects.
   result.add(new ArrayList<>(current))  NOT  result.add(current)
```
