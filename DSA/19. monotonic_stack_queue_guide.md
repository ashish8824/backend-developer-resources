# Part 1 — Foundations

# 1. What is a Monotonic Stack?

## The Core Idea

```
A MONOTONIC STACK is a stack that maintains elements
in either strictly increasing or strictly decreasing order
from bottom to top.

MONOTONIC INCREASING STACK (bottom to top: small → large):
  Elements are in increasing order.
  When a new element arrives:
    → Pop all elements LARGER than the new element first.
    → Then push the new element.
  Result: stack always increases from bottom to top.

MONOTONIC DECREASING STACK (bottom to top: large → small):
  Elements are in decreasing order.
  When a new element arrives:
    → Pop all elements SMALLER than the new element first.
    → Then push the new element.
  Result: stack always decreases from bottom to top.

WHAT HAPPENS DURING POPS?
  This is the key! When we pop element X because new element Y arrived:
    → For INCREASING stack: Y > X, so Y is the NEXT GREATER element of X.
    → For DECREASING stack: Y < X, so Y is the NEXT SMALLER element of X.
  
  The popping moment reveals a relationship between X and Y.
  This is exactly what we exploit.
```

## Visual Example

```
Array: [3, 1, 4, 1, 5, 9, 2, 6]
Build MONOTONIC INCREASING STACK (pop when new element is smaller):

Process 3: stack=[]       → push 3.    stack=[3]
Process 1: stack=[3]      → 1<3, pop 3 (3's next smaller = 1!)
                          → push 1.    stack=[1]
Process 4: stack=[1]      → 4>1, push. stack=[1,4]
Process 1: stack=[1,4]    → 1<4, pop 4 (4's next smaller = 1!)
                          → 1=1, equal: depends on strict/non-strict
                          → push 1.    stack=[1,1]
Process 5: stack=[1,1]    → push.      stack=[1,1,5]
Process 9: stack=[1,1,5]  → push.      stack=[1,1,5,9]
Process 2: stack=[1,1,5,9]→ 2<9, pop 9 (9's next smaller = 2!)
                          → 2<5, pop 5 (5's next smaller = 2!)
                          → 2>1, push. stack=[1,1,2]
Process 6: stack=[1,1,2]  → push.      stack=[1,1,2,6]

At the end, remaining elements in stack have NO next smaller element.

KEY INSIGHT: Every element is pushed once and popped at most once → O(n) total!
```

# 2. What is a Monotonic Queue?

```
A MONOTONIC QUEUE (usually implemented as a DEQUE) maintains
elements in order, supporting efficient front and back operations.

Unlike a stack (LIFO), a deque allows:
  - Add/remove from FRONT (pollFirst, peekFirst)
  - Add/remove from BACK (pollLast, peekLast)

MONOTONIC DECREASING QUEUE:
  Elements decrease from front to back.
  Front = maximum element in current window.
  When adding new element:
    → Remove from BACK all elements smaller than new element
      (they can never be maximum while new element is in window).
    → Add new element to BACK.
  When window slides:
    → Remove from FRONT if it's outside the window.

USE CASE: Sliding window maximum/minimum in O(n).

WHY DEQUE AND NOT STACK?
  Stack only allows access to one end.
  We need to:
    → ADD to back (new elements)
    → QUERY front (current max/min)
    → REMOVE from front (when outside window)
    → REMOVE from back (when a larger/smaller element arrives)
  Only a deque supports all four operations efficiently.
```

# 3. Why They Work — The Core Insight

## The Fundamental Property

```
MONOTONIC STACK KEY INSIGHT:
  For every element in the array, the monotonic stack tells us
  its "nearest" neighbor satisfying a condition.

  "Nearest" = closest in index direction (left or right)
  "Condition" = greater than, smaller than, etc.

  When element X is POPPED from the stack because element Y arrives:
    → Y is the FIRST element after X (to the right) that "breaks" the order.
    → For increasing stack (pop when smaller): Y < X → Y = Next Smaller of X
    → For decreasing stack (pop when greater): Y > X → Y = Next Greater of X

  Elements remaining in stack at the end:
    → They were never popped → no element to their right breaks the order.
    → They have NO next greater/smaller element.

MONOTONIC QUEUE KEY INSIGHT:
  In any sliding window, we only care about the maximum (or minimum).
  If element A came before element B and A ≤ B:
    → A can never be the maximum while B is in the window.
    → A is USELESS — remove it from the deque.
  This ensures the deque only holds "candidates" for window maximum.
  The front of deque = current window maximum.
```

## Why O(n) Not O(n²)?

```
NAIVE APPROACH (O(n²)):
  For each element, scan right to find next greater → O(n) per element.
  Total: O(n²).

MONOTONIC STACK (O(n)):
  Each element is pushed ONCE: O(n) pushes total.
  Each element is popped AT MOST ONCE: O(n) pops total.
  Total operations = O(n) pushes + O(n) pops = O(n).

This is the amortized analysis:
  Even though we might pop many elements for a single new element,
  across the entire array, total pops = total pushes = n.
  So amortized cost per element = O(1).
```

# 4. Increasing vs Decreasing — Which to Use?

```
QUESTION: "For each element, find its NEXT GREATER element to the right."
→ Use MONOTONIC DECREASING STACK.
  Keep large elements waiting. When smaller element arrives, it's not the answer.
  When LARGER element arrives → it's the next greater of stack's top.
  Pop on: new element > stack top.

QUESTION: "For each element, find its NEXT SMALLER element to the right."
→ Use MONOTONIC INCREASING STACK.
  Keep small elements waiting. When new SMALLER element arrives → it's the answer.
  Pop on: new element < stack top.

QUESTION: "Find PREVIOUS GREATER element."
→ Process left to right, use DECREASING STACK.
  Stack top = nearest previous greater.
  When we try to push element X: stack top (if exists) = previous greater of X.

QUESTION: "Find PREVIOUS SMALLER element."
→ Process left to right, use INCREASING STACK.
  Stack top (if exists) = previous smaller of X.

MEMORY TRICK:
  NEXT problem:       process left-to-right, POP when condition met
  PREVIOUS problem:   process left-to-right, CHECK top BEFORE pushing
  
  GREATER: use DECREASING stack (pop when new > top)
  SMALLER: use INCREASING stack (pop when new < top)

SUMMARY TABLE:
  Problem                  | Stack Type   | Pop Condition
  ─────────────────────────────────────────────────────────
  Next Greater Element     | Decreasing   | new > top
  Next Smaller Element     | Increasing   | new < top
  Previous Greater Element | Decreasing   | checked before push
  Previous Smaller Element | Increasing   | checked before push
  Largest Rectangle        | Increasing   | new < top
  Sliding Window Max       | Decreasing   | (queue, not stack)
  Sliding Window Min       | Increasing   | (queue, not stack)
```

# 5. The Mental Model That Makes Everything Click

```
THINK OF IT AS A "WAITING LINE":

Elements enter the stack/queue and WAIT for their answer.
When an element finds its answer (next greater/smaller), it leaves.

DECREASING STACK for Next Greater Element:
  - Element enters the "waiting room" (stack).
  - It waits for someone TALLER (greater) to come along.
  - When someone taller arrives, everyone shorter leaves (they found their answer).
  - The tall newcomer enters the waiting room.

  Example: [2, 1, 5, 6, 2, 3]
  
  "Who is 2 waiting for? Someone > 2."
  "Who is 1 waiting for? Someone > 1."
  When 5 arrives: both 2 and 1 find their answer (5).
  "Who is 5 waiting for? Someone > 5."
  When 6 arrives: 5 finds its answer.
  ...

INCREASING STACK for Next Smaller Element:
  - Elements wait for someone SHORTER (smaller) to come along.
  - A small element evicts all larger elements above it.

DECREASING QUEUE for Sliding Window Max:
  - Front of queue = current maximum (the "king").
  - New large element evicts smaller elements (they can never be king while it exists).
  - When window slides past front element, the king retires.
```

# 6. Pattern 1 — Next Greater Element

## Template

```java
// For each element, find the NEXT element to its RIGHT that is GREATER.
// Returns array where result[i] = next greater element of nums[i], or -1 if none.

int[] nextGreaterElement(int[] nums) {
    int n = nums.length;
    int[] result = new int[n];
    Arrays.fill(result, -1);  // Default: no next greater element

    Deque<Integer> stack = new ArrayDeque<>();  // Stack of INDICES

    for (int i = 0; i < n; i++) {
        // Pop all elements smaller than nums[i]
        // For them, nums[i] IS their next greater element
        while (!stack.isEmpty() && nums[stack.peek()] < nums[i]) {
            result[stack.pop()] = nums[i];
        }
        stack.push(i);
    }
    // Remaining elements in stack have no next greater element (result stays -1)
    return result;
}
```

**Trace:**
```
nums = [2, 1, 5, 6, 2, 3]

i=0: stack=[]. push 0. stack=[0]       (stack stores indices)
i=1: nums[0]=2 > nums[1]=1. push 1. stack=[0,1]
i=2: nums[1]=1 < 5 → pop 1, result[1]=5
     nums[0]=2 < 5 → pop 0, result[0]=5
     push 2. stack=[2]
i=3: nums[2]=5 < 6 → pop 2, result[2]=6
     push 3. stack=[3]
i=4: nums[3]=6 > 2. push 4. stack=[3,4]
i=5: nums[4]=2 < 3 → pop 4, result[4]=3
     nums[3]=6 > 3. push 5. stack=[3,5]

Remaining in stack: indices 3,5 → result[3]=result[5]=-1

result = [5, 5, 6, -1, 3, -1] ✓
```

## 🟢 LC #496 — Next Greater Element I

```java
// nums1 is a subset of nums2. For each element in nums1,
// find its next greater element in nums2.
public int[] nextGreaterElement(int[] nums1, int[] nums2) {
    Map<Integer, Integer> nextGreater = new HashMap<>();
    Deque<Integer> stack = new ArrayDeque<>();

    // Build next greater map for nums2
    for (int num : nums2) {
        while (!stack.isEmpty() && stack.peek() < num) {
            nextGreater.put(stack.pop(), num);
        }
        stack.push(num);
    }

    // Answer queries for nums1
    int[] result = new int[nums1.length];
    for (int i = 0; i < nums1.length; i++) {
        result[i] = nextGreater.getOrDefault(nums1[i], -1);
    }
    return result;
}
```

## 🟡 LC #503 — Next Greater Element II (Circular Array)

```java
// Array is circular (wrap around). Find next greater in circular sense.
public int[] nextGreaterElements(int[] nums) {
    int n = nums.length;
    int[] result = new int[n];
    Arrays.fill(result, -1);
    Deque<Integer> stack = new ArrayDeque<>();

    // Traverse TWICE to simulate circular array
    for (int i = 0; i < 2 * n; i++) {
        int num = nums[i % n];  // Wrap around using modulo
        while (!stack.isEmpty() && nums[stack.peek()] < num) {
            result[stack.pop()] = num;
        }
        if (i < n) stack.push(i);  // Only push indices from first pass
    }
    return result;
}
```

**Why traverse twice?** In a circular array, element at index n-1 might have its next greater at index 0. Going through the array twice ensures all elements can "look ahead" past the end.

## 🟡 LC #556 — Next Greater Element III

```java
// Given integer n, find the next permutation (smallest number > n using same digits)
public int nextGreaterElement(int n) {
    char[] digits = String.valueOf(n).toCharArray();
    int len = digits.length;

    // Find rightmost digit smaller than digit to its right
    int i = len - 2;
    while (i >= 0 && digits[i] >= digits[i + 1]) i--;
    if (i < 0) return -1;  // Already largest permutation

    // Find smallest digit to the right of i that is greater than digits[i]
    int j = len - 1;
    while (digits[j] <= digits[i]) j--;
    char temp = digits[i]; digits[i] = digits[j]; digits[j] = temp;

    // Reverse the suffix (right part of i)
    int l = i + 1, r = len - 1;
    while (l < r) { temp=digits[l]; digits[l++]=digits[r]; digits[r--]=temp; }

    long result = Long.parseLong(new String(digits));
    return result > Integer.MAX_VALUE ? -1 : (int) result;
}
```

## 🟡 LC #739 — Daily Temperatures

```java
// For each day, find how many days until a warmer temperature.
public int[] dailyTemperatures(int[] temperatures) {
    int n = temperatures.length;
    int[] result = new int[n];
    Deque<Integer> stack = new ArrayDeque<>();  // Indices of days waiting

    for (int i = 0; i < n; i++) {
        // Current day is warmer than days on stack → those days found their answer
        while (!stack.isEmpty() && temperatures[stack.peek()] < temperatures[i]) {
            int idx = stack.pop();
            result[idx] = i - idx;  // Days waited = current index - waiting index
        }
        stack.push(i);
    }
    return result;
    // Remaining in stack → no warmer day → result stays 0 (default)
}
```

# 8. Pattern 3 — Previous Greater Element

## Template

```java
// For each element, find the nearest element to its LEFT that is GREATER.
int[] previousGreaterElement(int[] nums) {
    int n = nums.length;
    int[] result = new int[n];
    Arrays.fill(result, -1);
    Deque<Integer> stack = new ArrayDeque<>();  // Decreasing stack

    for (int i = 0; i < n; i++) {
        // Remove elements that are NOT greater than nums[i]
        while (!stack.isEmpty() && stack.peek() <= nums[i]) {
            stack.pop();
        }
        // Stack top (if exists) = nearest previous greater
        result[i] = stack.isEmpty() ? -1 : stack.peek();
        stack.push(nums[i]);
    }
    return result;
}
```

**Key difference from "next" pattern:**
- Next: check condition WHEN popping (the popping moment reveals the answer)
- Previous: check stack top BEFORE pushing (the top is the answer)

## 🟡 LC #901 — Online Stock Span

```java
// Stock span = number of consecutive days (including today) with price ≤ today's price.
class StockSpanner {
    Deque<int[]> stack = new ArrayDeque<>();  // [price, span]

    public int next(int price) {
        int span = 1;

        // Absorb all previous prices ≤ current price
        // Their spans contribute to current span
        while (!stack.isEmpty() && stack.peek()[0] <= price) {
            span += stack.pop()[1];  // Add their span to current
        }

        stack.push(new int[]{price, span});
        return span;
    }
}
```

# 10. Pattern 5 — Largest Rectangle / Area Problems

## The Fundamental Idea

```
LARGEST RECTANGLE IN HISTOGRAM:
  Given bar heights, find the largest rectangle you can form.
  
  For each bar i, the widest rectangle using bar i as the SHORTEST bar:
    → Extends left to the first bar shorter than bar[i]
    → Extends right to the first bar shorter than bar[i]
  
  Area = height[i] × (rightBoundary[i] - leftBoundary[i] - 1)
  
  We need: for each bar, its PREVIOUS SMALLER and NEXT SMALLER bar.
  → Monotonic increasing stack gives both in one pass!

STACK APPROACH:
  Use increasing stack (pop when new bar is shorter).
  When we pop bar i (because bar j is shorter):
    → j = right boundary of i (next smaller to right)
    → stack top after pop = left boundary (previous smaller)
    → width = j - stack.peek() - 1 (or j if stack empty)
    → area = height[i] × width
```

## 🔴 LC #84 — Largest Rectangle in Histogram

```java
public int largestRectangleArea(int[] heights) {
    int n = heights.length;
    Deque<Integer> stack = new ArrayDeque<>();  // Indices, increasing by height
    int maxArea = 0;

    for (int i = 0; i <= n; i++) {
        int currHeight = (i == n) ? 0 : heights[i];  // Sentinel: 0 at end forces all pops

        while (!stack.isEmpty() && heights[stack.peek()] > currHeight) {
            int height = heights[stack.pop()];
            // Width: from (stack.peek()+1) to (i-1)
            int width = stack.isEmpty() ? i : i - stack.peek() - 1;
            maxArea = Math.max(maxArea, height * width);
        }
        stack.push(i);
    }
    return maxArea;
}
```

```
i=1: heights[0]=2 > 1 → pop 0.
     height=2, width=(stack empty? 1 : 1-(-1)-1)=1. area=2*1=2.
     push 1. stack=[1]
i=2: 1<5. push 2. stack=[1,2]
i=3: 5<6. push 3. stack=[1,2,3]
```

```java
// For each row, treat it as a histogram base, apply LC #84.
public int maximalRectangle(char[][] matrix) {
    int m = matrix.length, n = matrix[0].length;
    int[] heights = new int[n];
    int maxArea = 0;

    for (int r = 0; r < m; r++) {
```

# 11. Pattern 6 — Sum of Subarray Min/Max

## The Idea

```
PROBLEM: Find sum of minimum of every subarray.
         Equivalently: for each element, how many subarrays is it the MINIMUM?

For element A[i] to be minimum of subarray [l, r]:
  → All elements in [l, r] must be ≥ A[i].
  → The subarray cannot extend left past the previous smaller element.
  → The subarray cannot extend right past the next smaller element.

For element at index i:
  left[i]  = distance to previous element SMALLER than A[i]
  right[i] = distance to next element SMALLER THAN OR EQUAL to A[i]
  (Use strict < on one side and ≤ on other to avoid double counting equal elements)

Number of subarrays where A[i] is minimum = left[i] × right[i]
Contribution = A[i] × left[i] × right[i]
Answer = sum of all contributions.
```

---

## 🔴 LC #907 — Sum of Subarray Minimums

```java
public int sumSubarrayMins(int[] arr) {
    long MOD = 1_000_000_007L;
    int n = arr.length;

    // For each element, find:
    // left[i] = number of subarrays ending at i where arr[i] is minimum
    // right[i] = number of subarrays starting at i where arr[i] is minimum

    int[] left = new int[n];   // distance to previous SMALLER (strict)
    int[] right = new int[n];  // distance to next SMALLER OR EQUAL

    Deque<Integer> stack = new ArrayDeque<>();

    // Compute left[i]: previous smaller element
    for (int i = 0; i < n; i++) {
        while (!stack.isEmpty() && arr[stack.peek()] >= arr[i]) stack.pop();
        left[i] = stack.isEmpty() ? i + 1 : i - stack.peek();
        stack.push(i);
    }

    stack.clear();

    // Compute right[i]: next smaller or equal element
    for (int i = n - 1; i >= 0; i--) {
        while (!stack.isEmpty() && arr[stack.peek()] > arr[i]) stack.pop();
        right[i] = stack.isEmpty() ? n - i : stack.peek() - i;
        stack.push(i);
    }

    // Sum contributions
    long result = 0;
    for (int i = 0; i < n; i++) {
        result = (result + (long) arr[i] * left[i] * right[i]) % MOD;
    }
    return (int) result;
}
```

**Why strict on left and ≤ on right?** To avoid double-counting when equal elements exist. If `arr[i] == arr[j]` and i < j, only count the subarray once. The convention: left uses strict `<` (stop at equal), right uses `≤` (stop at equal or smaller).

```java
// Sum of (max - min) of every subarray = sum of max - sum of min
public long subArrayRanges(int[] nums) {

private long sumSubarrayMaxs(int[] arr) {
    int n = arr.length;
    long result = 0;
    Deque<Integer> stack = new ArrayDeque<>();

    for (int i = 0; i <= n; i++) {
        while (!stack.isEmpty() && (i == n || arr[stack.peek()] < arr[i])) {
            int mid = stack.pop();
            int left = stack.isEmpty() ? -1 : stack.peek();
            int right = i;
            result += (long) arr[mid] * (mid - left) * (right - mid);

        stack.push(i);
    }
    return result;
}

            int left = stack.isEmpty() ? -1 : stack.peek();
            int right = i;
            result += (long) arr[mid] * (mid - left) * (right - mid);
```

# 12. Pattern 7 — Sliding Window Maximum

## The Core Idea

```
PROBLEM: Given array and window size k, find maximum in each window.

NAIVE: For each window position, scan all k elements → O(nk).

DEQUE APPROACH: O(n)
  Maintain a DECREASING deque of indices.
  Front = index of maximum in current window.
  
  For each new element at index i:
    1. Remove indices from FRONT that are outside window (index < i-k+1).
    2. Remove indices from BACK where value ≤ current value
       (they can never be max while current element is in window).
    3. Add current index to BACK.
    4. If window is full (i >= k-1): front of deque = max index.

WHY REMOVE FROM BACK?
  If deque has index j with nums[j] ≤ nums[i]:
    → j came before i, so j will leave the window before i.
    → While i is in the window, j is never the max (nums[i] ≥ nums[j]).
    → j is useless — remove it.
  Result: deque is always DECREASING (front = max).
```

---

## 🔴 LC #239 — Sliding Window Maximum

```java
public int[] maxSlidingWindow(int[] nums, int k) {
    int n = nums.length;
    int[] result = new int[n - k + 1];
    Deque<Integer> deque = new ArrayDeque<>();  // Stores INDICES, decreasing by VALUE

    for (int i = 0; i < n; i++) {
        // Step 1: Remove indices outside current window from FRONT
        while (!deque.isEmpty() && deque.peekFirst() < i - k + 1) {
            deque.pollFirst();
        }

        // Step 2: Remove indices from BACK where value <= current
        // (they can never be max while nums[i] is in window)
        while (!deque.isEmpty() && nums[deque.peekLast()] <= nums[i]) {
            deque.pollLast();
        }

        // Step 3: Add current index
        deque.offerLast(i);

        // Step 4: Record result once window is full
        if (i >= k - 1) {
            result[i - k + 1] = nums[deque.peekFirst()];
        }
    }
    return result;
}
```

## 🔴 LC #1438 — Longest Continuous Subarray With Absolute Diff ≤ Limit

```java
public int longestSubarray(int[] nums, int limit) {
    Deque<Integer> maxDeque = new ArrayDeque<>();  // Decreasing (front = max)
    Deque<Integer> minDeque = new ArrayDeque<>();  // Increasing (front = min)
    int left = 0, result = 0;

    for (int right = 0; right < nums.length; right++) {
        // Maintain max deque
        while (!maxDeque.isEmpty() && nums[maxDeque.peekLast()] <= nums[right])
            maxDeque.pollLast();
        maxDeque.offerLast(right);

        // Maintain min deque
        while (!minDeque.isEmpty() && nums[minDeque.peekLast()] >= nums[right])
            minDeque.pollLast();
        minDeque.offerLast(right);

        // Shrink window if condition violated
        while (nums[maxDeque.peekFirst()] - nums[minDeque.peekFirst()] > limit) {
            if (maxDeque.peekFirst() == left) maxDeque.pollFirst();
            if (minDeque.peekFirst() == left) minDeque.pollFirst();
            left++;
        }

        result = Math.max(result, right - left + 1);
    }
    return result;
}
```

# 16. Trapping Rain Water

## The Key Insight

```
For each cell i, water trapped = min(maxLeft[i], maxRight[i]) - height[i]
(water is limited by the shorter of the two walls)

MONOTONIC STACK APPROACH:
  Think horizontally: water fills in layers.
  When we find a "valley" (previous height > current > next),
  water fills between the left wall and right wall.

  Use INCREASING stack:
    When new bar is TALLER: we found the right wall of a valley.
    Pop the valley bottom.
    Stack top after pop = left wall.
    Water = min(leftWall, rightWall) - valleyBottom) * width.
```

---

## 🔴 LC #42 — Trapping Rain Water

```java
// Monotonic stack approach
public int trap(int[] height) {
    int n = height.length;
    Deque<Integer> stack = new ArrayDeque<>();
    int water = 0;

    for (int i = 0; i < n; i++) {
        // While current bar is taller than stack top → found right wall
        while (!stack.isEmpty() && height[stack.peek()] < height[i]) {
            int bottom = stack.pop();             // Valley floor
            if (stack.isEmpty()) break;           // No left wall

            int leftWall = stack.peek();
            int boundedHeight = Math.min(height[leftWall], height[i]) - height[bottom];
            int width = i - leftWall - 1;
            water += boundedHeight * width;
        }
        stack.push(i);
    }
    return water;
}

// TWO POINTER approach (simpler, also O(n) O(1)):
public int trapTwoPointer(int[] height) {
    int left = 0, right = height.length - 1;
    int leftMax = 0, rightMax = 0, water = 0;

    while (left < right) {
        if (height[left] < height[right]) {
            if (height[left] >= leftMax) leftMax = height[left];
            else water += leftMax - height[left];
            left++;
        } else {
            if (height[right] >= rightMax) rightMax = height[right];
            else water += rightMax - height[right];
            right--;
        }
    }
    return water;
}
```

```
  Use DECREASING stack: pop when new digit is LARGER.
```

```java
// Remove k digits to make the smallest number
public String removeKdigits(String num, int k) {
    Deque<Character> stack = new ArrayDeque<>();

    for (char c : num.toCharArray()) {
        // Remove digits larger than current (greedy: smaller digits should come first)
        while (k > 0 && !stack.isEmpty() && stack.peek() > c) {
            stack.pop();
            k--;
        }
        stack.push(c);
    }

    // Remove k remaining digits from end (they're the largest remaining)
    while (k-- > 0) stack.pop();

    // Build result, skip leading zeros
    StringBuilder sb = new StringBuilder();
    for (char c : stack) sb.append(c);  // Iterates bottom to top
    // Note: Deque iteration order is from tail to head for push/pop,
    // but for-each goes bottom to top. Verify with your implementation.

    // Better: collect properly
    char[] result = new char[stack.size()];
    int i = stack.size() - 1;
    while (!stack.isEmpty()) result[i--] = stack.pop();
    // Remove leading zeros
    int start = 0;
    while (start < result.length - 1 && result[start] == '0') start++;
    return new String(result, start, result.length - start);
}
```

## 🟡 LC #316 — Remove Duplicate Letters

```java
// Remove duplicate letters so each appears once, result is smallest lexicographically
public String removeDuplicateLetters(String s) {
    int[] remaining = new int[26];  // Count of remaining occurrences
    boolean[] inStack = new boolean[26];
    for (char c : s.toCharArray()) remaining[c - 'a']++;

    Deque<Character> stack = new ArrayDeque<>();

    for (char c : s.toCharArray()) {
        remaining[c - 'a']--;

        if (inStack[c - 'a']) continue;  // Already in result

        // Remove larger chars that appear later (can be added back later)
        while (!stack.isEmpty() && stack.peek() > c && remaining[stack.peek() - 'a'] > 0) {
            inStack[stack.pop() - 'a'] = false;
        }

        stack.push(c);
        inStack[c - 'a'] = true;
    }

    // Build result from stack (bottom to top)
    char[] result = new char[stack.size()];
    int i = stack.size() - 1;
    while (!stack.isEmpty()) result[i--] = stack.pop();
    return new String(result);
}
```

```java
// Evaluate expression with +, -, *, / (no parentheses)
// Use stack to handle operator precedence
public int calculate(String s) {
    Deque<Integer> stack = new ArrayDeque<>();
    int num = 0;
    char op = '+';  // Track previous operator

    for (int i = 0; i < s.length(); i++) {
        char c = s.charAt(i);

        if (Character.isDigit(c)) {
            num = num * 10 + (c - '0');
        }

        if ((!Character.isDigit(c) && c != ' ') || i == s.length() - 1) {
            // Apply previous operator
            if (op == '+') stack.push(num);
            else if (op == '-') stack.push(-num);
            else if (op == '*') stack.push(stack.pop() * num);
            else if (op == '/') stack.push(stack.pop() / num);

            op = c;   // Update operator
            num = 0;  // Reset number
        }
    }

    // Sum everything in stack
    int result = 0;
    while (!stack.isEmpty()) result += stack.pop();
    return result;
}
```

```java
// Minimum sum of non-leaf nodes where each non-leaf = product of two largest leaves in subtrees
// Greedy: always multiply the two smallest adjacent elements
public int mctFromLeafValues(int[] arr) {
    int result = 0;
    Deque<Integer> stack = new ArrayDeque<>();
    stack.push(Integer.MAX_VALUE);  // Sentinel

    for (int num : arr) {
        while (stack.peek() <= num) {
            int mid = stack.pop();
            result += mid * Math.min(stack.peek(), num);
        }
        stack.push(num);
    }

    // Multiply remaining elements
    while (stack.size() > 2) {
        int top = stack.pop();
        result += top * stack.peek();
    }
    return result;
}
```

```java
// Find subarray of size >= k where minimum > threshold / k
public int validSubarraySize(int[] nums, int threshold) {
    int n = nums.length;
    int[] left = new int[n];   // Previous smaller element index
    int[] right = new int[n];  // Next smaller element index
    Arrays.fill(right, n);

    Deque<Integer> stack = new ArrayDeque<>();

    // Previous smaller
    for (int i = 0; i < n; i++) {
        while (!stack.isEmpty() && nums[stack.peek()] >= nums[i]) stack.pop();
        left[i] = stack.isEmpty() ? -1 : stack.peek();
        stack.push(i);
    }
    stack.clear();

    for (int i = n - 1; i >= 0; i--) {
        while (!stack.isEmpty() && nums[stack.peek()] >= nums[i]) stack.pop();
        right[i] = stack.isEmpty() ? n : stack.peek();
        stack.push(i);

    for (int i = 0; i < n; i++) {
        int k = right[i] - left[i] - 1;  // Subarray size where nums[i] is min
        if (nums[i] > threshold / k) return k;

}
```

# 23. How to Recognize Monotonic Stack/Queue Problems

```
SIGNAL WORDS → LIKELY PATTERN
──────────────────────────────────────────────────────────────────
"next greater element"                    → Decreasing stack
"next smaller element"                    → Increasing stack
"previous greater/smaller"                → Stack, check before push
"daily temperatures / days until warmer"  → Decreasing stack (NGE)
"stock span"                              → Decreasing stack (prev greater)
"largest rectangle"                       → Increasing stack (NSE)
"trapping rain water"                     → Increasing stack or two pointers
"remove k digits / make smallest"         → Increasing stack (greedy)
"remove duplicate letters"                → Increasing stack + frequency
"sliding window maximum"                  → Decreasing monotonic deque
"sliding window minimum"                  → Increasing monotonic deque
"max - min <= limit in window"            → Two deques
"sum of subarray min/max"                 → Contribution with prev/next smaller
"132 pattern"                             → Decreasing stack from right

DECISION FLOWCHART:
  Is it a WINDOW/RANGE problem?
    YES → Does window slide?
            YES → MONOTONIC DEQUE (max/min per window)
            NO  → Check others
    NO  → Is it about NEAREST element (left or right)?
            YES, to the right → MONOTONIC STACK (pop = find answer)
            YES, to the left  → MONOTONIC STACK (check top before push)
    NO  → Is it about AREA or counting (rectangles, rain water)?
            YES → INCREASING STACK (for NSE to define boundaries)
    NO  → Is it about LEXICOGRAPHIC order / removing elements?
            YES → INCREASING STACK (smaller replaces larger earlier)
    NO  → Is it CONTRIBUTION (sum of min/max of all subarrays)?
            YES → MONOTONIC STACK for prev/next smaller/greater

VERIFY IT'S MONOTONIC STACK IF:
  ✅ Brute force is O(n²) — for each element, scan all others.
  ✅ You need "nearest" neighbor satisfying a condition.
  ✅ Relationship between elements depends on relative order.
  ✅ Problem involves comparing heights, prices, temperatures, etc.
```

```java
// 4. SLIDING WINDOW MAXIMUM (Decreasing Deque)
// ============================================================
int[] slideMax(int[] nums, int k) {
    int n = nums.length;
    int[] res = new int[n - k + 1];
    Deque<Integer> dq = new ArrayDeque<>();
    for (int i = 0; i < n; i++) {
        while (!dq.isEmpty() && dq.peekFirst() < i - k + 1) dq.pollFirst();
        while (!dq.isEmpty() && nums[dq.peekLast()] <= nums[i]) dq.pollLast();
        dq.offerLast(i);
        if (i >= k - 1) res[i - k + 1] = nums[dq.peekFirst()];
    }
    return res;
}
```

## ⚡ Quick Reference: What to Pop, What to Keep

```
DECREASING STACK:
  Pop when: new element > stack top
  Maintains: large values at bottom, small at top
  Answers: Next Greater (on pop), Previous Greater (on peek before push)
  Use for: NGE, stock span, sliding window max (deque)

INCREASING STACK:
  Pop when: new element < stack top
  Maintains: small values at bottom, large at top
  Answers: Next Smaller (on pop), Previous Smaller (on peek before push)
  Use for: NSE, largest rectangle, trapping rain water, remove k digits

STORE INDICES not values when:
  → Need to compute widths/distances (area problems)
  → Need to know position of elements (next greater with index)
  → Sliding window (need to check if index is in window)

STORE VALUES when:
  → Only care about the value relationship
  → Stock span (accumulate spans)
  → Simple previous/next comparisons
```

## ⚡ Complexity Analysis

```
MONOTONIC STACK:
  Time: O(n) — each element pushed once, popped at most once.
  Space: O(n) — stack holds at most n elements.

MONOTONIC DEQUE (Sliding Window):
  Time: O(n) — each element added to deque once, removed at most once.
  Space: O(k) — deque holds at most k elements at any time.

COMPARE TO NAIVE:
  Next Greater (naive):        O(n²) → O(n) with stack
  Sliding Window Max (naive):  O(nk) → O(n) with deque
  Largest Rectangle (naive):   O(n²) → O(n) with stack
  Sum Subarray Min (naive):    O(n²) → O(n) with stack contribution
```

## ⚡ The 5 Golden Rules

```
1. STORE INDICES, NOT VALUES (usually):
   Storing indices lets you compute widths, check window bounds,
   and look up original values. Always prefer indices.

2. MARK BEFORE PUSH, CHECK ON POP:
   "Next" problems: the answer is revealed WHEN you pop.
   "Previous" problems: the answer is the CURRENT stack top BEFORE pushing.

3. SENTINEL VALUES prevent edge case bugs:
   Append 0 to histogram → forces all bars to be popped at end.
   Use INT_MAX/INT_MIN as boundaries → simplifies previous element logic.

4. DEQUE FOR SLIDING WINDOW, STACK FOR GLOBAL:
   If problem involves a fixed or sliding window → use deque.
   If problem involves the entire array → use stack.

5. WHEN IN DOUBT, DRAW:
   Draw the stack state at each step.
   Ask: "What does popping mean here? What relationship does it reveal?"
   The moment of popping is where the answer lives.
```

## ⚡ Common Mistakes and Fixes

```java
// ❌ MISTAKE 1: Pop condition wrong (>= vs >)
// For strict next greater: pop when nums[stack.peek()] < nums[i]
// For next greater or equal: pop when nums[stack.peek()] <= nums[i]
// Choose based on problem: "strictly greater" vs "greater or equal"

// ❌ MISTAKE 2: Not handling empty stack
while (!stack.isEmpty() && nums[stack.peek()] < nums[i]) {
    // ...
}
// Always check isEmpty() FIRST before comparing.

// ❌ MISTAKE 3: Wrong deque operation
// Front operations: peekFirst(), pollFirst() — for window boundary removal
// Back operations:  peekLast(), pollLast()  — for maintaining monotonic property
// Don't confuse which end does what!

// ❌ MISTAKE 4: Forgetting sentinel in area problems
// Without sentinel: elements remaining at end of array never get processed.
// Fix: add a 0-height bar at the end to force all pops.
for (int i = 0; i <= n; i++) {  // <= n, not < n
    int h = (i == n) ? 0 : heights[i];  // 0 sentinel at the end
    // ...
}

// ❌ MISTAKE 5: Using stack when deque needed
// If you need to add to one end AND remove from the other → use Deque.
// Java's ArrayDeque implements both stack and queue operations.
// Never use Stack class in interviews — use ArrayDeque instead.
Deque<Integer> stack = new ArrayDeque<>();  // Use as stack
Deque<Integer> queue = new ArrayDeque<>();  // Use as queue/deque
```
