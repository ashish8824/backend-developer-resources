# Part 1 — Foundations

# 1. What is an Interval Problem?

## The Core Idea

```
An interval is a range [start, end] representing something that:
  - Has a start point and end point
  - Can overlap with other intervals

Real examples:
  Meeting: [9, 11] means meeting from 9am to 11am
  Job:     [1, 3]  means job active from day 1 to day 3
  Range:   [2, 7]  means numbers 2 through 7

OVERLAP: Two intervals [a,b] and [c,d] overlap if and only if a <= d AND c <= b
  [1,4] and [3,6] → overlap (3 <= 4 and 1 <= 6)
  [1,2] and [3,6] → NO overlap (3 > 2)
  [1,3] and [3,6] → overlap (3 <= 3) ← touching counts as overlap usually
                                         (depends on problem — read carefully!)

NON-OVERLAP: [a,b] and [c,d] don't overlap if b < c (b comes strictly before c)
```

## Why Intervals Are Confusing

```
There are 5 COMPLETELY DIFFERENT types of interval problems:

1. MERGE:    Given overlapping intervals, merge them into non-overlapping ones
2. INSERT:   Given sorted intervals, insert a new one and merge if needed
3. REMOVE:   Find minimum intervals to remove so none overlap
4. SCHEDULE: Find minimum rooms/resources needed for all intervals
5. SWEEP:    Count events at each point, find peak, coverage problems

Each type needs a DIFFERENT approach.
Identifying the type = 90% of solving the problem.
```

# 2. The 5 Types of Interval Problems

```
TYPE 1 — MERGE OVERLAPPING INTERVALS
  Question asks: "Given intervals, merge all overlapping ones"
  Output: a new list of non-overlapping intervals
  Sort by: START time
  Example: [[1,3],[2,6],[8,10],[15,18]] → [[1,6],[8,10],[15,18]]

TYPE 2 — INSERT AN INTERVAL
  Question asks: "Insert new interval into sorted list"
  Output: sorted list with new interval inserted (and merged if overlapping)
  Sort by: already sorted, 3-phase scan
  Example: [[1,2],[3,5],[6,7]], insert [4,8] → [[1,2],[3,8]]

TYPE 3 — NON-OVERLAPPING / MIN REMOVAL
  Question asks: "Minimum intervals to remove so none overlap"
              OR "Maximum non-overlapping intervals"
  Output: count of removals, or count of kept intervals
  Sort by: END time (greedy keeps earliest-ending intervals)
  Example: [[1,2],[2,3],[3,4],[1,3]] → remove 1 interval

TYPE 4 — INTERVAL SCHEDULING (MEETING ROOMS)
  Question asks: "How many rooms/resources are needed?"
              OR "Can one person attend all meetings?"
  Output: minimum rooms needed, or true/false
  Approach: Sort starts and ends separately, or use min-heap of end times
  Example: [[0,30],[5,10],[15,20]] → need 2 rooms

TYPE 5 — SWEEP LINE / RANGE PROBLEMS
  Question asks: "At what point are the most intervals active?"
              OR "What is total coverage length?"
              OR "Count number of points covered by at least k intervals"
  Output: count, max overlap at any point, total coverage
  Approach: events array (start=+1, end=-1), sort, scan
  Example: [[1,3],[2,4],[3,5]] → max overlap at any point = 2
```

# 3. Core Interval Concepts You Must Know

## Overlap Check

```java
// Do [a,b] and [c,d] overlap?
// They DON'T overlap if: b < c (b before c) OR d < a (d before a)
// They DO overlap if: a <= d AND c <= b

boolean overlaps(int[] a, int[] b) {
    return a[0] <= b[1] && b[0] <= a[1];
}

// Variants:
// Strictly overlapping (touching NOT counted): a[1] < b[0] means no overlap
// Touching counted as overlap: a[1] <= b[0] means no overlap
// Read problem carefully!

// Example:
// [1,3] and [3,5]: a[1]=3, b[0]=3 → 3 <= 3 is TRUE → they overlap (if touching counts)
//                                   3 < 3 is FALSE → they don't overlap (if strict)
```

## Merge Two Intervals

```java
// Merge [a,b] and [c,d] that overlap
int[] merge(int[] a, int[] b) {
    return new int[]{Math.min(a[0], b[0]), Math.max(a[1], b[1])};
}

// merged start = min of both starts
// merged end   = max of both ends
```

## The Three Cases When Comparing Two Intervals

```
Given current interval [curStart, curEnd] and next interval [nextStart, nextEnd]:

CASE 1: No overlap (next starts after current ends)
  curEnd < nextStart
  [1,3]  [5,7]
  Action: save current, move to next

CASE 2: Overlap (next starts before or when current ends)
  nextStart <= curEnd
  [1,5]  [3,8]
  Action: merge → [1, max(5,8)] = [1,8]

CASE 3: Containment (next is completely inside current)
  nextStart >= curStart AND nextEnd <= curEnd
  [1,10]  [3,7]
  Action: keep current (it already contains next)
  Note: This is handled by CASE 2 automatically! max(10,7) = 10 ✓
```

# 4. Sorting — The Foundation of All Interval Problems

```java
int[][] intervals = {{3,5},{1,4},{2,6}};

// Sort by START time (ascending) — for merge, insert problems
Arrays.sort(intervals, (a, b) -> a[0] - b[0]);
// Result: [[1,4],[2,6],[3,5]]

// Sort by END time (ascending) — for non-overlapping, min-removal problems
Arrays.sort(intervals, (a, b) -> a[1] - b[1]);
// Result: [[1,4],[3,5],[2,6]]

// Sort by START, then by END if starts equal
Arrays.sort(intervals, (a, b) -> a[0] != b[0] ? a[0]-b[0] : a[1]-b[1]);

// Sort by END descending (for some greedy problems)
Arrays.sort(intervals, (a, b) -> b[1] - a[1]);

// CRITICAL: Use Integer.compare() to avoid overflow with large values
Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));

// WHY sort by START for merge?
// After sorting by start, any overlapping interval must be adjacent.
// We only need to compare current interval with the last merged one.

// WHY sort by END for non-overlapping?
// We want to keep intervals with earliest end (leaves most room for future).
// This is the greedy choice proven by exchange argument.
```

# Part 2 — The 5 Interval Patterns

# 5. Pattern 1 — Merge Overlapping Intervals

## The Algorithm

```
SORT by start time.
Maintain a "current" merged interval.
For each interval:
  If it overlaps with current → extend current's end (merge)
  Else → save current, start new current

Key: after sorting by start, overlapping intervals are ALWAYS adjacent.

[1,4] [2,6] [3,5] [8,10]
Sort by start: [1,4] [2,6] [3,5] [8,10]  ← [2,6] and [3,5] are adjacent

Process:
current = [1,4]
  [2,6]: 2 <= 4, overlap! current = [1, max(4,6)] = [1,6]
  [3,5]: 3 <= 6, overlap! current = [1, max(6,5)] = [1,6]  ← already contained
  [8,10]: 8 > 6, NO overlap! save [1,6], current = [8,10]
End: save [8,10]

Result: [[1,6],[8,10]] ✓
```

## 🟡 LC #56 — Merge Intervals

```java
public int[][] merge(int[][] intervals) {
    // Step 1: Sort by start time
    Arrays.sort(intervals, (a, b) -> a[0] - b[0]);

    List<int[]> merged = new ArrayList<>();
    int[] current = intervals[0];

    for (int i = 1; i < intervals.length; i++) {
        if (intervals[i][0] <= current[1]) {
            // Overlap: extend current's end
            current[1] = Math.max(current[1], intervals[i][1]);
        } else {
            // No overlap: save current, move to next
            merged.add(current);
            current = intervals[i];
        }
    }
    merged.add(current);  // Don't forget the last interval!

    return merged.toArray(new int[0][]);
}
```

**Dry run:**
```
Input: [[1,3],[2,6],[8,10],[15,18]]
Sorted: [[1,3],[2,6],[8,10],[15,18]]

current = [1,3]
i=1: [2,6]: 2 <= 3 → merge → current = [1,6]
i=2: [8,10]: 8 > 6 → save [1,6], current = [8,10]
i=3: [15,18]: 15 > 10 → save [8,10], current = [15,18]
End: save [15,18]

Output: [[1,6],[8,10],[15,18]] ✓
```

**Complexity:** Time O(n log n), Space O(n)

## 🟡 LC #986 — Interval List Intersections

```java
// Find all intersections of two sorted interval lists A and B
public int[][] intervalIntersection(int[][] firstList, int[][] secondList) {
    List<int[]> result = new ArrayList<>();
    int i = 0, j = 0;

    while (i < firstList.length && j < secondList.length) {
        // Intersection of [a1,a2] and [b1,b2]:
        int lo = Math.max(firstList[i][0], secondList[j][0]);
        int hi = Math.min(firstList[i][1], secondList[j][1]);

        if (lo <= hi) {
            result.add(new int[]{lo, hi});  // Valid intersection
        }

        // Advance the pointer with the smaller end (it's done contributing)
        if (firstList[i][1] < secondList[j][1]) i++;
        else j++;
    }

    return result.toArray(new int[0][]);
}
```

**Why advance smaller end?** An interval ending earlier cannot intersect with future intervals from the other list as much. It's "used up."

## 🟡 LC #1272 — Remove Interval

```java
// Remove interval [lo, hi] from a sorted list of intervals
public List<List<Integer>> removeInterval(int[][] intervals, int[] toBeRemoved) {
    List<List<Integer>> result = new ArrayList<>();
    int lo = toBeRemoved[0], hi = toBeRemoved[1];

    for (int[] iv : intervals) {
        if (iv[1] <= lo || iv[0] >= hi) {
            // No overlap with removed interval — keep entirely
            result.add(Arrays.asList(iv[0], iv[1]));
        } else {
            // Has overlap — keep left part (if any)
            if (iv[0] < lo) result.add(Arrays.asList(iv[0], lo));
            // Keep right part (if any)
            if (iv[1] > hi) result.add(Arrays.asList(hi, iv[1]));
        }
    }
    return result;
}
```

## 🔴 LC #759 — Employee Free Time

```java
// Find common free time across all employees' schedules
public List<Interval> employeeFreeTime(List<List<Interval>> schedule) {
    // Flatten and sort all intervals
    List<Interval> all = new ArrayList<>();
    for (List<Interval> emp : schedule) all.addAll(emp);
    all.sort((a, b) -> a.start - b.start);

    List<Interval> result = new ArrayList<>();
    int end = all.get(0).end;

    for (Interval iv : all) {
        if (iv.start > end) {
            // Gap found! [end, iv.start] is free time
            result.add(new Interval(end, iv.start));
        }
        end = Math.max(end, iv.end);
    }

    return result;
}
```

# 6. Pattern 2 — Insert an Interval

## The Algorithm

```
Input is ALREADY SORTED. No need to sort again.
Use 3-phase approach:

PHASE 1: Add all intervals that END before new interval starts
  (they come entirely before the new interval → no overlap)

PHASE 2: Merge all intervals that OVERLAP with the new interval
  (new interval absorbs them or they absorb it)

PHASE 3: Add all remaining intervals
  (they come entirely after the new interval → no overlap)

Key conditions:
  Phase 1 → 2 transition: intervals[i].end >= newInterval.start
  Phase 2 → 3 transition: intervals[i].start > newInterval.end
```

---

## 🟡 LC #57 — Insert Interval

```java
public int[][] insert(int[][] intervals, int[] newInterval) {
    List<int[]> result = new ArrayList<>();
    int i = 0, n = intervals.length;

    // PHASE 1: Add all intervals ending before newInterval starts
    while (i < n && intervals[i][1] < newInterval[0]) {
        result.add(intervals[i++]);
    }

    // PHASE 2: Merge all overlapping intervals with newInterval
    while (i < n && intervals[i][0] <= newInterval[1]) {
        newInterval[0] = Math.min(newInterval[0], intervals[i][0]);
        newInterval[1] = Math.max(newInterval[1], intervals[i][1]);
        i++;
    }
    result.add(newInterval);  // Add the merged interval

    // PHASE 3: Add all remaining intervals
    while (i < n) result.add(intervals[i++]);

    return result.toArray(new int[0][]);
}
```

**Dry run:**
```
intervals = [[1,2],[3,5],[6,7],[8,10],[12,16]], newInterval = [4,8]

Phase 1: [1,2]: 2 < 4 → add [1,2]. [3,5]: 5 >= 4 → stop. i=1
Phase 2:
  [3,5]: 3 <= 8 → merge → newInterval = [min(4,3), max(8,5)] = [3,8]. i=2
  [6,7]: 6 <= 8 → merge → newInterval = [min(3,6), max(8,7)] = [3,8]. i=3
  [8,10]: 8 <= 8 → merge → newInterval = [min(3,8), max(8,10)] = [3,10]. i=4
  [12,16]: 12 > 8 → stop. add [3,10].
Phase 3: add [12,16].

Result: [[1,2],[3,10],[12,16]] ✓
```

**Complexity:** Time O(n), Space O(n)

# 7. Pattern 3 — Non-Overlapping / Remove Intervals

## The Algorithm (Greedy)

```
GOAL: Find minimum intervals to REMOVE so the rest don't overlap.
Equivalently: find MAXIMUM non-overlapping intervals to KEEP.
Answer: total - maxKept = minRemovals

GREEDY KEY: Sort by END time. Always keep the interval with earliest end.
WHY? Earlier end = more room for future intervals. Never hurts to prefer it.

[1,2] [1,3] [2,4] [3,5]
Sort by end: [1,2] [1,3] [2,4] [3,5]

Keep [1,2] (first, always keep it). prevEnd = 2
  [1,3]: starts at 1 ≤ prevEnd=2 → OVERLAPS → REMOVE
  [2,4]: starts at 2 ≤ prevEnd=2 → OVERLAPS → REMOVE
  [3,5]: starts at 3 > prevEnd=2 → NO overlap → KEEP. prevEnd=5

Kept: [1,2] [3,5] = 2 intervals
Removed: 4 - 2 = 2 ← minimum removals
```

## 🟡 LC #435 — Non-overlapping Intervals

```java
public int eraseOverlapIntervals(int[][] intervals) {
    // Sort by END time — key greedy insight
    Arrays.sort(intervals, (a, b) -> a[1] - b[1]);

    int kept = 1;  // Always keep the first interval
    int prevEnd = intervals[0][1];

    for (int i = 1; i < intervals.length; i++) {
        if (intervals[i][0] >= prevEnd) {
            // No overlap with previous kept interval → KEEP this one
            kept++;
            prevEnd = intervals[i][1];
        }
        // Else: overlap → SKIP (remove this one)
    }

    return intervals.length - kept;  // Minimum removals
}
```

**Dry run:**
```
intervals = [[1,2],[2,3],[3,4],[1,3]]
Sort by end: [[1,2],[2,3],[1,3],[3,4]]

kept=1, prevEnd=2
i=1: [2,3]: 2 >= 2 → KEEP. kept=2, prevEnd=3
i=2: [1,3]: 1 < 3 → SKIP (remove)
i=3: [3,4]: 3 >= 3 → KEEP. kept=3, prevEnd=4

kept=3, removals = 4-3 = 1 ✓
```

## 🟡 LC #452 — Minimum Number of Arrows to Burst Balloons

```java
// Balloons are intervals. Arrow at x bursts all balloons covering x.
// Find minimum arrows needed to burst all balloons.
public int findMinArrowShots(int[][] points) {
    // Sort by END position
    Arrays.sort(points, (a, b) -> Integer.compare(a[1], b[1]));

    int arrows = 1;
    int arrowPos = points[0][1];  // First arrow at end of first balloon

    for (int i = 1; i < points.length; i++) {
        if (points[i][0] > arrowPos) {
            // This balloon starts AFTER current arrow → need new arrow
            arrows++;
            arrowPos = points[i][1];  // Place arrow at end of this balloon
        }
        // Else: current arrow bursts this one too (start <= arrowPos <= end)
    }

    return arrows;
}
```

**Key insight:** Each arrow placed at the earliest possible position (end of first balloon in a group) can burst the maximum number of overlapping balloons.

**Dry run:**
```
Balloons: [[10,16],[2,8],[1,6],[7,12]]
Sort by end: [[1,6],[2,8],[7,12],[10,16]]

arrows=1, arrowPos=6
i=1: [2,8]: 2 <= 6 → same arrow bursts it
i=2: [7,12]: 7 > 6 → NEW arrow. arrows=2, arrowPos=12
i=3: [10,16]: 10 <= 12 → same arrow bursts it

Result: 2 arrows ✓
```

## Connection Between LC #435 and LC #452

```
These two problems are almost identical!

LC #435 (non-overlapping): sort by end, count intervals to REMOVE
LC #452 (arrows): sort by end, count GROUPS of overlapping intervals
```

## Type 4a: Can One Person Attend All? (LC #252)

```java
public boolean canAttendMeetings(int[][] intervals) {
    Arrays.sort(intervals, (a, b) -> a[0] - b[0]);
    for (int i = 1; i < intervals.length; i++) {
        if (intervals[i][0] < intervals[i-1][1]) {

        }
    }
    return true;
}
```

## Type 4b: Minimum Rooms Needed (LC #253) — Most Important

```
APPROACH 1: Sort Starts and Ends Separately (Clean O(n log n))

Key insight: At any time t, rooms needed = meetings started by t - meetings ended by t.
Sort starts → when each meeting begins
Sort ends   → when each meeting ends (a room becomes available)

Use two pointers: one on starts, one on ends.
When a new meeting starts before the earliest-ending meeting ends → need new room.
When a new meeting starts after or when earliest ends → reuse that room.

[0,30] [5,10] [15,20]
starts = [0, 5, 15]
ends   = [10, 20, 30]

s=0: starts[0]=0 < ends[0]=10 → new room. rooms=1
s=1: starts[1]=5 < ends[0]=10 → new room. rooms=2
s=2: starts[2]=15 >= ends[0]=10 → reuse room. e++. rooms still 2

Answer: 2 rooms ✓
```

```java
public int minMeetingRooms(int[][] intervals) {
    int n = intervals.length;
    int[] starts = new int[n];
    int[] ends   = new int[n];

    for (int i = 0; i < n; i++) {
        starts[i] = intervals[i][0];
        ends[i]   = intervals[i][1];
    }

    Arrays.sort(starts);
    Arrays.sort(ends);

    int rooms = 0, endPtr = 0;

    for (int i = 0; i < n; i++) {
        if (starts[i] < ends[endPtr]) {
            rooms++;          // No room freed up → need new room
        } else {
            endPtr++;         // A room freed up → reuse it
        }
    }

    return rooms;
}
```

## Type 4b Alternative: Min-Heap Approach

```java
// Min-heap stores end times of ongoing meetings
// Top of heap = earliest-ending ongoing meeting
public int minMeetingRoomsHeap(int[][] intervals) {
    Arrays.sort(intervals, (a, b) -> a[0] - b[0]);
    PriorityQueue<Integer> heap = new PriorityQueue<>();  // Min-heap of end times

    for (int[] iv : intervals) {
        if (!heap.isEmpty() && heap.peek() <= iv[0]) {
            heap.poll();  // This room is free (meeting ended before new one starts)
        }
        heap.offer(iv[1]);  // Allocate room (or reuse) until iv[1]
    }

    return heap.size();  // Rooms in use = rooms needed
}
```

**Dry run with heap:**
```
Meetings sorted by start: [[0,30],[5,10],[15,20]]

[0,30]:  heap empty → add 30. heap=[30]
[5,10]:  heap.peek()=30 > 5 → can't reuse → add 10. heap=[10,30]
[15,20]: heap.peek()=10 <= 15 → room free! remove 10, add 20. heap=[20,30]

heap.size() = 2 → 2 rooms needed ✓
```

## Type 4c: Maximum CPU Load (Weighted Intervals)

```java
// Each job [start, end, cpuLoad]. Find peak CPU load at any point.
public int findMaxCPULoad(List<List<Integer>> jobs) {
    jobs.sort((a, b) -> a.get(0) - b.get(0));
    int maxLoad = 0, currentLoad = 0;
    PriorityQueue<List<Integer>> heap = new PriorityQueue<>((a,b) -> a.get(1)-b.get(1));

    for (List<Integer> job : jobs) {
        // Remove jobs that ended before this one starts
        while (!heap.isEmpty() && heap.peek().get(1) <= job.get(0)) {
            currentLoad -= heap.poll().get(2);
        }
        heap.offer(job);
        currentLoad += job.get(2);
        maxLoad = Math.max(maxLoad, currentLoad);
    }
    return maxLoad;
}
```

## 🟡 LC #1235 — Maximum Profit in Job Scheduling

```java
// Schedule jobs to maximize profit. No two jobs can overlap.
public int jobScheduling(int[] startTime, int[] endTime, int[] profit) {
    int n = startTime.length;
    int[][] jobs = new int[n][3];
    for (int i = 0; i < n; i++) jobs[i] = new int[]{startTime[i], endTime[i], profit[i]};

    // Sort by end time
    Arrays.sort(jobs, (a, b) -> a[1] - b[1]);

    // dp[i] = max profit using first i jobs (sorted by end)
    int[] dp = new int[n + 1];

    for (int i = 0; i < n; i++) {
        // Binary search: find last job that ends <= current job's start
        int lo = 0, hi = i;
        while (lo < hi) {
            int mid = lo + (hi - lo) / 2;
            if (jobs[mid][1] <= jobs[i][0]) lo = mid + 1;
            else hi = mid;
        }
        // dp[i+1] = max(skip this job, take this job + best profit before it)
        dp[i + 1] = Math.max(dp[i], dp[lo] + jobs[i][2]);
    }

    return dp[n];
}
```

## 🟡 LC #2402 — Meeting Rooms III

```java
// n rooms. Schedule meetings. Delayed meetings go to lowest-index free room.
// Return room that held most meetings.
public int mostBooked(int n, int[][] meetings) {
    Arrays.sort(meetings, (a, b) -> a[0] - b[0]);

    // minHeap of [endTime, roomIndex] for occupied rooms
    PriorityQueue<long[]> occupied = new PriorityQueue<>((a, b) ->
        a[0] != b[0] ? Long.compare(a[0], b[0]) : Long.compare(a[1], b[1]));

    // minHeap of room indices for available rooms
    PriorityQueue<Integer> available = new PriorityQueue<>();
    for (int i = 0; i < n; i++) available.offer(i);

    int[] count = new int[n];

    for (int[] m : meetings) {
        long start = m[0], end = m[1];

        // Free rooms that have ended by start
        while (!occupied.isEmpty() && occupied.peek()[0] <= start) {
            available.offer((int) occupied.poll()[1]);
        }

        if (!available.isEmpty()) {
            int room = available.poll();
            occupied.offer(new long[]{end, room});
            count[room]++;
        } else {
            // All rooms occupied — wait for earliest-ending room
            long[] earliest = occupied.poll();
            long newEnd = earliest[0] + (end - start);
            int room = (int) earliest[1];
            occupied.offer(new long[]{newEnd, room});
            count[room]++;
        }
    }

    int maxRoom = 0;
    for (int i = 1; i < n; i++) if (count[i] > count[maxRoom]) maxRoom = i;
    return maxRoom;
}
```

# 9. Pattern 5 — Sweep Line

## What is a Sweep Line?

```
Imagine a vertical line sweeping left-to-right across all intervals.
At each "event" (interval start or end), we update a counter.

STEP 1: Create events:
  For each interval [s, e]: add event (s, +1) and (e, -1)
  +1 means "interval starts here" (one more active interval)
  -1 means "interval ends here"  (one fewer active interval)

STEP 2: Sort events by time (and within same time, process +1 before -1 for "overlap at point")

STEP 3: Scan events, maintain running count.
  Max count = maximum intervals active at any point.

Example: [[1,3],[2,4],[3,6]]
Events: (1,+1), (2,+1), (3,-1), (3,+1), (4,-1), (6,-1)
Sort:   (1,+1), (2,+1), (3,-1 or +1 — depends), (4,-1), (6,-1)

If we process starts before ends at same time:
  t=1: count=1
  t=2: count=2
  t=3: -1 then +1 → count=2 (or +1 then -1 → count=2) — depends on problem
  t=4: count=1
  t=6: count=0

Max = 2 (two intervals active at any point between t=2 and t=3)
```

```java
// Given buildings [left, right, height], find the skyline outline
public List<List<Integer>> getSkyline(int[][] buildings) {
    // Create events: [x, height]
    // Building start: negative height (to sort starts before ends)
    // Building end:   positive height (= 0 for end)
    List<int[]> events = new ArrayList<>();
    for (int[] b : buildings) {
        events.add(new int[]{b[0], -b[2]});  // Start
        events.add(new int[]{b[1],  b[2]});  // End
    }
    // Sort by x; if same x, sort by height (negative first = starts first)
    events.sort((a, b) -> a[0] != b[0] ? a[0]-b[0] : a[1]-b[1]);

    // Max-heap of current building heights
    TreeMap<Integer, Integer> heightMap = new TreeMap<>(Collections.reverseOrder());
    heightMap.put(0, 1);  // Ground level always present
    int prevMax = 0;
    List<List<Integer>> result = new ArrayList<>();

    for (int[] e : events) {
        if (e[1] < 0) {
            // Building start: add height
            heightMap.merge(-e[1], 1, Integer::sum);
        } else {
            // Building end: remove height
            int h = e[1];
            heightMap.merge(h, -1, Integer::sum);
            if (heightMap.get(h) == 0) heightMap.remove(h);
        }
        int currMax = heightMap.firstKey();
        if (currMax != prevMax) {
            result.add(Arrays.asList(e[0], currMax));
            prevMax = currMax;
        }
    }
    return result;
}
```

# 11. Priority Queue (Min-Heap) for Intervals

```
USE CASES:
  - Minimum rooms/resources (meeting rooms)
  - Maximum profit job scheduling
  - K-way merge of interval lists

PATTERN: Sort intervals by START. Use min-heap to track END times.
  When processing new interval: if heap's minimum end <= new interval's start,
  a resource is free (pop and reuse). Otherwise, allocate new resource (push).
```

```java
// Template: interval scheduling with min-heap
int minResources(int[][] intervals) {
    Arrays.sort(intervals, (a,b) -> a[0]-b[0]);
    PriorityQueue<Integer> endTimes = new PriorityQueue<>();

    for (int[] iv : intervals) {
        if (!endTimes.isEmpty() && endTimes.peek() <= iv[0]) {
            endTimes.poll();  // Reuse freed resource
        }
        endTimes.offer(iv[1]);  // Assign resource ending at iv[1]
    }

    return endTimes.size();  // Total resources in use
}
```

# 12. Difference Array / Sweep Line for Range Updates

```
PROBLEM TYPE: Apply range updates efficiently, then query points.
"Add 1 to all elements in [l, r]" done for many ranges.
After all updates, find the maximum value, or total non-zero positions.

DIFFERENCE ARRAY:
  diff[l] += val
  diff[r+1] -= val
  Prefix sum of diff gives actual values at each point.

TIME: O(n + q) where n = range size, q = number of queries
Better than naive O(n*q) per query.

Example:
  Array of size 6, initially all zeros.
  Add 3 to [1,3]: diff[1]+=3, diff[4]-=3
  Add 2 to [2,5]: diff[2]+=2, diff[6]-=2
  
  diff = [0, 3, 2, 0, -3, 0, -2]
  prefix sum (actual values) = [0, 3, 5, 5, 2, 2, 0]
  Maximum = 5 (at indices 2 and 3)
```

```java
// LC #1109 — Corporate Flight Bookings
// bookings[i] = [first_i, last_i, seats_i]: add seats to flights [first, last]
public int[] corpFlightBookings(int[][] bookings, int n) {
    int[] diff = new int[n + 2];  // +2 for safety

    for (int[] b : bookings) {
        diff[b[0]] += b[2];      // Range start: add seats
        diff[b[1] + 1] -= b[2]; // Range end+1: remove seats
    }

    // Prefix sum to get actual seat counts
    int[] result = new int[n];
    int running = 0;
    for (int i = 1; i <= n; i++) {
        running += diff[i];
        result[i - 1] = running;
    }
    return result;
}
```

```java
int findInterval(int[][] intervals, int q) {
    // Binary search for rightmost interval with start <= q
    int lo = 0, hi = intervals.length - 1, result = -1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        if (intervals[mid][0] <= q) {
            result = mid;
            lo = mid + 1;
        } else {
            hi = mid - 1;
        }
    }
    // Check if result interval actually contains q
    if (result != -1 && intervals[result][1] >= q) return result;
    return -1;
}
```

```java
// Attend at most 1 event per day. Maximize events attended.
// Greedy: each day, attend the event with earliest deadline
public int maxEvents(int[][] events) {
    Arrays.sort(events, (a,b) -> a[0]-b[0]);  // Sort by start
    PriorityQueue<Integer> minHeap = new PriorityQueue<>();  // Min-heap of end days

    int day = 0, i = 0, result = 0;

    while (i < events.length || !minHeap.isEmpty()) {
        // If heap empty, jump to next event's start
        if (minHeap.isEmpty()) day = events[i][0];

        // Add all events starting on or before today
        while (i < events.length && events[i][0] <= day) {
            minHeap.offer(events[i++][1]);
        }

        // Remove expired events
        while (!minHeap.isEmpty() && minHeap.peek() < day) minHeap.poll();

        // Attend the event with earliest deadline
        if (!minHeap.isEmpty()) {
            minHeap.poll();
            result++;
            day++;
        }
    }
    return result;
}
```

```java
// For each row, compute histogram heights, find max rectangle
public int maximalRectangle(char[][] matrix) {
    int m = matrix.length, n = matrix[0].length;
    int[] heights = new int[n];
    int maxArea = 0;

    for (int i = 0; i < m; i++) {
        // Update histogram heights
        for (int j = 0; j < n; j++) {
            heights[j] = matrix[i][j] == '1' ? heights[j] + 1 : 0;
        }
        maxArea = Math.max(maxArea, largestRectangleInHistogram(heights));
    }
    return maxArea;
}

private int largestRectangleInHistogram(int[] heights) {
    Deque<Integer> stack = new ArrayDeque<>();
    int maxArea = 0;
    for (int i = 0; i <= heights.length; i++) {
        int h = i == heights.length ? 0 : heights[i];
        while (!stack.isEmpty() && heights[stack.peek()] > h) {
            int height = heights[stack.pop()];
            int width = stack.isEmpty() ? i : i - stack.peek() - 1;
            maxArea = Math.max(maxArea, height * width);
        }
        stack.push(i);
    }
    return maxArea;
}
```

```java
// Add integers one by one. Always return current list of disjoint intervals.
class SummaryRanges {
    TreeMap<Integer, Integer> map = new TreeMap<>();  // start → end

    public void addNum(int val) {
        if (map.containsKey(val)) return;

        Integer lo = map.lowerKey(val);   // Largest start < val
        Integer hi = map.higherKey(val);  // Smallest start > val

        boolean mergeLeft  = lo != null && map.get(lo) >= val - 1;
        boolean mergeRight = hi != null && hi <= val + 1;

        if (mergeLeft && mergeRight) {
            // Merge both neighbors: extend lo's interval to hi's end
            map.put(lo, map.get(hi));
            map.remove(hi);
        } else if (mergeLeft) {
            map.put(lo, Math.max(map.get(lo), val));
        } else if (mergeRight) {
            map.put(val, map.get(hi));
            map.remove(hi);
        } else {
            map.put(val, val);  // New standalone interval
        }
    }

    public int[][] getIntervals() {
        int[][] result = new int[map.size()][2];
        int i = 0;
        for (Map.Entry<Integer, Integer> e : map.entrySet()) {
            result[i][0] = e.getKey();
            result[i++][1] = e.getValue();
        }
        return result;
```

```java
class RangeModule {

    public void addRange(int left, int right) {

        if (lo != null && ranges.get(lo) >= left) left = lo;  // Extend left
        Integer hi = ranges.floorKey(right);
        if (hi != null && ranges.get(hi) > right) right = ranges.get(hi);  // Extend right

        ranges.subMap(left, right).clear();
        ranges.put(left, right);

        return lo != null && ranges.get(lo) >= right;

        Integer hi = ranges.floorKey(right);
```

## ⚡ The Fundamental Sort Orders

```java
// MERGE INTERVALS → sort by START
Arrays.sort(intervals, (a, b) -> a[0] - b[0]);

// NON-OVERLAPPING / ARROWS → sort by END
Arrays.sort(intervals, (a, b) -> a[1] - b[1]);

// MEETING ROOMS (heap approach) → sort by START
Arrays.sort(intervals, (a, b) -> a[0] - b[0]);

// SWEEP LINE → events sorted by time, then type
Arrays.sort(events, (a, b) -> a[0] != b[0] ? a[0]-b[0] : a[1]-b[1]);
```

## ⚡ Overlap Check Reference

```
Two intervals [a,b] and [c,d]:
  OVERLAP (touching counts): a <= d AND c <= b
  OVERLAP (strict, touching doesn't count): a < d AND c < b
  NO OVERLAP: b < c OR d < a (a's end before b's start, or vice versa)

After sorting by start:
  Current=[s1,e1], Next=[s2,e2] with s1<=s2:
  OVERLAP if s2 <= e1   (next starts before current ends)
  NO OVERLAP if s2 > e1 (next starts after current ends)
```

## ⚡ Algorithm Choice Summary

```

──────────────────────────────────────────────────────────

Insert interval                   (sorted)  3-phase
Min intervals to remove           End       Greedy count kept
Min arrows to burst balloons      End       Greedy count groups
Can attend all meetings           Start     Adjacent overlap check
Min rooms for meetings            Start     Heap or separate sorts
Max concurrent events             Either    Sweep line events
Range updates (bulk add)          N/A       Difference array

```

## ⚡ The 5 Golden Rules of Interval Problems

```
1. SORT FIRST — always sort before processing.
   Merge/Insert → sort by START.
   Non-overlap/Remove → sort by END.

2. OVERLAP CHECK: intervals [a,b] and [c,d] overlap iff a<=d AND c<=b.
   After sorting by start: just check intervals[i][0] <= current[1].

3. MERGE = extend end: current[1] = Math.max(current[1], next[1]).
   NOT current[1] = next[1]! (next might be contained in current)

4. FOR NON-OVERLAP: sort by END and keep earliest-ending intervals.
   This greedy is optimal — provable by exchange argument.

5. FOR SCHEDULING (meeting rooms):
   Answer = maximum number of intervals overlapping at any single point.
   Use sweep line or sort starts+ends separately with two pointers.
```
