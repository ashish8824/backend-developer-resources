# 1. What is Sorting and Why Does It Matter?

## The Core Idea

**Sorting** means arranging elements in a specific order (usually ascending or descending). It is one of the most fundamental operations in computer science.

```
Unsorted: [64, 34, 25, 12, 22, 11, 90]
Sorted:   [11, 12, 22, 25, 34, 64, 90]
```

## Why Sorting Matters

```
BEFORE SORTING               AFTER SORTING
Search: O(n) linear          Search: O(log n) binary search
Find min/max: O(n)           Find min/max: O(1)
Find duplicates: O(n²)       Find duplicates: O(n)
Find pairs with sum k: O(n²) Find pairs with sum k: O(n)
Merge two lists: O(n²)       Merge two lists: O(n)

Sorting is the GATEWAY to efficient algorithms.
50%+ of algorithm problems become easier once the input is sorted.
```

# 2. Key Properties of Sorting Algorithms

## Stability

A sort is **stable** if equal elements maintain their **original relative order**.

```
Input:  [(Alice, 25), (Bob, 30), (Carol, 25), (Dave, 20)]
Sort by age:

STABLE sort:   [(Dave,20), (Alice,25), (Carol,25), (Bob,30)]
                                        ↑ Alice before Carol (original order preserved)

UNSTABLE sort: [(Dave,20), (Carol,25), (Alice,25), (Bob,30)]
                                        ↑ Carol before Alice (order not preserved)

Why stability matters:
  Multi-key sorting: sort by name first, then by age.
  Stable sort preserves the name order within same-age groups.
```

## In-Place vs Out-of-Place

```
IN-PLACE: Sorts within the original array, O(1) extra space.
  → Bubble, Selection, Insertion, Quick Sort, Heap Sort

OUT-OF-PLACE: Requires extra array/space.
  → Merge Sort O(n), Counting Sort O(k), Radix Sort O(n+k)
```

## Adaptive

```
ADAPTIVE: Runs faster when input is already partially sorted.
  → Insertion Sort (O(n) for nearly sorted)
  → Tim Sort (used by Java)

NON-ADAPTIVE: Performance independent of input order.
  → Selection Sort (always O(n²))
  → Heap Sort (always O(n log n))
```

# 4. Bubble Sort

## How It Works

Repeatedly **compare adjacent elements** and **swap** them if they're in the wrong order. After each pass, the **largest unsorted element "bubbles up"** to its correct position.

```
Array: [64, 34, 25, 12, 22]

Pass 1:
  [64,34] → swap → [34,64,25,12,22]
  [64,25] → swap → [34,25,64,12,22]
  [64,12] → swap → [34,25,12,64,22]
  [64,22] → swap → [34,25,12,22,64]  ← 64 is in correct position!

Pass 2:
  [34,25] → swap → [25,34,12,22,64]
  [34,12] → swap → [25,12,34,22,64]
  [34,22] → swap → [25,12,22,34,64]  ← 34 is in correct position!

...continues until sorted
```

## Implementation

```java
public class BubbleSort {

    // Basic Bubble Sort — O(n²) always
    public static void bubbleSort(int[] arr) {
        int n = arr.length;

        for (int i = 0; i < n - 1; i++) {           // n-1 passes
            for (int j = 0; j < n - 1 - i; j++) {   // last i elements are sorted
                if (arr[j] > arr[j + 1]) {
                    // Swap adjacent elements
                    int temp = arr[j];
                    arr[j] = arr[j + 1];
                    arr[j + 1] = temp;
                }
            }
        }
    }

    // OPTIMIZED Bubble Sort — O(n) best case (already sorted)
    public static void bubbleSortOptimized(int[] arr) {
        int n = arr.length;

        for (int i = 0; i < n - 1; i++) {
            boolean swapped = false;  // Track if any swap happened

            for (int j = 0; j < n - 1 - i; j++) {
                if (arr[j] > arr[j + 1]) {
                    int temp = arr[j];
                    arr[j] = arr[j + 1];
                    arr[j + 1] = temp;
                    swapped = true;
                }
            }

            // If no swap happened → array is already sorted → STOP
            if (!swapped) break;
        }
    }

    public static void main(String[] args) {
        int[] arr = {64, 34, 25, 12, 22, 11, 90};
        bubbleSortOptimized(arr);
        System.out.println(Arrays.toString(arr));
        // [11, 12, 22, 25, 34, 64, 90]
    }
}
```

## When to Use Bubble Sort

```
✅ Teaching/learning purposes
✅ Nearly sorted arrays (optimized version is O(n))
✅ Very small arrays
❌ Production code (almost never)
❌ Large arrays
```

**Complexity:**
- Time: O(n²) average/worst, O(n) best (optimized)
- Space: O(1) — in-place
- Stable: Yes

# 5. Selection Sort

## How It Works

Find the **minimum element** in the unsorted portion and **swap** it to its correct position. Simple but always O(n²).

```
Array: [64, 25, 12, 22, 11]

Pass 1: Find min in [64,25,12,22,11] = 11 → swap with index 0
        [11, 25, 12, 22, 64]

Pass 2: Find min in [25,12,22,64] = 12 → swap with index 1
        [11, 12, 25, 22, 64]

Pass 3: Find min in [25,22,64] = 22 → swap with index 2
        [11, 12, 22, 25, 64]

Pass 4: Find min in [25,64] = 25 → already in place
        [11, 12, 22, 25, 64] ✅
```

## Implementation

```java
public class SelectionSort {

    public static void selectionSort(int[] arr) {
        int n = arr.length;

        for (int i = 0; i < n - 1; i++) {
            // Find minimum element in arr[i..n-1]
            int minIdx = i;
            for (int j = i + 1; j < n; j++) {
                if (arr[j] < arr[minIdx]) {
                    minIdx = j;
                }
            }

            // Swap minimum element with first element of unsorted portion
            if (minIdx != i) {
                int temp = arr[minIdx];
                arr[minIdx] = arr[i];
                arr[i] = temp;
            }
        }
    }

    public static void main(String[] args) {
        int[] arr = {64, 25, 12, 22, 11};
        selectionSort(arr);
        System.out.println(Arrays.toString(arr));
        // [11, 12, 22, 25, 64]
    }
}
```

## Key Insight: Selection Sort Makes Fewest Swaps

```
Selection Sort: at most n-1 swaps (one per pass)
Bubble Sort:    up to n(n-1)/2 swaps

Use Selection Sort when write operations are expensive
(e.g., writing to flash memory).
```

**Complexity:**
- Time: O(n²) always (no optimization possible)
- Space: O(1) — in-place
- Stable: No (swapping can break relative order)

# 6. Insertion Sort

## How It Works

Build a **sorted portion** on the left one element at a time. For each new element, **insert** it into its correct position in the sorted portion (like sorting playing cards in hand).

```
Array: [12, 11, 13, 5, 6]

Start: sorted=[12]  unsorted=[11,13,5,6]

Step 1: Take 11 → insert into [12]
        11 < 12, shift 12 right → [11, 12, 13, 5, 6]

Step 2: Take 13 → insert into [11,12]
        13 > 12, already in place → [11, 12, 13, 5, 6]

Step 3: Take 5 → insert into [11,12,13]
        5 < 13 → shift 13, 5 < 12 → shift 12, 5 < 11 → shift 11
        → [5, 11, 12, 13, 6]

Step 4: Take 6 → insert into [5,11,12,13]
        6 < 13 → shift, 6 < 12 → shift, 6 < 11 → shift, 6 > 5 → stop
        → [5, 6, 11, 12, 13] ✅
```

## Implementation

```java
public class InsertionSort {

    public static void insertionSort(int[] arr) {
        int n = arr.length;

        for (int i = 1; i < n; i++) {
            int key = arr[i];   // Element to insert
            int j = i - 1;

            // Shift all elements greater than key one position to the right
            while (j >= 0 && arr[j] > key) {
                arr[j + 1] = arr[j];  // Shift right
                j--;
            }

            // Insert key at its correct position
            arr[j + 1] = key;
        }
    }

    // BINARY INSERTION SORT — use binary search to find position
    // Reduces comparisons to O(n log n) but shifts are still O(n²)
    public static void binaryInsertionSort(int[] arr) {
        int n = arr.length;
        for (int i = 1; i < n; i++) {
            int key = arr[i];
            // Binary search for correct position in arr[0..i-1]
            int left = 0, right = i;
            while (left < right) {
                int mid = left + (right - left) / 2;
                if (arr[mid] <= key) left = mid + 1;
                else right = mid;
            }
            // Shift elements to make room
            for (int j = i; j > left; j--) arr[j] = arr[j - 1];
            arr[left] = key;
        }
    }

    public static void main(String[] args) {
        int[] arr = {12, 11, 13, 5, 6};
        insertionSort(arr);
        System.out.println(Arrays.toString(arr));
        // [5, 6, 11, 12, 13]
    }
}
```

## When to Use Insertion Sort

```
✅ Nearly sorted arrays → O(n) in the best case!
✅ Small arrays (n < 15) — faster than O(n log n) due to low overhead
✅ Online sorting (sort as elements arrive)
✅ As a base case in Tim Sort (Java uses it for small subarrays)
❌ Large unsorted arrays
```

**Complexity:**
- Time: O(n²) average/worst, **O(n) best** (nearly sorted)
- Space: O(1) — in-place
- Stable: Yes
- Adaptive: Yes

# 7. Merge Sort

## How It Works

**Divide** the array in half, **recursively sort** each half, then **merge** the two sorted halves. Classic divide and conquer.

```
Array: [38, 27, 43, 3, 9, 82, 10]

DIVIDE:
[38, 27, 43, 3, 9, 82, 10]
        ↙           ↘
[38, 27, 43]      [3, 9, 82, 10]
   ↙    ↘           ↙       ↘
[38]  [27,43]    [3,9]   [82,10]
       ↙  ↘      ↙ ↘     ↙   ↘
     [27] [43]  [3] [9] [82] [10]

MERGE (bottom up):
[27] + [43] → [27, 43]
[3]  + [9]  → [3, 9]
[82] + [10] → [10, 82]

[38] + [27, 43] → [27, 38, 43]
[3, 9] + [10, 82] → [3, 9, 10, 82]

[27, 38, 43] + [3, 9, 10, 82] → [3, 9, 10, 27, 38, 43, 82] ✅
```

## Implementation

```java
public class MergeSort {

    public static void mergeSort(int[] arr, int left, int right) {
        if (left >= right) return;  // Base case: 1 element

        int mid = left + (right - left) / 2;

        // Divide
        mergeSort(arr, left, mid);        // Sort left half
        mergeSort(arr, mid + 1, right);   // Sort right half

        // Conquer (merge)
        merge(arr, left, mid, right);
    }

    private static void merge(int[] arr, int left, int mid, int right) {
        // Create temp arrays for left and right halves
        int[] leftArr  = Arrays.copyOfRange(arr, left, mid + 1);
        int[] rightArr = Arrays.copyOfRange(arr, mid + 1, right + 1);

        int i = 0, j = 0, k = left;

        // Merge two sorted arrays back into arr
        while (i < leftArr.length && j < rightArr.length) {
            if (leftArr[i] <= rightArr[j]) {  // <= preserves stability
                arr[k++] = leftArr[i++];
            } else {
                arr[k++] = rightArr[j++];
            }
        }

        // Copy remaining elements
        while (i < leftArr.length)  arr[k++] = leftArr[i++];
        while (j < rightArr.length) arr[k++] = rightArr[j++];
    }

    // COUNT INVERSIONS using Merge Sort
    // (Classic problem: pairs where arr[i] > arr[j] for i < j)
    static long inversions = 0;

    public static void mergeSortInversions(int[] arr, int left, int right) {
        if (left >= right) return;
        int mid = left + (right - left) / 2;
        mergeSortInversions(arr, left, mid);
        mergeSortInversions(arr, mid + 1, right);
        mergeCount(arr, left, mid, right);
    }

    private static void mergeCount(int[] arr, int left, int mid, int right) {
        int[] leftArr  = Arrays.copyOfRange(arr, left, mid + 1);
        int[] rightArr = Arrays.copyOfRange(arr, mid + 1, right + 1);
        int i = 0, j = 0, k = left;

        while (i < leftArr.length && j < rightArr.length) {
            if (leftArr[i] <= rightArr[j]) {
                arr[k++] = leftArr[i++];
            } else {
                // leftArr[i] > rightArr[j]: ALL remaining in leftArr form inversions with rightArr[j]
                inversions += (leftArr.length - i);
                arr[k++] = rightArr[j++];
            }
        }
        while (i < leftArr.length)  arr[k++] = leftArr[i++];
        while (j < rightArr.length) arr[k++] = rightArr[j++];
    }

    public static void main(String[] args) {
        int[] arr = {38, 27, 43, 3, 9, 82, 10};
        mergeSort(arr, 0, arr.length - 1);
        System.out.println(Arrays.toString(arr));
        // [3, 9, 10, 27, 38, 43, 82]

        // Count inversions
        int[] arr2 = {2, 4, 1, 3, 5};
        inversions = 0;
        mergeSortInversions(arr2, 0, arr2.length - 1);
        System.out.println("Inversions: " + inversions); // 3
    }
}
```

## When to Use Merge Sort

```
✅ When stability is required
✅ Sorting linked lists (no random access needed)
✅ External sorting (data doesn't fit in memory)
✅ Guaranteed O(n log n) — no worst-case degradation
✅ Count inversions problem
❌ Memory-constrained environments (requires O(n) extra space)
```

**Complexity:**
- Time: O(n log n) — always
- Space: O(n) — extra arrays for merging
- Stable: Yes
- Adaptive: No

# 8. Quick Sort

## How It Works

Choose a **pivot** element, **partition** array so elements smaller than pivot are on the left, larger on the right, then **recursively sort** each partition.

```
Array: [10, 7, 8, 9, 1, 5]

Choose pivot = 5 (last element)

Partition:
  Walk with i=-1, j=0 to n-2:
  j=0: arr[0]=10 > 5 → skip
  j=1: arr[1]=7  > 5 → skip
  j=2: arr[2]=8  > 5 → skip
  j=3: arr[3]=9  > 5 → skip
  j=4: arr[4]=1  ≤ 5 → i++, swap arr[0] and arr[4] → [1,7,8,9,10,5]
  End: swap pivot (arr[5]=5) with arr[i+1=1] → [1,5,8,9,10,7]
       Wait - redo: pivot=5 at index 5
       After partition: [1, 5, 8, 9, 10, 7] → pivot 5 at index 1

Now recursively sort:
  Left:  [1]       → already sorted
  Right: [8,9,10,7] → sort recursively

Final: [1, 5, 7, 8, 9, 10] ✅
```

## Implementation

```java
public class QuickSort {

    public static void quickSort(int[] arr, int low, int high) {
        if (low < high) {
            int pivotIdx = partition(arr, low, high);
            quickSort(arr, low, pivotIdx - 1);   // Sort left partition
            quickSort(arr, pivotIdx + 1, high);  // Sort right partition
        }
    }

    // LOMUTO PARTITION — simple implementation (pivot = last element)
    private static int partition(int[] arr, int low, int high) {
        int pivot = arr[high];  // Choose last element as pivot
        int i = low - 1;        // i tracks boundary of smaller elements

        for (int j = low; j < high; j++) {
            if (arr[j] <= pivot) {
                i++;
                swap(arr, i, j);  // Move smaller element to left side
            }
        }

        swap(arr, i + 1, high);  // Place pivot in correct position
        return i + 1;            // Return pivot's final index
    }

    // HOARE PARTITION — original, more efficient (fewer swaps)
    private static int partitionHoare(int[] arr, int low, int high) {
        int pivot = arr[low];  // Choose first element as pivot
        int i = low - 1;
        int j = high + 1;

        while (true) {
            do { i++; } while (arr[i] < pivot);
            do { j--; } while (arr[j] > pivot);
            if (i >= j) return j;
            swap(arr, i, j);
        }
    }

    // RANDOMIZED Quick Sort — avoids worst case on sorted arrays
    public static void quickSortRandom(int[] arr, int low, int high) {
        if (low < high) {
            // Pick random pivot and swap with last element
            int randIdx = low + (int)(Math.random() * (high - low + 1));
            swap(arr, randIdx, high);
            int pivotIdx = partition(arr, low, high);
            quickSortRandom(arr, low, pivotIdx - 1);
            quickSortRandom(arr, pivotIdx + 1, high);
        }
    }

    // THREE-WAY PARTITION (Dutch National Flag) — efficient for duplicates
    public static void quickSort3Way(int[] arr, int low, int high) {
        if (low >= high) return;

        int lt = low, gt = high, i = low + 1;
        int pivot = arr[low];

        while (i <= gt) {
            if      (arr[i] < pivot) swap(arr, lt++, i++);
            else if (arr[i] > pivot) swap(arr, i,  gt--);
            else                     i++;
        }
        // arr[low..lt-1] < pivot == arr[lt..gt] < arr[gt+1..high]
        quickSort3Way(arr, low, lt - 1);
        quickSort3Way(arr, gt + 1, high);
    }

    private static void swap(int[] arr, int i, int j) {
        int temp = arr[i]; arr[i] = arr[j]; arr[j] = temp;
    }

    public static void main(String[] args) {
        int[] arr = {10, 7, 8, 9, 1, 5};
        quickSort(arr, 0, arr.length - 1);
        System.out.println(Arrays.toString(arr));
        // [1, 5, 7, 8, 9, 10]

        // Three-way partition for arrays with many duplicates
        int[] arr2 = {4, 2, 4, 4, 1, 4, 3, 4};
        quickSort3Way(arr2, 0, arr2.length - 1);
        System.out.println(Arrays.toString(arr2));
        // [1, 2, 3, 4, 4, 4, 4, 4]
    }
}
```

## Pivot Selection Strategies

```
BAD pivot choice: always pick min or max → O(n²) every time
  → Happens on sorted arrays with first/last pivot!

GOOD strategies:
  1. Random pivot     → Expected O(n log n), avoids adversarial inputs
  2. Median of three → Pick median of arr[low], arr[mid], arr[high]
  3. Introselect     → Adaptive (Java uses this internally)

WORST CASE for Quick Sort:
  Already sorted array + pivot = last element → O(n²)
  Solution: Always use randomized pivot!
```

## When to Use Quick Sort

```
✅ General-purpose sorting (fastest in practice)
✅ Cache-friendly (good locality of reference)
✅ In-place (O(log n) stack space only)
✅ When average performance matters more than worst case
❌ When stability is required
❌ When worst case O(n²) is unacceptable (use Merge Sort instead)
```

**Complexity:**
- Time: O(n log n) average, **O(n²) worst** (bad pivot)
- Space: O(log n) — recursion stack
- Stable: No
- Adaptive: No (but can be with randomization)

# 9. Heap Sort

## How It Works

Build a **max-heap** from the array, then repeatedly **extract** the maximum element and place it at the end.

```
Array: [12, 11, 13, 5, 6, 7]

Step 1: Build Max-Heap
        13
       /   \
     11     12
    /  \   /
   5    6 7

  Array: [13, 11, 12, 5, 6, 7]

Step 2: Extract max repeatedly
  Swap arr[0]=13 with arr[5]=7 → [7,11,12,5,6,|13]
  Heapify [7,11,12,5,6]: 12 > 7 → swap
  Array: [12,11,7,5,6,|13]

  Swap arr[0]=12 with arr[4]=6 → [6,11,7,5,|12,13]
  Heapify [6,11,7,5]: 11 > 6 → swap
  Array: [11,6,7,5,|12,13]

  ...continue until fully sorted
  Final: [5, 6, 7, 11, 12, 13] ✅
```

## Implementation

```java
public class HeapSort {

    public static void heapSort(int[] arr) {
        int n = arr.length;

        // Step 1: Build max-heap (heapify from bottom up)
        // Start from last non-leaf node (n/2 - 1) and go up
        for (int i = n / 2 - 1; i >= 0; i--) {
            heapify(arr, n, i);
        }

        // Step 2: Extract elements from heap one by one
        for (int i = n - 1; i > 0; i--) {
            // Move current root (max) to end
            swap(arr, 0, i);
            // Heapify reduced heap
            heapify(arr, i, 0);
        }
    }

    // Heapify subtree rooted at index i, heap size = n
    private static void heapify(int[] arr, int n, int i) {
        int largest = i;        // Assume root is largest
        int left  = 2 * i + 1; // Left child
        int right = 2 * i + 2; // Right child

        // If left child is larger than root
        if (left < n && arr[left] > arr[largest]) largest = left;

        // If right child is larger than current largest
        if (right < n && arr[right] > arr[largest]) largest = right;

        // If largest is not root, swap and continue heapifying
        if (largest != i) {
            swap(arr, i, largest);
            heapify(arr, n, largest);
        }
    }

    private static void swap(int[] arr, int i, int j) {
        int temp = arr[i]; arr[i] = arr[j]; arr[j] = temp;
    }

    public static void main(String[] args) {
        int[] arr = {12, 11, 13, 5, 6, 7};
        heapSort(arr);
        System.out.println(Arrays.toString(arr));
        // [5, 6, 7, 11, 12, 13]
    }
}
```

## When to Use Heap Sort

```
✅ When O(n log n) GUARANTEED with O(1) space is needed
✅ Embedded systems with strict memory limits
✅ Finding k largest/smallest elements (partial heap sort)
❌ When stability is needed
❌ Cache-sensitive environments (poor locality compared to Quick Sort)
```

**Complexity:**
- Time: O(n log n) — always
- Space: O(1) — in-place
- Stable: No

# 10. Counting Sort

## How It Works

Count occurrences of each value, then reconstruct the sorted array. Only works when the **range of values is limited**.

```
Array: [4, 2, 2, 8, 3, 3, 1]  (values 1-8)

Step 1: Count occurrences
  count[1]=1, count[2]=2, count[3]=2, count[4]=1, count[8]=1

Step 2: Prefix sums (position in output)
  count[1]=1, count[2]=3, count[3]=5, count[4]=6, count[8]=7

Step 3: Build output
  Process input right to left (for stability):
  arr[6]=1: output[count[1]-1=0]=1, count[1]--
  arr[5]=3: output[count[3]-1=4]=3, count[3]--
  ...
  Output: [1, 2, 2, 3, 3, 4, 8] ✅
```

## Implementation

```java
public class CountingSort {

    // Counting Sort for non-negative integers
    public static void countingSort(int[] arr) {
        if (arr.length == 0) return;

        // Find range
        int max = arr[0];
        for (int num : arr) max = Math.max(max, num);

        // Count occurrences
        int[] count = new int[max + 1];
        for (int num : arr) count[num]++;

        // Reconstruct sorted array
        int idx = 0;
        for (int i = 0; i <= max; i++) {
            while (count[i]-- > 0) arr[idx++] = i;
        }
    }

    // STABLE counting sort (for use in Radix Sort)
    public static int[] stableCountingSort(int[] arr, int exp) {
        int n = arr.length;
        int[] output = new int[n];
        int[] count  = new int[10];

        // Count digits
        for (int num : arr) count[(num / exp) % 10]++;

        // Prefix sums
        for (int i = 1; i < 10; i++) count[i] += count[i - 1];

        // Build output (right to left for stability)
        for (int i = n - 1; i >= 0; i--) {
            int digit = (arr[i] / exp) % 10;
            output[--count[digit]] = arr[i];
        }

        return output;
    }

    // Count sort for chars/strings
    public static void countingSortChars(char[] arr) {
        int[] count = new int[256];  // ASCII range
        for (char c : arr) count[c]++;
        int idx = 0;
        for (int i = 0; i < 256; i++) {
            while (count[i]-- > 0) arr[idx++] = (char) i;
        }
    }

    public static void main(String[] args) {
        int[] arr = {4, 2, 2, 8, 3, 3, 1};
        countingSort(arr);
        System.out.println(Arrays.toString(arr));
        // [1, 2, 2, 3, 3, 4, 8]
    }
}
```

## When to Use Counting Sort

```
✅ Values in a small, known range (e.g., 0-100, ages, grades)
✅ When O(n) time is needed and range is limited
✅ Sorting characters or small integers
❌ Large range of values (e.g., sort billions of different numbers)
❌ Floating-point numbers
```

**Complexity:**
- Time: O(n + k) where k = range of values
- Space: O(k)
- Stable: Yes (with prefix-sum implementation)

# 11. Radix Sort

## How It Works

Sort numbers **digit by digit** from least significant to most significant, using stable counting sort at each step.

```
Array: [170, 45, 75, 90, 802, 24, 2, 66]

Sort by units digit (exp=1):
  17[0], 9[0], 80[2], [2], 8[0]2, 2[4], 4[5], 7[5], 6[6]
  → [170, 90, 802, 2, 24, 45, 75, 66]

Sort by tens digit (exp=10):
  1[7]0, [9]0, 8[0]2, [0]2, [2]4, [4]5, [7]5, [6]6
  → [802, 2, 24, 45, 66, 170, 75, 90]

Sort by hundreds digit (exp=100):
  [8]02, [0]02, [0]24, [0]45, [0]66, [1]70, [0]75, [0]90
  → [2, 24, 45, 66, 75, 90, 170, 802] ✅
```

## Implementation

```java
public class RadixSort {

    public static void radixSort(int[] arr) {
        int max = Arrays.stream(arr).max().getAsInt();

        // Sort by each digit position (1s, 10s, 100s, ...)
        for (int exp = 1; max / exp > 0; exp *= 10) {
            arr = countingSortByDigit(arr, exp);
        }
    }

    private static int[] countingSortByDigit(int[] arr, int exp) {
        int n = arr.length;
        int[] output = new int[n];
        int[] count  = new int[10];

        // Count occurrences of digits at position exp
        for (int num : arr) count[(num / exp) % 10]++;

        // Prefix sums → positions in output
        for (int i = 1; i < 10; i++) count[i] += count[i - 1];

        // Build output right to left (stability!)
        for (int i = n - 1; i >= 0; i--) {
            int digit = (arr[i] / exp) % 10;
            output[--count[digit]] = arr[i];
        }

        return output;
    }

    public static void main(String[] args) {
        int[] arr = {170, 45, 75, 90, 802, 24, 2, 66};
        radixSort(arr);
        System.out.println(Arrays.toString(arr));
        // [2, 24, 45, 66, 75, 90, 170, 802]
    }
}
```

**Complexity:**
- Time: O(d × (n + k)) where d = digits, k = base (10)
- Space: O(n + k)
- Stable: Yes

# 12. Bucket Sort

## How It Works

Distribute elements into **buckets** based on value range, sort each bucket (with insertion sort), then concatenate.

```
Array: [0.78, 0.17, 0.39, 0.26, 0.72, 0.94, 0.21]
Range: [0, 1)

Buckets (10 buckets, 0.0-0.1, 0.1-0.2, ...):
  Bucket 1: [0.17]
  Bucket 2: [0.26, 0.21]
  Bucket 3: [0.39]
  Bucket 7: [0.78, 0.72]
  Bucket 9: [0.94]

Sort each bucket:
  Bucket 2: [0.21, 0.26]
  Bucket 7: [0.72, 0.78]

Concatenate: [0.17, 0.21, 0.26, 0.39, 0.72, 0.78, 0.94] ✅
```

## Implementation

```java
public class BucketSort {

    // Bucket sort for floats in [0, 1)
    public static void bucketSort(float[] arr) {
        int n = arr.length;
        List<List<Float>> buckets = new ArrayList<>();
        for (int i = 0; i < n; i++) buckets.add(new ArrayList<>());

        // Distribute into buckets
        for (float num : arr) {
            int bucketIdx = (int)(num * n);
            buckets.get(bucketIdx).add(num);
        }

        // Sort each bucket using Collections.sort (Tim Sort)
        int idx = 0;
        for (List<Float> bucket : buckets) {
            Collections.sort(bucket);
            for (float num : bucket) arr[idx++] = num;
        }
    }

    // Bucket sort for integers in range [min, max]
    public static void bucketSortInt(int[] arr, int numBuckets) {
        int min = arr[0], max = arr[0];
        for (int num : arr) { min = Math.min(min, num); max = Math.max(max, num); }

        int range = (max - min) / numBuckets + 1;
        List<List<Integer>> buckets = new ArrayList<>();
        for (int i = 0; i < numBuckets; i++) buckets.add(new ArrayList<>());

        for (int num : arr) {
            int idx = (num - min) / range;
            buckets.get(idx).add(num);
        }

        int idx = 0;
        for (List<Integer> bucket : buckets) {
            Collections.sort(bucket);
            for (int num : bucket) arr[idx++] = num;
        }
    }

    public static void main(String[] args) {
        float[] arr = {0.78f, 0.17f, 0.39f, 0.26f, 0.72f, 0.94f, 0.21f};
        bucketSort(arr);
        System.out.println(Arrays.toString(arr));
        // [0.17, 0.21, 0.26, 0.39, 0.72, 0.78, 0.94]
    }
}
```

**Complexity:**
- Time: O(n + k) average, O(n²) worst (all in one bucket)
- Space: O(n + k)
- Best for: Uniformly distributed data

# 13. Tim Sort (Java's Arrays.sort)

## What is Tim Sort?

Tim Sort is a **hybrid** algorithm combining **Merge Sort** and **Insertion Sort**. It's Java's actual sorting algorithm for objects.

```
Key ideas:
1. Divide array into "runs" (already sorted subsequences)
2. Runs smaller than minRun (32-64 elements) are sorted with Insertion Sort
3. Merge adjacent runs using Merge Sort

Why it's fast:
→ Real-world data often has partially sorted sections (runs)
→ Insertion Sort is faster than Merge Sort for small arrays
→ Merging sorted runs is more efficient than sorting from scratch

Java usage:
  Arrays.sort(int[])    → Dual-Pivot Quick Sort (for primitives)
  Arrays.sort(Object[]) → Tim Sort (for objects, stability required)
  Collections.sort()    → Tim Sort
```

# 14. Shell Sort

## How It Works

A **generalization of Insertion Sort** that sorts elements far apart first, then progressively smaller gaps. Reduces the number of shifts needed.

```java
public class ShellSort {

    public static void shellSort(int[] arr) {
        int n = arr.length;

        // Start with large gap, reduce by half each time
        for (int gap = n / 2; gap > 0; gap /= 2) {
            // Insertion sort with this gap
            for (int i = gap; i < n; i++) {
                int key = arr[i];
                int j = i;
                while (j >= gap && arr[j - gap] > key) {
                    arr[j] = arr[j - gap];
                    j -= gap;
                }
                arr[j] = key;
            }
        }
    }

    public static void main(String[] args) {
        int[] arr = {12, 34, 54, 2, 3};
        shellSort(arr);
        System.out.println(Arrays.toString(arr));
        // [2, 3, 12, 34, 54]
    }
}
```

**Complexity:** O(n log² n) with good gap sequence — between O(n log n) and O(n²)

```java
        Arrays.sort(words, (a, b) -> a.compareTo(b));           // Alphabetical
        Arrays.sort(words, Comparator.comparingInt(String::length)
                                     .thenComparing(Comparator.naturalOrder()));

        // ===== 4. SORT 2D ARRAY =====
        int[][] intervals = {{1,3},{2,6},{8,10},{15,18}};
        Arrays.sort(intervals, (a, b) -> a[0] - b[0]);  // By start time
        Arrays.sort(intervals, (a, b) -> {               // Multi-key sort
            if (a[0] != b[0]) return a[0] - b[0];
            return a[1] - b[1];
        });

        // ===== 5. SORT LIST =====
        List<Integer> list = new ArrayList<>(Arrays.asList(5,2,8,1,9));
        Collections.sort(list);
        list.sort(Comparator.reverseOrder());
        list.sort((a, b) -> b - a);  // Descending

        // ===== 6. SORT OBJECTS =====
        // Implement Comparable OR use Comparator
        List<int[]> points = Arrays.asList(new int[]{3,1}, new int[]{1,2}, new int[]{2,3});
        points.sort((a, b) -> a[0] != b[0] ? a[0]-b[0] : a[1]-b[1]);

        // ===== 7. PARTIAL SORT — find k-th element =====
        // Use PriorityQueue for k-th largest
        int[] nums = {3, 2, 1, 5, 6, 4};
        int k = 2;
        PriorityQueue<Integer> minHeap = new PriorityQueue<>();
        for (int num : nums) {
            minHeap.offer(num);
            if (minHeap.size() > k) minHeap.poll();
        }
        System.out.println("K-th largest: " + minHeap.peek()); // 5
    }
```

```java
public int[][] merge(int[][] intervals) {
    // Sort by start time
    Arrays.sort(intervals, (a, b) -> a[0] - b[0]);

    List<int[]> merged = new ArrayList<>();
    int[] current = intervals[0];

    for (int i = 1; i < intervals.length; i++) {
        if (intervals[i][0] <= current[1]) {
            // Overlapping — extend end if needed
            current[1] = Math.max(current[1], intervals[i][1]);
        } else {
            merged.add(current);
            current = intervals[i];
        }
    }
    merged.add(current);
    return merged.toArray(new int[0][]);
}
```

```java
public List<List<Integer>> threeSum(int[] nums) {
    Arrays.sort(nums);  // Sort first!
    List<List<Integer>> result = new ArrayList<>();

    for (int i = 0; i < nums.length - 2; i++) {
        if (i > 0 && nums[i] == nums[i-1]) continue;  // Skip duplicates

        int left = i + 1, right = nums.length - 1;
        while (left < right) {
            int sum = nums[i] + nums[left] + nums[right];
            if (sum == 0) {
                result.add(Arrays.asList(nums[i], nums[left], nums[right]));
                while (left < right && nums[left] == nums[left+1]) left++;
                while (left < right && nums[right] == nums[right-1]) right--;
                left++; right--;
            } else if (sum < 0) left++;
            else right--;
        }
    }
    return result;
}
```

## 🟡 LC #215 — Kth Largest Element in Array

```java
// Method 1: Quick Select — O(n) average, O(n²) worst
public int findKthLargest(int[] nums, int k) {
    return quickSelect(nums, 0, nums.length-1, nums.length-k);
}

private int quickSelect(int[] arr, int low, int high, int k) {
    if (low == high) return arr[low];
    int pivot = partition(arr, low, high);
    if      (pivot == k) return arr[pivot];
    else if (pivot < k)  return quickSelect(arr, pivot+1, high, k);
    else                 return quickSelect(arr, low, pivot-1, k);
}

private int partition(int[] arr, int low, int high) {
    int pivot = arr[high], i = low - 1;
    for (int j = low; j < high; j++) {
        if (arr[j] <= pivot) { i++; int t=arr[i]; arr[i]=arr[j]; arr[j]=t; }
    }
    int t=arr[i+1]; arr[i+1]=arr[high]; arr[high]=t;
    return i+1;
}

// Method 2: Min-Heap — O(n log k)
public int findKthLargestHeap(int[] nums, int k) {
    PriorityQueue<Integer> minHeap = new PriorityQueue<>();
    for (int num : nums) {
        minHeap.offer(num);
        if (minHeap.size() > k) minHeap.poll();
    }
    return minHeap.peek();
}
```

## 🟡 LC #164 — Maximum Gap

```java
public int maximumGap(int[] nums) {
    if (nums.length < 2) return 0;
    int n = nums.length;
    int min = Arrays.stream(nums).min().getAsInt();
    int max = Arrays.stream(nums).max().getAsInt();
    if (min == max) return 0;

    // Bucket sort: n-1 buckets, each of size (max-min)/(n-1)
    int bucketSize = Math.max(1, (max - min) / (n - 1));
    int bucketCount = (max - min) / bucketSize + 1;

    int[] bucketMin = new int[bucketCount];
    int[] bucketMax = new int[bucketCount];
    Arrays.fill(bucketMin, Integer.MAX_VALUE);
    Arrays.fill(bucketMax, Integer.MIN_VALUE);

    for (int num : nums) {
        int idx = (num - min) / bucketSize;
        bucketMin[idx] = Math.min(bucketMin[idx], num);
        bucketMax[idx] = Math.max(bucketMax[idx], num);
    }

    int maxGap = 0, prevMax = min;
    for (int i = 0; i < bucketCount; i++) {
        if (bucketMin[i] == Integer.MAX_VALUE) continue;  // Empty bucket
        maxGap = Math.max(maxGap, bucketMin[i] - prevMax);
        prevMax = bucketMax[i];
    }
    return maxGap;
}
```

# 22. How to Choose the Right Sort

```
DECISION GUIDE:

Is the VALUE RANGE SMALL? (e.g., 0-100, ages, scores)
  → Counting Sort — O(n + k)

Are values FLOATS uniformly distributed in [0,1)?
  → Bucket Sort — O(n) average

Are values INTEGERS with many digits?
  → Radix Sort — O(d * n)

Do you need STABILITY?
  → Merge Sort (always stable)
  → Tim Sort (Java's built-in for objects)
  → Insertion Sort (for small arrays)
  → NOT Quick Sort or Heap Sort

Do you have MEMORY CONSTRAINTS?
  → Quick Sort — O(log n) stack space
  → Heap Sort  — O(1) space
  → NOT Merge Sort (O(n) extra)

Is the array NEARLY SORTED?
  → Insertion Sort — O(n) for nearly sorted!
  → Tim Sort — exploits existing runs

Need GUARANTEED O(n log n) worst case?
  → Merge Sort
  → Heap Sort
  → NOT Quick Sort (can be O(n²))

Is SPEED in practice most important?
  → Quick Sort (best cache behavior, fastest in practice)
  → Tim Sort (what Java actually uses)

Just need to SORT IN AN INTERVIEW and don't want to implement?
  → Arrays.sort() for primitives (Dual-Pivot Quick Sort)
  → Arrays.sort() with Comparator for objects (Tim Sort)
```

## ⚡ Implementations (5-minute versions for interviews)

```java
// MERGE SORT (stable, guaranteed O(n log n))
void mergeSort(int[] arr, int l, int r) {
    if (l >= r) return;
    int m = l + (r-l)/2;
    mergeSort(arr, l, m);
    mergeSort(arr, m+1, r);
    merge(arr, l, m, r);
}
void merge(int[] arr, int l, int m, int r) {
    int[] left = Arrays.copyOfRange(arr, l, m+1);
    int[] right = Arrays.copyOfRange(arr, m+1, r+1);
    int i=0, j=0, k=l;
    while (i<left.length && j<right.length)
        arr[k++] = left[i]<=right[j] ? left[i++] : right[j++];
    while (i<left.length) arr[k++]=left[i++];
    while (j<right.length) arr[k++]=right[j++];
}

// QUICK SORT (fast in practice)
void quickSort(int[] arr, int l, int r) {
    if (l >= r) return;
    int p = partition(arr, l, r);
    quickSort(arr, l, p-1);
    quickSort(arr, p+1, r);
}
int partition(int[] arr, int l, int r) {
    int pivot=arr[r], i=l-1;
    for (int j=l; j<r; j++)
        if (arr[j]<=pivot) { i++; int t=arr[i]; arr[i]=arr[j]; arr[j]=t; }
    int t=arr[i+1]; arr[i+1]=arr[r]; arr[r]=t;
    return i+1;
}

// COUNTING SORT (for small range)
void countingSort(int[] arr, int max) {
    int[] count = new int[max+1];
    for (int n : arr) count[n]++;
    int idx = 0;
    for (int i=0; i<=max; i++)
        while (count[i]-- > 0) arr[idx++] = i;
}
```

## ⚡ The 5 Golden Rules of Sorting in Interviews

```
1. In LeetCode, just call Arrays.sort() unless asked to implement.
   Save time for the actual algorithm.

2. Comparator overflow trap:
   WRONG:  (a, b) -> a - b     ← overflows for large negative numbers!
   RIGHT:  (a, b) -> Integer.compare(a, b)   ← safe always

3. Sort by end time for interval greedy problems (meeting rooms, arrows).
   Sort by start time for merge intervals.

4. When you see "k-th largest/smallest": Quick Select O(n) or Min-Heap O(n log k).
   Don't full-sort O(n log n) when you don't need to.

5. Three-way partition (Dutch National Flag) for arrays with
   many duplicates or exactly 3 distinct values.
```
