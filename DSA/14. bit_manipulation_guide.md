# 1. What is Bit Manipulation?

## The Core Idea

Every integer in a computer is stored as a sequence of **bits** (0s and 1s). **Bit manipulation** means directly operating on those bits using special operators.

```
Why use bit manipulation?
→ O(1) operations (single CPU instruction)
→ Extremely memory efficient (pack 32 booleans into 1 int)
→ Elegant solutions to problems that seem complex otherwise
→ Common in competitive programming and system-level code
→ Frequently asked in interviews at top companies
```

## Example: The Power

```
Number 13 in binary: 1 1 0 1
                     ↑ ↑ ↑ ↑
                     8 4 2 1  ← position values (powers of 2)

13 = 8 + 4 + 0 + 1 = 1101 in binary

Operations on these bits directly:
  13 & 7  = 1101 & 0111 = 0101 = 5   (AND)
  13 | 7  = 1101 | 0111 = 1111 = 15  (OR)
  13 ^ 7  = 1101 ^ 0111 = 1010 = 10  (XOR)
  ~13     = ...10010 = -14            (NOT)
  13 << 1 = 11010 = 26               (Left shift = multiply by 2)
  13 >> 1 = 110 = 6                   (Right shift = divide by 2)
```

# 2. Binary Number System

## Converting Decimal to Binary

```
Method: Repeatedly divide by 2, collect remainders bottom-up

13 ÷ 2 = 6 remainder 1  ← LSB (Least Significant Bit)
 6 ÷ 2 = 3 remainder 0
 3 ÷ 2 = 1 remainder 1
 1 ÷ 2 = 0 remainder 1  ← MSB (Most Significant Bit)

Read remainders bottom-up: 1101 = 13 ✅

Quick powers of 2 to memorize:
2^0  = 1
2^1  = 2
2^2  = 4
2^3  = 8
2^4  = 16
2^5  = 32
2^6  = 64
2^7  = 128
2^8  = 256
2^10 = 1024    (~1 thousand)
2^20 = 1048576 (~1 million)
2^30 ≈ 1 billion
2^31 - 1 = 2147483647 = Integer.MAX_VALUE
```

## Converting Binary to Decimal

```
1 1 0 1 (binary)
↓ ↓ ↓ ↓
8+4+0+1 = 13 (decimal)

Trick: Sum the positional values where bit = 1
Position: 3 2 1 0
Value:     8 4 2 1
```

## Hexadecimal (Base 16)

```
0-9 → 0-9,  A=10, B=11, C=12, D=13, E=14, F=15

1 hex digit = 4 bits (nibble)
0xF = 1111 = 15
0xFF = 11111111 = 255
0xFFFF = 16 ones = 65535

In Java:
  int x = 0xFF;    // hex literal
  System.out.println(Integer.toHexString(255)); // "ff"
  System.out.println(Integer.toBinaryString(13)); // "1101"
  System.out.println(Integer.bitCount(13));        // 3 (three 1-bits)
```

# 3. Two's Complement (Negative Numbers)

## How Java Stores Negative Numbers

```
Java uses 32-bit integers with TWO'S COMPLEMENT representation.

Positive 5:  00000000 00000000 00000000 00000101
Negative 5: flip all bits and add 1
  Flip:     11111111 11111111 11111111 11111010
  Add 1:    11111111 11111111 11111111 11111011  ← this is -5

Key facts:
  MSB (leftmost bit) = sign bit: 0=positive, 1=negative
  ~n = -(n+1)       → ~5 = -6
  -n = ~n + 1       → -5 = ~5 + 1
  Integer.MAX_VALUE = 01111...1 = 2^31 - 1
  Integer.MIN_VALUE = 10000...0 = -2^31
  Overflow: MAX_VALUE + 1 = MIN_VALUE (wraps around!)

Range of int: -2,147,483,648 to 2,147,483,647
```

# 4. All Bitwise Operators Explained

## AND (&)

```
Output is 1 only when BOTH bits are 1.

  1 0 1 1   (11)
& 0 1 1 0   (6)
---------
  0 0 1 0   (2)

Uses:
  Check if bit is set:   n & (1 << k)
  Keep only certain bits: n & 0xFF  (keep lowest 8 bits)
  Check even/odd:        n & 1  (0=even, 1=odd)
```

## OR (|)

```
Output is 1 when AT LEAST ONE bit is 1.

  1 0 1 1   (11)
| 0 1 1 0   (6)
---------
  1 1 1 1   (15)

Uses:
  Set a bit:     n | (1 << k)
  Combine flags: READ | WRITE | EXECUTE
```

## XOR (^)

```
Output is 1 when bits are DIFFERENT (eXclusive OR).

  1 0 1 1   (11)
^ 0 1 1 0   (6)
---------
  1 1 0 1   (13)

MAGIC PROPERTIES:
  a ^ 0 = a          (XOR with 0 → identity)
  a ^ a = 0          (XOR with itself → zero!)
  a ^ b ^ a = b      (XOR is commutative + associative)
  a ^ b ^ b = a      (XOR with same value twice → original)

Uses:
  Find single non-duplicate: XOR all → pairs cancel out
  Toggle a bit: n ^ (1 << k)
  Swap without temp: a^=b; b^=a; a^=b;
```

## NOT (~)

```
Flip all bits (complement).

  ~5 = ~00000101 = 11111010 = -6

Rule: ~n = -(n+1)
  ~0 = -1
  ~5 = -6
  ~(-1) = 0

Uses:
  Clear a bit: n & ~(1 << k)
  Get all 1s mask: ~0 = -1 = 111...1
```

## Left Shift (<<)

```
Shift all bits LEFT, fill right with 0s.

  13 << 1:  0001101 → 0011010 = 26   (×2)
  13 << 2:  0001101 → 0110100 = 52   (×4)
  13 << 3:  0001101 → 1101000 = 104  (×8)

Rule: n << k = n × 2^k

Uses:
  Create bitmask: 1 << k  (bit k is set)
  Fast multiplication by 2
  Create large numbers: 1 << 30 = 2^30
```

## Right Shift (>> and >>>)

```
SIGNED right shift (>>): fills with sign bit (arithmetic)
  -8 >> 1 = -4  (preserves sign)
   8 >> 1 =  4

UNSIGNED right shift (>>>): always fills with 0 (logical)
  -8 >>> 1 = large positive number (sign bit becomes 0)
   8 >>> 1 = 4

Rule: n >> k = n / 2^k (for positive n)

Uses:
  Fast division by 2: n >> 1
  Extract digit: (n >> k) & 1  (get bit at position k)
  Mid calculation: (left + right) >> 1  (avoids overflow vs /2)
```

# 6. Trick 1 — Get a Bit

## Read the bit at position k (0-indexed from right)

```
n = 13 = 1 1 0 1
         3 2 1 0  ← bit positions

Get bit at position 2:
  1 << 2 = 0100
  13 & (1<<2) = 1101 & 0100 = 0100 ≠ 0  → bit is 1 ✅

Get bit at position 1:
  1 << 1 = 0010
  13 & (1<<1) = 1101 & 0010 = 0000 = 0  → bit is 0 ✅
```

```java
// Get bit at position k
public int getBit(int n, int k) {
    return (n >> k) & 1;  // Shift right, then check last bit
    // Alternative: return (n & (1 << k)) != 0 ? 1 : 0;
}

// Examples
getBit(13, 0) // = (1101 >> 0) & 1 = 1101 & 1 = 1
getBit(13, 1) // = (1101 >> 1) & 1 = 0110 & 1 = 0
getBit(13, 2) // = (1101 >> 2) & 1 = 0011 & 1 = 1
getBit(13, 3) // = (1101 >> 3) & 1 = 0001 & 1 = 1
```

# 8. Trick 3 — Clear a Bit

## Force a bit to 0 at position k

```
n = 13 = 1 1 0 1
Clear bit at position 3:
  1 << 3 = 1000
  ~(1<<3) = 0111
  13 & 0111 = 1101 & 0111 = 0101 = 5
```

```java
public int clearBit(int n, int k) {
    return n & ~(1 << k);  // AND with mask that has 0 at position k
}

clearBit(13, 3) // 1101 & ~1000 = 1101 & 0111 = 0101 = 5
clearBit(13, 0) // 1101 & ~0001 = 1101 & 1110 = 1100 = 12

// Clear all bits from position k to MSB (keep low k bits)
public int clearBitsFromKToMSB(int n, int k) {
    return n & ((1 << k) - 1);  // Mask: 000...0111...1 (k ones)
}

// Clear all bits from 0 to k (keep high bits)
public int clearBitsFromZeroToK(int n, int k) {
    return n & (~((1 << (k+1)) - 1));
}
```

# 9. Trick 4 — Toggle a Bit

## Flip a bit at position k (0→1, 1→0)

```
n = 13 = 1 1 0 1
Toggle bit at position 1:
  1 << 1 = 0010
  13 ^ 0010 = 1101 ^ 0010 = 1111 = 15

Toggle again:
  15 ^ 0010 = 1111 ^ 0010 = 1101 = 13  (back to original!)
```

```java
public int toggleBit(int n, int k) {
    return n ^ (1 << k);  // XOR with mask
}

toggleBit(13, 1) // 1101 ^ 0010 = 1111 = 15
toggleBit(15, 1) // 1111 ^ 0010 = 1101 = 13  (toggled back!)
```

# 10. Trick 5 — Check if Power of 2

## A power of 2 has exactly ONE bit set

```
Powers of 2:
1  = 0001
2  = 0010
4  = 0100
8  = 1000
16 = 10000

Non-powers of 2:
3  = 0011  (two bits)
5  = 0101  (two bits)
6  = 0110  (two bits)

KEY INSIGHT:
For power of 2 n: n has exactly one 1-bit
Therefore: n & (n-1) removes the lowest set bit
If n is power of 2: n & (n-1) = 0!

8  & 7  = 1000 & 0111 = 0000 = 0  ✅ (power of 2)
12 & 11 = 1100 & 1011 = 1000 ≠ 0  ❌ (not power of 2)
```

```java
public boolean isPowerOfTwo(int n) {
    return n > 0 && (n & (n - 1)) == 0;
    // n > 0 handles the case n=0 (not a power of 2)
}

isPowerOfTwo(1)  // true  (2^0)
isPowerOfTwo(8)  // true  (2^3)
isPowerOfTwo(12) // false
isPowerOfTwo(0)  // false
isPowerOfTwo(-8) // false
```

# 11. Trick 6 — Count Set Bits (Brian Kernighan's Algorithm)

## Count the number of 1-bits

```
NAIVE: Check each of 32 bits → O(32) = O(1) but slow constant

BRIAN KERNIGHAN: n & (n-1) removes the LOWEST set bit each time
  Count iterations until n = 0

Example n = 13 = 1101:
  Iteration 1: 1101 & 1100 = 1100 (12), count=1
  Iteration 2: 1100 & 1011 = 1000 (8),  count=2
  Iteration 3: 1000 & 0111 = 0000 (0),  count=3
  n=0, stop!  Answer: 3 set bits ✅

Time: O(number of set bits) — faster than O(32) for sparse numbers
```

```java
// Brian Kernighan's Algorithm
public int countBits(int n) {
    int count = 0;
    while (n != 0) {
        n = n & (n - 1);  // Remove lowest set bit
        count++;
    }
    return count;
}

// Java built-in (uses POPCNT instruction on modern CPUs)
int count = Integer.bitCount(n);  // O(1) — single CPU instruction!

// Examples
countBits(0)  // 0
countBits(1)  // 1
countBits(7)  // 3  (111)
countBits(13) // 3  (1101)
```

# 12. Trick 7 — XOR Magic

## The Most Powerful Bit Trick

```
XOR Properties (MEMORIZE THESE):
  a ^ 0 = a          → XOR with 0 changes nothing
  a ^ a = 0          → XOR with itself = 0
  a ^ b = b ^ a      → Commutative
  (a^b)^c = a^(b^c)  → Associative
  a ^ b ^ a = b      → Cancel pairs!

CONSEQUENCE: XOR all numbers in array where each appears twice
             except one → pairs cancel out → single element remains!

[2, 3, 5, 3, 2]:
  2^3^5^3^2
  = (2^2)^(3^3)^5
  = 0^0^5
  = 5  ← the single number!
```

```java
// Find single number (all others appear twice)
public int singleNumber(int[] nums) {
    int result = 0;
    for (int num : nums) result ^= num;
    return result;
}

// XOR of 1 to n has a pattern:
// n%4==0: n
// n%4==1: 1
// n%4==2: n+1
// n%4==3: 0
public int xorUpTo(int n) {
    int[] pattern = {n, 1, n+1, 0};
    return pattern[n % 4];
}
```

# 13. Trick 8 — Lowest Set Bit

## Extract or remove the rightmost 1-bit

```
n = 12 = 1100

Lowest set bit:
  n & (-n) = n & (~n + 1)
  12 & (-12) = 1100 & 0100 = 0100 = 4  ← lowest set bit!

Why? -n in two's complement flips all bits and adds 1.
The trailing zeros become 1s, and the lowest 1 stays as 1.
Everything above the lowest 1 becomes 0.

Remove lowest set bit:
  n & (n-1) = 1100 & 1011 = 1000 = 8

Count trailing zeros:
  Integer.numberOfTrailingZeros(12) = 2
  (12 = 1100 has two trailing zeros)
```

```java
// Lowest set bit as value
int lowBit = n & (-n);          // e.g., 12 → 4
// OR:
int lowBit2 = n & (~n + 1);     // same thing

// Remove lowest set bit
int removeLowest = n & (n - 1); // e.g., 12 → 8

// Position of lowest set bit (0-indexed)
int pos = Integer.numberOfTrailingZeros(n);  // 12 → 2

// Uses in Fenwick Tree (Binary Indexed Tree):
// i += i & (-i)  → move to parent
// i -= i & (-i)  → move to previous
```

# 14. Trick 9 — Swap Without Temp Variable

```
Using XOR:
  a = 5 = 101
  b = 3 = 011

  a ^= b  → a = 101^011 = 110 = 6
  b ^= a  → b = 011^110 = 101 = 5  ← b now has original a!
  a ^= b  → a = 110^101 = 011 = 3  ← a now has original b!
```

```java
// XOR swap (don't use in practice — less readable, same speed)
public void xorSwap(int[] arr, int i, int j) {
    if (i != j) {  // Must check! XOR swap fails if i == j
        arr[i] ^= arr[j];
        arr[j] ^= arr[i];
        arr[i] ^= arr[j];
    }
}

// Regular swap (cleaner, compiler optimizes same way)
public void swap(int[] arr, int i, int j) {
    int temp = arr[i];
    arr[i] = arr[j];
    arr[j] = temp;
}
```

# 15. Trick 10 — Multiply / Divide by Powers of 2

```
MULTIPLY: Left shift
  n << k = n × 2^k
  5 << 1 = 10  (×2)
  5 << 2 = 20  (×4)
  5 << 3 = 40  (×8)

DIVIDE: Right shift (only for positive integers)
  n >> k = n / 2^k  (integer division, floor)
  20 >> 1 = 10  (÷2)
  20 >> 2 = 5   (÷4)
  20 >> 3 = 2   (÷8, floor of 20/8=2.5)

USEFUL PATTERNS:
  n / 2   → n >> 1
  n * 2   → n << 1
  n % 2   → n & 1   (0=even, 1=odd)
  n % 4   → n & 3
  n % 8   → n & 7
  n % 2^k → n & (2^k - 1)  (bitwise modulo)

MIDPOINT (avoid overflow):
  mid = (left + right) / 2  ← can overflow if left+right > MAX_INT!
  mid = left + (right - left) / 2  ← safe
  mid = (left + right) >>> 1  ← unsigned right shift, also safe
```

```java
public int singleNumber(int[] nums) {
    int result = 0;
    for (int i = 0; i < 32; i++) {
```

## 🟡 LC #260 — Single Number III (two singles)

```java
// Two numbers appear once, all others twice. Find both.
public int[] singleNumber(int[] nums) {
    int xor = 0;
    for (int num : nums) xor ^= num;
    // xor = a ^ b (a and b are the two single numbers)

    // Find any bit where a and b differ (use lowest set bit)
    int diffBit = xor & (-xor);

    // Divide numbers into two groups by this bit
    int a = 0, b = 0;
    for (int num : nums) {
        if ((num & diffBit) == 0) a ^= num;  // Group 1
        else                      b ^= num;  // Group 2
    }
    return new int[]{a, b};
}
```

```java
// Array has 0..n with one missing. Find it.
// Method 1: XOR — XOR all indices AND all values
public int missingNumber(int[] nums) {
    int result = nums.length;  // XOR with n first
    for (int i = 0; i < nums.length; i++) {
        result ^= i ^ nums[i];  // XOR index and value cancel if both present
    }
    return result;
}

// Method 2: Math — Expected sum - actual sum
public int missingNumberMath(int[] nums) {
    int n = nums.length;
    int expected = n * (n + 1) / 2;
    int actual = 0;
    for (int num : nums) actual += num;
    return expected - actual;
}
```

```java
public int[] countBits(int n) {
```

```java
public int totalHammingDistance(int[] nums) {
```

# 18. Pattern 3 — Bitmask (Represent Subsets)

## What is a Bitmask?

Use an **integer as a set of booleans**. Bit i = 1 means element i is "selected."

```
Elements: [A, B, C, D]  (indices 0, 1, 2, 3)

Bitmask 0110 = {B, C} (B and C selected)
Bitmask 1010 = {B, D}
Bitmask 1111 = {A, B, C, D} (all selected)

For n elements: 2^n possible subsets (0 to 2^n - 1)

Operations on bitmask 'mask':
  Check if element i is in set:  (mask >> i) & 1
  Add element i to set:           mask | (1 << i)
  Remove element i from set:      mask & ~(1 << i)
  Toggle element i:               mask ^ (1 << i)
  Is set empty?                   mask == 0
  Full set (all n elements):      (1 << n) - 1
  Iterate all subsets of mask:    for(int s=mask; s>0; s=(s-1)&mask)
```

---

## 🟡 LC #78 — Subsets (Bitmask Approach)

```java
public List<List<Integer>> subsets(int[] nums) {
    int n = nums.length;
    List<List<Integer>> result = new ArrayList<>();

    // Each number from 0 to 2^n - 1 represents a subset
    for (int mask = 0; mask < (1 << n); mask++) {
        List<Integer> subset = new ArrayList<>();
        for (int i = 0; i < n; i++) {
            if ((mask >> i & 1) == 1) {  // If bit i is set
                subset.add(nums[i]);
            }
        }
        result.add(subset);
    }
    return result;
}
// mask=000 → []
// mask=001 → [1]
// mask=010 → [2]
// mask=011 → [1,2]
// mask=100 → [3]
// ...etc
```

## 🟡 LC #1239 — Maximum Length of a Concatenated String with Unique Characters

```java
public int maxLength(List<String> arr) {
    List<Integer> masks = new ArrayList<>();
    masks.add(0);  // Empty set
    int result = 0;

    for (String s : arr) {
        // Build bitmask for this string (bit i = letter i-'a')
        int mask = 0, len = 0;
        for (char c : s.toCharArray()) {
            int bit = 1 << (c - 'a');
            if ((mask & bit) != 0) { mask = 0; break; }  // Duplicate in s
            mask |= bit;
            len++;
        }
        if (mask == 0) continue;  // String had duplicates

        // Try combining with existing valid masks
        int newMasks = masks.size();
        for (int i = 0; i < newMasks; i++) {
            int existing = masks.get(i);
            if ((existing & mask) == 0) {  // No overlap
                masks.add(existing | mask);
                result = Math.max(result, Integer.bitCount(existing | mask));
            }
        }
    }
    return result;
}
```

## 🟡 LC #1986 — Minimum Number of Work Sessions to Finish the Tasks

```java
public int minSessions(int[] tasks, int sessionTime) {
    int n = tasks.length;
    int full = (1 << n) - 1;

    // valid[mask] = can these tasks fit in one session?
    int[] time = new int[1 << n];
    for (int mask = 1; mask <= full; mask++) {
        for (int i = 0; i < n; i++) {
            if ((mask >> i & 1) == 1) {
                time[mask] = time[mask ^ (1 << i)] + tasks[i];
                break;
            }
        }
    }

    // dp[mask] = min sessions to finish tasks in mask
    int[] dp = new int[1 << n];
    Arrays.fill(dp, Integer.MAX_VALUE);
    dp[0] = 0;

    for (int mask = 1; mask <= full; mask++) {
        // Try all submasks as the "last session"
        for (int sub = mask; sub > 0; sub = (sub - 1) & mask) {
            if (time[sub] <= sessionTime && dp[mask ^ sub] != Integer.MAX_VALUE) {
                dp[mask] = Math.min(dp[mask], dp[mask ^ sub] + 1);
            }
        }
    }
    return dp[full];
}
```

# 19. Pattern 4 — Bit DP

## What is Bitmask DP?

Dynamic programming where the **state includes a bitmask** representing which elements have been used/visited. Used when n ≤ 20.

```
State: dp[mask][last] = min cost to visit all cities in 'mask', ending at 'last'
Transition: dp[mask | (1<<next)][next] = min(dp[mask][last] + dist[last][next])
```

## 🔴 LC #847 — Shortest Path Visiting All Nodes

```java
public int shortestPathLength(int[][] graph) {
    int n = graph.length;
    int full = (1 << n) - 1;

    // BFS: state = (node, visited_mask)
    Queue<int[]> queue = new ArrayDeque<>();
    boolean[][] visited = new boolean[n][1 << n];

    // Start BFS from all nodes simultaneously
    for (int i = 0; i < n; i++) {
        int mask = 1 << i;
        queue.offer(new int[]{i, mask, 0});
        visited[i][mask] = true;
    }

    while (!queue.isEmpty()) {
        int[] curr = queue.poll();
        int node = curr[0], mask = curr[1], dist = curr[2];

        if (mask == full) return dist;  // All nodes visited!

        for (int next : graph[node]) {
            int newMask = mask | (1 << next);
            if (!visited[next][newMask]) {
                visited[next][newMask] = true;
                queue.offer(new int[]{next, newMask, dist + 1});
            }
        }
    }
    return -1;
}
```

## 🔴 LC #943 — Find the Shortest Superstring

```java
public String shortestSuperstring(String[] words) {
    int n = words.length;

    // overlap[i][j] = max overlap of words[i] suffix with words[j] prefix
    int[][] overlap = new int[n][n];
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++) {
            if (i != j) {
                int m = Math.min(words[i].length(), words[j].length());
                for (int k = m; k > 0; k--) {
                    if (words[i].endsWith(words[j].substring(0, k))) {
                        overlap[i][j] = k;
                        break;
                    }
                }
            }
        }
    }

    // dp[mask][i] = max overlap to visit all words in mask, ending at i
    int[][] dp = new int[1 << n][n];
    int[][] parent = new int[1 << n][n];
    for (int[] row : parent) Arrays.fill(row, -1);

    for (int mask = 1; mask < (1 << n); mask++) {
        for (int last = 0; last < n; last++) {
            if ((mask >> last & 1) == 0) continue;
            int prev = mask ^ (1 << last);
            if (prev == 0) continue;
            for (int prev_last = 0; prev_last < n; prev_last++) {
                if ((prev >> prev_last & 1) == 0) continue;
                int val = dp[prev][prev_last] + overlap[prev_last][last];
                if (val > dp[mask][last]) {
                    dp[mask][last] = val;
                    parent[mask][last] = prev_last;
                }
            }
        }
    }

    // Find best ending node
    int full = (1 << n) - 1;
    int best = 0, last = 0;
    for (int i = 0; i < n; i++) {
        if (dp[full][i] > best) { best = dp[full][i]; last = i; }
    }

    // Reconstruct path
    List<Integer> path = new ArrayList<>();
    int mask = full;
    while (last != -1) {
        path.add(last);
        int prev = parent[mask][last];
        mask ^= (1 << last);
        last = prev;
    }
    Collections.reverse(path);

    // Build result string
    StringBuilder sb = new StringBuilder(words[path.get(0)]);
    for (int i = 1; i < path.size(); i++) {
        int ov = overlap[path.get(i-1)][path.get(i)];
        sb.append(words[path.get(i)].substring(ov));
    }
    return sb.toString();
}
```

```java
// Two words can't share letters → AND of their bitmasks = 0
public int maxProduct(String[] words) {
    int n = words.length;
    int[] masks = new int[n];

    for (int i = 0; i < n; i++) {
        for (char c : words[i].toCharArray()) {
            masks[i] |= 1 << (c - 'a');  // Set bit for each letter
        }
    }

    int max = 0;
    for (int i = 0; i < n; i++) {
        for (int j = i + 1; j < n; j++) {
            if ((masks[i] & masks[j]) == 0) {  // No shared letters
                max = Math.max(max, words[i].length() * words[j].length());
            }
        }
    }
    return max;
}
```

## 🟡 LC #421 — Maximum XOR of Two Numbers in Array

```java
// Greedy: for each bit from MSB to LSB, try to set it in result
public int findMaximumXOR(int[] nums) {
    int max = 0, mask = 0;

    for (int i = 31; i >= 0; i--) {
        mask |= (1 << i);
        Set<Integer> prefixes = new HashSet<>();
        for (int num : nums) prefixes.add(num & mask);

        int candidate = max | (1 << i);  // Try setting this bit
        for (int prefix : prefixes) {
            if (prefixes.contains(prefix ^ candidate)) {
                max = candidate;  // Found two numbers whose XOR achieves this
                break;
            }
        }
    }
    return max;
}
```

```java
public int minFlips(int a, int b, int c) {
    int flips = 0;
    while (a > 0 || b > 0 || c > 0) {
        int bitA = a & 1, bitB = b & 1, bitC = c & 1;
        if (bitC == 0) {
            flips += bitA + bitB;  // Both a and b bits must be 0
        } else {
            if (bitA == 0 && bitB == 0) flips++;  // Need at least one to be 1
        }
        a >>= 1; b >>= 1; c >>= 1;
    }
    return flips;
}
```

## 🟡 LC #1442 — Count Triplets That Can Form Two Arrays of Equal XOR

```java
// XOR property: if a^b = 0 (they're equal), then a^...^b in between is 0
// For any triplet (i,j,k): if arr[i]^...^arr[k] = 0, count += (k-i)
public int countTriplets(int[] arr) {
    int n = arr.length, count = 0;
    for (int i = 0; i < n - 1; i++) {
        int xor = arr[i];
        for (int k = i + 1; k < n; k++) {
            xor ^= arr[k];
            if (xor == 0) count += (k - i);  // j can be i+1 to k
        }
    }
    return count;
}
```

## 🔴 LC #41 — First Missing Positive (Cyclic Sort / Bit)

```java
// Using cyclic sort (already in sorting guide)
// Bit approach for O(n) time O(1) space using array itself as bitset:
public int firstMissingPositive(int[] nums) {
    int n = nums.length;

    // Replace non-positive and > n values with n+1
    for (int i = 0; i < n; i++) if (nums[i] <= 0 || nums[i] > n) nums[i] = n + 1;

    // Use sign as presence marker for values 1..n
    for (int i = 0; i < n; i++) {
        int val = Math.abs(nums[i]);
        if (val <= n) nums[val - 1] = -Math.abs(nums[val - 1]);
    }

    // First positive index+1 is the answer
    for (int i = 0; i < n; i++) if (nums[i] > 0) return i + 1;
    return n + 1;
}
```

```java
// Uses Gray code / recursion
public int minimumOneBitOperations(int n) {
    if (n == 0) return 0;
    int k = 0, curr = 1;
    while (curr * 2 <= n) { curr <<= 1; k++; }
    // Operations to turn leading bit 0: 2^(k+1) - 1 - minimumOneBitOperations(n ^ curr)
    return (1 << (k + 1)) - 1 - minimumOneBitOperations(n ^ curr);
}
```

```java
public int[] cycleLengthQueries(int n, int[][] queries) {
```

## ⚡ XOR Properties (Memorize!)

```
a ^ 0   = a      Identity
a ^ a   = 0      Self-inverse
a ^ b   = b ^ a  Commutative
(a^b)^c = a^(b^c) Associative
a^b^a  = b      Cancel pairs

a^b^b  = a      Second occurrence cancels
XOR all of 1..n:
  n%4==0 → n
  n%4==1 → 1
  n%4==2 → n+1
  n%4==3 → 0
```

## ⚡ Signed vs Unsigned Right Shift

```java
int  n = -8;            // 11111111 11111111 11111111 11111000
n >> 1  = -4;           // 11111111 11111111 11111111 11111100 (sign preserved)
n >>> 1 = 2147483644;   // 01111111 11111111 11111111 11111100 (fills with 0)

Use >>> when you want to treat the integer as unsigned bits.
Use >> for arithmetic right shift (divides by 2, preserves sign).
```

## ⚡ The 5 Golden Rules of Bit Manipulation

```
1. Operator precedence: ALWAYS use parentheses around bit operations.
   WRONG:  n & 1 == 0   (== evaluated first: n & (1==0) = n & 0 = 0!)
   RIGHT:  (n & 1) == 0

2. Use >>> (unsigned) when you don't want sign extension.
   Safe midpoint: (left + right) >>> 1

3. n & (n-1) removes the lowest set bit — the foundation of many tricks.
   n & (-n) isolates the lowest set bit.

4. For "single number" type problems, always think XOR first.
   XOR cancels pairs: a^a=0, remaining is the answer.

5. For n ≤ 20, bitmask DP/enumeration is usually the intended solution.
   2^20 = ~1 million — completely feasible.
```
