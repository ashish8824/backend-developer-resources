# 1. What is a Greedy Algorithm?

## The Core Idea

A **Greedy Algorithm** makes the **locally optimal choice at each step** with the hope (and proof!) that this leads to a **globally optimal solution**.

```
Think of it like this:

YOU'RE HIKING TO THE HIGHEST PEAK:
  Greedy:  At each fork, always take the path going UP
  Problem: You might get stuck on a local peak, not the global peak
  
FOR SOME PROBLEMS: The local best = global best (and we can PROVE it)
FOR OTHERS:        Need DP or backtracking

Real example — Making change:
  Target: 41 cents
  Coins:  [25, 10, 5, 1]
  
  Greedy: Take largest coin that fits:
    41 → take 25 → 16 remaining
    16 → take 10 → 6 remaining
     6 → take  5 → 1 remaining
     1 → take  1 → 0 remaining
  
  Total: 4 coins ✅ (optimal for standard coin system)
  
  BUT! With coins [1, 3, 4], target = 6:
    Greedy: 4+1+1 = 3 coins
    Optimal: 3+3   = 2 coins ❌ Greedy FAILS!
  
Key insight: Greedy works when the coin system has "canonical" property.
             Use DP when greedy might fail.
```

# 2. Greedy vs Dynamic Programming vs Backtracking

```
┌──────────────────┬──────────────────────────┬────────────────────────┐
│                  │ GREEDY                   │ DYNAMIC PROGRAMMING    │
├──────────────────┼──────────────────────────┼────────────────────────┤
│ How it works     │ One pass, local choice   │ All subproblems solved │
│ When it works    │ Greedy choice is safe    │ Overlapping subproblems│
│ Efficiency       │ O(n log n) or O(n)       │ O(n²) or O(n³)         │
│ Proof needed     │ YES (exchange argument)  │ Recurrence relation     │
│ Examples         │ Intervals, scheduling    │ Knapsack, LCS, coins   │
└──────────────────┴──────────────────────────┴────────────────────────┘

BACKTRACKING: Tries ALL possibilities
  → Use when: must find ALL solutions or no greedy/DP structure
  → Cost: exponential

GREEDY: Takes best available option at each step
  → Use when: provably safe to make local choices
  → Cost: O(n) or O(n log n) typically

DP: Breaks into overlapping subproblems, stores results
  → Use when: optimal substructure + overlapping subproblems
  → Cost: polynomial

RULE: Try greedy first (if you can prove it). 
      Fall back to DP if greedy fails.
      Fall back to backtracking if DP is too complex.
```

# 3. When Does Greedy Work?

## Two Key Properties

```
A problem has a greedy solution when it has:

1. GREEDY CHOICE PROPERTY:
   A globally optimal solution can be obtained by making locally optimal choices.
   → "The first choice I make will always be part of some optimal solution."

2. OPTIMAL SUBSTRUCTURE:
   An optimal solution contains optimal solutions to its subproblems.
   → "After making the greedy choice, the remaining problem is the same type."
```

## Proof Technique: Exchange Argument

```
How to PROVE greedy works:

ASSUME: Greedy solution G differs from optimal solution OPT
SHOW:   We can "exchange" the differing choices in OPT with G's choices
        without making the solution worse.
CONCLUDE: G is at least as good as OPT → G IS optimal.

Example: Activity Selection (maximize non-overlapping intervals)
  Greedy: Always pick the interval with the EARLIEST END TIME.
  
  Proof (exchange argument):
    Suppose OPT doesn't pick the earliest-ending activity A.
    OPT picks some other activity B that starts later.
    We can SWAP B with A in OPT:
      A ends before B ends → A doesn't conflict with future choices more than B
      → OPT with A has at least as many activities → exchange is valid ✅
  
  Conclusion: Always picking earliest end time is optimal.
```

## Common Greedy Proofs in Interviews

```
1. INTERVAL SCHEDULING: Sort by end time → always take earliest ending
   Proof: Earlier end = more room for future activities

2. JUMP GAME: At each step, greedily maximize reach
   Proof: A farther reach can never be worse than a shorter reach

3. HUFFMAN CODING: Always merge two smallest frequency trees
   Proof: Smaller freq → deeper in tree → fewer bits used more often

4. TASK SCHEDULER: Always schedule the most frequent remaining task
   Proof: Most frequent task is the bottleneck — tackle it first

5. FRACTIONAL KNAPSACK: Take highest value/weight ratio first
   Proof: If we took less of the best item, we could swap it for worse
          item and improve the solution
```

# 4. The Greedy Framework

```
STEP 1: IDENTIFY the greedy choice
  What is the locally optimal decision at each step?
  → "Always pick the X" where X is the key insight

STEP 2: PROVE it's safe (exchange argument or intuition)
  Why won't this greedy choice hurt us later?

STEP 3: REDUCE to a smaller problem
  After making the greedy choice, what's left?
  Is it the same type of problem?

STEP 4: IMPLEMENT
  Usually sort + iterate, or use a priority queue

Template:
  sort(input by some criterion)
  for each element:
      if (greedy condition):
          make_greedy_choice()
  return result
```

# 5. Pattern 1 — Activity Selection / Interval Scheduling

## The Key Insight

```
Sort intervals by END TIME.
Greedily select each interval that starts AFTER the previous one ends.

Why end time? Earlier end = more room for future intervals.
If we chose an interval ending later, we'd have less room — never better.

[1,3], [2,4], [3,5], [4,6]
Sorted by end: [1,3], [2,4], [3,5], [4,6]

Take [1,3] (first, always take it)
  [2,4]: starts at 2, but previous ends at 3 → SKIP (2 < 3, overlaps)
  [3,5]: starts at 3, previous ends at 3 → TAKE (3 >= 3, no overlap)
  [4,6]: starts at 4, previous ends at 5 → SKIP (4 < 5, overlaps)

Result: [[1,3], [3,5]] = 2 intervals ✅ (maximum possible)
```

## 🟡 LC #435 — Non-overlapping Intervals (Minimum Removals)

**Problem:** Find minimum intervals to remove so the rest don't overlap.

```java
public int eraseOverlapIntervals(int[][] intervals) {
    if (intervals.length == 0) return 0;

    // Sort by END time — key to greedy interval problems
    Arrays.sort(intervals, (a, b) -> a[1] - b[1]);

    int count = 0;       // Count of intervals to KEEP (not remove)
    int prevEnd = Integer.MIN_VALUE;

    for (int[] interval : intervals) {
        if (interval[0] >= prevEnd) {
            // No overlap → KEEP this interval
            count++;
            prevEnd = interval[1];
        }
        // Overlap → REMOVE this interval (don't update prevEnd)
    }

    return intervals.length - count;  // Removals = total - kept
}
```

**Why sort by END?** We want to keep as many intervals as possible. Keeping intervals with earlier ends leaves more room for future intervals.

**Complexity:** Time O(n log n), Space O(1)

```java
public int findMinArrowShots(int[][] points) {

    Arrays.sort(points, (a, b) -> Integer.compare(a[1], b[1]));

    int arrows = 1;
    int arrowPos = points[0][1];  // First arrow at end of first balloon

    for (int i = 1; i < points.length; i++) {
        if (points[i][0] > arrowPos) {

        // Else: current arrow bursts this balloon too (starts before or at arrow)
    }

    return arrows;
}
```

## 🟡 LC #56 — Merge Intervals

**Problem:** Merge all overlapping intervals.

```java
public int[][] merge(int[][] intervals) {
    // Sort by START time
    Arrays.sort(intervals, (a, b) -> a[0] - b[0]);

    List<int[]> merged = new ArrayList<>();
    int[] current = intervals[0];

    for (int i = 1; i < intervals.length; i++) {
        if (intervals[i][0] <= current[1]) {
            // Overlap → merge (extend end)
            current[1] = Math.max(current[1], intervals[i][1]);
        } else {
            // No overlap → save current, move to next
            merged.add(current);
            current = intervals[i];
        }
    }
    merged.add(current);

    return merged.toArray(new int[0][]);
}
```

## 🟡 LC #57 — Insert Interval

**Problem:** Insert a new interval into a sorted non-overlapping list of intervals.

```java
public int[][] insert(int[][] intervals, int[] newInterval) {
    List<int[]> result = new ArrayList<>();
    int i = 0, n = intervals.length;

    // Phase 1: Add all intervals ending before newInterval starts
    while (i < n && intervals[i][1] < newInterval[0]) {
        result.add(intervals[i++]);
    }

    // Phase 2: Merge all overlapping intervals with newInterval
    while (i < n && intervals[i][0] <= newInterval[1]) {
        newInterval[0] = Math.min(newInterval[0], intervals[i][0]);
        newInterval[1] = Math.max(newInterval[1], intervals[i][1]);
        i++;
    }
    result.add(newInterval);

    // Phase 3: Add remaining intervals (start after newInterval ends)
    while (i < n) result.add(intervals[i++]);

    return result.toArray(new int[0][]);
}
```

## 🔴 LC #253 — Meeting Rooms II (Min Rooms Needed)

**Problem:** Find minimum number of conference rooms required.

```java
public int minMeetingRooms(int[][] intervals) {
    int n = intervals.length;
    int[] starts = new int[n];
    int[] ends   = new int[n];

    for (int i = 0; i < n; i++) {
        starts[i] = intervals[i][0];
        ends[i]   = intervals[i][1];
    }

    Arrays.sort(starts);
    Arrays.sort(ends);

    int rooms = 0, endPtr = 0;

    for (int i = 0; i < n; i++) {
        if (starts[i] < ends[endPtr]) {
            rooms++;  // New meeting starts before earliest ending → need new room
        } else {
            endPtr++; // A room freed up → reuse it
        }
    }

    return rooms;
}
```

**Why this works:** At any point, the number of concurrent meetings = rooms needed. Sort starts and ends separately; greedily track when rooms free up.

## 🟡 LC #1235 — Maximum Profit in Job Scheduling

**Problem:** Schedule jobs with start/end/profit. No two jobs overlap. Maximize total profit.

```java
public int jobScheduling(int[] startTime, int[] endTime, int[] profit) {
    int n = startTime.length;
    int[][] jobs = new int[n][3];
    for (int i = 0; i < n; i++) jobs[i] = new int[]{startTime[i], endTime[i], profit[i]};

    // Sort by end time
    Arrays.sort(jobs, (a, b) -> a[1] - b[1]);

    // dp[i] = max profit considering jobs 0..i-1
    int[] dp = new int[n + 1];

    for (int i = 0; i < n; i++) {
        // Binary search: find last job that ends <= current job's start
        int lo = 0, hi = i;
        while (lo < hi) {
            int mid = lo + (hi - lo) / 2;
            if (jobs[mid][1] <= jobs[i][0]) lo = mid + 1;
            else hi = mid;
        }
        // dp[i+1] = max(skip this job, take this job + best before it)
        dp[i + 1] = Math.max(dp[i], dp[lo] + jobs[i][2]);
    }

    return dp[n];
}
```

**Complexity:** O(n log n)

# 6. Pattern 2 — Fractional Knapsack

## The Key Insight

```
Unlike 0/1 Knapsack (DP), fractional allows partial items.
Greedy: Sort by value/weight ratio. Take best ratio first.
        When bag fills, take fraction of next item.

Example:
Items: [(value=60, weight=10), (value=100, weight=20), (value=120, weight=30)]
Capacity: 50

Ratios: 6.0, 5.0, 4.0
Sort:   [(60,10), (100,20), (120,30)]

Take all of item 1: weight=10, value=60, remaining=40
Take all of item 2: weight=20, value=100, remaining=20
Take 2/3 of item 3: weight=20, value=80, remaining=0

Total: 240 (optimal!)
```

```java
public double fractionalKnapsack(int W, int[] weights, int[] values) {
    int n = weights.length;

    // Create (value/weight ratio, index) pairs and sort by ratio descending
    Integer[] indices = new Integer[n];
    for (int i = 0; i < n; i++) indices[i] = i;
    Arrays.sort(indices, (a, b) -> Double.compare(
        (double) values[b] / weights[b],
        (double) values[a] / weights[a]
    ));

    double totalValue = 0;
    int remaining = W;

    for (int idx : indices) {
        if (remaining <= 0) break;

        if (weights[idx] <= remaining) {
            // Take whole item
            totalValue += values[idx];
            remaining  -= weights[idx];
        } else {
            // Take fraction
            totalValue += (double) values[idx] * remaining / weights[idx];
            remaining = 0;
        }
    }

    return totalValue;
}
```

# 7. Pattern 3 — Huffman Coding (Greedy Trees)

## The Key Insight

```
Huffman coding assigns shorter bit sequences to more frequent characters.
Greedy: Always merge the two lowest frequency nodes.

Characters: A(5), B(9), C(12), D(13), E(16), F(45)

Step 1: Min-heap: [A(5), B(9), C(12), D(13), E(16), F(45)]
Step 2: Merge A+B → AB(14): [C(12), D(13), AB(14), E(16), F(45)]
Step 3: Merge C+D → CD(25): [AB(14), E(16), CD(25), F(45)]
Step 4: Merge AB+E → ABE(30): [CD(25), ABE(30), F(45)]
Step 5: Merge CD+ABE → CDABE(55): [F(45), CDABE(55)]
Step 6: Merge F+CDABE → root(100)

Result tree:
         root(100)
        /          \
     F(45)        CDABE(55)
                  /       \
              CD(25)      ABE(30)
              / \          / \
           C(12) D(13)  AB(14) E(16)
                        / \
                      A(5) B(9)

Huffman codes:
  F: 0      (1 bit)
  C: 100    (3 bits)
  D: 101    (3 bits)
  A: 1100   (4 bits)
  B: 1101   (4 bits)
  E: 111    (3 bits)
```

```java
import java.util.*;

public class HuffmanCoding {

    static class Node implements Comparable<Node> {
        char ch;
        int freq;
        Node left, right;

        Node(char ch, int freq) { this.ch = ch; this.freq = freq; }
        Node(int freq, Node left, Node right) {
            this.ch = '\0'; this.freq = freq;
            this.left = left; this.right = right;
        }

        @Override
        public int compareTo(Node other) { return this.freq - other.freq; }
    }

    public static Node buildHuffmanTree(char[] chars, int[] freqs) {
        PriorityQueue<Node> minHeap = new PriorityQueue<>();
        for (int i = 0; i < chars.length; i++) {
            minHeap.offer(new Node(chars[i], freqs[i]));
        }

        while (minHeap.size() > 1) {
            Node left  = minHeap.poll();  // Smallest frequency
            Node right = minHeap.poll();  // Second smallest
            // Merge into internal node
            minHeap.offer(new Node(left.freq + right.freq, left, right));
        }

        return minHeap.poll();  // Root of Huffman tree
    }

    public static void printCodes(Node root, String code) {
        if (root == null) return;
        if (root.left == null && root.right == null) {
            System.out.println(root.ch + ": " + code);
            return;
        }
        printCodes(root.left,  code + "0");
        printCodes(root.right, code + "1");
    }

    public static void main(String[] args) {
        char[] chars = {'A', 'B', 'C', 'D', 'E', 'F'};
        int[]  freqs = { 5,   9,  12,  13,  16,  45};
        Node root = buildHuffmanTree(chars, freqs);
        printCodes(root, "");
    }
}
```

## 🟡 LC #1167 — Minimum Cost to Connect Sticks

```java
public int connectSticks(int[] sticks) {
    PriorityQueue<Integer> minHeap = new PriorityQueue<>();
    for (int s : sticks) minHeap.offer(s);

    int totalCost = 0;

    while (minHeap.size() > 1) {
        int first  = minHeap.poll();  // Two smallest sticks
        int second = minHeap.poll();
        int cost   = first + second;  // Cost to connect
        totalCost += cost;
        minHeap.offer(cost);          // Put combined stick back
    }

    return totalCost;
}
```

**Greedy proof:** Connecting smallest sticks first minimizes how many times each stick is counted. Shorter sticks that get combined early contribute less to total cost.

# 8. Pattern 4 — Jump Game / Reach Problems

## The Key Insight

```
Maintain the FARTHEST REACHABLE position.
At each step, if we can reach current index, update max reach.
If we fall behind the max reach → stuck (can't reach end).

Array: [2, 3, 1, 1, 4]
maxReach starts at 0.

i=0: nums[0]=2, can reach 0+2=2. maxReach=max(0,2)=2
i=1: i<=maxReach(2)✅, nums[1]=3, can reach 1+3=4. maxReach=max(2,4)=4
i=2: i<=maxReach(4)✅, nums[2]=1, can reach 2+1=3. maxReach=max(4,3)=4
i=3: i<=maxReach(4)✅, nums[3]=1, can reach 3+1=4. maxReach=max(4,4)=4
i=4: i<=maxReach(4)✅ → reached end! return true
```

## 🟢 LC #55 — Jump Game

```java
public boolean canJump(int[] nums) {
    int maxReach = 0;

    for (int i = 0; i < nums.length; i++) {
        if (i > maxReach) return false;  // Can't reach this index
        maxReach = Math.max(maxReach, i + nums[i]);  // Update max reach
        if (maxReach >= nums.length - 1) return true; // Can reach end
    }

    return true;
}
```

## 🟡 LC #45 — Jump Game II (Minimum Jumps)

```java
public int jump(int[] nums) {
    int jumps = 0;
    int currentEnd = 0;   // End of current jump's range
    int farthest = 0;     // Farthest we can reach

    for (int i = 0; i < nums.length - 1; i++) {
        farthest = Math.max(farthest, i + nums[i]);

        if (i == currentEnd) {
            // Must make a jump here (explored all positions in current range)
            jumps++;
            currentEnd = farthest;
            if (currentEnd >= nums.length - 1) break;
        }
    }

    return jumps;
}
```

**Visual:**
```
nums = [2, 3, 1, 1, 4]

Jump 1: from index 0, reach up to index 2 (max reach from here = 4)
Jump 2: from range [1,2], reach up to index 4 (end!) → done

Minimum jumps = 2
```

## 🟡 LC #1024 — Video Stitching

```java
public int videoStitching(int[][] clips, int time) {
    // Sort by start time, then by longest end for same start
    Arrays.sort(clips, (a, b) -> a[0] != b[0] ? a[0]-b[0] : b[1]-a[1]);

    int jumps = 0, curEnd = 0, farthest = 0;
    int i = 0;

    while (curEnd < time) {
        // Find clip that starts <= curEnd and extends farthest
        while (i < clips.length && clips[i][0] <= curEnd) {
            farthest = Math.max(farthest, clips[i][1]);
            i++;
        }
        if (farthest == curEnd) return -1;  // Can't extend
        jumps++;
        curEnd = farthest;
    }

    return jumps;
}
```

## 🔴 LC #1326 — Minimum Number of Taps to Open to Water Garden

```java
public int minTaps(int n, int[] ranges) {
    // Convert to interval problem: each tap i covers [i-ranges[i], i+ranges[i]]
    // Then solve like Jump Game II
    int[] maxReach = new int[n + 1];  // maxReach[i] = farthest right reachable starting from i

    for (int i = 0; i <= n; i++) {
        int left  = Math.max(0, i - ranges[i]);
        int right = Math.min(n, i + ranges[i]);
        maxReach[left] = Math.max(maxReach[left], right);
    }

    int taps = 0, curEnd = 0, farthest = 0;

    for (int i = 0; i <= n; i++) {
        if (i > farthest) return -1;  // Gap in coverage
        farthest = Math.max(farthest, maxReach[i]);
        if (i == curEnd && i < n) {
            taps++;
            curEnd = farthest;
        }
    }

    return taps;
}
```

## 🟡 LC #621 — Task Scheduler

**Problem:** Schedule tasks (A-Z) with cooldown n between same tasks. Minimize total time.

```java
public int leastInterval(char[] tasks, int n) {
    int[] freq = new int[26];
    for (char t : tasks) freq[t - 'A']++;
    Arrays.sort(freq);

    int maxFreq = freq[25];           // Most frequent task
    int idleSlots = (maxFreq - 1) * n; // Idle slots needed

    // Fill idle slots with other tasks
    for (int i = 24; i >= 0 && idleSlots > 0; i--) {
        idleSlots -= Math.min(freq[i], maxFreq - 1);
    }

    idleSlots = Math.max(0, idleSlots);
    return tasks.length + idleSlots;
}
```

**Why this works:**
```
Tasks: [A,A,A,B,B], n=2

Most frequent: A (3 times)
Structure: A _ _ A _ _ A
           ↑
           2 cooldown slots needed between each A

Fill with B: A B _ A B _ A
One idle slot remains: A B _ A B _ A = 7 total time

Formula: max(tasks.length, (maxFreq-1)*(n+1) + count_of_maxFreq_tasks)
```

```java
public int maximumUnits(int[][] boxTypes, int truckSize) {
    // Greedy: take boxes with most units per box first
    Arrays.sort(boxTypes, (a, b) -> b[1] - a[1]);  // Sort by units DESC

    int totalUnits = 0, remaining = truckSize;

    for (int[] box : boxTypes) {

        remaining  -= take;
        if (remaining == 0) break;
    }

    return totalUnits;
}
```

## 🟡 LC #11 — Container With Most Water

```java
public int maxArea(int[] height) {
    int left = 0, right = height.length - 1, maxWater = 0;

    while (left < right) {
        int water = Math.min(height[left], height[right]) * (right - left);
        maxWater = Math.max(maxWater, water);

        // Greedy: move the SHORTER side (can only gain by finding taller)
        if (height[left] < height[right]) left++;
        else right--;
    }

    return maxWater;
}
```

**Greedy proof:** Moving the taller side can only reduce width with no benefit. Moving the shorter side might find a taller wall — the only way to improve.

## 🟡 LC #881 — Boats to Save People

**Problem:** Each boat holds at most 2 people with total weight ≤ limit. Find minimum boats.

```java
public int numRescueBoats(int[] people, int limit) {
    Arrays.sort(people);
    int left = 0, right = people.length - 1;
    int boats = 0;

    while (left <= right) {
        if (people[left] + people[right] <= limit) {
            left++;   // Lightest person fits with heaviest
        }
        right--;      // Heaviest person always gets a boat (alone or with lightest)
        boats++;
    }

    return boats;
}
```

**Greedy:** Always pair the heaviest with the lightest (if possible). If they can't pair, heaviest goes alone.

```java
public int numSubseq(int[] nums, int target) {
    Arrays.sort(nums);
    int MOD = 1_000_000_007;
    int n = nums.length;

    // Precompute powers of 2
    int[] pow = new int[n];
    pow[0] = 1;
    for (int i = 1; i < n; i++) pow[i] = pow[i-1] * 2 % MOD;

    int result = 0, left = 0, right = n - 1;

    while (left <= right) {
        if (nums[left] + nums[right] <= target) {
            // nums[left] as min: can choose any subset of [left+1..right] as max
            result = (result + pow[right - left]) % MOD;
            left++;
        } else {
            right--;  // Sum too large, reduce right
        }
    }

    return result;
}
```

## 🔴 LC #321 — Create Maximum Number

```java
// From two arrays of digits, create max number of length k
public int[] maxNumber(int[] nums1, int[] nums2, int k) {
    int m = nums1.length, n = nums2.length;
    int[] result = new int[k];

    for (int i = Math.max(0, k - n); i <= Math.min(k, m); i++) {
        int[] seq1 = maxSubsequence(nums1, i);
        int[] seq2 = maxSubsequence(nums2, k - i);
        int[] merged = merge(seq1, seq2);
        if (compare(merged, 0, result, 0) > 0) result = merged;
    }
    return result;
}

private int[] maxSubsequence(int[] nums, int k) {
    Deque<Integer> stack = new ArrayDeque<>();
    int drop = nums.length - k;
    for (int num : nums) {
        while (drop > 0 && !stack.isEmpty() && stack.peek() < num) {
            stack.pop(); drop--;
        }
        stack.push(num);
    }
    int[] result = new int[k];
    for (int i = k-1; i >= 0; i--) result[i] = stack.pop();
    return result;
}

private int[] merge(int[] seq1, int[] seq2) {
    int[] result = new int[seq1.length + seq2.length];
    int i = 0, j = 0, k = 0;
    while (i < seq1.length || j < seq2.length) {
        result[k++] = compare(seq1, i, seq2, j) >= 0 ? seq1[i++] : seq2[j++];
    }
    return result;
}

private int compare(int[] a, int i, int[] b, int j) {
    while (i < a.length && j < b.length && a[i] == b[j]) { i++; j++; }
    if (i == a.length) return j == b.length ? 0 : -1;
    if (j == b.length) return 1;
    return a[i] - b[j];
}
```

```java
        while (k > 0 && !stack.isEmpty() && stack.peek() > c) {
            stack.pop();
            k--;
        }
        stack.push(c);

    while (k-- > 0) stack.pop();

    // Build result, skip leading zeros
    StringBuilder sb = new StringBuilder();

public String removeKdigits2(String num, int k) {

        while (top > 0 && k > 0 && stack[top-1] > c) { top--; k--; }
        stack[top++] = c;
```

```java
public String reorganizeString(String s) {

    for (char c : s.toCharArray()) freq[c-'a']++;

    // Max frequency character
    int maxFreq = 0, maxChar = 0;
    for (int i = 0; i < 26; i++) {
        if (freq[i] > maxFreq) { maxFreq = freq[i]; maxChar = i; }

    if (maxFreq > (s.length() + 1) / 2) return "";

    // Fill remaining positions

        while (freq[i] > 0) {
            if (idx >= result.length) idx = 1;
            result[idx] = (char)('a'+i); idx+=2; freq[i]--;
        }
    }

    return new String(result);
```

## 🟡 LC #1642 — Furthest Building You Can Reach

```java
public int furthestBuilding(int[] heights, int bricks, int ladders) {
    // Min-heap: track the k largest climbs (use ladders for these)
    PriorityQueue<Integer> ladderClimbs = new PriorityQueue<>();

    for (int i = 0; i < heights.length - 1; i++) {
        int diff = heights[i+1] - heights[i];
        if (diff <= 0) continue;  // Going down or same height

        ladderClimbs.offer(diff);

        if (ladderClimbs.size() > ladders) {
            // Too many climbs for ladders: use bricks for the smallest one
            bricks -= ladderClimbs.poll();
        }

        if (bricks < 0) return i;  // Ran out of bricks
    }

    return heights.length - 1;
}
```

**Greedy:** Use ladders for the largest jumps. Since we don't know future jumps, maintain a min-heap of ladder-assigned jumps. When we have more jumps than ladders, demote the smallest to bricks.

```java
public boolean isPossible(int[] target) {
    PriorityQueue<Long> maxHeap = new PriorityQueue<>(Collections.reverseOrder());
    long sum = 0;
```

# 19. How to Identify Greedy Problems

```
Keywords / Clues → Greedy Pattern
──────────────────────────────────────────────────────────────
"maximize/minimize" + "greedy structure" → Greedy
"intervals" + "minimum removals/arrows"  → Sort by end time
"intervals" + "merge"                    → Sort by start time
"intervals" + "minimum rooms"            → Sort starts & ends
"jump/reach" + "minimum jumps"           → Greedy max reach
"tasks with cooldown"                    → Frequency-based
"assign to maximize matches"             → Sort both + match
"fractional items"                       → Sort by value/weight
"make change" with specific coins        → Try greedy, verify
"build string with min cost"             → Fill from min cost end
"remove k digits for smallest"           → Monotonic stack
"k largest/heaviest first"               → Max-heap greedy

WHEN GREEDY WORKS:
→ Can you PROVE that local best = global best? (exchange argument)
→ Does making the greedy choice NOT hurt future choices?
→ Can you sort the input to make greedy obvious?

WHEN GREEDY FAILS:
→ Coin change with non-standard coins → DP
→ 0/1 Knapsack (can't take fractions) → DP
→ Edit distance → DP
→ Matrix chain multiplication → DP

GREEDY CHECKLIST:
1. Identify what to optimize (maximize/minimize what?)
2. What is the greedy choice? (always pick X)
3. WHY is it safe? (exchange argument)
4. What sort order makes greedy obvious?
5. Code: sort + iterate (or priority queue)
```

```java
int jumps=0, curEnd=0, farthest=0;
for (int i=0; i<n-1; i++) {
    farthest = Math.max(farthest, i+nums[i]);
    if (i==curEnd) { jumps++; curEnd=farthest; }
}
return jumps;

// STOCK PROFIT II (multiple transactions)
int profit=0;
for (int i=1; i<n; i++)
    if (prices[i]>prices[i-1]) profit+=prices[i]-prices[i-1];
return profit;

// MEETING ROOMS II (min rooms)
Arrays.sort(starts); Arrays.sort(ends);
int rooms=0, e=0;
for (int s : starts) { if (s<ends[e]) rooms++; else e++; }
return rooms;
```

## ⚡ The 5 Golden Rules of Greedy

```
1. SORT FIRST — 90% of greedy problems require sorting as the first step.
   Think: "What order makes the greedy choice obvious?"

2. PROVE IT — always convince yourself (and the interviewer) WHY the
   greedy choice is safe. Use the exchange argument:
   "If optimal differs from greedy, we can swap and not get worse."

3. INTERVAL PROBLEMS:
   Sort by END → maximize non-overlapping count / minimize removals
   Sort by START → merge overlapping intervals

4. JUMP PROBLEMS: Track max reachable index. If current index > max
   reachable, return false. Else update max reachable.

5. WHEN IN DOUBT: Sort and try greedy. If it gives wrong answer on
   small examples, switch to DP. Never assume greedy works without
   checking edge cases.
```
